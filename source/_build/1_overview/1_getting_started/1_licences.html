Lizenzen für GameMaker Studio 2


Es gibt verschiedene Lizenztypen, die Sie für GameMaker Studio 2 können. Die Auswahl hängt weitgehend von der Zielgruppe ab, die Sie ansprechen möchten, und davon, ob Sie nur Spaß am Erstellen von Spielen haben oder diese verkaufen möchten verteilen sie auch. Nachfolgend finden Sie Details zu jedem Lizenztyp.

GameMaker Studio 2 wird mit einer Testlizenz ausgeliefert, und Sie können je nach Bedarf verschiedene Upgrades erwerben. Die Testlizenz unterliegt bestimmten Einschränkungen. Sie können jedoch die wichtigsten Funktionen und Prototypprojekte testen. Bei der Testlizenz des Produkts ist zu beachten, dass Sie zwar auf Ihren Computern Spiele testen können, jedoch keine endgültigen ausführbaren Pakete für eine Zielplattform erstellen können. Abgesehen davon sind Sie auch hinsichtlich der verfügbaren Ressourcen eingeschränkt:

  • Gegenstände - 15
  • Sprites - 20 (keine dynamische Belastung zulässig, keine SWF- oder Spine-Sprites zulässig)
  • Sounds - 10 (keine Audiopuffer erlaubt)
  • Fliesensets - 2
  • Drehbücher - 10
  • Pfade - 5 (keine dynamische Pfaderstellung zulässig)
  • Zeitleisten - 2 (keine dynamische Zeitleistenerstellung zulässig)
  • Schriftarten - 5 (keine dynamische Zeichensatzerstellung zulässig)
  • Rooms - 5 (keine dynamische Raumerstellung erlaubt)
  • Shader - 0
  • Enthaltene Dateien - 0
  • Erweiterungen - 0
  • Konfigurationen - 0


Abgesehen von diesen Ressourcenbeschränkungen fehlen in der IDE einige andere Elemente:

  • Sie können keine Textur-Seiten erstellen
  • Sie können keine Texturgruppen erstellen
  • Sie können keine Audiogruppen erstellen
  • Sie können vorhandene Ressourcen nicht über das Kontextmenü hinzufügen
  • Sie können die integrierte Quellcodeverwaltung nicht verwenden
  • Sie können keine Assets vom Marketplace herunterladen
  • Sie können keine alten Projekte aus früheren Versionen von GameMaker Studio importieren
  • Sie können keine YYZ-Pakete importieren
  • Sie können den YoYo Games Splash Screen nicht ändern
  • Sie können anonyme IDE-Analysen nicht deaktivieren

Mit der Creator- Version von GameMaker Studio 2 meisten Einschränkungen der Testlizenz (siehe oben) aufgehoben, außer der Möglichkeit zum Ändern des Begrüßungsbildschirms, der Deaktivierung anonymer IDE-Statistiken und der Verwendung des YYC-Ziels. Nach dem Kauf können Sie Projekte zu einem der folgenden Ziele zusammenstellen:

  • Windows
  • Mac OS

Die verfügbare Zielplattform hängt von der Version der IDE ab, die Sie gekauft haben ( Creator für macOS oder Creator für Windows ), und verwendet die VM (ein Runner + interpretierter Code), um Ihre Spiele auszuführen und zu erstellen. Informationen zum Einrichten finden Sie im Helpdesk-Abschnitt " Erforderliche SDKs" oder im Abschnitt " Desktop-Ziele".


Mit der Desktop- Version von GameMaker Studio 2 alle Einschränkungen der Testlizenz (siehe oben) aufgehoben, und Sie können Projekte mit den folgenden Desktop-Zielen zusammenstellen:

  • Windows
  • Mac OS
  • Ubuntu

Alle Ziele verfügen über zwei Kompilierungsoptionen, eine ist die VM, die einen Läufer und interpretierten Code verwendet, und der andere ist der YYC ( YoYo Compiler), der nativen Code und eine kompilierte ausführbare Datei erstellt. Beachten Sie, dass für die YYC-Option eine zusätzliche Einrichtung erforderlich ist. Weitere Informationen finden Sie im Abschnitt Erforderliche SDKs oder im Helpdesk-Abschnitt zu Desktop-Zielen.


Mit der Web- Version von GameMaker Studio 2 alle Einschränkungen der Testlizenz (siehe oben) aufgehoben und Sie können Projekte zum HTML5- Ziel kompilieren. Spiele werden mit JavaScript kompiliert. Weitere Informationen finden Sie im YoYo Games-Helpdesk: Einrichten für das Web (HTML5).


Mit der Amazon Fire Version von GameMaker Studio 2 alle Einschränkungen der Testlizenz (siehe oben) aufgehoben, und Sie können Projekte mit dem Amazon Fire Ziel zusammenstellen. Spiele werden mit dem Android SDK erstellt und können entweder mit der VM (einem Läufer mit interpretiertem Code) oder dem YYC ( YoYo Compiler, der nativen Code erstellt) kompiliert werden. Weitere Informationen zum Einrichten finden Sie im Abschnitt Erforderliche SDKs oder im Helpdesk-Abschnitt der Amazon Fire Lizenz.


Mit der Mobile- Version von GameMaker Studio 2 alle Einschränkungen der Testlizenz (siehe oben) aufgehoben, und Sie können Projekte für folgende Ziele zusammenstellen:

  • Android
  • Amazon Fire
  • iOS

Alle Ziele verfügen über zwei Kompilierungsoptionen, eine ist die VM, die einen Läufer und interpretierten Code verwendet, und der andere ist der YYC ( YoYo Compiler), der nativen Code und eine kompilierte ausführbare Datei erstellt. Beachten Sie, dass für die YYC-Option eine zusätzliche Einrichtung erforderlich ist. Weitere Informationen zum Einrichten finden Sie im Abschnitt " Erforderliche SDKs" oder im Helpdesk-Abschnitt " Mobile Targets".


Mit der UWP- Version von GameMaker Studio 2 alle Einschränkungen der Testlizenz (siehe oben) aufgehoben, und Sie können Projekte als UWP-kompatible ausführbare Dateien kompilieren. Diese ausführbaren Dateien können für die Ausführung auf einer von drei UWP-Plattformen konfiguriert werden:

  • Windows 10 Desktop
  • Windows 10 Mobile
  • XBox One

Das UWP-Ziel verfügt über zwei Kompilierungsoptionen, eine ist die VM, die einen Läufer und interpretierten Code verwendet, und die andere ist der YYC ( YoYo Compiler), der nativen Code und eine kompilierte ausführbare Datei erstellt. Beachten Sie, dass für die YYC-Option eine zusätzliche Einrichtung erforderlich ist. Weitere Informationen zum Einrichten finden Sie im Abschnitt " Erforderliche SDKs" oder im Helpdesk-Abschnitt " UWP-Ziele".


Mit der Consoles- Version von GameMaker Studio 2 alle Einschränkungen der Testlizenz (siehe oben) aufgehoben und Sie können Projekte für die folgenden Zielplattformen kompilieren

  • XBox One
  • Playstation 4
  • Nintendo-Schalter

Jedes Konsolenziel verfügt über zwei Kompilierungsoptionen, eine ist die VM, die einen Läufer und interpretierten Code verwendet, und der andere ist der YYC ( YoYo Compiler), der nativen Code und eine kompilierte ausführbare Datei erstellt. Beachten Sie, dass für die YYC-Option eine zusätzliche Einrichtung erforderlich ist. Weitere Informationen zum Einrichten finden Sie im Abschnitt Erforderliche SDKs.

WICHTIG! Aufgrund der Einschränkungen der verschiedenen NDAs, die mit dem Export der Konsole verbunden sind, finden Sie keine Informationen zum Einrichten oder Programmieren dieser Ziele im GameMaker Studio 2 Handbuch. Wenn Sie ein registrierter Entwickler für die ausgewählte Plattform sind, wenden Sie sich bitte mit Ihrem YoYo Konto an das Helpdesk und fordern Sie den Zugang zur entsprechenden Dokumentation an.

GameMaker Studio 2 kann auch in einer College-Umgebung verwendet werden. Für weitere Informationen hierzu wenden Sie sich bitte an YoYo Games über das YoYo Games Education Portal.


WICHTIG! Wenn Sie eine Testlizenz für GameMaker Studio 2 und dann eine der anderen Lizenzen erwerben (Ersteller, Desktop, Amazon Fire, Mobile, UWP, Web oder Konsole), müssen Sie sich von der IDE GameMaker Studio 2 und GameMaker Studio 2 neu starten GameMaker Studio 2 Wenn Sie das Programm neu starten und sich erneut anmelden, sollte die neue Lizenz korrekt angewendet werden. Sie können sich vom Datei-Menü abmelden.
