Einstellungen - Runtime-Feeds


Die Runtime-Feeds-Einstellungen dienen zum Anzeigen und Ändern der aktuell installierten Laufzeiten sowie der aktuellen Laufzeit. GameMaker Studio 2 können mehrere verschiedene Runtime-Tools gleichzeitig installiert sein, und Sie können festlegen, welches von diesen Tools verwendet werden soll. Idealerweise möchten Sie die aktuellste Runtime verwenden, aber manchmal kann ein Problem auftreten, oder Sie müssen möglicherweise ein älteres für die Vergleichbarkeit usw. verwenden, damit Sie die Laufzeit-Tools von GameMaker Studio 2 ändern können GameMaker Studio 2 verwendet, um Ihr Spiel hier zu bauen.

Der erste Abschnitt dieser Einstellung bezieht sich auf den Ort, von dem aus Sie die Laufzeittools abrufen, und erfordert die Eingabe einer URL in einen RSS-Feed. Dieser Feed wird dann nach Updates für die Runtime-Tools gescannt, die dann heruntergeladen und installiert werden können. In diesem Abschnitt wird standardmäßig die Master-Laufzeitfeed-URL angezeigt, Sie können jedoch auf die Schaltfläche Hinzufügen klicken, um einen neuen Laufzeitfeed zu erstellen. Geben Sie den Namen und die URL für den RSS-Feed ein und klicken Sie auf Anwenden, um GameMaker Studio 2 den Zugriff zu gewähren. Sie müssen das Fenster "Einstellungen" schließen und erneut öffnen, damit der neue Feed angezeigt wird.

Sobald ein Runtime-Feed erstellt wurde, wird er in diesem Abschnitt als Unterkategorie aufgeführt, und Sie sollten immer mindestens einen Feed aufgelistet haben (standardmäßig wäre dies der Master-Feed). Die Optionen für den Laufzeit-Feed sind unten für den Master angegeben, sie sind jedoch für alle anderen identisch:

Mit den Voreinstellungen für den Master Runtime-Feed werden die verschiedenen Runtime-Tools ausgewählt, installiert und deinstalliert (diese Seite ist auch für weitere Runtime-Feeds, die Sie hinzugefügt haben, identisch). Ganz oben befinden sich die aktuellen Laufzeitwerkzeuge, das Erstellungsdatum und der Speicherort auf der Festplatte. Unter diesen Details finden Sie eine Liste aller verfügbaren Laufzeiten, und Sie können eine beliebige aus dieser Liste auswählen, um zu der Laufzeit zu werden, die zum Kompilieren Ihrer Projekte verwendet wird.

Um eine andere zu verwendende Laufzeit auszuwählen, suchen Sie sie einfach in der Liste und doppelklicken Sie dann auf darauf. Wenn die Laufzeitumgebung nicht heruntergeladen und installiert wurde, werden Sie dazu aufgefordert. Nach der Installation werden Sie aufgefordert, die IDE zu schließen und neu zu starten, damit die neue Laufzeitumgebung initialisiert werden kann.

In dieser Laufzeitliste werden alle verfügbaren Laufzeiten, ihre Versionsnummer, das Veröffentlichungsdatum und, falls installiert, der Pfad zu dem Ort, an dem sie installiert wurden, aufgeführt. Die rechte Seite zeigt auch die verschiedenen Statussymbole, die wie folgt aussehen:

Dies kennzeichnet die aktuelle Laufzeit, die beim Erstellen Ihrer Projekte ausgewählt wurde.
Wenn Sie darauf klicken, wird der Download und die Installation der ausgewählten Runtime gestartet (eine Statusleiste wird oben rechts in der IDE angezeigt).
Wenn Sie darauf klicken, wird die ausgewählte Laufzeit von Ihrer Festplatte entfernt.

Sie können die Laufzeitliste jederzeit aus dem angegebenen RSS-Feed aktualisieren, indem Sie auf die Schaltfläche zum Aktualisieren klicken .
