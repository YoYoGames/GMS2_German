Texturgruppen


Die Textur Group Manager sind von dem verfügbaren Werkzeugen Menü in der IDE. Hier können Sie hinzufügen, löschen und Texture Gruppen umbenennen. GameMaker Studio 2 können Sie die verschiedenen Grafikressourcen (Sprites, Tilesets und Schriftarten) verschiedenen Texturgruppen zuweisen, um die Anzahl der Textur-Swaps, die die Hardware zu einem bestimmten Zeitpunkt in Ihrem Spiel ausführt, zu optimieren. Damit dies funktioniert, müssen Sie in diesem Fenster die erforderlichen Texturgruppen definieren und dann die entsprechenden Ressourcentypen durchsuchen und die gewünschte Gruppe aus dem Dropdown-Menü auswählen.

WICHTIG! Texturgruppen sind mit der Testlizenz des Produkts nicht verfügbar.


Standardmäßig werden alle Bilder in Ihrem Spiel der Texturgruppe "Standard" hinzugefügt, die immer vorhanden ist. Sie können jedoch auf die Schaltfläche " Neu hinzufügen" klicken, um eine neue Texturgruppe zu erstellen. Wenn Sie den Namen der Gruppe auswählen, können Sie dies ändern. Um mit der Gruppe zu beginnen, wird "diese Gruppe ist leer" für jeden Abschnitt der Gruppe angezeigt (Sprites, Tilesets und Schriftarten), und Sie müssen dann Ihre Ressourcen durchsuchen und hinzufügen. Sie können eine Ressource mithilfe der Schaltfläche Ressource hinzufügen am unteren Rand hinzufügen oder indem Sie die Texturgruppe aus dem Ressourceneditor für das Bild, den Kachelsatz oder die Schriftart auswählen. Andere Texturgruppen als "Standard" können auch jederzeit gelöscht werden, wodurch ihre Inhalte in die Gruppe "Standard" verschoben werden. Wenn Sie versehentlich eine Gruppe löschen, können Sie sie verwenden / + Z um die Aktion rückgängig zu machen).

Nachdem Sie Texturgruppen und zugewiesene Sprites definiert haben, sehen Sie diese links im Hauptfenster des Texturgruppen-Editors. Wenn Sie auf eines der Sprites in dieser Liste klicken, werden sie im rechten Vorschaufenster angezeigt. Dies ist besonders hilfreich, wenn Sie an großen Projekten arbeiten, und Sie müssen sehen können, welche Sprites zu welcher Gruppe zugewiesen wurden. Wenn die der Gruppe zugewiesenen Sprites mehrere Unterbilder haben (dh sie sind animiert), wird nur das erste Unterbild im Vorschaufenster angezeigt. Sie können auch mit der rechten Maustaste klicken auf einer Ressource, um ein kleines Menü zu öffnen, das Ihnen die Möglichkeit gibt, es in eine andere Texturgruppe zu verschieben oder den Editor für dieses Asset zu öffnen.

Es ist erwähnenswert, dass, obwohl die Assets in der Liste als separate Elemente angezeigt werden, alle Sprites, Tilesets und Schriftarten derselben Texturseite (oder denselben Texturseiten) hinzugefügt werden und keine individuelle Seite für jeden Ressourcentyp erhalten.


Gruppeneinstellungen

Die Gruppeneinstellungen werden verwendet, um genau festzulegen, wie die Texturseiten für eine bestimmte Gruppe mit den folgenden verfügbaren Optionen erstellt werden sollen:

  • Scaling erlauben - Wird diese GameMaker deaktiviert, wird GameMaker Studio 2 gezwungen, die Texturgruppe unter keinen Umständen zu skalieren. Dies kann insbesondere bei universellen Apps hilfreich sein, da Sie bestimmte Ressourcen wie Zeichensätze in einer eigenen Gruppe verwenden können, wenn die App auf einem bestimmten Gerät ausgeführt wird. Zum Beispiel können Schriften auf einem Retina-Display gut aussehen, aber wenn sie auf ein normales Display verkleinert werden, können sie schrecklich aussehen, so dass Sie sie einer speziellen nicht skalierenden Gruppe zuweisen können - wenn Sie für das iPhone 6 entwickeln möchten, dass dies überprüft wird, da Apps aufgrund von unscharfen Bildern durch Texturskalierung abgelehnt wurden. Dies ist standardmäßig aktiviert.

  • Automatisches Zuschneiden - Wenn Sie Bilder zu GameMaker Studio 2 hinzufügen, wenn diese einen "Abstand" haben (dh 100% Transparenz), wird diese automatisch zugeschnitten, wenn sie der Texturseite hinzugefügt wird und "Automatisch zuschneiden" markiert ist. Wenn Sie beispielsweise das Bild als sich nicht wiederholende Textur für Grundelemente oder ähnliches verwenden, bedeutet dies, dass die UV-Koordinaten für die Texturzuordnung nicht korrekt dargestellt werden. In diesen Fällen können Sie diese Option deaktivieren und das Sprite wird der Texturseite "so wie es ist" hinzugefügt, wobei alle Transparenzen beibehalten werden, die an den Kanten vorhanden sind. Dies ist standardmäßig aktiviert.

  • Mipmaps generieren - Wenn diese Option markiert ist (standardmäßig deaktiviert ), wird GameMaker Studio 2 mitgeteilt, dass die ursprüngliche Textur-Seite skaliert und in mehrere Auflösungen innerhalb der Textur-Datei gefiltert werden soll. Dies ist normalerweise deaktiviert, aber wenn Sie Ansichten zum Vergrößern oder Verkleinern großer Bereiche des Spielraums verwenden oder wenn Sie mit einem 3D-Spiel arbeiten, möchten Sie dies möglicherweise aktivieren und dann mithilfe der Mipmapping-Funktionen anpassen, wie Das Spiel beschäftigt sich mit den verschiedenen Mipmap Levels. Wenn Sie dieses Kontrollkästchen aktivieren, wird der Wert für "Border Size" auf 8 gesetzt (wenn er ursprünglich auf weniger als 8 gesetzt wurde). Dies liegt daran, dass Sie bei Verwendung von Mipmaps größere Ränder zwischen Bildern auf den Texturseiten benötigen, da sie sonst bei höheren Mip-Werten zusammen bluten. Der Standardwert von 8 ermöglicht es, das Bild auf etwa ein Achtel seiner ursprünglichen Größe zu reduzieren, bevor die Teilbilder zusammen zu bluten beginnen. Sie können dies jedoch auf jeden Wert ändern, der Ihren Anforderungen entspricht. Beachten Sie, dass Sie, wenn Sie die Option " Getrennte Texturseite " für Sprites verwenden, nicht jede Textur hier durchgehen und sie zum Erstellen von Mipmaps kennzeichnen müssen. Stattdessen können Sie zu den allgemeinen Spieloptionen gehen und die Option " Generate Mipmaps für separate Texturseiten ".

  • Rahmengröße - Die Rahmengröße ändert die Überlappung (oder "Pixelverdopplung") um die Ränder aller Elemente auf den Seiten, die für die Texturgruppe generiert wurden, so dass beim Skalieren der Bilder keine Lücken (oder Nähte) sichtbar sind Sie. Der Standardwert ist 2 Pixel, aber in extremen Fällen, in denen Sie die Ansicht stark vergrößern oder verkleinern, müssen Sie diesen Wert möglicherweise erhöhen. Beachten Sie, dass der Kachelsatz-Editor eine Option zum Festlegen des Ausgabebegrenzers für Kacheln hat. Dies ist unabhängig von der Rahmeneinstellung. Grundsätzlich werden Kacheln mit einem Ausgabebegrenzungsrahmen für jede Kachel in einer temporären Textur generiert und dann hinzugefügt die Texturseite mit der hier angegebenen Grenze um die Außenkante. Beachten Sie außerdem, dass beim Aktivieren von MipMaps automatisch die Standardgröße für die Texturseite auf 8 festgelegt wird, um ein "Ausbluten" zwischen Texturen bei höheren Mip-Werten zu verhindern. Dies kann jedoch geändert werden.

  • Parent Texture Group - Hier können Sie eine der vorhandenen Gruppen als "Parent" -Gruppe für die ausgewählte Gruppe auswählen. "Untergeordnete" Gruppen erben alle Änderungen, die an der übergeordneten Strukturgruppe vorgenommen wurden, und dienen hauptsächlich der Ressourcenverwaltung bei der Arbeit mit plattformübergreifenden Spielen. Eine Texturgruppe mit einem übergeordneten Element ist immer noch eine eigenständige Texturgruppe und generiert Texturseiten wie jede andere Gruppe. Wenn Sie ihnen jedoch ein Elternelement zuweisen, können Sie große Segmente für verschiedene Plattformen ausschließen. Zum Beispiel können Sie einer Gruppe "Spieler" eine ganze Ladung Android Sprites hinzufügen und Android Gegner zu einer Gruppe "Feind" sprites, dann verwenden beide eine "Android" -Muttergruppe. Wenn Sie dann die Elterngruppe "android" von der Windows- oder anderen Plattform-Version ausschließen, werden die untergeordneten Gruppen ebenfalls ausgeschlossen.

Abgesehen von diesen Einstellungen können Sie auch festlegen, dass eine bestimmte Texturgruppe auf eine bestimmte Plattform exportiert (oder nicht) wird, indem Sie einfach die entsprechenden Kontrollkästchen im Bereich Gruppe verwenden am... aktivieren oder deaktivieren. Beachten Sie jedoch, dass die <Default> -Texturgruppe immer auf jede Plattform exportiert wird, und Sie können keine davon aktivieren / deaktivieren (nur benutzerdefinierte Texturgruppen erlauben dies).


Konfigurationen

Texturgruppen sind auch mit den Konfigurationseinstellungen verknüpft. Das bedeutet, dass Sie Texturgruppen für jede einzelne Konfiguration Ihres Spiels definieren können und dann die Sprite-, Kachelsatz- oder Zeichensatz-Ressourcenzuweisungen auf diese verschiedenen Gruppen festlegen können und diese "erinnert" werden, wenn Sie zu einer anderen Konfiguration wechseln. Das Gleiche gilt für alle Exportoptionen, die Sie im Fenster "Texturgruppe" ausgewählt haben. So können Sie festlegen, dass die Texturgruppen für bestimmte Plattformen pro Konfiguration exportiert werden.

Eine Anwendung hierfür ist zum Beispiel, dass Sie eine iPad Konfiguration machen könnten und dafür schaffen 2048x2048px Textur Seiten ( das Einrichten dieser Funktion über die Spieloptionen ). Sie würden dann alle Grafikelemente der entsprechenden Texturgruppe zuweisen, und wenn Sie fertig sind, würden Sie dann zu einer anderen Konfiguration wechseln (z. B. für HTML5) und im Texturgruppenfenster neue Texturgruppen erstellen, die 512x512px sind Vermögenswerte zu diesen neuen Gruppen. Es gibt auch eine Schaltfläche am unteren Rand der Seite, auf der Sie Texturgruppeneinstellungen von einer Konfiguration in eine andere kopieren können.
