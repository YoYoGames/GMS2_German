Plattformspezifische Einstellungen

In diesem Abschnitt des Handbuchs werden die verschiedenen plattformspezifischen Voreinstellungen beschrieben, die festgelegt werden müssen, bevor Sie Ihre Spiele für bestimmte Ziele zusammenstellen können. Diese müssen in der Regel nur einmal eingerichtet werden (bei zukünftigen Aktualisierungen der Laufzeiten müssen jedoch möglicherweise einige Änderungen vorgenommen werden, die jedoch in den Versionshinweisen erwähnt werden ). Sie dienen dazu, auf erforderliche SDKs zu verweisen und grundlegende Optionen festzulegen wie Sie möchten, dass GameMaker Studio 2 Ihre Spielprojekte für eine bestimmte Plattform zusammenstellt. Beachten Sie, dass die verfügbaren Plattformen von Ihrer Lizenz abhängen und möglicherweise nicht alle verfügbar sind.

Nachfolgend finden Sie eine Liste aller verfügbaren Zielplattformen und ihrer Präferenzen:

Die Amazon Fire Einstellungen sind in verschiedene Abschnitte unterteilt. Einige davon beziehen sich auf Android Tools, da das AMazon Fire-Ziel von Android abhängt und dieselben SDK-Anforderungen aufweist.

Oben in den Einstellungen haben Sie die Möglichkeit, die Einstellungen aus dem Android Export zu synchronisieren. Dies ist für Personen, die über eine Mobile-Lizenz verfügen und den Android Export bereits eingerichtet und verwendet haben, da Amazon Fire fast die gleichen Build-Tools wie Android. Klicken Sie einfach auf diese Option, um alle entsprechenden Einstellungen von Android zu importieren, und überprüfen Sie sie, um sicherzustellen, dass sie korrekt sind.

Unter der Schaltfläche Synchronisieren können Sie die Größe des Memory Heap für das Android JDK einstellen. Als Nächstes müssen Sie die verschiedenen Pfade einrichten, damit GameMaker Studio 2 wissen kann, wo Sie nach den verschiedenen Tools suchen müssen, die zum Erstellen des endgültigen ausführbaren Pakets und zum Testen Ihres Spiels erforderlich sind. Dies ist in drei Abschnitte für Android SDK, Android NDK und Java JDK unterteilt. Diese Pfade sollten für Sie automatisch ausgefüllt worden sein. Sollten Sie jedoch Probleme haben, können Sie auf die Schaltfläche rechts klicken, um den Dateiexplorer zu öffnen und die Speicherorte für jedes Element manuell auszuwählen. Details dazu, welche Versionen für jedes SDK-Element erforderlich sind, finden Sie hier. Pfade sollten beim Anwenden validiert werden, und in einer Meldung wird "Gefunden" angezeigt oder Sie werden über etwaige Fehler informiert (in diesem Fall sollten Sie alle Informationen überarbeiten, um sicherzustellen, dass sie korrekt sind).

Darunter haben Sie die verschiedenen Verpackungsoptionen:

  • Führen Sie immer eine vollständige Installation von APK durch: Wenn Sie dieses GameMaker Studio 2, werden alle vorherigen Läufer Ihres Spiels von GameMaker Studio 2 sowie alle zugehörigen Dateien (z. B. *.ini-Dateien) von Ihrem Testgerät entfernt, bevor Sie eine neue Version von installieren das Spiel.

  • Install on package: Wenn diese Option markiert ist, wird eine endgültige ausführbare Datei erstellt *.apk Das Paket installiert es automatisch auf dem aktuell ausgewählten Zielgerät.

Zum Schluss kommen wir zum KeyStore. Dies ist eine Datei, die zum "Signieren" aller Amazon Fire Apps verwendet wird. Füllen Sie daher bitte alle Angaben korrekt aus und beachten Sie, dass diese Datei für das Erstellen und Aktualisieren aller Ihrer Amazon Fire Apps in der Zukunft benötigt wird Nach dem Setup wird empfohlen, einen Screenshot der verwendeten Einstellungen zu erstellen und eine Sicherungskopie der endgültigen Datei zu erstellen. Wenn Sie diese Datei verlieren Sie keine vorhandenen Spiele aktualisieren können, die auf den Google Play Store hochgeladen.

HINWEIS: Derselbe Keystore kann für Android und Amazon Fire Ziele verwendet werden.


Um eine neue Keystore-Datei zu erstellen, sind folgende Details erforderlich:

  • Dateiname: Dies ist der Name der KeyStore-Datei. HINWEIS: Dies ist nicht Ihr Name oder der Firmenname, sondern der Name der zu generierenden Datei! Wenn Sie bereits über eine Keystore-Datei aus früheren Projekten verfügen, können Sie hier auf die Schaltfläche des Dateibrowsers klicken und GameMaker Studio 2 auf die alte Keystore-Datei verweisen (oder auf die Schaltfläche Importieren klicken). In diesem Fall müssen Sie immer noch den Rest der erforderlichen Informationen ausfüllen (wie bei der ersten Erstellung der Datei), aber Sie dürfen nicht auf die Schaltfläche Generate Key Hash klicken!. Dadurch wird eine neue Keystore-Datei erstellt, die die vorherige überschreibt.

  • Common Name: Dies wäre normalerweise Ihr Name.

  • Passwort: Ihr Sicherheitspasswort für die KeyStore-Datei, die mindestens sechs Zeichen lang sein muss.

  • Alias: Dies ist ein anderer Name, der für die KeyStore-Datei verwendet werden kann und sich von dem oben eingegebenen Namen unterscheiden sollte.

  • Alias-Passwort: Ihr Alias-Sicherheitskennwort für die KeyStore-Datei, die mindestens sechs Zeichen lang sein muss.

  • Organisationseinheit: Die Abteilung innerhalb des Unternehmens, zu der Sie gehören.

  • Organisation: Der Name Ihres Unternehmens.

  • Ort: Der Name der Stadt, in der Sie ansässig sind.

  • Land: Der Standard-Zwei-Buchstaben-Code für das Land, in dem Sie ansässig sind.

Wenn diese Informationen ausgefüllt sind und Sie damit zufrieden sind, klicken Sie auf die Schaltfläche Generate Key Hash for GameMaker Studio 2, um die erforderliche KeyStore-Datei zu erstellen. Dies kann einen Moment dauern, aber wenn es fertig ist, muss es normalerweise nicht mehr geändert werden. Beachten Sie, dass Sie auf die Schaltfläche Importieren klicken, können Sie eine bereits erstellte Schlüsselspeicher - Datei zu importieren (den Dateinamen Abschnitt, siehe oben), und Sie können auch die Schlüsselspeicher SHA1 - Hash abrufen, indem Sie die Show Key Hash - Schaltfläche klicken. Wenn Sie auf diese Schaltfläche drücken, werden zwei spezielle Sicherheits-Hash-Schlüssel generiert, die direkt mit Ihrer Keystore-Datei verknüpft sind, die einige Websites für die korrekte Funktionalität benötigen (z. B. Facebook oder Google Play für den Sha1-Hash). Wenn Sie dies benötigen, drücken Sie einfach die Taste, kopieren und fügen Sie die generierten Hash-Schlüssel ein.

Weitere Informationen zum Einrichten der erforderlichen Amazon Fire SDKs finden Sie in diesem Artikel in der YoYo Games Knowledge Base.

Die HTML5- Einstellungen haben folgende Optionen:

  • Standard- Pakettyp: Beim Erstellen eines endgültigen ausführbaren HTML5 Pakets können Sie zwischen Paket als lose Dateien wählen. Dabei werden einfach alle erforderlichen Dateien in einem Ordner generiert - oder als ZIP- Paket. Dabei wird eine komprimierte ZIP-Datei erstellt, die das Spiel enthält und alle zusätzlichen Dateien innerhalb. Standardmäßig haben Sie die Wahl, welche der beiden Optionen Sie beim Kompilieren auswählen möchten. Sie können diese Einstellung jedoch so einstellen, dass immer die eine oder die andere verwendet wird. Wenn diese Option auf " Dialog anzeigen" eingestellt ist, wird im aktuellen Dialogfenster, das beim Kompilieren angezeigt wird, bei Auswahl von "Verpackungsoption merken" diese Einstellung auf die von Ihnen gewählte Option gesetzt.

Die iOS- Einstellungen haben folgende Optionen:

  • Standard-Teamkennung: Hier können Sie Ihre Standard-Teamkennung hinzufügen , die Ihnen von Apple zugewiesen wurde. Diese Team-ID wird verwendet, wenn Ihre Spieldateien an Xcode gesendet werden, um die App zu erstellen, und Xcode kann die erforderlichen Signaturzertifikate generieren. Beachten Sie, dass diese Einstellung standardmäßig für alle Spiele, die für iOS entwickelt wurden, angewendet wird. Sie kann jedoch projektbezogen mit den allgemeinen iOS-Spieloptionen außer Kraft gesetzt werden.

  • Erstellen und Ausführen unterdrücken: Wenn Sie ein iOS-Projekt erstellen und anschließend ein endgültiges ausführbares Paket erstellen, erstellt GameMaker Studio 2 standardmäßig das Paket und versucht, es auf allen verbundenen Geräten auszuführen. Wenn Sie diese Option aktivieren, wird die Erstellung und Ausführung der App auf dem Gerät unterdrückt. Erstellen Sie einfach ein XCode-Projekt auf dem Host-Mac

  • macOS- Installationspfad: Der Installationspfad für die Projektdateien auf dem Build-Mac.


Die macOS- Einstellungen haben folgende Optionen:

  • Standard-Teamkennung: Hier können Sie Ihre Standard-Teamkennung hinzufügen , die Ihnen von Apple zugewiesen wurde. Diese Team-ID wird verwendet, wenn Ihre Spieldateien an Xcode gesendet werden, um die App zu erstellen, und Xcode kann die erforderlichen Signaturzertifikate generieren. Beachten Sie, dass diese Einstellung standardmäßig auf alle für macOS erstellten Spiele angewendet wird. Sie kann jedoch auf Projektebene in den allgemeinen macOS-Spieloptionen außer Kraft gesetzt werden.


Das Steam SDK wird für Windows, Mac OS and Ubuntu (Linux) Builds Windows, Mac OS and Ubuntu (Linux), die über die Steam Plattform verteilt werden sollen. Dieser Abschnitt enthält die Einrichtungsoptionen, die erforderlich sind, damit es in Ihren Projekten funktioniert:

  • Pfad zum Steam SDK: Der Systempfad zu dem Ort, an dem Sie das Steam SDK installiert haben. Nachdem Sie diesen Pfad zum SDK festgelegt haben, müssen Sie Steam in den Spieloptionen aktivieren und dann für jede Plattform die App-ID hinzufügen.


Die Windows Einstellungen haben folgende Optionen:

  • Visual Studio-Pfad: Mit dieser Option können Sie GameMaker Studio 2 auf die MS Visual Studio-Installation auf dem Build-PC zeigen.

  • Standard- Pakettyp: Beim Erstellen eines endgültigen ausführbaren Windows Pakets können Sie wählen, ob Sie ein Installationsprogramm (mit NSIS ) oder eine komprimierte ZIP-Datei erstellen, in der sich das Spiel und weitere Dateien befinden. Standardmäßig haben Sie die Wahl, welche der beiden Optionen Sie beim Kompilieren auswählen möchten. Sie können diese Einstellung jedoch so einstellen, dass immer die eine oder die andere verwendet wird. Wenn diese Option auf " Dialog anzeigen" eingestellt ist, wird im aktuellen Dialogfenster, das beim Kompilieren angezeigt wird, bei Auswahl von "Verpackungsoption merken" diese Einstellung auf die von Ihnen gewählte Option gesetzt.


Die Windows UWP Voreinstellungen haben folgende Optionen:

  • Visual Studio-Speicherort: Mit dieser Option können Sie GameMaker Studio 2 auf die MS Visual Studio-Installation auf dem Build-PC zeigen.

  • Standardpaketarchitektur: Beim Erstellen eines endgültigen ausführbaren Windows UWP Pakets können Sie wählen, ob Sie ein x86-, x64- oder ARM- Paket erstellen möchten, das das Spiel und alle zusätzlichen Dateien enthält. Hier können Sie einen Standardwert für das zu verwendende Paket festlegen oder " Dialog anzeigen" auswählen, um ein Fenster zu öffnen, wenn Sie auf die Schaltfläche zum Kompilieren klicken, um die Auswahl zu bestätigen. Im Allgemeinen möchten Sie x86 für Windows 10 Computer, x64 für Xbox One und ARM für Windows 10 Geräte als Ziel festlegen.

  • Windows-UWP-Tools-Speicherort: Mit dieser Option wird GameMaker Studio 2 auf den Installationspfad der UWP-Tools auf dem Build-PC aufmerksam gemacht.

  • Automatische Antwort auf das Erstellen eines Store-Pakets: Beim Erstellen eines endgültigen ausführbaren Windows UWP Pakets werden Sie gefragt, ob Sie ein Store-Paket erstellen möchten oder nicht. Sie können diese Option so einstellen, dass dieses Popup standardmäßig standardmäßig mit Ja oder Nein übersprungen wird, oder Sie können Dialog anzeigen so einstellen, dass das Fenster immer angezeigt wird, wenn Sie auf die Schaltfläche zum Kompilieren klicken.

Beachten Sie, dass Sie neben den Voreinstellungen auch die Geräte einrichten müssen, auf die einige dieser Plattformen abzielen. Andernfalls können Sie Ihre Projekte möglicherweise nicht kompilieren oder testen. Weitere Informationen finden Sie im Abschnitt zum Kompilieren und im Abschnitt zum Geräte-Manager.

Wenn Sie über eine Konsolenlizenz verfügen, können Sie diese Informationen vom YoYo Games Helpdesk erhalten. Sie müssen sich jedoch zunächst mit Ihrer Konsolen-ID-E-Mail an YoYo Games wenden und die Berechtigung zum Anzeigen der verschiedenen Abschnitte anfordern.
