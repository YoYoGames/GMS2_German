Konfigurationen

Aufgrund der Anforderungen der plattformübergreifenden Entwicklung und der Notwendigkeit, dass ein Produkt für verschiedene Clients und / oder Geräte verfügbar und angepasst werden kann, enthält GameMaker Studio 2 Konfigurationen, um die Dinge einfacher zu machen. Mit diesem System können Sie schnell zwischen verschiedenen Begrüßungsbildschirmen, Symbolen und enthaltenen Dateien wechseln und nur diejenigen testen oder exportieren, die für die Zielplattform oder den Client zu jeder Zeit relevant sind, und das alles aus demselben Basiscode.

WICHTIG! Konfigurationen sind mit der Testlizenz des Produkts nicht verfügbar.


Zum Einrichten Ihrer Konfigurationen wird empfohlen, zunächst eine "Standard" -Konfiguration vorzubereiten, indem Sie alle konfigurierbaren Elemente für das Projekt in GameMaker Studio 2, einschließlich der Symbole und Startbildschirme (über die Spieloptionen ), der enthaltenen Dateien und der Erweiterungen Sie brauchen vielleicht, und Sie können auch die Ausgabeeigenschaften für Sound-Ressourcen oder die Einstellungen für die Texturgruppe und viele andere Optionen ändern... Fast alles, was in der IDE konfiguriert werden kann, wird in der aktuell ausgewählten Konfiguration gespeichert.

HINWEIS: Mit den enthaltenen Dateien und Erweiterungen können Sie eine Zielplattform für den Export auswählen. Das sollte auch so eingestellt sein, dass es beim Erstellen individueller Konfigurationen einfach darum geht, diejenigen zu deaktivieren, die Sie nicht benötigen.


Nachdem Sie die Standardkonfiguration eingerichtet haben, können Sie dem Ressourcenbaum einen neuen hinzufügen, indem Sie mit der rechten Maustaste klicken im Konfigurationsordner und wählen Sie Hinzufügen. Sobald dies geschehen ist, können Sie die neue Konfiguration oben im Ressourcenbaum auswählen und nach der Auswahl können Sie sie so anpassen, dass sie sich mit den Konfigurationen ändern. Dafür sollten Sie nun die Spieloptionen für jede Zielplattform überprüfen und Symbole, Begrüßungsbildschirme und andere für die Konfiguration relevante Informationen hinzufügen, entfernen oder ändern sowie die Texturgruppen für jede Konfiguration einrichten. Wenn Sie für jede Konfiguration Texturgruppen festgelegt haben, müssen Sie auch alle Grafikelemente (Sprites und Hintergründe) durchlaufen und sie neu zuweisen. Jede Änderung, die Sie an diesen Einstellungen vornehmen, wird vom Konfigurationsmanager gespeichert und bei jedem Konfigurationswechsel abgerufen.

Es ist erwähnenswert, dass GameMaker Studio 2 jede gegebene Konfiguration automatisch auf eine bestimmte Zielplattform anwenden kann. Dies geschieht, indem Sie im Targets-Fenster die Konfiguration auswählen, die Sie für diese Plattform benötigen. Sobald Sie eine Konfiguration festgelegt haben, die auf diese Weise verwendet werden soll, wird jedes Mal, wenn Sie für dieses Ziel kompilieren, entweder zum Testen oder zum Erstellen eines endgültigen Asset-Pakets, die Konfiguration verwendet. Weitere Informationen zum Targets-Fenster finden Sie im Abschnitt Compilieren.
