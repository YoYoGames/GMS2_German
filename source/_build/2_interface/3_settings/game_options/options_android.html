Android Optionen


In diesem Abschnitt werden die verschiedenen verfügbaren Optionen beschrieben, die steuern, wie Ihre Android Spielprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Der erste Teil des Abschnitts " Allgemein " ist den Produktinformationen und den Build-Einstellungen gewidmet. Sie sollten die Produktinformation, die den Anzeigenamen des Projekts enthält, sowie die Paketinformationen, die aus einer Domäne, einem Unternehmen und einem Produkt bestehen, ausfüllen. Die Paketinformationen können nur aus Buchstaben von AZ und den Zahlen 0-9 bestehen, ohne Symbole oder andere Sonderzeichen, und sie dürfen auch keine Großbuchstaben enthalten. Diese Details werden dann verwendet, um eine Paket-ID mit dem Format " com.company.myappname ".

Sie müssen dann die SDK-Versionen Ziel, Minimum und Kompilieren festlegen. Die Compile SDK-Version ist die Version der API, für die das Projekt kompiliert wird. Dies bedeutet, dass Sie die in dieser Version der API enthaltenen Android API-Funktionen verwenden können. Wenn Sie beispielsweise versuchen, API 16-Funktionen zu verwenden, aber Compile SDK 15 festlegen, erhalten Sie einen Kompilierungsfehler. Wenn Sie das Kompilierungs-SDK auf 16 setzen, können Sie die App weiterhin auf einem API 15-Gerät (sowie allen anderen früheren Versionen) ausführen.

Das Ziel-SDK hat jedoch nichts damit zu tun, wie Ihre App kompiliert wird oder welche APIs Sie verwenden können. Das Target SDK gibt an, dass Sie Ihre App bis zur angegebenen Version (bis einschließlich) getestet haben. Dies ist einfach, um dem Android Betriebssystem eine Idee zu geben, wie es mit Ihrer App in Bezug auf Betriebssystemfunktionen umgehen sollte. Für alle praktischen Zwecke werden die meisten Apps Target SDK auf die neueste veröffentlichte Version der API festlegen (wie von der Compile SDK-Einstellung verwendet). Dadurch wird sichergestellt, dass Ihre App auf den neuesten Android Geräten so gut wie möglich aussieht.

Die Minimum SDK-Version ist die minimale API-Ebene, auf der Ihr Projekt ausgeführt wird. Dies ist standardmäßig auf 9 eingestellt, und im Allgemeinen ist dies in Ordnung, aber wenn Sie Erweiterungen hinzufügen, benötigen sie möglicherweise eine höhere Mindest-API-Ebene.

Der zweite Teil des Abschnitts " Allgemein " behandelt die zu erstellenden CPU- Architekturen mit den folgenden verfügbaren Optionen (Sie können alle oder nur diejenigen auswählen, die Sie benötigen, aber beachten Sie, dass jeder einzelne zur endgültigen Größe des kompilierten Programms beiträgt apk ):

  • Build für ARMV5 (standardmäßig aktiviert)
  • Build für ARMV7 (standardmäßig aktiviert)
  • Für x86 erstellen
  • Für Mips bauen

Nach dem Einrichten können Sie die verschiedenen Orientierungen auswählen, auf denen Ihr Spiel ausgeführt werden kann, um die beste Benutzererfahrung zu erzielen. Die verfügbaren Optionen sind:

  • Landschaft
  • Porträt
  • Landschaft-umgedreht
  • Porträt-umgedreht

Zum Schluss haben Sie folgende verschiedene Optionen:

  • Bluetooth / iCade-Unterstützung aktivieren: Dies wird Unterstützung für Bluetooth- oder iCade-Gamepads zu Ihrem Spielprojekt hinzufügen. Dies ist standardmäßig aktiviert.

  • Aktivierung der MOGA Gamepad-Unterstützung: Dies ermöglicht die Unterstützung von MOGA-Controllern. Beachten Sie, dass auf alle MOGA-Funktionen über eine Erweiterung zugegriffen wird. Wenn Sie diese Option zum ersten Mal aktivieren, werden Sie aufgefordert, die MOGA-Erweiterung vom Marktplatz herunterzuladen und zu installieren. Diese Option ist standardmäßig deaktiviert.

  • Lint-Code-Analyse ausführen: Wenn diese Option markiert ist, führen die Erstellungs-Tools beim Erstellen des Projekts eine Analyse des Lint-Codes aus. Dies führt zu einer gründlichen Fehlerüberprüfung und zeigt mehr Informationen im Ausgabefenster an, wenn das Projekt kompiliert wird Datei wird ebenfalls erstellt, wobei der Pfad zu der Datei am Ende des Kompilierens im Ausgabefenster angezeigt wird). Beachten Sie, dass nicht alles, was durch die Lint-Analyse als Fehler gekennzeichnet ist, für Ihr Spiel problematisch ist. Wenn Sie dies aktivieren, wird Ihr Spiel möglicherweise nicht mehr kompiliert, obwohl es kompiliert und in Ordnung ist. Im Wesentlichen dient dies nur zum Debuggen, und die Option ist standardmäßig deaktiviert.

  • Installationsort: Hier können Sie den Standardinstallationsort für Ihr Spiel festlegen. Dies kann entweder auf Automatisch gesetzt werden, in diesem Fall wird das Spiel an dem Ort installiert, den der Benutzer auf seinem Gerät ausgewählt hat, oder es kann auf Bevorzugter External eingestellt werden, der immer versucht das Spiel auf dem externen Speicher des Gerätes zu installieren.

Sie können hier auch die Schlafgrenze für Android festlegen. Diese Option bezieht sich auf das Reduzieren von Stottern, wenn Sie Ihr Spiel auf bestimmten Systemen ausführen. Wenn Ihr Spiel schneller läuft als Ihre Zimmergeschwindigkeit, "schläft" GameMaker Studio 2 für die verbleibende Zeit, aber dieser Schlaf kann sehr ungenau sein und Sie können oft länger als nötig schlafen, was dazu führt, dass Ihr Bild länger dauert verursacht Stottern. Um dies zu umgehen, können wir für kürzere Zeit schlafen und dann für den Rest der Zeit in einer engen Schleife sitzen, um es genauer zu machen - obwohl das Problem beim Sitzen in einer Schleife ist, dass es die CPU-Auslastung erhöht, was wiederum kann verursachen Sie, dass Ihre CPU-Temperaturen steigen und Ihr Gerät mehr Batterie verbraucht. Standardmäßig ist dieser Wert auf 4 gesetzt, und in 99,99% der Fälle ist dies kein Problem, aber für Low-End-Geräte oder Geräte mit vielen Hintergrundprozessen ist dies möglicherweise keine ideale Lösung und kein Wert von 5 bis 10 kann erforderlich sein. Beachten Sie jedoch, dass dies eine gerätespezifische Konfiguration ist und dass das, was auf Ihrem Build-Gerät funktioniert, möglicherweise nicht für einen anderen Benutzer geeignet ist und als solches im Zweifelsfall auf 4 gesetzt bleibt.


Hier können Sie die folgenden Details zur Anzeige Ihres Spiels mit den folgenden Optionen ändern:

  • Interpolieren von Farben zwischen Pixeln: Schaltet die Interpolation ein, die Pixel im Grunde "glättet". Für knackige Pixelgrafiken sollte es ausgeschaltet sein, aber wenn Sie nette Alpha-Mischungen und geglättete Kantengrafiken haben, ist es besser, auf sie zu bleiben. Standard ist deaktiviert.

  • Screen Color Depth: Hier kann die Farbtiefe für das Rendering auf 16 Bit oder 24 Bit eingestellt werden. Dies wirkt sich auf die Kompatibilität mit einigen älteren Geräten aus, wenn es auf 24 Bit eingestellt ist, und erhöht auch den benötigten Aschem (dies ist der gemeinsame Speicher auf Android Geräten).

  • Geräteunterstützung Diese Option legt fest, ob nur Geräte mit dedizierter GPU oder alle Geräte unterstützt werden. Wenn Ihr Spiel fortgeschrittene Zeichenfunktionen wie Alpha Blending, additive Blending, Oberflächen usw. hat, sollten Sie es auf die Geräte beschränken, die eine GPU haben, da sonst das Risiko besteht, negative Kommentare und schlechte Bewertungen für Ihr Spiel zu erhalten auf Grafikfehler oder Abstürze auf älteren Geräten.

  • Skalierung: Ihr Spiel kann so konfiguriert werden, dass der Zeichenbereich automatisch skaliert wird, um das Seitenverhältnis im Browser beizubehalten, oder Sie können auswählen, dass der Bereich gestreckt werden soll. Die Stretch-to-Fit-Option wird das Spiel im Browser nicht im Vollbildmodus anzeigen, sondern das, was gezeichnet wird, an die Leinwandgröße anpassen, wie sie im ersten Raum des Spiels definiert ist.

Schließlich gibt es noch die Möglichkeit, die Größe der Textur-Seite festzulegen. Die Standardgröße (und die meist kompatible) ist 2048x2048, aber Sie können zwischen 256x256 und 4096x4096 wählen. Es gibt auch eine Schaltfläche mit der Bezeichnung Vorschau, die die Strukturseiten für diese Plattform erzeugt und dann ein Fenster öffnet, damit Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen möchten, wie die Strukturseiten strukturiert sind und um zu verhindern, dass Strukturseiten größer (oder kleiner) als nötig sind.

HINWEIS: Beachten Sie, je größer die Texturseite ist, desto weniger kompatibel wird Ihr Spiel sein.

In diesem Abschnitt können Sie die verschiedenen Symbolbilder hinzufügen, die Ihr Projekt für die verschiedenen Geschäftsseiten und Geräte benötigt. Alle Icon-Images sollten als 24bit PNG-Dateien in den entsprechenden Größen erstellt werden.

Es ist erwähnenswert, dass GameMaker Studio 2 ein Project Image Generator- Tool besitzt, mit dem automatisch alle Icons erstellt werden können, die für die verschiedenen Zielplattformen benötigt werden, auf denen das Spiel kompiliert wird. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen.


Hier können Sie Ihrer finalen Spieldatei sowohl im Quer- als auch im Hochformat einen Begrüßungsbildschirm hinzufügen, der angezeigt wird, während das Spiel auf dem Gerät geladen wird. Dieser Bildschirm sollte eine 24-Bit-PNG-Datei sein und sollte die gleiche Größe wie der erste Raum (oder die erste Ansicht) Ihres Spiels haben. Wenn Sie möchten, dass der Begrüßungsbildschirm für eine bestimmte Zeit angezeigt wird, können Sie ihn auch hier von 0 bis 10 Sekunden einstellen (die Standardzeit von 0 bedeutet, dass er nur für die Dauer des Ladens des Assets angezeigt wird).

Eine der Funktionen von Google Play für Ihre Android Spiele ist die Lizenzierung Ihrer Spiele. Das bedeutet, dass Ihre Spiele von Google kontrolliert werden, sodass nur Nutzer, die es über ihren Markt heruntergeladen haben, in der Lage sind, das Spiel zu spielen. Auf diese Weise wird verhindert, dass illegale Kopien Ihres Produkts verteilt werden.

Um diese Funktion in Ihrem Spiel zu aktivieren, müssen Sie zunächst das Kontrollkästchen Google-Lizenz aktivieren aktivieren und dann Ihren öffentlichen Schlüssel in das Feld mit dem öffentlichen Google-Lizenzschlüssel kopieren. Diese Lizenz erhalten Sie, wenn Sie auf Ihre Google Play-Entwicklerseite gehen und dann das Spiel auswählen, das Sie bearbeiten möchten. Scrollen Sie auf der Spielprofilseite nach unten zum Abschnitt "Dienste & APIs". Dort sollten Sie den öffentlichen Schlüssel finden, der für die Aktivierung dieser Funktionalität erforderlich ist.

Wenn jemand, der nicht berechtigt ist, Ihr Spiel zu spielen, versucht, es zu spielen, wird ihm ein Popup-Meldungsfeld angezeigt, das ihn darüber informiert, dass die Lizenz nicht auf seinem Gerät gefunden werden kann, und ihnen dann die Möglichkeit geben, es erneut zu versuchen und teste es erneut oder verlasse das Spiel.

Ein weiteres Feature der Lizenzierung ist, dass Ihr Spiel dann APK-Erweiterungen verwenden kann. Dies ist eine Möglichkeit, die maximale Größe von 50 MB zu umgehen, die von Google Play (und nur von Google Play) auferlegt wird. Es wird dein Spiel in zwei Teilen erstellen: ein kleines *.apk Datei und ein größeres *.zip Datei, die alle Ihre Spielgüter enthält, die beide in den Laden hochgeladen werden. Wenn der Benutzer das Spiel zum ersten Mal ausführt, wird die Zip heruntergeladen und entpackt. Weitere Details zu dieser Option finden Sie hier.


Auf dieser Registerkarte können Sie einige der Berechtigungen ändern, die Ihr Spiel im Android Manifest anfordern kann. GameMaker Studio 2 weist die Berechtigungen automatisch nach Bedarf zu. Möglicherweise benötigen Sie jedoch eine Berechtigung für Ihr Spiel, die nicht korrekt zugewiesen wurde (z. B. wenn Sie benutzerdefinierte Erweiterungen verwenden). In diesem Fall aktivieren Sie das entsprechende Berechtigungsfeld. In der folgenden Liste sind die verfügbaren Optionen aufgeführt:

  • WRITE_EXTERNAL_STORAGE: Erlaubt Ihrem Spiel, auf externen Speicher zu schreiben.

  • READ_PHONE_STATE: Lesezugriff auf den Telefonstatus zulassen.

  • ACCESS_NETWORK_STATE: Erlaube deinem Spiel, auf Informationen über Netzwerke zuzugreifen.

  • INTERNET: Gestatten Sie Ihrem Spiel, Netzwerk-Sockets zu öffnen.

  • BLUETOOTH: Erlaubt Ihrem Spiel, sich mit gepaarten Bluetooth-Geräten zu verbinden.

  • RECORD AUDIO: Erlaubt Ihrem Spiel, Audio-Input vom Mikrofon aufzunehmen.

HINWEIS: Wenn Sie nicht sicher sind, ob Sie diese benötigen, sollten Sie sie standardmäßig GameMaker lassen und GameMaker: Studio die Berechtigungen für Ihr Spiel GameMaker.

In diesem Abschnitt können Sie verschiedene soziale Funktionen in Ihrem Android Spiel aktivieren. Folgende Optionen können gewählt werden:

  • Facebook: Wenn Sie die Facebook-Funktionen benötigen, müssen Sie dieses Kontrollkästchen aktivieren und dann die Facebook-App-ID und den Facebook-App-Namen angeben. Beachten Sie, dass auf alle Facebook-Funktionen über eine Durchwahl zugegriffen wird. Wenn Sie das Kontrollkästchen Facebook verwenden aktivieren, werden Sie aufgefordert, die Facebook-Erweiterung vom Marktplatz herunterzuladen und zu installieren.

  • Google - Push-Benachrichtigungen aktivieren: Damit Ihr Spiel Push-Benachrichtigungen verwenden kann, müssen sie zuerst hier aktiviert werden, sonst funktionieren sie nicht. Für lokale Benachrichtigungen unter Android keine "GCM Sender ID" erforderlich, Sie müssen sie jedoch hinzufügen, wenn Sie Remote-Benachrichtigungen erstellen möchten.

  • Google - Google-Dienste aktivieren: Wenn Sie Erfolge und Ranglisten für den Google App Store aktivieren möchten, müssen Sie das Kontrollkästchen Google-Dienste aktivieren aktivieren. Sobald Sie diese Option ausgewählt haben, werden Sie aufgefordert, die entsprechende Erweiterung vom Marktplatz herunterzuladen und zu installieren. Anschließend können Sie die Funktionalität mit den Funktionen der Erweiterungskarten in Ihrem Spiel codieren. Beachten Sie, dass Sie bei Verwendung von Google Play Ihre App-ID auf dieser Registerkarte hinzufügen müssen, damit die Dienste funktionieren (diese finden Sie im Google Play-Entwicklerdashboard).

  • Google - Cloud-Speicher aktivieren: Wenn Sie diese Option aktivieren, wird Google Cloud Saving aktiviert. Sie werden (wie oben) aufgefordert, die entsprechende Erweiterung vom Marketplace herunterzuladen und zu installieren.

  • Amazon - Amazon IAPs aktivieren: Wenn Sie Amazon Services verwenden, müssen Sie dieses Kontrollkästchen aktivieren. Sobald Sie dies ausgewählt haben, werden Sie aufgefordert, die entsprechende Erweiterung vom Marketplace herunterzuladen und zu installieren, und alle erforderlichen Funktionen sind als Teil des Erweiterungspakets enthalten.
