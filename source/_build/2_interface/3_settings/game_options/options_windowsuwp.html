Windows UWP Optionen


In diesem Abschnitt werden die verschiedenen verfügbaren Optionen beschrieben, die steuern, wie Ihre UWP (Universal Windows Platform) Spielprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Im allgemeinen Abschnitt der UWP-Optionen können Sie die Produktinformationen einstellen:

  • Anzeigename: Gibt den Anzeigenamen für die App an, der Benutzern angezeigt wird (diese Zeichenfolge ist lokalisierbar).

  • Paketname: Dies ist der Name des Pakets, das Sie erstellt haben, und sollte, dass für den Entwicklerschlüssel verwendet übereinstimmen, die Sie für das Spiel erstellt haben (siehe Abschnitt Verpackung, unten).

  • Name der Paketanzeige: Dies ist der "Kurzname" des Pakets und sollte Idealerweise dem Anzeigenamen entsprechen

  • Publisher-Anzeigename: Dies ist der Name Ihres Unternehmens oder der Person, die Sie als Spiele-Publisher zuordnen möchten.

  • Version: Die Versionsnummer Ihres Spiels.

Nach dem Einrichten können Sie die verschiedenen Orientierungen auswählen, auf denen Ihr Spiel ausgeführt werden kann, um die beste Benutzererfahrung zu erzielen. Die verfügbaren Optionen sind:

  • Landschaft
  • Porträt
  • Landschaft-umgedreht
  • Porträt-umgedreht

Standardmäßig werden alle diese Optionen ausgewählt, und Sie sollten diejenigen deaktivieren, die Sie nicht benötigen.

Nachdem Sie die Ausrichtungsoptionen festgelegt haben, können Sie den verschiedenen Berechtigungen, die Ihre UWP-App benötigt, die folgenden Optionen signalisieren:

  • Internetzugriff aktivieren: Kennzeichnet das Spiel als mit dem Internet verbunden. Dies ist standardmäßig deaktiviert.

  • Enable Microphone Permission: Kennzeichnet das Spiel als Zugang zum Mikrofon des Geräts. Dies ist standardmäßig deaktiviert.

  • IAP-Sandbox aktivieren: Kennzeichnet das Spiel als Verwendung von Sandbox-In-App-Käufen. Dies ist standardmäßig deaktiviert.

Schließlich haben Sie zwei Möglichkeiten, um die Ziel- und Mindestversion der Windows-Plattform festzulegen. Im Allgemeinen möchten Sie diese nicht anfassen und sollten sie auf ihren Standardwerten belassen, aber wenn Sie Build-Probleme haben (insbesondere, wenn Sie das Windows SDK-Set nicht standardmäßig installieren können, müssen Sie diese ändern zu dem, was Sie installiert haben), dann möchten Sie vielleicht diese auf andere Werte setzen. Wenn Sie sich nicht sicher sind, welche Werte festgelegt werden sollen, erstellen Sie ein neues UWP-Projekt in Visual Studio und sehen Sie sich an, wofür die Werte festgelegt werden.


Die Grafikoptionen sind diejenigen, die Sie konfigurieren sollten, um zu bestimmen, wie Ihr Spiel die Grafikkarte Ihres Ziel-UWP-Geräts verwendet. Die folgenden Optionen sind enthalten, die Sie ändern können:

  • Interpolieren von Farben zwischen Pixeln: Schaltet die Interpolation ein, die Pixel im Grunde "glättet". Für knackige Pixelgrafiken sollte es ausgeschaltet sein, aber wenn Sie schöne Alpha-Mischungen und geglättete Kantengrafiken haben, ist es besser eingeschaltet. Dies ist standardmäßig deaktiviert.

  • Cursor anzeigen: Wenn diese Option aktiviert ist, wird der standardmäßige OS-Cursor in Ihrem Spiel angezeigt. Wenn dies deaktiviert ist, dann ist, während sich die Maus über dem Spielfenster befindet, kein Cursor sichtbar, es sei denn, Sie haben einen als Teil des Spielprojekts erstellt (standardmäßig aktiviert).

  • Fullscreen starten: Wenn dies aktiviert ist, startet das Spiel im Vollbildmodus. Beachten Sie, dass dies keine sichtbaren Auswirkungen auf etwas anderes als ein Windows 10 Desktop-Gerät haben kann.

  • Fullscreen-Umschaltung zulassen: Wenn dies aktiviert ist, können Sie das Spiel zwischen den Vollbild- und Fenstermodi mit den entsprechenden Tastenkombinationen wechseln. Beachten Sie, dass dies keine sichtbaren Auswirkungen auf etwas anderes als ein Windows 10 Desktop-Gerät haben kann.

  • Verwenden Sie die Synchronisierung, um ein Einreißen zu vermeiden: Diese Option ist vorhanden, wurde aber aufgrund der Tatsache, dass die UWP-Plattform immer aktuell aktiviert sein muss, deaktiviert.

  • Skalierung: Hier können Sie das Seitenverhältnis beibehalten (so dass ein 4: 3-Raum auf einem 16: 9 "Letterboxed" ist) oder vollständig skalieren (das Bild auf den gesamten Bildschirm ausdehnen).
    WARNUNG! Wenn Sie die Oberfläche der Anwendung ausschalten, werden alle in den UWP-Spieloptionen festgelegten Skalierungsoptionen deaktiviert, bis sie wieder eingeschaltet werden. Weitere Informationen finden Sie unter Die Anwendungsoberfläche.

Schließlich gibt es noch die Möglichkeit, die Größe der Textur-Seite festzulegen. Die Standardgröße (und die meist kompatible) ist 2048x2048, aber Sie können zwischen 256x256 und 8192x8192 wählen! Es gibt auch eine Schaltfläche mit der Bezeichnung View, die die Textur-Seiten für diese Plattform generiert und dann ein Fenster öffnet, damit Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen möchten, wie die Strukturseiten strukturiert sind und um zu verhindern, dass Strukturseiten größer (oder kleiner) als nötig sind.

HINWEIS: Je größer die Texturseite ist, desto weniger kompatibel ist dein Spiel auf UWP-Geräten mit niedrigeren Spezifikationen.

Der erste Teil des Images- Bereichs dient zur Definition des Spiels Logo. Dies ist im Wesentlichen das Symbol, das Ihr Spiel innerhalb des UWP-Ökosystems verwendet, und es ist erwähnenswert, dass diese Bilder transparent sein können .png Dateien, in diesem Fall werden sie über die in diesem Abschnitt angegebene Hintergrundfarbe gezeichnet. Diese Hintergrundfarbe wird auch auf die Schaltflächenfarbe in allen App-eigenen Dialogfeldern und auf die App-Beschreibungsseite im Store auf Geräten angewendet.

Nach dem Einrichten des Logos können Sie den Begrüßungsbildschirm für das Spiel einstellen. Dies ist der Bildschirm, der beim Starten des Spiels angezeigt wird. Er darf nicht größer als 620 x 300 Pixel sein und wie das Logo transparent sein .png Datei, in welchem Fall die Hintergrundfarbe Sie shoose wird dafür verwendet werden.

Schließlich haben Sie die Möglichkeit, die verschiedenen Kacheln einzustellen, die Ihr Spiel auf den verschiedenen Geräten verwenden wird, auf denen UWP laufen kann. Sie können Folgendes festlegen:

  • Das kleine Kachelbild ist das Logo, das neben dem App-Namen in der Suchleiste und anderen Bereichen der Benutzeroberfläche angezeigt wird. Es sollte 44 x 44 Pixel sein.

  • Das mittlere Kachelbild sollte ein 71x71 Pixel Bild sein.

  • Das nächste Kachelbild ist die Store-Logo-Kachel und sollte 50 x 50 Pixel groß sein.

  • Die breite Bildkachel ist diejenige, die erscheint, wenn die Kachel in ihrem breiten Format ist und 310 × 150 Pixel groß sein sollte. Wenn dieses Bild nicht bereitgestellt wird, kann die Kachel nur im quadratischen Format angezeigt werden und kann keine Benachrichtigungen basierend auf umfangreichen Vorlagentypen akzeptieren. Da der Benutzer entscheiden kann, welches Format die Kachel verwendet, wird empfohlen, eine breite Bildkachel aufzunehmen. Beachten Sie außerdem, dass die Kachel anfangs in ihrem breiten Format angezeigt wird, wenn eine breite Bildkachel bereitgestellt wird.

  • Die große Bildkachel ist das große Live-Kachelformat und sollte 310 x 310 Pixel groß sein. Beachten Sie, dass dies nur auf Desktop-Betriebssystemen verwendet wird.

Es ist erwähnenswert, dass GameMaker Studio 2 ein Project Image Generator- Tool enthält, mit dem automatisch alle Bilder erstellt werden können, die für die verschiedenen Zielplattformen benötigt werden, auf denen das Spiel kompiliert wird. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen.


GameMaker Studio 2 bietet einen Standard- Entwicklerschlüssel für die WindowsUWP-Entwicklung, aber es wird empfohlen, dass Sie hier Ihre eigenen erstellen und darauf verweisen. Erstellen Sie dazu ein leeres Projekt in Visual Studio Express, und speichern Sie eine sichere Kopie des automatisch generierten Objekts *.pfx Datei, die in das leere Projekt aufgenommen wurde (dies ist der Entwicklerschlüssel). Sie sollten das dann verlinken *.pfx Datei hier.

Nach dem Verknüpfen mit der Datei sollten Sie auf die zugehörige Schaltfläche Installieren klicken und dann den Anweisungen in der Eingabeaufforderung folgen, die geöffnet wird. Wenn Sie dies nicht tun, wird die Verwendung der Run / Debug-Schaltflächen (oder das Drücken von F5) fehlschlagen und Ihr Spiel kann nicht auf der UWP-Plattform getestet werden (wenn Sie jedoch Create Executable verwenden, wird der Zertifizierungsinstallationsprozess automatisch gestartet *.pfx muss noch registriert werden).

HINWEIS: Wenn es darum geht, Ihre App in den Laden zu laden, müssen Sie die richtige Lösung erhalten *.pfx für deinen Titel. Dies geschieht durch Verknüpfen Ihrer App mit dem Windows Store in Visual Studio, wodurch Visual Studio ein neues generiert *.pfx speziell für die App. Es ist auch erwähnenswert, dass Sie jede Anwendung mit dem Windows Store in Visual Studio verknüpfen können, was bedeutet, dass Sie ein leeres Projekt mit der Windows Store App verknüpfen und einfach das generierte kopieren können pfx an einen sicheren Ort, der dann innerhalb der UWP Spieloptionen verwendet werden kann. Anweisungen dazu finden Sie in der YoYo Games Knowledge Base.

Im Xbox Live- Bereich der UWP-Spieloptionen können Sie Ihr UWP-Projekt für die Verwendung auf der Xbox One-Konsole aktivieren. Wenn Sie das Projekt für die Xbox hier aktivieren, können Sie Folgendes tun, wenn das Projekt auf einer Xbox One ausgeführt wird:

  • Melden Sie sich über die App bei Xbox Live an

  • Verwenden Sie die Xbox Live-Benutzeridentität in irgendeiner Weise im Projekt (z. B. das Gamertag- oder Gamer-Bild)

Bevor Sie diese Dinge jedoch tun können, müssen Sie sicherstellen, dass Sie das Kontrollkästchen Xbox Live aktiviert haben, und wenn Sie Teil des Microsoft Creators-Programms sind, sollten Sie auch die Creators App- Option überprüfen. Wenn Sie Teil des Creators-Programms sind, müssen Sie die Titel-ID und die Dienstkonfigurations-ID hinzufügen (weitere Informationen hierzu finden Sie bei Microsoft ).

Beachten Sie, dass Sie zum Testen der Xbox Live-Funktionalität in Ihrem Projekt " Create Executable" statt " Run" verwenden müssen, da die UWP Xbox-Ausgabe das Windows Store-Signaturzertifikat verwenden muss, das Sie als Teil der Packaging- Informationen (siehe oben) angeben das temporäre, das mit GameMaker Studio 2 geliefert wird. GameMaker Studio 2 wird verwendet, wenn Sie Run drücken.

Sobald Sie XBox Live aktiviert haben, können Sie die XBox Live Funktionen in Ihrem Projekt verwenden.
