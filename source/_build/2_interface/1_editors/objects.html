Der Objekteditor


Objekte sind die Ressource, die wir verwenden, um Aspekte eines Spiels zu kontrollieren und bestimmte Dinge zu tun. Meistens ist ihnen ein Sprite zugeordnet, so dass Sie sie in den Spielräumen sehen können, aber manchmal werden sie als Controller "hinter den Kulissen" verwendet, um Dinge zu erledigen, die sich auf den Benutzer beziehen, oder für das Timing usw. Ihnen können Verhaltensweisen gegeben werden, die auf bestimmte Ereignisse sowie aufeinander reagieren können. Die meisten Dinge, die Sie in einem Spiel sehen, basieren auf Objekten und deren Interaktionen. Jetzt sagen wir "basierend auf", weil Sie keine Objekte direkt in den Spielräumen platzieren, sondern Instanzen dieser Objekte, die im Wesentlichen Kopien (oder Klone, wenn Sie möchten) der Objektressource sind. Dies ist eine sehr wichtige Sache, die Sie sich merken sollten, da Instanzen und Objekte nicht dasselbe sind und jeder seine eigenen Funktionen hat, die sich darauf auswirken können.

Um dies zu verdeutlichen, stellen Sie sich ein Spiel vor, das mit GameMaker Studio 2 erstellt wurde. Alle Charaktere, Monster, Bälle, Wände usw. sind Objekte, die Sie im Ressourcenbaum erstellen. Anschließend ziehen Sie das Objekt in einen Raum im Raumeditor, um eine Instanz zu erstellen, die aus der Basisobjektvorlage kopiert wird. Die Instanz im Raum kann dann im Editor selbst oder durch Code geändert werden, sodass sie skaliert, farbig oder durch das Objekt "base" mit anderen Details modifiziert werden kann. Wenn wir also über etwas sprechen, das eine Instanz beeinflusst oder ändert, meinen wir, dass es sich um eine bestimmte Kopie eines Objekts in einem Raum handelt. Nur diese einzige Instanz ist betroffen, der Rest nicht. Wenn wir jedoch über das Beeinflussen oder Ändern eines Objekts sprechen, meinen wir, dass wir das Objekt im Ressourcenbaum ändern und alles, was wir dafür tun, wird sich auch in allen Fällen widerspiegeln, die an diesem Punkt erstellt wurden. Das Objekt ist also die Vorlage für die Instanz, und die Instanz ist das, was wir in einem Raum platzieren, um unser Spiel zu erstellen.

Um ein Objekt in Ihrem Spiel zu erstellen, können Sie mit der rechten Maustaste klicken Klicken Sie im Objektressourcenordner auf die Option Erstellen. Dadurch wird das oben auf dieser Seite abgebildete Fenster mit den folgenden Abschnitten geöffnet:

Hier können Sie Ihrem Objekt einen Namen geben. Dieser Name sollte weder Leerzeichen noch unberechtigte Zeichen enthalten (nur alphanumerische Zeichen und der Unterstrich "_" sind zulässig) und kurz, aber einprägsam sein, damit Sie später Ressourcen leicht identifizieren können, wenn Sie über viele verfügen. Zum Beispiel verwenden viele Personen ein Präfix oder Suffix, um die Unterschiede zwischen Ressourcen zu identifizieren, wie "obj_Ship" oder "Ship_obj".

Als Nächstes können Sie das Sprite für das Objekt auswählen, falls Sie eines haben möchten. Klicken Sie mit der linken Maustaste In der Sprite-Box und im Asset Explorer wird eine Liste aller verfügbaren Sprites aus dem Ressourcenbaum angezeigt:

Sie können dann das Objekt auswählen, das Sie für das Objekt verwenden möchten. Wenn dem Objekt ein Sprite zugewiesen ist, können Sie auf die Schaltfläche Sprite bearbeiten klicken Um die Sprite-Eigenschaften zu ändern, oder wenn Sie kein Sprite für das Objekt haben, aber eines erstellen möchten, klicken Sie auf die Schaltfläche Sprite hinzufügen um eine neue Sprite-Ressource zu erstellen und diese zuzuweisen. Sie können das zugewiesene Sprite-Ressourcenbild auch bearbeiten, indem Sie auf die Schaltfläche Bild bearbeiten klicken um den Bildeditor zu öffnen.


Unter dem Sprite-Zuweisungsbereich haben Sie eine Schaltfläche zum Auswählen einer bestimmten Kollisionsmaske. Standardmäßig verwendet ein Objekt die Kollisionseigenschaften des ihm zugewiesenen Sprites. In einigen Fällen möchten Sie jedoch, dass Kollisionen auf einer anderen Maskenform basieren. Beispielsweise haben Sie möglicherweise ein Top-Down-Spiel, bei dem das Haupt-Sprite eine rechteckige Kollisionsmaske hat. Um Kollisionen jedoch glatter zu gestalten, ist eine kreisförmige Maske möglicherweise besser. In diesen Fällen können Sie die Kollisionsmaske Schaltfläche klicken und ein anderes Sprite Ressource auswählen, so dass nun Instanzen des Objekts wird die zugewiesene Sprite, sondern Basis Kollisionen aus der Maske Sprite zeichnen.

Sichtbar gibt an, ob Instanzen dieses Objekts sichtbar sind, wenn der Raum beginnt. Normalerweise sind die meisten Fälle sichtbar, aber manchmal ist es nützlich, unsichtbare Fälle zu haben. Sie können sie beispielsweise als Wegpunkte verwenden, um ein sich bewegendes Monster zu steuern oder die Werte nachverfolgen und bestimmte zeitgesteuerte Aktionen ausführen. Unsichtbare Objekte reagieren immer noch auf Ereignisse. Wenn ihnen ein Sprite oder eine Maske zugewiesen ist, reagieren sie auch, wenn andere Instanzen mit ihnen kollidieren. Sie können einfach nicht gesehen werden und führen kein Zeichnungsereignis aus. Das sichtbare Flag ist standardmäßig aktiviert.


Wenn Sie eine Instanz als solide GameMaker Studio 2, teilen Sie GameMaker Studio 2 dass sie versuchen sollte, Kollisionen zu lösen, bevor Sie ein Kollisionsereignis auslösen. Dies ist eine sehr grundlegende Funktionalität, und in der Realität muss die Instanz nur in die Position zurückbewegt werden, in der sie vor dem Erkennen einer Kollision erkannt wurde, bevor Code oder DnD™ ausgeführt wird, die sich auf das Ereignis selbst beziehen begrenzte Verwendung.


Persistent - Ein persistentes Objekt ist ein Objekt, das beim Ändern des Raums nicht verschwindet, sondern "persistiert" und in den neuen Raum übertragen wird. Es wird nur verschwinden, wenn Sie es explizit zerstören, entweder mit einer Zerstörungsaktion oder im Code. Das bedeutet, dass, wenn Sie ein persistentes Objekt in einem Raum platzieren, dieses weiterhin in allen nachfolgenden Räumen verfügbar ist und weiterhin alle Aktionen ausführt, die den Ereignissen in den Objekteigenschaften zugewiesen sind, mit Ausnahme des Erzeugungsereignisses, das nur einmal ausgelöst wird wird anfangs erstellt und nicht, wenn Sie in einen neuen Raum wechseln.

Dies ist praktisch, wenn Sie beispielsweise eine Hauptfigur haben, die sich von Raum zu Raum bewegt und Sie die Variablen in diesem Objekt gleich beibehalten möchten. Sie sollten jedoch sehr sorgfältig darauf achten, persistente Objekte korrekt zu erstellen und zu löschen, da dies problemlos möglich ist Fehler erhalten, die sich aus Versehen in Ihr Projekt einschleichen.

Beachten Sie auch, dass bei einem persistenten Objekt weiterhin die Ereignisse Spielstart, Spielende, Raumstart und Raumende ausgelöst werden. Wenn Sie das Spiel jedoch neu starten (z. B. mit der Funktion game_restart ()), werden nur die persistenten Objekte entfernt existieren, wenn sie vom Spiel erneut erstellt werden. Beachten Sie außerdem, dass das Deaktivieren eines persistenten Objekts nicht mehr von einem Raum in einen anderen erfolgt, wenn es nicht erneut aktiviert wird, bevor das Room End-Ereignis ausgelöst wird.


Wenn Sie das Kontrollkästchen " Physik verwenden" aktivieren, teilen Sie GameMaker Studio 2 dass dieses Objekt Teil einer physischen Welt sein sollte. Wenn Sie dies aktivieren, wird auch ein weiteres verkettetes Fenster geöffnet, in dem Sie die physikalischen Eigenschaften aller Instanzen dieses Objekts definieren können. Weitere Informationen hierzu finden Sie weiter unten auf dieser Seite im Bereich Physik.

Was sind also Ereignisse? Grundsätzlich handelt es sich dabei um diskrete Momente in der Spielschleife, in denen die Dinge auf der Grundlage der für Sie programmierten Dinge geschehen. Nachdem Sie Ihre Form eingerichtet haben, müssen Sie die grundlegenden physikalischen Eigenschaften definieren, die Ihr Objekt haben soll. Diese werden durch Ändern der folgenden Parameter konfiguriert:

Die Dichte von etwas wird als Masse pro Volumeneinheit definiert, was im Wesentlichen bedeutet, wie viel Masse in den Weltraum eingepfercht wird. Ein Ballon hätte also eine geringe Dichte, da er eine geringe Masse hat, die einen großen Raum einnimmt, während eine Führungsstange eine hohe Dichte haben würde, da sie eine große Masse hat, jedoch wenig Platz beansprucht. In der 2D-Physik-Welt von GameMaker Studio 2 wird die Masse automatisch aus den von Ihnen für die Dichte eingegebenen Werten und der Oberfläche der Form berechnet, die Sie für das Fixture definieren. Dies hat einen direkten Einfluss auf die Trägheit einer Vorrichtung sowie darauf, wie sie auf Stöße und Kräfte reagiert. Wenn Sie also eine kleine Form mit hoher Dichte herstellen, hat sie eine sehr große Masse (wie ein Bleibalken). Wenn Sie jedoch eine große Form mit geringer Dichte definieren, hat sie eine viel geringere Masse (wie ein Ballon).


In der Physik wird Restitution definiert als " die Rückkehr eines Objekts oder Systems in seinen ursprünglichen Zustand nach elastischer Verformung ", aber da die Befestigungen in GameMaker Studio 2 wirklich starre Körper sind und nicht deformiert werden können, ist Restitution eine Art zu sagen, wie "Bouncy" ist das Fixture. Diese Einstellung wirkt sich darauf aus, wie stark ein Objekt "springt", wenn es mit anderen Objekten kollidiert, und ist abhängig von anderen Kräften, die auf die Instanz wirken, wie beispielsweise Schwerkraft und Reibung.


Standardmäßig ist die Kollisionsgruppe für alle Physikobjekte 0, was bedeutet, dass sie wie gewohnt interagieren, was wiederum bedeutet, dass sie ein Kollisionsereignis haben müssen, um interagieren zu können (ein einfacher Kommentar löst die Kollision aus), andernfalls wird keine Kollision auftreten angemeldet sein. Sie können jedoch auch angeben, dass ein Objekt zu einer bestimmten Kollisionsgruppe gehören soll. Durch das Zuweisen von Objekten zu einer positiven Kollisionsgruppe (dh, drei Objekte befinden sich alle in der Gruppe " 2 "), teilen Sie GameMaker Studio 2 mit, dass Instanzen dieser Objekte immer kollidieren sollten, selbst wenn sie keine Kollisionsereignisse aufweisen, und dabei den physikalischen Eigenschaften folgen Wenn Sie Objekte mit einer negativen Kollisionsgruppe haben (z. B. vier Objekten, denen " -1 " zugewiesen ist), teilen Sie GameMaker Studio 2 dass diese niemals kollidieren sollen, und auch keine Kollision Ereignisse zwischen ihnen werden ignoriert.

ANMERKUNG: Durch die Verwendung von Kollisionsgruppen wird die vom Physiksystem erforderliche Verarbeitung drastisch erhöht und sollte nur dann verwendet werden, wenn es absolut notwendig ist. Sie sollten so wenige Gruppen wie möglich haben.

Dämpfung wird verwendet, um die Weltgeschwindigkeit der Physik einer Instanz zu reduzieren, und unterscheidet sich von Reibung dadurch, dass Reibung nur auftritt, wenn zwei Instanzen mit Kollisionsformen (Vorrichtungen) in Kontakt sind. Dämpfung ist auch viel billiger zu simulieren als Reibung. Beachten Sie jedoch, dass Dämpfung kein Ersatz für Reibung ist. Die beiden Effekte können und sollten zusammen verwendet werden.


Wenn Sie an ein sich drehendes Objekt in der "realen Welt" denken, wird es mit der Zeit aufgrund des Einflusses äußerer Kräfte (z. B. Reibung mit der umgebenden Luft) langsamer, wenn es keinen Motor hat oder sich im Weltraum befindet. Wir können set verwenden, um diesen Effekt zu simulieren und die Rotationsgeschwindigkeit von Instanzen in der Welt der Physik zu reduzieren, da sich sonst jede rotierende Instanz unendlich weit drehen würde.


Reibung ist die Kraft, die sich der relativen Bewegung von gegeneinander gleitenden Elementen widersetzt, was in der GameMaker Studio 2 als Impulsverlust übersetzt wird, der durch die Kollision zweier Instanzen mit daran gebundenen Fixtures verursacht wird. Wenn also zwei Fälle kollidieren, wird ihre Bewegung durch diesen Wert beeinflusst, wobei eine hohe Reibung einen größeren Impulsverlust als einen niedrigeren Wert verursacht.

Wenn Sie dieses Kontrollkästchen aktivieren, weisen Sie GameMaker Studio 2 an, die physischen Eigenschaften des Objekts im Wesentlichen zu "ignorieren", aber dennoch ein Kollisionsereignis mit den Objekten in der Umgebung zurückgeben. Auf diese Weise können Sie eine Instanz ohne physische Präsenz im Raum haben, die jedoch auf die Kollision einer anderen Instanz wie dem Spieler reagieren und etwas tun kann, beispielsweise eine Tür öffnen, oder eine Aktion in einer anderen auslösen Platz im Raum.

HINWEIS: Ein Sensor-Fixture löst das Kollisionsereignis nur beim ersten Auftreten der Kollision aus. Dies bedeutet, dass keine Kollisionsereignisse auftreten, da sich die beiden Körper weiterhin überlappen (was normalerweise der Fall wäre). Wenn sie sich nicht mehr überlappen und anschließend überlappen, wird ein anderes Kollisionsereignis ausgelöst.

Dies ist standardmäßig aktiviert und stellt den anfänglichen "Zustand" des Geräts am Anfang des Raums dar, in dem es aufgestellt wurde. Normalerweise möchten Sie, dass es wach ist (dh an der Physik-Welt teilnimmt), aber von Anfang an Dies kann aufgrund des abrupten Starts der Physiksimulation zu bestimmten Instabilitäten und unerwünschten Effekten führen. Um diese Effekte zu vermeiden, können Sie dieses Flag deaktivieren. Die Instanz wird mit dem schlafenden Gerät erstellt und nimmt nicht an der Physiksimulation teil, es sei denn, etwas wird davon beeinflusst.


In einem auf Physik basierenden Spiel gibt es bestimmte Fälle, in denen Sie sich bewegen möchten, aber nicht von Kräften wie der Schwerkraft oder Kräften, die durch Kollisionen mit dynamischen Objekten verursacht werden (z. B. Bewegungsplattformen in einem Plattformspiel) ). Wenn Sie für solche Objekte einfach die Dichte des Geräts auf 0 setzen, bedeutet dies, dass die Physik davon ausgeht, dass das Objekt statisch sein soll, und dass es auf nichts reagiert. Wenn Sie dieses Kontrollkästchen aktivieren, wird eine statische Objektkinematik erstellt. Obwohl Kollision und Schwerkraft sie nicht beeinträchtigen, kann sie mit den entsprechenden Variablen immer noch verschoben oder gedreht werden.

Die verschiedenen Variablentypen, die Sie erstellen können, sind unten aufgeführt:

  • Real: Eine reelle Zahl ist eine beliebige Zahl, die keine ganze Ganzzahl ist und positiv oder negativ sein kann. 124.5, 45639.566546456, 0.9, -45.5 usw. sind Beispiele für reelle Zahlen. Alle reellen Zahlen werden als 64-Bit-Gleitkommawerte (Doppelwerte) gespeichert. Beachten Sie, dass Sie in den Variablenoptionen einen Wertebereich für die Ausgabe festlegen können (weitere Einzelheiten siehe unten).

  • Ganzzahl: Eine Ganzzahl ist eine ganze Zahl und kann positiv oder negativ sein, z. B. 30004, 19, 0, -300. Beachten Sie, dass Sie in den Variablenoptionen einen Wertebereich für die Ausgabeganzzahl festlegen können (weitere Einzelheiten siehe unten).

  • String: Ein String ist alles, was in doppelten Anführungszeichen gesetzt wurde, zum Beispiel „Fisch“, „Hallo Welt“ oder „12345“.

  • Boolean: Ein Boolean ist entweder ein Wert true oder false. Im Fenster Objekt-Editor-Variablen wird dies einfach als Feld angezeigt, nach dem Sie suchen true und deaktivieren Sie für false.

  • Ausdruck: Ein Ausdruck ist ein mathematischer Ausdruck, der gewöhnliche Zahlen, Variablen, Strings oder GML-Funktionen sowie einen oder mehrere Operatoren enthalten kann. Zum Beispiel sqrt(85 * 6) + 5.5 ist ein Ausdruck.

  • Ressource: Eine Ressource ist einfach eine der Ressourcen, die Sie im Ressourcenbaum definiert haben. Wenn Sie diesen Ressourcentyp auswählen, können Sie auf den Open Asset Explorer klicken , um den Asset Explorer zu öffnen und die gewünschte Ressource auszuwählen. Beachten Sie, dass Sie Filter für die im Asset Explorer angezeigten Assets über die variablen Optionen festlegen können (weitere Einzelheiten siehe unten).

  • Liste: Wenn Sie einen Listentyp für die Eingabe auswählen, können Sie eine Auswahl von Werten erstellen (dies können Strings, Reals, Ausdrücke usw. sein), von denen Sie einen oder mehrere Werte auswählen können, die von der Variablen zurückgegeben werden sollen. Um die Elemente zu definieren, die in der Liste enthalten sind, müssen Sie sie zuerst in den Variablenoptionen hinzufügen (weitere Informationen siehe unten). Anschließend können Sie das Element als Standardwert für die Variable im Hauptfenster aus dem Drop-Fenster auswählen Menü unten: Wenn Sie in den Listenoptionen (siehe unten) die Option Mehrfachauswahl aktiviert haben, wird die Variable zu einem Array aller ausgewählten Optionen.

  • Farbe: Mit dem Farbtyp können Sie einen Farbwert definieren, der in der Variablen gespeichert werden soll. Sie können eine reelle Zahl (von 0 bis 16777216, dies hat einen Alphawert von 255) oder einen Hexadezimalwert (im Format $ RRGGBBAA) eingeben, oder Sie können doppelklicken Öffnen Sie auf dem Farbfeld den Farbwähler und legen Sie dort die Farbe fest.
