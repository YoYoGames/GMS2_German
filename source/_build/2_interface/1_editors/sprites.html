Der Sprite-Editor


Im Sprite-Editor legen Sie die allgemeinen Eigenschaften für jede Sprite-Ressource fest, die Sie erstellen. Sie können dieses Fenster öffnen, indem Sie ein neues Sprite erstellen (Rechtsklick) Klicken Sie im Sprite-Ressourcenordner auf "Erstellen" und doppelklicken Sie darauf auf einem vorhandenen Sprite oder durch Klicken mit der rechten Maustaste ein vorhandenes Sprite und wählen Sie "Eigenschaften". Die Teile, aus denen der Sprite-Editor besteht, lauten wie folgt:

Hier können Sie das Sprite benennen. Der Sprite-Name wird in Ihrem gesamten Spielprojekt verwendet, um das Sprite als Ressource zu identifizieren, und kann nur alphanumerisch sein und darf nur das Symbol "_" verwenden. Während Sie im Sprites-Namen Zahlen verwenden können, darf der Name nicht mit einer Zahl beginnen, da dies bei Verwendung von GML oder DnD™ zu einem ungültigen Ressourcennamen würde.


In diesem Bereich können Sie die Bildgröße festlegen, indem Sie auf die Schaltfläche Sprite Size klicken . Dies öffnet das folgende Fenster:

Hier sehen Sie eine Vorschau des Sprites vor und nach dem Anwenden der ausgewählten Transformationen, und der folgende Abschnitt dient zum Festlegen der neuen Sprite-Eigenschaften und ist in zwei Teile aufgeteilt:

  • Bild skalieren: Mit dieser Option können Sie die Größe des Sprite-Zeichnungsbereichs ändern und alle auf dem Sprite enthaltenen Bilder werden gestreckt und skaliert, um sie an die neue Größe anzupassen. Sie können wählen, ob das Seitenverhältnis des Basisbildes skaliert und beibehalten werden soll, und Sie können auch wählen, ob Sie in Pixel oder in Prozent skalieren möchten. Abhängig von der Art des Bildes, mit dem Sie arbeiten, möchten Sie möglicherweise auch die lineare Interpolation aktivieren oder nicht (im Allgemeinen wird lineare Interpolation "glättet" Pixel über einen Bereich und so ist am besten für Pixel Kunst links).

  • Leinwandgröße anpassen: Diese Option ändert die Größe der Zeichenfläche auf die neue Größe, ohne den Inhalt zu skalieren. Sie können den Ankerpunkt für die Größenänderung mit den Richtungspfeilen festlegen und dann den Größenwert festlegen. Es gibt eine Option zum Beibehalten des Seitenverhältnisses bei der Größenänderung der Zeichenfläche und unten können Sie die Größenänderungswerte in Pixeln oder als Prozentsatz festlegen.

Wenn Sie mit der neuen Größe zufrieden sind, können Sie auf Übernehmen klicken, um die Änderungen zu übernehmen und das Fenster Größenänderungseigenschaften zu schließen.

HINWEIS: Diese Option ist nur für Bitmap-Sprites verfügbar und wird für SWF- oder Skeleton-Animations-Sprites ausgegraut.

In diesem Bereich können Sie entweder auch öffnen Image Editor auf Ihrem Sprite zu zeichnen, oder Sie können auf die Schaltfläche „Importieren“ klicken, um ein zuvor erstelltes Sprite zu importieren. Sprites können aus den gängigsten Bitmap-Formaten importiert werden ( PNG, JPG, GIF usw., aber GameMaker Studio 2 unterstützt auch Vektorsprites (SWF) und Skelettanimationsprites (Spine). Weitere Informationen zu diesen Formaten finden Sie im Abschnitt Importieren von Nicht-Bitmap-Sprites. Beachten Sie, dass beim Importieren von Bitmap-Sprites mehrere Bilddateien im Dateiexplorer ausgewählt werden können und diese jeweils demselben Sprite als separate Teilbilder (Frames) hinzugefügt werden.


In diesem Abschnitt des Sprite-Eigenschaftenfensters wird erläutert, wie GameMaker Studio 2 die Bilder speichert, aus denen Ihr Sprite besteht, auf Texturseiten zur Verwendung mit Geräten und Browsern. Für Desktop-Plattformen ist dies in der Regel nicht so wichtig, aber wenn Sie mit der Entwicklung für Mobilgeräte oder Web beginnen, wird die ordnungsgemäße Verwaltung Ihrer Image-Assets (Texturen) sehr wichtig, da schlecht verwaltete Texturen nachteilige Auswirkungen auf Ihr Spiel haben können.

Die Kontrollkästchen Kachel: Horizontal und Kachel: Vertikal sind standardmäßig nicht so oft aktiviert, wie Sie Sprites nicht kacheln möchten. Unter bestimmten Umständen möchten Sie jedoch, dass sie kacheln, was bedeutet, dass Sie diese Optionen überprüfen sollten, insbesondere wenn Sie die Ansicht oder den Raum skalieren, da Skalierung Artefakte in die Grafik eines Spiels einbringen kann, wenn die Texturseite nicht generiert wird richtig.

Wenn Ihr Sprite als Textur verwendet wird, sollten Sie das Kontrollkästchen Separate Textur-Seite aktivieren und das Sprite erhält eine eigene Textur-Seite (beachten Sie, dass jedes Teil-Bild auch eine eigene Textur-Seite erhält, Wenn dein Sprite also 10 Bilder hat, wirst du 10 Texturseiten erzeugen. Dies wird die Texturspeicheranforderungen Ihres Spiels enorm erhöhen, und daher muss bei der Verwendung dieser Option sehr vorsichtig vorgegangen werden. Beachten Sie auch, dass im Allgemeinen, wenn Ihr Sprite diese Option für 3D verwendet, die Texturen eine Potenz von 2 haben sollten (dh> 128x128, 256x256, 512x512 etc...).

Als nächstes können Sie die Texturgruppe auswählen, zu der die Sprite-Ressource gehören soll. Grundsätzlich können Sie eine Texturgruppe (zuvor im Texturgruppen- Manager definiert ) so einrichten, dass alle Bildressourcen, die Sie für bestimmte Räume oder Ebenen Ihres Spiels benötigen, zusammen gespeichert werden können. So können Sie zum Beispiel alle Ihre Bilder der Ebene 1 in einer Texturgruppe, alle Bilder der Ebene 2 in einer anderen usw. haben und GameMaker Studio 2 wird versuchen, alle diese gruppierten Ressourcen auf der gleichen Texturseite zu platzieren, um die Texturseite zu reduzieren Tauschen Sie, während Ihr Spiel auf der gewählten Zielplattform läuft.

ANMERKUNG: Dies ist möglicherweise nicht immer notwendig und die Leistungssteigerung von dieser Methode hängt davon ab, ob das Zielgerät CPU-gebunden oder GPU-gebunden ist.

Die nächste Option ist die Aktivierung von vormultipliziertem Alpha für die Sprite-Bilder. Wenn diese Option aktiviert ist, wird das Alpha aller Unterbilder des Sprites vorab multipliziert. Dies ist normalerweise nur für das Zeichnen von Sprites auf Oberflächen oder für bestimmte Spezialeffekte erforderlich. Für normale Sprite-Verwendung sollten Sie keinen erkennbaren Unterschied zwischen dem normalen Sprite und dem vor-multiplizierten sehen.

Endlich haben wir Kantenfilterung. Diese Option ist speziell, wenn Sie lineare Interpolation für Ihr Spiel aktiviert haben und Sie "Halos" von Farbe um Ihre Sprites sehen. Dies wird durch die Hardware verursacht, die gleichzeitig von mehreren Quelltexeln auf der Texturseite interpoliert wird, und kann besonders offensichtlich sein, wenn Sie Spiele in der Auflösung skalieren. Wenn Sie dies aktivieren, werden die Kantenpixel so gefiltert, dass sie die Farbe des nächsten vollständigen Alpha-Sprite-Pixels annehmen und so die interpolierten Pixel ohne den zuvor vorhandenen Lichthof überblenden.


Alle Sprites haben eine Kollisionsmaske, die dem Bereich des Sprites entspricht, der zum Erkennen von Kollisionen verwendet wird, wenn das Sprite einer Instanz zugewiesen ist. Sie können zuerst den Modus für die Kollisionsmaske festlegen. Dies kann einer der folgenden sein:

  • Automatisch: GameMaker Studio 2 berechnet die Kollisionsmaske automatisch basierend auf den Sprite-Alpha-Werten (dies hängt von der unten beschriebenen Toleranzeinstellung ab ).

  • Vollständiges Bild: Die Maske wird so berechnet, dass ihre Grenzen den linken, rechten, oberen und unteren Grenzen des Sprites entsprechen.

  • Manuell: Wenn Sie dies auswählen, müssen Sie die Grenzen der Kollisionsmaske mithilfe der Eingabewerte für links, rechts, oben und unten festlegen.

Neben der Kollisionsmaske Mode gibt es verschiedene Arten von Kollisionsmasken, die Sie aus dem Dropdown-Menü auswählen können:

  • Rechteck: Eine einfache rechteckige Kollisionsmaske. Dies ist am schnellsten zu beheben, wenn es in Ihrem Projekt verwendet wird.

  • Ellipse: Erzeugt eine elliptische Kollisionsmaske. Dies ist langsamer zu lösen.

  • Diamant: Erzeugt eine "Diamant" -Kollisionsmaske. Dies ist langsamer zu lösen und generiert mehr CPU-Overhead.

  • Präzise: Dies erstellt eine präzise Kollisionsmaske basierend auf dem Umriss des Sprites. Wenn das Sprite mehrere Teilbilder hat, wird dies eine Zusammensetzung der Kanten aller Teilbilder sein, die übereinander angeordnet sind. Dies ist sehr langsam und erzeugt eine Menge CPU-Overhead.

  • Präzise (pro Bild): Dies erzeugt pro Frame eine präzise Kollisionsmaske (dh die Maske ändert sich so, dass sie dem Umriss jedes anderen Bildes entspricht). Dies ist bei weitem der langsamste Typ der zu lösenden Kollisionsmaske und sollte vermieden werden, außer in sehr, sehr spezifischen und seltenen Fällen.

Die Kollisionsüberprüfungsoptionen sind sehr wichtig für Ihr Spiel, da sie direkt beeinflussen, wie Ihre Objekte interagieren und wie Ihr Spiel läuft, wobei die falschen Einstellungen sogar negative Auswirkungen auf die Gesamtleistung haben. Warum das? Nun, wenn zwei Instanzen sich treffen und beide Instanzen eine gültige Maske haben, wird ein Kollisionsereignis erzeugt, indem die Überlappung der Maske überprüft wird, die entweder präzise sein kann oder nicht und an den Bildindex angepasst werden kann oder nicht. Unten ist ein Bild um dies zu verdeutlichen:

Darunter befindet sich der Alpha-Toleranz- Schieberegler, der die Toleranz für die Erstellung der Maske in Bezug auf die Transparenz der Sprite-Pixel angibt. Bei einer höheren Toleranz bleiben Pixel, die teilweise transparent sind, außerhalb der Maske, während die Maske bei einer niedrigeren Toleranz liegt wird diese Pixel berücksichtigen. Dies beeinflusst sowohl die Begrenzungsbox (wenn nicht manuell eingestellt) als auch die Maske für die präzise Kollisionsprüfung.

Schließlich können Sie die Begrenzungsrahmen- Grenzen für die Maske festlegen. Der Begrenzungsrahmen ist der Name für den rechteckigen Bereich, in den die Sprite-Maske passt. Nur Pixel innerhalb dieses Begrenzungsrahmens werden für die Maske verwendet und sind an der Kollisionsprüfung beteiligt. Standardmäßig wird dies automatisch unter Berücksichtigung der Alpha-Toleranz berechnet Konto. Wenn Sie genau überprüft haben, ist dies auch für jedes Teilbild separat. Wenn Sie es selbst einstellen, können Sie die Werte in die Felder für links, rechts oben und unten eingeben oder Sie können klicken in der Sprite-Ansicht und ziehen, um es über das Sprite "zu zeichnen". Beachten Sie, dass beim Zeichnen der Begrenzungsbox die Maske automatisch so generiert wird, dass sie in die Endwerte passt.

Es ist erwähnenswert, dass Sie, wenn Sie zum Vorschaufenster gehen, die Kollisionsmaske mithilfe der "Griffe" in jeder Ecke anpassen können. Wenn Sie auf diese klicken und ziehen, wird die Kollisionsmaske genau so angepasst, wie die Eingabe von Werten für die Bounding Box.


Die obere Leiste des Sprite-Editors enthält die Eigenschaften des Sprite-Ursprungs und die Zoom-Steuerelemente für das Vorschaufenster. Hier können Sie die Breite und Höhe des Sprites sehen und den Ursprung festlegen. Dies ist der Punkt im Sprite, der seiner Position innerhalb des Raums entspricht, dh wenn Sie eine Instanz an einer bestimmten x / y-Position erstellen, wird der Ursprung des Sprites dort platziert. Standardmäßig ist es die obere linke Ecke des Sprites, aber es ist oft bequemer, das Zentrum zu verwenden, oder Sie möchten vielleicht eine andere Position verwenden, damit Sie klicken können Wählen Sie im Dropdown-Menü Ursprung einen der Standardwerte aus. Sie können den Ursprung auch manuell festlegen, indem Sie auf klicken im Sprite-Vorschaubild, das das Kreuz an den Punkt bewegt, auf den Sie geklickt haben, oder indem Sie in die entsprechenden Felder verschiedene Werte für x und y eingeben. Beachten Sie, dass Sie auch einen Ursprung außerhalb des Sprite-Bereichs festlegen können, indem Sie negative Zahlen (für links und oben) oder positive Zahlen größer als die Breite und Höhe des Sprites (für rechts und unten) verwenden, was bei Objekten sehr hilfreich sein kann müssen zusammengesetzte Sprites zeichnen.

Oben rechts befinden sich die Zoomsteuerelemente, mit denen Sie das Vorschaubild nach Bedarf vergrößern oder verkleinern können. Drücke den Symbol, um das Bild 1: 1 mit den tatsächlichen Pixeln zu machen.


Das Sprite-Vorschaufenster zeigt das Sprite, wie es im Spiel aussehen wird, und kann mit dem Mausrad ein- oder ausgezoomt werden und herumgewirbelt mit dem oder indem man hält + und ziehen. Sie können die Frame-Steuerelemente (unten erläutert) verwenden, um zu ändern, wie das Vorschaubild angezeigt wird.


Dieser Teil des Sprite-Editors zeigt alle verschiedenen Frames an, aus denen ein Sprite besteht. Sie können klicken Klicken Sie auf einen beliebigen Rahmen und ziehen Sie ihn nach links oder rechts, um die Reihenfolge zu ändern, in der er animiert werden soll, und Sie können auch doppelklicken auf einem beliebigen Rahmen, um den Image Editor zu starten und zu bearbeiten. Ein ausgewählter Rahmen hat ein Löschen-Symbol in der oberen Ecke, und Sie können darauf klicken, um den Rahmen aus dem Sprite zu entfernen. Sie können auch klicken / + auf mehrere Frames, um sie zusammen auszuwählen. In diesem Fall werden alle ausgewählten Frames gelöscht, wenn Sie auf das Symbol Löschen klicken.


Die Frame-Steuerelemente dienen zur Wiedergabe der Animation der Frames im Vorschaufenster, wobei die Play-Schaltfläche zum Starten / Stoppen der Animation und die Loop-Schaltfläche verwendet wird verwendet, um festzulegen, ob das Sprite loops (geht zurück zu Frame 1, wenn der letzte Frame erreicht ist) oder Ping-Pongs (Geht beim Erreichen des letzten Frames rückwärts durch die Frames) und Sie können die Geschwindigkeit, mit der sie abgespielt werden sollen, über das Eingabefeld links einstellen. Die Frame-Geschwindigkeit kann entweder auf "Frames pro Sekunde" oder "Frames pro Spiel-Frame" basieren. Die Gesamtzahl der verfügbaren Bilder wird rechts neben dem aktuell ausgewählten Bild angezeigt, und Sie können mehrere Bilder auswählen, mit denen Sie Bilder auswählen können / + .
