Enthaltene Dateien


GameMaker Studio 2 ist nicht nur auf die Ressourcen beschränkt, die Sie innerhalb des Programms erstellen. Es hat einen speziellen Abschnitt im Ressourcenbaum des Hauptfensters für Included Files. Hier können Sie verschiedene Dateitypen in GameMaker Studio 2 laden (und sogar Unterverzeichnisse für sie erstellen), die dann als Teil des Pakets für die ausgewählte Zielplattform in das endgültige Spiel einbezogen werden. So werden sie beispielsweise in einem Windows - Spiel innerhalb der *.exe und exportiert, wenn sie ausgeführt wird, oder in HTML5 werden sie in die html5_game Ordner und auf dem Server gespeichert.

Um eine Datei zu Ihren Ressourcen hinzuzufügen, müssen Sie mit der rechten Maustaste klicken Klicken Sie im Abschnitt Included Files des Ressourcenbaums auf Included File einfügen oder, wenn Sie ein Unterverzeichnis zum Speichern anderer Dateien erstellen möchten, Create Group. Dadurch werden Sie aufgefordert, eine Datei oder mehrere Dateien zur Liste hinzuzufügen, wenn Sie mehrere auswählen. Sobald sie hinzugefügt wurden, können Sie mit der rechten Maustaste klicken darauf und wählen Sie Eigenschaften, um die Eigenschaften der Datei zu bearbeiten. Hier können Sie die Datei umbenennen, wenn Sie durch Ändern des Dateinamen zu wählen. Neben dem Dateinamen gibt es auch eine Reihe von Optionen für die Zielplattform, auf die diese Datei kopiert werden soll. Wenn Sie möchten, dass es nur exportiert wird, wenn Sie das Spiel in einem Browser ausführen, dann würden Sie nur das HTML5-Feld ankreuzen, und wenn Sie möchten, dass es nur auf Android und iOS-Ziele exportiert wird, würden Sie diese markieren und so weiter. Wenn Sie keine Ziele ankreuzen, wird diese Datei überhaupt nicht exportiert. Diese Option ist besonders nützlich für diejenigen, die den Konfigurations-Manager verwenden möchten, um bestimmte Konfigurationen zu speichern und spezifische Dateien zu exportieren (klicken Sie auf die Konfiguration, die Sie verwenden möchten, aktivieren / deaktivieren Sie die Plattformen, die Sie benötigen und dann wenn Sie ändern Sie die Konfigurationen, die für jeden ausgewählten Dateien werden gespeichert).

Sobald Sie auf "OK" klicken, wird die Datei mit Ihrem aktuellen Projekt in einem eigenen Ordner gespeichert data-files im Projektverzeichnis gefunden. Wenn Sie den Namen der Datei beim Hinzufügen zum Ressourcenbaum geändert haben, wird die Datei nun unter diesem neuen Namen gespeichert. Bitte beachten Sie, dass Sie ab jetzt die in diesem Ordner gespeicherten Dateien und nicht die Originaldateien bearbeiten sollten (es sei denn, Sie beabsichtigen, sie erneut einzubeziehen, aber im Allgemeinen ist dies nicht erforderlich).

Weitere Informationen darüber, wie die enthaltenen Dateien in Ihrem Spiel gespeichert sind und wie sie aufgerufen werden, finden Sie in dem Abschnitt des Handbuchs, der dem Dateisystem gewidmet ist.

WICHTIG! Wenn Sie Sounddateien einschließen, sollten Sie sicherstellen, dass die Dateien Namen vermeiden, die mit Streaming- Soundressourcen kollidieren. Zum Beispiel wird eine Sound-Ressource namens "Musik" und eine enthaltene Datei namens "music.ogg" sein undefined wenn das Projekt ausgeführt wird.


Es ist erwähnenswert, dass Sie DLL (auf Windows) oder DyLib (auf Mac OS) Dateien als enthaltene Dateien hinzufügen und dann Ihre eigenen benutzerdefinierten Funktionen erstellen können, um mit ihnen umzugehen external_define. Diese können dann beim Aufruf des external_call Funktion.

WARNUNG! Die Verwendung von enthaltenen Dateien, um zusätzliche Funktionalität auf diese Weise zu erstellen, ist inkompatibel mit dem Extension- System, insofern Sie das nicht verwenden können external_ Funktionen mit Dateien in einer Erweiterung hinzugefügt, da Erweiterungen bereits eine Methode zum Hinzufügen von Konstanten und Funktionen haben.
