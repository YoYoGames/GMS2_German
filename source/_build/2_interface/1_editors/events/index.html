Nun, Sie würden dies für ein anderes Ereignis (möglicherweise das Ereignis zum Erstellen) eines Objekts codieren und dieses Objekt dann während des Wartens eine Ladebalken zeichnen lassen, wobei das entsprechende asynchrone Ereignis abgefragt wird, bis der Rückruf, der GameMaker Studio 2 das Laden der Datei mitteilt. Veranstaltungsreihenfolge

Bei der Betrachtung von Ereignissen in GameMaker Studio 2 ist zu beachten, dass die genaue Reihenfolge, in der die Ereignisse in den einzelnen Schritten auftreten, nicht eindeutig angegeben werden kann. Dies liegt einfach daran, dass dies von den internen Funktionen von GameMaker Studio 2 abhängt Die Software entwickelt sich. Es gibt jedoch bestimmte Ereignisse, die immer in derselben Reihenfolge ausgeführt werden. Die erste Gruppe von Ereignissen, die immer auf dieselbe Art und Weise ausgeführt wird, sind die Ereignisse, die beim ersten Betreten eines Raums auftreten. Die Reihenfolge der verschiedenen Ereignisse lautet:

  • Objektvariablen / Instanzvariablen werden initialisiert (dies sind die Variablen, die Sie in der IDE des Objekteditors oder des Raumeditors definieren können ). Wenn Sie ein Objekt mit angehängten Variablen haben, werden diese vor allen anderen Objekten erstellt. Anschließend werden alle Instanzen, denen Variablen zugeordnet sind, definiert, sodass Sie Objektvariablen mit instanzspezifischen Variablen im Raumeditor überschreiben können.

  • Ereignis erstellen, gefolgt vom Instanzerstellungscode jeder Instanz - Wenn jede Instanz erstellt wird, führt sie zuerst ihr Erstellungsereignis und dann ihren Instanzerstellungscode aus, bevor sie zur nächsten zu erstellenden Instanz übergeht

  • Spiel Start - Ereignis - dieses Ereignis wird ausgelöst, wenn im ersten Raum des Spiels für alle Instanzen im Raum aus dem Raum Editor platziert (beachten Sie, dass Berufung game_restart() wird dieses Ereignis erneut auslösen.

  • Room Creation Code ( Raumerstellungscode) - Dies ist der einmalige Code, der in den Raumeditor geschrieben wird, wenn ein Raum zum ersten Mal eingegeben wird

  • Raumstartereignis aller Instanzen - eine der "anderen" Ereigniskategorien und wird für alle Instanzen ausgelöst, dauerhaft oder anderweitig


Es ist auch erwähnenswert, dass Sie auch die Reihenfolge festlegen können, in der bestimmte Instanzen im Raumeditor selbst erstellt werden, indem Sie sie in der Liste des Fensters Instance Layer Properties nach oben oder unten verschieben.

Abgesehen von diesen spezifischen Ereignissen ist die einzige bekannte Reihenfolge immer gleich, unabhängig davon, was zu den drei Schrittereignissen und den verschiedenen Zeichnungsereignissen gehört. Diese werden immer konsistent bleiben. Wenn Sie also Code haben, der in jedem Schritt des Spiels auf ein bestimmtes Timing angewiesen ist, sollten Sie Folgendes verwenden:

  • Beginnen Sie das Schrittereignis

  • Schrittereignis (Beachten Sie, dass das Schrittereignis ausgeführt wird, bevor Instanzen an ihren neuen Positionen platziert werden.)

  • Endereignis beenden


Alle Unterereignisse zum Zeichnen werden auch immer in der gleichen Reihenfolge wie folgt behandelt (mit Ausnahme des Resize- Ereignisses, das anders ausgelöst wird):

  • Pre Draw Event

  • Startereignis zeichnen

  • Ereignis zeichnen

  • Endereignis zeichnen

  • Post Draw Event

  • GUI-Startereignis zeichnen

  • GUI-Ereignis zeichnen

  • GUI End Event zeichnen
