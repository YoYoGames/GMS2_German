Ladepuffer


Beschreibung

Mit dieser Aktion können Sie einen Puffer laden, der zuvor mit der Aktion Puffer speichern gespeichert wurde. Die Aktion erstellt einen neuen Puffer, lädt die Datendatei hinein und gibt den eindeutigen ID-Wert zurück, um den Puffer für die Variable zu identifizieren. Sie geben den Namen der zu ladenden Datei (als Zeichenfolge) und eine Variable zum Speichern der zurückgegebenen Puffer-ID an. Wenn Sie die Variable als temporäre lokale Variable kennzeichnen, müssen Sie sicherstellen, dass der von dieser Funktion erstellte Puffer gelöscht wird vor dem Ende des Aktionsskriptes oder -ereignisses, andernfalls verlieren Sie den Verweis darauf und erhalten ein Speicherleck, wenn die Variable verworfen wird.


Aktionssyntax:


Argumente:

Streit Beschreibung
Filename Der Name (als String) der zu öffnenden Datei
Target Die Zielvariable, die den eindeutigen ID-Wert für den beim Laden erstellten Puffer enthält


Beispiel:

Der obige Aktionsblockcode wird eine Variable zum Halten von Pufferdaten erzeugen und dann prüfen, ob zuvor eine Pufferspeicherdatei erstellt wurde. Wenn die Datei vorhanden ist, wird sie in die von uns erstellte Puffervariable geladen, aber wenn sie nicht existiert, wird sie erstellt, geschrieben und dann zum nächsten Laden geladen.
