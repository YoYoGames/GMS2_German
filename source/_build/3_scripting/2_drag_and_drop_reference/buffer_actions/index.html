Pufferbibliothek


Ein Puffer ist eine Region von physischem Speicher, die verwendet wird, um Daten temporär zu halten, während sie von einem Ort zu einem anderen bewegt oder auf irgendeine Weise manipuliert wird, zum Beispiel können Sie einen Puffer verwenden, um Spielzustandsdaten zu halten. GameMaker Studio 2 bietet eine Reihe von spezifischen Aktionen zum Erstellen, Ändern und Laden / Speichern von Puffern. Bitte beachten Sie vor der Verwendung die folgenden Punkte:

  • Wenn ein Puffer erstellt wird, wird er automatisch gelöscht und auf 0 gesetzt.
  • Das Argument "size" der Pufferfunktionen bezieht sich immer auf die Größe in Bytes des Puffers.

  • DnD™ Puffer sind immer 1 Byte ausgerichtet. Die "Ausrichtung" bezieht sich darauf, wie Daten in einem Puffer gespeichert werden und bei Verwendung der GML- Pufferfunktionen verwendet werden, aber für das Ziehen und Ablegen ist dies nicht relevant.

  • Die Daten in den Puffern ausgelesen und in einer sequentiellen Weise geschrieben (dh: Sie nehmen den Anruf Write Buffer Aktion dreimal, dann wird es die ersten Daten schreiben, vorwärts bewegen, rechts die zweiten Daten, vorwärts bewegen und dann die letzten Daten schreiben), mit etwas, das der "Such" -Wert genannt wird, um Ihnen zu sagen, wo in dem Puffer das nächste Lesen / Schreiben stattfinden wird.

  • Alle Daten werden in den Puffer geschrieben und als Zeichenfolge vom Puffer zurückgegeben. Wenn Sie also reelle Zahlen gespeichert haben, müssen Sie eine der Datentyp-Aktionen verwenden, um sie von der Zeichenfolge in den Wert zu konvertieren.

Der Speicher, der zum Erstellen von Puffern verwendet wird, ist Systemspeicher. Selbst wenn das Spiel keinen Fokus hat, sollten alle in einem Puffer gespeicherten Daten sicher sein. Wenn die App jedoch geschlossen oder neu gestartet wird, geht sie verloren.
