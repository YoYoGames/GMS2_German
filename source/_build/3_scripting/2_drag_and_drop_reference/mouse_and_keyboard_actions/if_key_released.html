Wenn Schlüssel freigegeben wird


Beschreibung

Diese Aktion überprüft, ob eine Tastaturtaste losgelassen wurde. Es wird zurückkehren true auf dem einzelnen Spiel ankreuzen, dass der Schlüssel auf und freigegeben wird false zu allen anderen Zeiten. Wenn Sie das aktivieren not Modifikator, diese Aktion wird umgekehrt, und prüfen, ob der Schlüssel nicht freigegeben wurde und zurückkehrt false auf dem Spiel ticken, dass es auf und veröffentlicht wurde true zu allen anderen Zeiten.

Beachten Sie, dass Aktionen zum "if" -Block hinzugefügt werden sollten, wie in der folgenden Abbildung gezeigt:
Diese Aktionen werden jetzt ausgeführt, wenn das "if" ausgewertet wird true, während alle anderen Aktionen nach dem "if" -Block ausgeführt werden.


Aktionssyntax:


Argumente:

Streit Beschreibung
Key Der Name des zu überprüfenden Schlüssels.
Not Annulliere die Prüfung (wahr wird falsch und umgekehrt)


Beispiel:

Der obige Aktionsblockcode fragt den Tastaturstatus bei jedem Schritt ab und wenn eine Taste gedrückt gehalten wird, prüft er, ob die ursprüngliche Taste gedrückt ist. Wenn der Scheck ist true Beim ersten Drücken der Taste wird ein Ton wiedergegeben, und während die Taste gedrückt gehalten wird, wird die Sprite-Überblendung in Rot geändert. Wenn die Taste nicht gedrückt gehalten wird, erfolgt eine Überprüfung der Tastenfreigabe, um die Mischfarbe auf Weiß zurückzusetzen.
