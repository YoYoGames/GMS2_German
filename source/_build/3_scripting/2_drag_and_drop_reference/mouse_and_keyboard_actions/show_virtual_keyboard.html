Virtuelle Tastatur anzeigen


Beschreibung

Diese Aktion kann verwendet werden, um die virtuelle Tastatur auf dem Gerät anzuzeigen, auf dem das Spiel ausgeführt wird. Wenn Sie diese Aktion aufrufen, werden Sie aufgefordert, für jedes der ersten drei Argumente eine der folgenden Konstanten anzugeben:

  • Tastaturtyp: Mit dieser Konstante wird festgelegt, welches Keyset auf der virtuellen Tastatur verfügbar ist. Die verfügbaren Tastaturtypen sind:
    Tastaturtyp Beschreibung
    kbv_type_default Der Standardtastaturtyp für das aktuelle System.
    kbv_type_ascii Eine Nur-ASCII-Tastatur.
    kbv_type_url Eine normale, für die URL-Eingabe optimierte Tastatur. In der Regel sind ".com" oder andere Domänenschlüssel sowie "/" und "." Schlüssel.
    kbv_type_email Eine normale, für die E-Mail-Eingabe optimierte Tastatur. Normalerweise Funktionen "@" und "." Zeichen.
    kbv_type_numbers Eine Nur-Nummern-Tastatur, die normalerweise als Nummernblock angezeigt wird.
    kbv_type_phone Eine Telefontastatur. Normalerweise nur Zahlen mit den Tasten "*" und "#".
    kbv_type_phone_name Eine Tastatur, die für die Eingabe einer Telefonnummer und eines Namens optimiert ist. Normalerweise ähnlich einer ASCII-Tastatur, jedoch mit einer begrenzten Auswahl an Sonderzeichen.

  • Rückgabetyp: Diese Konstante wird verwendet, um festzulegen, was auf der Rückgabe- / Aktionstaste der virtuellen Tastatur angezeigt wird. Die verfügbaren Rückgabetypen sind:
    Rückgabetyp Beschreibung
    kbv_returnkey_default Der Standardschlüssel für den Rückgabeschlüssel für das aktuelle System.
    kbv_returnkey_go Setzt den Return-Key-Titel auf "Go".
    kbv_returnkey_google Setzt für den Return-Key-Titel "Google" oder in einigen Fällen ein allgemeines Suchsymbol.
    kbv_returnkey_join Setzt den Return-Key-Titel auf "Go".
    kbv_returnkey_next Setzt den Return-Key-Titel auf "Next".
    kbv_returnkey_route Setzt den Return-Key-Titel auf "Route".
    kbv_returnkey_search Setzt den Return-Key-Titel auf "Search" oder in einigen Fällen auf ein generisches Suchsymbol.
    kbv_returnkey_send Setzt den Return-Key-Titel auf "Senden".
    kbv_returnkey_yahoo Setzt den Return-Key-Titel auf "Yahoo" oder in einigen Fällen auf ein generisches Suchsymbol.
    kbv_returnkey_done Setzt den Return-Key-Titel auf "Done".
    kbv_returnkey_continue Setzt den Return-Key-Titel auf "Continue".
    kbv_returnkey_emergency Setzt den Return-Key-Titel auf "Notruf".

  • Autokapitalisierungstyp: Mit dieser Konstante wird festgelegt, wie die über die virtuelle Tastatur eingegebenen Wörter autokapitalisiert werden sollen. Die verfügbaren Autokapitalisierungsarten sind:
    Autokapitalisierungstyp Beschreibung
    kbv_autocapitalize_none Autokapitalisierung ist deaktiviert.
    kbv_autocapitalize_words Wörter werden automatisch groß geschrieben.
    kbv_autocapitalize_sentences Sätze werden automatisch aktiviert.
    kbv_autocapitalize_characters Alle Zeichen werden groß geschrieben.

Das letzte Argument ist das Aktivieren / Deaktivieren von Vorhersagetext, und dies würde auf festgelegt werden true um es zu erlauben, und false Andernfalls, beachten Sie jedoch, dass die Verwendung nur zulässig ist, weil sie von den Präferenzen des Benutzers auf dem Gerät abhängt. Im Modus "Predictive Text" generiert die virtuelle Tastatur keine normalen KeyPress-Events von GameMaker Studio 2. Stattdessen werden nur die zuletzt gedrückten Zeichen und die Tastaturzeichenfolgenvariablen aktualisiert. Dies liegt daran, dass nicht erkannt werden kann, ob eine Änderung im internen Textfeld, das zum Erkennen von Tastendrücken verwendet wird, von einer tatsächlichen virtuellen Tastatur oder einem Textvorschlag stammt. In diesen Fällen möchten Sie das lesen keyboard_string Eingabe im Gegensatz zum Lesen einer beliebigen rohen Tasteneingabe.

Es ist auch wichtig zu beachten, dass der Benutzer je nach Betriebssystem der Plattform unterschiedliche Tastaturen mit unterschiedlichen Funktionen erhält, wobei die folgenden Einschränkungen je Ziel zu beachten sind:

  • UWP: Die UWP-Implementierung wird nicht unterstützt return_key_type. autocapitalization_type oder predictive_text_enabled. Das Übergeben dieser Parameter in eine Funktion verursacht keine Fehler, sie haben nur Auswirkungen auf die Tastatur.

  • Android / Amazon Fire: Bei Android der Tastaturtyp kbv_type_phone_name wird nicht unterstützt und zeigt stattdessen eine ASCII-Tastatur an, und der Rückgabetyp kann nur angegeben werden kbv_returnkey_go. kbv_returnkey_search. kbv_returnkey_next. kbv_returnkey_send oder kbv_returnkey_done. Wenn andere Rückgabeschlüsseltypen auf dieser Plattform verwendet werden, verwendet das System standardmäßig die folgenden Ersatzrückgabeschlüssel:
    Nicht verfügbarer Return-Schlüssel Ersatzschlüssel
    kbv_returnkey_google
    kbv_returnkey_yahoo     		kbv_returnkey_search
    kbv_returnkey_join
    kbv_returnkey_route
    kbv_returnkey_emergency     		kbv_returnkey_go
    kbv_returnkey_continue kbv_returnkey_next


  • Android TV / FireTV: Benutzerdefinierte Rückgabetasten werden auf ASCII-Tastaturen nicht unterstützt. Die Standard-Rückgabetaste wird immer angezeigt.

Beim Aufruf dieser Funktion wird ein asynchrones Systemereignis generiert, in dem das async_load Die DS-Karte wird mit den folgenden Schlüssel / Wert-Paaren aufgefüllt:

  • " event_type " - Dies ist der Status der virtuellen Tastatur, wenn er durch virtuelle Tastaturaktionen ausgelöst wird.

  • " screen_height " - die Höhe der virtuellen Tastatur (in Pixel). Dies ist 0, wenn die Tastatur unsichtbar ist.

  • " keyboard_status " - der aktuelle Status der Tastatur, der als eine der folgenden Zeichenfolgen zurückgegeben wird:

    • "verstecken"
    • "versteckt"
    • "zeigen"
    • "sichtbar"


Aktionssyntax:


Argumente:

Streit Beschreibung
keyboard_type Bestimmt das auf der virtuellen Tastatur verfügbare Keyset.
return_key_type Legt fest, was auf der Rückkehr- / Aktionstaste der virtuellen Tastatur angezeigt wird.
autocapitalization_type Legt fest, wie / ob die über die virtuelle Tastatur eingegebenen Wörter automatisch mit Kapital versehen werden.
predictive_text_enabled Einstellen true / false vorhersagende Texteingabe aktivieren / deaktivieren.


Beispiel:

Der obige Aktionsblockcode prüft, ob die Maus heruntergefahren ist. Wenn eine erkannt wird, wird geprüft, ob die virtuelle Tastatur des Betriebssystems angezeigt wird. Ist dies nicht der Fall, wird die Anzeige mithilfe eines numerischen Tastenfelds festgelegt. Wenn dies bereits der Fall ist, wird sie ausgeblendet.
