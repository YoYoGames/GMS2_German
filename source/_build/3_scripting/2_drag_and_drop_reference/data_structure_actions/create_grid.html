Gitter erstellen


Beschreibung

Diese Aktion erstellt eine neue Rasterdatenstruktur und gibt den Indexwert zurück, sodass Sie später über die anderen Datenstrukturaktionen auf das Raster zugreifen können. Der Rasterindex wird an die von Ihnen angegebene Zielvariable zurückgegeben, die zuvor mit Zuweisungsvariable oder Deklariertemperatur erstellt wurde. Sie können das Kontrollkästchen "Temp" aktivieren, um eine temporäre lokale Variable zu benennen und zu erstellen, in der der Wert bis gespeichert wird das Ende des Skripts oder Ereignisses Beachten Sie, dass im Gegensatz zu anderen Datenstrukturen Grids nicht "leer" erstellt werden und jede Zelle eines neu erstellten Grids den Wert 0 hat. Sie können die Zellendaten ändern oder das Raster mit der Aktion Clear auf einen anderen Wert zurücksetzen Gitter. Beachten Sie, dass Sie zusätzliche DS-Raster erstellen können, indem Sie auf das Plus-Symbol klicken neben der Aktion, und wählen Sie eine andere Variable, um die Raster-ID zu halten und die Werte für Breite und Höhe erneut einzugeben.

Wichtig! Das Erstellen einer beliebigen Datenstruktur verbraucht Speicherplatz auf der Zielplattform und daher sollten alle Datenstrukturen frei sein, wenn sie nicht länger mit der Aktion Freie Datenstruktur benötigt werden. Andernfalls entsteht ein Speicherverlust, der die Spieleleistung beeinträchtigen oder sogar zum Absturz bringen kann. Dies ist besonders relevant, wenn temporäre lokale Variablen zum Speichern von Datenstrukturindizes verwendet werden, da diese Variablen am Ende des Codes oder des Ereignisses entfernt werden. Dies bedeutet jedoch nicht, dass auch die Datenstruktur entfernt wird. Die Datenstruktur wird weiterhin existieren, nur Sie haben keine Möglichkeit, sie zu referenzieren. Verwenden Sie entweder eine Instanzvariable und geben Sie die Struktur zu einem späteren Zeitpunkt frei oder geben Sie die Struktur vor dem Ende des Ereignisses oder Skripts frei, wenn der Index gespeichert ist eine temporäre Variable.
