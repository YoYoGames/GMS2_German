Setzen Sie die Instanzvariable


Beschreibung

Mit dieser Aktion können Sie eine der eingebauten Variablen festlegen. Alle Instanzen aller Objekte haben eine Anzahl von Variablen, die bereits in ihnen erzeugt wurden, um wesentliche Informationen wie die aktuelle x / y-Position, die Geschwindigkeit, die Richtung, die Sichtbarkeit und viele andere Dinge zu speichern. Die Instanzvariablen, die Sie verwenden können, sind:

  • X-Koordinate - Die Position entlang der x-Achse im Raum.

  • Y-Koordinate - Die Position entlang der Y-Achse im Raum.

  • Richtung - Die Bewegungsrichtung für die Instanz (Standard ist 0, rechts ist 0 °, oben ist 90 °, links ist 180 ° und unten ist 270 °).

  • Geschwindigkeit - Die Geschwindigkeit (in Pixeln pro Spiel), in der sich die Instanz bewegt.

  • Horizontal Speed ( Horizontale Geschwindigkeit) - Die horizontale Geschwindigkeit (in Pixeln pro Spiel-Frame), in der sich die Instanz bewegt. Eine negative Geschwindigkeit bedeutet, dass es sich nach links bewegt, eine positive Geschwindigkeit ist richtig und der Standardwert ist 0.

  • Vertikale Geschwindigkeit - Die vertikale Geschwindigkeit (in Pixel pro Spiel), in der sich die Instanz bewegt. Eine negative Geschwindigkeit bedeutet, dass sie sich nach oben bewegt, eine positive Geschwindigkeit ist ab und der Standardwert ist 0.

  • Sichtbar - Die Umschaltvariable für die Sichtbarkeit (Standard ist true Wenn im Objekt-Editor nicht anders festgelegt, setzen Sie den Wert auf false unsichtbar machen). Beachten Sie, dass eine unsichtbare Instanz weiterhin alle Ereignisse mit Ausnahme des Draw-Ereignisses ausführt.

  • Persistent - Die Umschaltvariable für die Persistenz (Standardwert ist false Wenn im Objekt-Editor nicht anders festgelegt, setzen Sie den Wert auf true hartnäckig machen). Dadurch wird eine Instanz von Raum zu Raum "bestehen bleiben", dh die Instanz wird nicht am Raumende aufgeräumt, sondern in den nächsten Raum übertragen. Beachten Sie, dass das Ereignis "Create not Destroy" nicht aufgerufen wird, wenn eine Instanz beibehalten wird. Sie ruft jedoch die Ereignisse "Clean Up", "Room End" und "Room Start" auf.

  • Ebene - Die Ebene, der die Instanz zugewiesen wurde.

  • Tiefe - Der Tiefenwert, der dem Layer zugewiesen ist.

  • Sprite - Die der Instanz zugewiesene Sprite-Ressource.

  • Bildindex - Der Bildindex des zugewiesenen Sprites. Dieser Wert legt den Rahmen des der Instanz zugewiesenen Sprites fest, und wenn der Wert größer als die Anzahl der Rahmen in der Animation ist, "schleift" er so, dass ein Sprite mit 5 Rahmen, wenn er auf Rahmen 7 gesetzt ist, Rahmen 2 zeigt.

  • Bildwinkel - Der Winkel, dass die Instanz gesetzt werden sollte (Standard: 0, wobei rechts von 0 ° bis 90 °, links 180 ° beträgt und ab 270 °).

  • Image Alpha - Der Alpha-Wert für die Instanz (Standardwert ist 1) von 0 bis 1, wobei 0 vollständig transparent und 1 vollständig undurchsichtig ist.

  • Bildüberblendung - Die Farbe, die mit dem Instanz-Sprite gemischt werden soll (Standard ist Weiß).

  • Bildgeschwindigkeit - Die Animationsgeschwindigkeit für die Instanz.

  • Horizontale Skalierung - Die horizontale Skalierung, mit der das Instanz-Sprite gezeichnet werden soll (dies wirkt sich auch auf die Instanz-Begrenzungsbox und auf Kollisionen aus). Eine Skala von 1 zeigt keine Skalierung an (1: 1), kleinere Werte werden verkleinert (0,5, zum Beispiel die halbe Breite des Sprites), größere Werte skalieren und negative Werte spiegeln das Sprite und skalieren es, es sei denn, die Der verwendete Wert ist genau -1 (in diesem Fall wird das Sprite nur entlang der y-Achse ohne Skalierung gespiegelt).

  • Vertikale Skalierung - Die vertikale Skalierung, mit der das Instanz-Sprite gezeichnet werden soll (dies wirkt sich auch auf die Bounding-Box und die Kollisionen der Instanz aus). Eine Skalierung von 1 zeigt keine Skalierung an (1: 1), kleinere Werte werden verkleinert (0,5, z. B. wird die Hälfte der Höhe des Sprites), größere Werte skalieren und negative Werte spiegeln das Sprite und skalieren es, es sei denn, Der verwendete Wert ist genau -1 (in diesem Fall wird das Sprite einfach ohne Skalierung entlang der y-Achse gedreht).

  • Maske - Der Maskenindex für die Instanz. Der Standardwert ist -1 (was bedeutet, dass die Instanz die Kollisionsmaske für den aktuell zugewiesenen Sprite-Index verwendet). Wenn sie jedoch auf eine andere Sprite-Ressource eingestellt ist, wird die Kollisionsmaske für diese Instanz anstelle des dem Sprite zugewiesenen Sprites verwendet Index.

  • Reibung - Die Reibung, um jeden Spielrahmen des Spiels zu verwenden, um die Instanz zu verlangsamen. Dieser Wert wird einfach von der aktuellen Geschwindigkeit jedes Spielrahmens abgezogen (oder addiert, wenn der Geschwindigkeitswert negativ ist).

  • Schwerkraft - Schwerkraft wendet eine konstante Kraft in Richtung der Schwerkraft (siehe unten) der Instanz an und beeinflusst ihre Geschwindigkeit (Standard ist 0). Beachten Sie, dass die Schwerkraft eine kumulative Kraft ist und das Objekt beschleunigt, wenn Sie die Endgeschwindigkeit nicht begrenzen.

  • Gravitationsrichtung - Dies kann verwendet werden, um die Bewegungsrichtung festzulegen, wenn die Schwere der Instanzen größer als 0 ist (Standard ist 270). Beachten Sie, dass die Anweisungen in GameMaker Studio 2 so berechnet werden, dass 0 ° rechts ist, 90 ° oben, 180 ° links und 270 ° unten liegen.

  • Ergebnis - Legen Sie die score Variable.

  • Leben - Setzen Sie die lives Variable.

  • Gesundheit - Legen Sie die health Variable.

Nachdem Sie die zu verwendende integrierte Variable ausgewählt haben, legen Sie den Wert für diese Variable fest. Der von Ihnen verwendete Wert hängt von der ausgewählten Variablen ab (siehe die obige Liste für Standardwerte und Bereiche), und Sie können den Wert als relativ zum aktuellen Wert für die Variable kennzeichnen. Sie können auch weitere Variablen hinzufügen, die in derselben Aktion festgelegt werden, indem Sie auf das Pluszeichen klicken neben der Aktion, und wählen Sie eine andere Instanzvariable und geben Sie einen Wert.
