Instanzvariable abrufen


Beschreibung

Mit dieser Aktion können Sie eine der aufgeführten integrierten Variablen abrufen. Alle Instanzen aller Objekte haben eine Anzahl von Variablen, die bereits in ihnen erzeugt wurden, um wesentliche Informationen wie die aktuelle x / y-Position, die Geschwindigkeit, die Richtung, die Sichtbarkeit und viele andere Dinge zu speichern. Die Instanzvariablen, von denen Sie die Werte erhalten können, sind:

  • X-Koordinate - Die Position entlang der x-Achse im Raum.

  • Y-Koordinate - Die Position entlang der Y-Achse im Raum.

  • Richtung - Die Bewegungsrichtung für die Instanz (Standard ist 0, rechts ist 0 °, oben ist 90 °, links ist 180 ° und unten ist 270 °).

  • Geschwindigkeit - Die Geschwindigkeit (in Pixeln pro Spiel), in der sich die Instanz bewegt.

  • Horizontal Speed ( Horizontale Geschwindigkeit) - Die horizontale Geschwindigkeit (in Pixeln pro Spiel-Frame), in der sich die Instanz bewegt. Eine negative Geschwindigkeit bedeutet, dass es sich nach links bewegt, eine positive Geschwindigkeit ist richtig und der Standardwert ist 0.

  • Vertikale Geschwindigkeit - Die vertikale Geschwindigkeit (in Pixel pro Spiel), in der sich die Instanz bewegt. Eine negative Geschwindigkeit bedeutet, dass sie sich nach oben bewegt, eine positive Geschwindigkeit ist ab und der Standardwert ist 0.

  • Sichtbar - Die Umschaltvariable für die Sichtbarkeit (Standard ist true Wenn im Objekt-Editor nicht anders festgelegt, setzen Sie den Wert auf false unsichtbar machen). Beachten Sie, dass eine unsichtbare Instanz weiterhin alle Ereignisse mit Ausnahme des Draw-Ereignisses ausführt.

  • Persistent - Die Umschaltvariable für die Persistenz (Standardwert ist false Wenn im Objekt-Editor nicht anders festgelegt, setzen Sie den Wert auf true hartnäckig machen). Dadurch wird eine Instanz von Raum zu Raum "bestehen bleiben", dh die Instanz wird nicht am Raumende aufgeräumt, sondern in den nächsten Raum übertragen. Beachten Sie, dass das Ereignis "Create not Destroy" nicht aufgerufen wird, wenn eine Instanz beibehalten wird. Sie ruft jedoch die Ereignisse "Clean Up", "Room End" und "Room Start" auf.

  • Ebene - Die Ebene, der die Instanz zugewiesen wurde.

  • Tiefe - Der Tiefenwert, der dem Layer zugewiesen ist.

  • Sprite - Die der Instanz zugewiesene Sprite-Ressource.

  • Bildindex - Der Bildindex des zugewiesenen Sprites. Dieser Wert ruft den Rahmen des Sprites ab, das der Instanz zugewiesen ist. Der zurückgegebene Wert liegt zwischen 0 und der Anzahl der Bilder für das Sprite. Wenn die Bildgeschwindigkeit nicht genau eins ist, kann der zurückgegebene Wert eine Dezimalzahl sein.

  • Bildwinkel - Der Winkel, dass die Instanz zur Zeit eingestellt ist (Standard: 0, wobei rechts von 0 ° bis 90 °, links 180 ° beträgt und ab 270 °).

  • Image Alpha - Der aktuelle Alpha-Wert für die Instanz (Standard ist 1) von 0 bis 1, wobei 0 voll transparent und 1 vollständig undurchsichtig ist.

  • Bildüberblendung - Die aktuelle Farbe wird mit dem Instanz-Sprite gemischt (Standard ist Weiß).

  • Bildgeschwindigkeit - Die aktuelle Animationsgeschwindigkeit für das Sprite.

  • Horizontale Skalierung - Die horizontale Skalierung, mit der das Instanz-Sprite gezeichnet wird (dies betrifft auch die Instanz-Begrenzungsbox und Kollisionen). Eine Skala von 1 zeigt keine Skalierung (1: 1) an, kleinere Werte werden herunterskaliert (0,5, zum Beispiel, wird die halbe Breite des Sprites), größere Werte werden hochskaliert und negative Werte spiegeln das Sprite und skalieren es, es sei denn Rückgabewert ist genau -1 (in diesem Fall wird das Sprite nur entlang der y-Achse ohne Skalierung gespiegelt).

  • Vertikale Skalierung - Die vertikale Skalierung, mit der das Instanz-Sprite gezeichnet wird (dies betrifft auch die Bounding-Box und die Kollisionen der Instanz). Eine Skala von 1 zeigt keine Skalierung an (1: 1), kleinere Werte werden herunterskaliert (0,5, z. B. wird die halbe Höhe des Sprites), größere Werte werden hochskaliert und negative Werte spiegeln das Sprite und skalieren es, es sei denn, Der zurückgegebene Wert ist genau -1 (in diesem Fall wird das Sprite gerade ohne Skalierung entlang der y-Achse gekippt).

  • Maske - Der Maskenindex für die Instanz. Der standardmäßig zurückgegebene Wert ist -1 (was bedeutet, dass die Instanz die Kollisionsmaske für den aktuell zugewiesenen Sprite-Index verwendet). Wenn sie jedoch auf eine andere Sprite-Ressource gesetzt wird, ist der zurückgegebene Wert der Sprite-Index für die verwendete Ressource.

  • Reibung - Die Reibung, die bei jedem Spiel verwendet wird, um die Instanz zu verlangsamen. Standard ist 0.

  • Schwerkraft - Der zurückgegebene Wert ist eine konstante Schwerkraft, die auf die Instanz angewendet wird. Standard ist 0.

  • Gravitationsrichtung - Die Richtung, in der die Gravitation die Instanz "zieht". Der zurückgegebene Wert liegt zwischen 0 und 360, wobei rechts 0 °, aufwärts 90 °, links 180 ° und unten 270 ° ist. Der Standardwert ist 270.

  • Score - Holen Sie sich das score Variable.

  • Leben - Holen Sie sich das lives Variable.

  • Gesundheit - Holen Sie sich das health Variable.

Sobald Sie die integrierte Variable zum Abrufen ausgewählt haben, legen Sie die Zielvariable so fest, dass sie den zurückgegebenen Wert enthält. Wenn Sie diese Variable als temporäre Variable kennzeichnen, wird sie in der Aktion für Sie erstellt und ist nur bis zum Ende des Ereignis- oder Aktionsskripts verfügbar. Sie können die Werte zusätzlicher Variablen auch in derselben Aktion abrufen, indem Sie auf das Pluszeichen klicken neben der Aktion, und wählen Sie eine andere Instanzvariable und geben Sie eine begleitende Variable, um den zurückgegebenen Wert zu speichern.
