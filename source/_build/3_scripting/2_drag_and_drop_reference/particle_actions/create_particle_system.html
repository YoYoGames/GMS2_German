Partikelsystem erstellen


Beschreibung

Mit dieser Aktion wird das Partikelsystem initialisiert, mit dem Sie Partikeltypen und Emitter erstellen und verwenden können. Sie geben den Layernamen als String an - zum Beispiel "Particle Layer" (der im Raumeditor vorhanden sein muss) - und kennzeichnen das System dann als Persistent oder nicht. Wenn das System als Persistent gekennzeichnet ist, wird es nach der Erstellung in allen Räumen beibehalten und eine verwaltete Schicht zugewiesen, wenn die ursprüngliche Schicht in nachfolgenden Räumen nicht vorhanden ist. Wenn die Persistenz nicht markiert ist, wird das System am Ende des Raums, in dem es erstellt wurde, automatisch zerstört, ebenso wie alle Emitter, die mit dem System verbunden sind. Wenn Sie das System manuell aus dem Speicher entfernen möchten, müssen Sie die Aktion Partikel System zerstören verwenden.

Sie müssen auch eine Zielvariable angeben, die den eindeutigen Partikelsystem-ID-Wert enthält. Dieser ID-Wert wird in anderen Funktionen verwendet, um Eigenschaften des Systems GameMaker Studio 2 oder um GameMaker Studio 2 mitzuteilen, welches System beim Erstellen von Partikeln verwendet werden soll. Sie können diese Variable als temporäre lokale Variable kennzeichnen. Dies bedeutet jedoch, dass sie am Ende des Aktionsskripts oder -Ereignisses verworfen wird. Dies bedeutet, dass das System zwar weiterhin vorhanden ist, Sie aber ohne Zugriffsmöglichkeit bleiben Vorsicht ist bei der Verwendung geboten (im Allgemeinen müssen Systeme niemals in einer temporären lokalen Variablen gespeichert werden, aber für bestimmte Dinge, wie das Zeichnen von Partikeln auf eine Oberfläche, kann es erforderlich sein).

HINWEIS: Ein persistentes Partikelsystem bleibt nach der Erstellung für die Dauer des Spiels im Systemspeicher, und wenn Sie es an einem beliebigen Punkt erneut erstellen müssen (z. B. wenn Sie die Aktion " Spiel neu starten" aufrufen), müssen Sie dies tun Zerstöre zuerst das aktuelle System, sonst bekommst du ein Speicherleck. Ein Speicherleck tritt auf, wenn eine Ressource dem Speicher hinzugefügt und dann "de-referenziert" wird, was bedeutet, dass GameMaker Studio 2 keine Möglichkeit mehr hat, mit dieser Ressource zu interagieren. Dies führt dazu, dass der Speicher durch unzugängliche Ressourcen verstopft wird, was wiederum Probleme mit Ihrem Spiel verursachen kann. Um ein persistentes Partikelsystem zu zerstören, wenn es nicht mehr benötigt wird, verwenden Sie die Aktion Partikel-System zerstören.
