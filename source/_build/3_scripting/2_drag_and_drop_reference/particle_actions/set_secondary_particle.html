Stellen Sie das sekundäre Partikel ein


Beschreibung

Mit dieser Aktion können Sie einen zweiten Partikeltyp zusammen mit dem Primärpartikel verwenden. Sie geben den primären Partikeltyp-ID-Wert an (der dem beim Erstellen des Partikels mit der Aktion "Partikeltyp erstellen" verwendeten Wert entsprechen sollte) und Sie können dann für jeden Spielframe, für den er existiert, einen sekundären Partikeltyp ausgeben lassen / oder am Ende der Lebensdauer jedes Primärpartikels. Sie müssen einen weiteren Partikel-ID-Wert für "Sekundär" und dann eine Zahl angeben, die in "Anzahl" erstellt werden soll.

Wenn das ausgewählte Ereignis "Pro Step" ist, dann wird jedes Spiel, das ein primäres Partikel auf dem Bildschirm existiert, die angegebene Anzahl sekundärer Partikel emittieren - zum Beispiel könnten Sie eine Rakete mit "Flammen" -Teilchen haben, und jedes Flammenpartikel emittiert a sekundäres "Rauch" -Teilchen. Wenn das Ereignis auf "On Death" eingestellt ist, werden die Sekundärteilchen nur dann emittiert, wenn ein Teilchen des Primärtyps das Ende seiner Lebensdauer erreicht - zum Beispiel könnten Sie ein "Explosions-" Teilchen haben, das einen "Staub" emittiert "Partikel am Ende ihres Lebens.

Es ist wichtig zu beachten, dass Sie nicht dieselbe Partikeltyp-ID wie das Primärpartikel wie für Sekundärpartikel verwenden können und dass bei dieser Aktion sehr schnell Leistungsprobleme auftreten können, da auf diese Weise schnell Tausende von Partikeln erstellt werden können Achten Sie darauf, die Lebensdauer und die Anzahl der Partikel, die von den primären und sekundären Partikeltypen erzeugt werden, zu begrenzen. Beachten Sie auch, dass Sie jedem Partikeltyp ein Sekundärpartikel zuweisen können und so "Partikelketten" erzeugen können, wobei ein Partikel bei jedem Schritt einen anderen erzeugt und dieses Partikel dann ein anderes erzeugt, wenn es das Ende seiner Lebensdauer erreicht Bei der Verkettung von Partikeln sollte darauf geachtet werden, dass die Leistung nicht leidet. Sie können diese Aktion auch zweimal für einen einzelnen Partikeltyp aufrufen und den Stream bei Schritt sowie beim Tod platzen lassen.


Aktionssyntax:


Argumente:

Streit Beschreibung
Type ID Der ID-Wert für den Partikeltyp, dem das Sekundärpartikel zugewiesen werden soll
Event Wann man das Sekundärteilchen erzeugt, entweder "Per Step" oder "On Death"
Death Type Der ID-Wert für den Partikeltyp, der im ausgewählten sekundären Event ausgegeben werden soll
Count Die Anzahl der sekundären Partikel, die im ausgewählten sekundären Ereignis erstellt werden sollen
