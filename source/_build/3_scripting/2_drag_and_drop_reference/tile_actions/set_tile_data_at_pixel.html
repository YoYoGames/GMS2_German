Kacheldaten bei Pixel festlegen


Beschreibung

Mit dieser Aktion können Sie die Kacheldaten für eine Kachelzelle festlegen, die an einer bestimmten Position auf einer Kachelkartenebene gefunden wird. Das tiledata ist einfach ein Wert, der den Index der Kachel zusammen mit den Werten für Spiegeln, Spiegeln und Drehen sowie den benutzerdefinierten Kachelmasken, die Sie verwendet haben, widerspiegelt. Mit dieser Aktion versorgen Sie die Ebene zuerst mit dem Namen der Ebene (eine Zeichenkette), wie im Raum-Editor definiert, und dann mit der x- und y-Achse im Raum, für die die Fläche festgelegt werden soll. Beachten Sie, dass das tiledata für die Tilemap "Zelle" gesetzt wird, in der die gegebene Position landet. Wenn Ihre Tileset-Kacheln zum Beispiel 16x16 sind, enthält der Tilemap-Layer Kacheln in Zellen von 16x16 Pixeln. Wenn Sie also diese Aktion zum Targeting der Raumposition (8, 8) verwenden, ändern Sie die Kachel in der Zelle ( 0, 0) der Kachelkarte.

HINWEIS: Die ausgewählte Ebene muss im Raumeditor als Kachelkarte definiert worden sein, da sonst Fehler auftreten können.


Das letzte Argument für die Aktion ist die tiledate selbst. Normalerweise würden Sie die tiledata für die Kachelmap-Zelle mit der Aktion Get Tile Data At Pixel abrufen und dann mit der Aktion Kachel-Datentransformation festlegen bearbeiten, bevor Sie sie mit dieser Aktion erneut festlegen. Weitere Informationen zu tiledata finden Sie im GML-Abschnitt zu Tilemaps - Tile-Funktionen.


Aktionssyntax:


Argumente:

Streit Beschreibung
Layer Die Ebene mit der zu bearbeitenden Kachelkarte
X Die Position entlang der horizontalen (X) Achse, für die die Elemente festgelegt werden sollen
Y Die Position entlang der vertikalen Achse (Y), um die Fliesen zu setzen
Data Das tiledata, das zum Festlegen der Zelle an der Position verwendet werden soll


Beispiel:

Der obige Action-Block-Code testet auf einen Mausklick und wenn einer empfangen wird, ruft er die tiledata für die Zelle ab, die mit der Mausposition übereinstimmt. Diese Daten werden dann geändert und an die Tilemap-Ebene zurückgegeben, wobei die Kachel in dieser Zelle rotiert wird.
