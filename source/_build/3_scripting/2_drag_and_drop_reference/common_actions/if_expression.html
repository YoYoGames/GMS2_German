Wenn Ausdruck


Beschreibung

Mit dieser Aktion wird überprüft, ob ein Ausdruck als true oder false und Sie können hier jeden logischen Ausdruck überprüfen, solange er den einen oder anderen Wert auswertet. Zum Beispiel könnte jeder der folgenden Ausdrücke verwendet werden:

x < (room_width / 2)
a == b
++val >= 100

Wenn Sie das Argument "Not" angeben, werden die obigen Ausdrücke negiert, zum Beispiel "==" wird zu " not =". Beachten Sie auch, dass der Ausdruck nicht auf numerische Werte beschränkt ist. Sie können Zeichenfolgen, Zeiger, Ressourcen-IDs oder andere Funktionen, die eine Funktion zurückgeben oder verwenden kann, überprüfen (weitere Informationen finden Sie im Abschnitt Datentypen ). Sie können jedoch nicht zwei verschiedene Daten vergleichen Typen, ohne einen Fehler zu erhalten - wenn Sie beispielsweise versuchen, einen String und eine Ganzzahl im Ausdruck zu verwenden, erhalten Sie einen Fehler. Weitere Informationen zu Ausdrücken finden Sie hier.

WICHTIGER HINWEIS: Aufgrund von Gleitkommaprozisionsproblemen kann die Überprüfung, ob zwei Werte exakt gleich sind, zurückgegeben werden false, da eine genau 1 sein kann, während die andere 1,00000000000001 sein kann. Dies kann vermieden werden, indem Sie die Aktion Dezimal zu Ganzzahl verwenden, bevor Sie die Ausdrücke "größer als (>)" oder "kleiner als (<)" überprüfen oder verwenden.

Diese Aktionen werden jetzt ausgeführt, wenn das "if" ausgewertet wird true, während alle anderen Aktionen nach dem "if" -Block ausgeführt werden. Beachten Sie, dass Sie Ausdrücke in derselben Aktion "stapeln" können, indem Sie auf das Plus-Symbol klicken neben der Aktion, und geben Sie einen anderen Ausdruck zu überprüfen. Dies überprüft, ob alle angegebenen Ausdrücke zurückgegeben werden true in einem Scheck und führen Sie den addierten Aktionscode nur aus, wenn sie es tun (siehe das unten angegebene Beispiel).
