Temporäre Variable deklarieren


Beschreibung

Diese Aktion wird verwendet, um eine temporäre Variable (auch als lokale Variable bezeichnet ) zu deklarieren und ihr einen Anfangswert zu geben. Dies ist eine Variable, die nur für die Dauer des Ereignisses oder Skripts existiert, in dem sie deklariert wurde. Temporäre Variablen sind sehr nützlich, um Werte zu speichern, die nur für eine einzelne Operation benötigt werden, und verhindern, dass Ihre Objekte mit Variablen überladen werden, die nur einmal verwendet werden dürfen. Sie sind auch sehr nützlich, um eigene komplexe Operationen zu unterbrechen oder um Werte in anderen Instanzen zu setzen, wenn Sie den Aktionsbereich ändern. Dies liegt daran, Variablen als temporäre deklariert sind nicht auf eine bestimmte Instanz scoped, sondern scoped statt auf das Ereignis oder das Skript selbst, wenn Sie also eine temporäre Variable erstellen und dann den Umfang Aktion ändern mit der Anwenden auf... Aktion, können Sie immer noch Überprüfen Sie die temporäre Variable ohne Probleme und ohne ihren Aktionsbereich zu ändern.

Es ist erwähnenswert, dass wenn wir von "Werten" sprechen, wir nicht nur numerische Werte meinen, da eine Variable eine Zeichenkette, ein Zeiger, eine Ressourcen-ID oder irgendetwas anderes, das eine Funktion zurückgeben oder benutzen kann, auch ein Ausdruck sein kann. Weitere Informationen finden Sie im Abschnitt zu Datentypen.

Beachten Sie, dass viele Aktionen über ein Kontrollkästchen "Temp" verfügen, mit dem Sie eine neue temporäre lokale Variable erstellen können, die den Rückgabewert der Aktion enthält. Auf diese Variable kann dann wie alle anderen Variablen im Aktionsskript oder -ereignis zugegriffen werden. Beachten Sie außerdem, dass temporäre lokale Variablen auf das Skript oder das Ereignis, in dem sie erstellt wurden, beschränkt sind und daher von anderen Instanzen im selben Block verwendet werden können, wenn Sie Anwenden auf verwenden.

Sie können auch zusätzliche Variablen in derselben Aktion hinzufügen, indem Sie auf das Plus-Symbol klicken neben der Aktion, und geben Sie einen anderen temporären lokalen Variablennamen und -wert. Weiterführende Informationen zu Variablen finden Sie im Abschnitt Variablen und Variablenumfang.
