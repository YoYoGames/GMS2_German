Accessoren


Die GameMaker Sprache (GML) erlaubt Ihnen auch den Zugriff auf bestimmte Datenstrukturen und Puffer durch die Verwendung von logischen Ausdrücken, die Accessoren genannt werden. Dies geschieht auf ähnliche Weise wie beim Arbeiten mit Arrays. Wir verwenden vor dem ersten Argument ein Bezeichnersymbol, um GameMaker Studio 2 mitzuteilen, dass Sie an einer (zuvor erstellten) Datenstruktur arbeiten.

Datenstrukturen

Sie können Werte und Schlüssel für die folgenden Datenstrukturtypen hinzufügen, ersetzen und abrufen: ds_maps, ds_lists und ds_grids. Jedem wurde ein eigenes Symbol und eine Methode für den Zugriff, das Ändern oder Einstellen von Werten zugewiesen, wie unten gezeigt.

Die Syntax für Listen ist:

list_index[| index]


Also wenn du benutzt hast ds_list_create() Um Ihre Liste zu erstellen, verwenden Sie den Listenindex (den Sie in einer Variablen gespeichert haben), um ihn zu referenzieren, wobei der Wert "index" die Position in der Liste ist, die Sie festlegen oder hinzufügen möchten. Der folgende Code erstellt beispielsweise eine Liste und fügt dann 10 Einträge hinzu, wobei jeder Eintrag auf eine Zufallszahl zwischen 0 und 9 gesetzt wird:

ds = ds_list_create();
var index = 0;
repeat(10)
   {
   ds[| index++] = irandom(9);
   }


Sobald Sie Ihre Listenstruktur erstellt und mit Daten gefüllt haben, können Sie Werte aus der Liste abrufen:

value = ds[| 5];


Das obige wird den Wert von Position 5 (der sechste Index, da Listen bei 0 beginnen) erhalten und in einer Variablen speichern. Wenn Sie einen Ausdruck verwenden, um einen Verweis auf einen Index hinzuzufügen, der bereits einen Wert enthält, wird der vorherige Wert ersetzt, statt der Liste einen weiteren Index hinzuzufügen. Um weitere Einträge hinzuzufügen, müssen Sie die Größe von ds_list kennen und am Ende hinzufügen. Es ist auch erwähnenswert, dass Sie einen Listenindex festlegen können, der größer ist als die Größe der referenzierten Liste. Dadurch wird dieser Wert festgelegt, die Liste wird gleichzeitig erweitert und alle Positionen in der Liste werden initialisiert zu dem angegebenen Index als 0.

Die Syntax für Karten ist:

map_index[? key]


Nach dem Erstellen Ihrer Karte mit ds_map_create(), würden Sie den Map-Index verwenden, den Sie in einer Variablen gespeichert haben, um ihn zu referenzieren, wobei der "Schlüssel" -Wert der Map-Schlüssel zum Setzen oder Abrufen ist. Der folgende Code erstellt beispielsweise eine Map und fügt dann mit dieser Syntax einige Einträge hinzu:

ds = ds_map_create();
ds[? "Name"] = "Hamish";
ds[? "Company"] = "MacSeweeny Games";
ds[? "Game"] = "Catch The Haggis";


Sobald Sie Ihre Kartenstruktur erstellt und mit Daten gefüllt haben, können Sie Werte aus einem bestimmten Kartenschlüssel erhalten:

value = ds[? "Name"];


Das Obige erhält den Wert vom Schlüssel "Name" und speichert es in einer Variablen. Beachten Sie, dass wenn die Map bereits denselben Schlüsselwert enthält, den Sie hinzufügen möchten, wird kein doppelter Schlüssel mit dem neuen Wert erstellt, sondern der vorherige Wert wird ersetzt.

Die Syntax für Raster lautet:

grid_index[# xpos, ypos]


Nach dem Erstellen Ihres Rasters mit dem ds_grid_create() Funktion verwenden Sie den Gitterindex, den Sie in einer Variablen gespeichert haben, um ihn zu referenzieren, wobei "xpos" und "ypos" die Position innerhalb des Gitters sind, um einen Wert zu erhalten oder festzulegen. Der folgende Code erstellt beispielsweise ein Raster, löscht es auf 0 und fügt dann einige Einträge hinzu:

ds = ds_grid_create();
ds_grid_clear(ds, 0);
var gw = ds_grid_width(ds) - 1;
var gh = ds_grid_height(ds) - 1;
repeat(10)
   {
   var xx = irandom(gw);
   var yy = irandom(gh);
   if ds[# xx, yy] == 0 ds[# xx, yy] = 1;
   }


Nachdem Sie Ihre Gitterstruktur erstellt und mit Daten gefüllt haben, können Sie Werte aus einer bestimmten Gitterposition erhalten:

value = ds[# mouse_x div 16, mouse_y div 16];


Das obige wird den Wert von dem gegebenen ds_grid basierend auf der Mausposition erhalten (geteilt durch die "Zellen" Breite in dem Raum, um den korrekten Ort zu erhalten).


Arrays

Arrays haben auch eigene Accessoren, die ähnlich wie die oben für Datenstrukturen aufgeführten arbeiten. Array-Zugriffsmethoden haben jedoch eine interessante Eigenschaft, mit der Sie ein Array von einem Skript aus ändern können, ohne es kopieren zu müssen. Wenn Sie ein Array an ein Skript übergeben, wird es als Referenz übergeben. Dies bedeutet, dass das Array selbst nicht in das Skript kopiert wird, sondern nur referenziert wird, um die Daten zu erhalten. Wenn Sie das Array dann ändern müssen, wird es normalerweise in das Skript kopiert. Anschließend müssen Sie das kopierte Array für das ursprüngliche Array, das aktualisiert werden soll, zurückgeben. Dies kann teuren Verarbeitungsaufwand verursachen, und Sie können stattdessen den Accessor verwenden, da dies das ursprüngliche Array direkt ändern wird, ohne dass es kopiert werden muss. In den folgenden Beispielen sehen Sie, wie das funktioniert.

Die Syntax für Arrays lautet:

array[@ xpos]
array[@ xpos, ypos]

Nachdem Sie Ihr Array in einer Instanz erstellt haben, können Sie es als Referenz an ein Skript übergeben und den Accessor verwenden @ um es direkt zu ändern. Zum Beispiel würden Sie das Array erstellen und das Skript wie folgt aufrufen:

array[99] = 0;
scr_Array_Populate(array);

Das Skript selbst würde so etwas haben:

var a = argument0; var i = 0; repeat(25)
   {
   i = irandom(99);
   while (a[i] != 0)
      {
      i = irandom(99);
      }
   a[@ i] = 100;
   }

Das ganze Skript macht 25 zufällige Positionen im Array und setzt den Wert der gewählten Array-Position auf 100.
