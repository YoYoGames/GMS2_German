Ausdrücke


Ein Ausdruck ist ein mathematischer Ausdruck, der gewöhnliche Zahlen, Variablen, Strings oder GML-Funktionen sowie einen oder mehrere Operatoren (wie add, subtract, multiplizieren usw.) enthalten kann. Die in einem Ausdruck verwendeten Werte können reelle Zahlen (z. B. 3,4 * 6), hexadezimale Zahlen sein, die mit einem $ -Zeichen beginnen (z. B. $00FFAA | $88FFAA ), Strings zwischen Anführungszeichen (z. B. "Hallo" + "Welt") oder komplizierter Ausdrücke mit mehreren Operatoren und Werten.

Für Ausdrücke gibt es folgende Operatoren (in der Reihenfolge ihrer Priorität):

= - Wird verwendet, um einer Variablen einen Wert zuzuweisen. Beachten Sie, dass dies auch zum Vergleichen von Variablen in GameMaker Studio 2 kann. Dies kann in Beispielen und anderen GameMaker Studio 2 angezeigt werden. Dies ist jedoch ein Erbe aus alten GameMaker Versionen. Sie sollten die Operatoren == zum Vergleichen und = zum Zuweisen verwenden, wie in den folgenden Beispielen gezeigt:

a = 12;
speed = 5;
val = (old_val + 5);


&&, ||, ^^ (und, oder und xor) - Kombinieren Sie boolesche Werte, um entweder "true" oder "false" zu erhalten. Wenn sich eines der folgenden Beispiele in löst true dann würde der Code ausgeführt werden:

if (a == b && c == d) {do something...}// and
if (a == b || c ==d) {do something...}// or
if (a == b ^^ c == d) {do something...} // xor


<, <=, ==,! =,>,> = - Dies sind Vergleiche und können nur ein geben true oder false result (wobei true auch als 1 und false als 0 interpretiert werden kann). Anwendungsbeispiele:

if (a < b) {do something...}
if (a != b) {do something...}


|, &, ^, <<, >>: Mit diesen können Sie bitweise Operationen ausführen, wobei | = bitweise oder, & = bitweise und, ^ = bitweise xor, << = nach links verschieben, >> = nach rechts verschieben. Anwendungsbeispiele:

x = (x & $ffffffe0) + 32;
if (y ^ $1f) > 0 {do something...};


Weitere Informationen zur Verwendung der bitweisen Operatoren und ihrer Funktionsweise finden Sie im Abschnitt: Bitweise Operatoren.

+, -, *, / - Addiere, subtrahiere, multipliziere und dividiere. Anwendungsbeispiele:

c = a * b;
str = a + "world";


++, - - Addiere oder subtrahiere einen Wert. Es ist erwähnenswert, dass die Platzierung vor oder nach dem Wert, zu dem ein Wert hinzugefügt oder von diesem abgezogen werden soll, etwas andere Ergebnisse hat. Zum Beispiel:

  • ++ a erhöht die Variable und gibt den inkrementierten Wert zurück.
  • a ++ erhöht die Variable, gibt jedoch den Wert zurück, bevor er erhöht wurde.
Deshalb, wenn Sie so etwas haben:

var a = 1;
show_debug_message(string(a++));
show_debug_message(string(++a));


Die Debug-Ausgabe wäre 1 und 3. Hier einige Anwendungsbeispiele:

for (var i = 0; i < 10; i++;) {do something...};
if hit == true score--;


HINWEIS: Auf den Zielplattformen des YoYo Compilers (mit den markierten (YYC)) werden diese Ausdrücke von links nach rechts ausgewertet, während sie auf allen anderen Zielplattformen von rechts nach links ausgewertet werden.

val = max(num, ++num, num++);


gibt je nach Plattform unterschiedliche Ergebnisse.

div, mod (%) - Division und Modulo, wobei div gibt den Betrag an, den ein Wert so einteilen kann, dass nur ein ganzzahliger Quotient erzeugt wird mod gibt Ihnen nur den Rest einer Division. Beachten Sie, dass Sie nur können div oder mod mit ganzzahligen Werten. Anwendungsbeispiele:

secs = time mod 60;
time_str = string(time div 60);


Es gibt auch die folgenden unären Operatoren:
  • !: boolean "nicht", so !true == false
  • -: negiert den nächsten reellen oder ganzzahligen Wert (nicht gültig für Strings oder Booleans)
  • ~: negiert den nächsten Wert bitweise

Als Werte in allen Ausdrücken können Sie Zahlen, Variablen oder Funktionen verwenden, die einen Wert zurückgeben. Unterausdrücke können auch in Klammern stehen. Alle Operatoren arbeiten für echte Werte, Vergleiche funktionieren jedoch auch für Strings und "+" verkettet Strings.


Reihenfolge der Operationen

Wenn Sie mehrere Vorgänge in einem Ausdruck ausführen, ist es sehr wichtig, dass Sie die Reihenfolge der Vorgänge in eckige Klammern () einfügen, da diese von unterschiedlichen Plattformen unterschiedlich ausgeführt werden können, sofern dies nicht ausdrücklich angegeben ist. Betrachten Sie zum Beispiel den folgenden Code:

a = b == c || d;

Die verschiedenen Zielcompiler führen die Operationen in unterschiedlicher Reihenfolge aus, da sie nicht explizit angezeigt werden. Dies führt zu "ungeraden" Ergebnissen, die Sie möglicherweise nicht erwarten, wenn Sie Ihr Spiel spielen. Um dies zu vermeiden, trennen Sie die Teile mit der Taste () wie folgt:

a = (b == c || d); //better
a = ((b == c) || d); //best

Hier sind einige abschließende Beispiele für die verschiedenen Ausdrücke:

{
x = 23;
colour = $FFAA00;
str = "hello world";
y += 5;
x *= y;
x = y << 2;
x = 23 * ((2 + 4) / sin(y));
str = "hello" + "world";
b = (x < 5) && !(x==2 || x==4);
}


Accessoren

Es gibt auch den Ausdruck "Abkürzungen", die als Accessoren bezeichnet werden, die für bestimmte Datenstrukturen und Arrays verwendet werden. Damit können Sie Daten innerhalb dieser Formate schnell und einfach hinzufügen oder ersetzen, ohne dass Funktionsaufrufe erforderlich sind.
