Ausdrücke


Ein Ausdruck ist eine mathematische Phrase, die gewöhnliche Zahlen, Variablen, Strings oder GML-Funktionen sowie einen oder mehrere Operatoren (wie Addieren, Subtrahieren, Multiplizieren usw.) enthalten kann. Die in einem Ausdruck verwendeten Werte können reelle Zahlen (zB 3,4 * 6), hexadezimale Zahlen, die mit einem $ -Zeichen (zB $00FFAA | $88FFAA ) beginnen, Strings zwischen doppelten Anführungszeichen (zB "Hallo" + "Welt") oder komplizierter sein Ausdrücke, die mehrere Operatoren und Werte verwenden.

Für Ausdrücke existieren die folgenden Operatoren (in der Reihenfolge ihrer Priorität):

= - Wird verwendet, um einer Variablen einen Wert zuzuweisen. Beachten Sie, dass dies auch für den Vergleich von Variablen in GameMaker Studio 2 kann. Sie können dies in Beispielen und anderen Codes sehen. Dies ist jedoch ein Vermächtnis alter GameMaker Versionen und Sie sollten die Operatoren == zum Vergleichen und = zum Zuweisen verwenden, wie in diesen Beispielen gezeigt:

a = 12;
speed = 5;
val = (old_val + 5);


&&, ||, ^^ (und, oder und xor) - Boolesche Werte kombinieren, um entweder wahr oder falsch zu ergeben. Wenn eines der folgenden Beispiele aufgelöst wird true dann würde der Code ausgeführt werden:

if (a == b && c == d) {do something...}// and
if (a == b || c ==d) {do something...}// or
if (a == b ^^ c == d) {do something...} // xor


<, <=, ==,! =,>,> = - Dies sind Vergleiche und können nur einen geben true oder false Ergebnis (wobei true auch als 1 und false als 0 interpretiert werden kann). Anwendungsbeispiele:

if (a < b) {do something...}
if (a != b) {do something...}


|, &, ^, <<, >>: Mit diesen können Sie bitweise Operationen ausführen, wobei | = bitweise oder, & = bitweise und, ^ = bitweise xor, << = nach links verschieben, >> = nach rechts verschieben. Anwendungsbeispiele:

x = (x & $ffffffe0) + 32;
if (y ^ $1f) > 0 {do something...};


Weitere Informationen zur Verwendung der bitweisen Operatoren und ihrer Funktionsweise finden Sie im Abschnitt: Bitweise Operatoren.

+, -, *, / - Addieren, subtrahieren, multiplizieren und dividieren. Anwendungsbeispiele:

c = a * b;
str = a + "world";


++, - - Addiere oder subtrahiere eins von einem Wert. Es ist erwähnenswert, dass die Platzierung vor oder nach dem hinzuzufügenden oder zu subtrahierenden Wert leicht unterschiedliche Ergebnisse hat. Beispielsweise:

  • ++ a erhöht die Variable und gibt den inkrementierten Wert zurück.
  • a ++ erhöht die Variable, gibt aber den Wert zurück, bevor sie inkrementiert wurde.
Wenn Sie also etwas haben:

var a = 1;
show_debug_message(string(a++));
show_debug_message(string(++a));


Die Debug-Ausgabe wäre 1 und 3. Hier sind einige Anwendungsbeispiele:

for (var i = 0; i < 10; i++;) {do something...};
if hit == true score--;


HINWEIS: Auf den Zielplattformen des YoYo Compilers (die mit (YYC) gekennzeichnet sind) werden diese Ausdrücke von links nach rechts ausgewertet, während sie auf allen anderen Zielplattformen von rechts nach links ausgewertet werden, was bedeutet:

val = max(num, ++num, num++);


Je nach Plattform ergeben sich unterschiedliche Ergebnisse.

div, mod (%) - Division und modulo, wobei dir div die Menge gibt, in die ein Wert geteilt werden kann, um nur einen ganzzahligen Quotienten zu erzeugen, während mod dir nur den Rest einer Division gibt. Anwendungsbeispiele:

secs = time mod 60;
time_str = string(time div 60);


Außerdem gibt es die folgenden unären Operatoren:
  • !: boolean "nicht", also !true == false
  • -: negiert den nächsten reellen oder ganzzahligen Wert (nicht gültig für Strings oder Booleans)
  • ~: negiert den nächsten Wert bitweise

Als Werte in allen Ausdrücken können Sie Zahlen, Variablen oder Funktionen verwenden, die einen Wert zurückgeben, und Unterausdrücke können auch in Klammern gesetzt werden. Alle Operatoren arbeiten für reale Werte, aber Vergleiche funktionieren auch für Strings und "+" verkettet Strings.


Reihenfolge der Operationen

Wenn Sie mehrere Operationen in einem einzelnen Ausdruck ausführen, ist es sehr wichtig, dass Sie eckige Klammern () verwenden, um die Reihenfolge der Operationen voneinander zu trennen, da verschiedene Plattformen sie möglicherweise anders ausführen, wenn sie nicht explizit auf diese Weise angegeben werden. Betrachten Sie beispielsweise den folgenden Code:

a = b == c || d;

Die verschiedenen Ziel-Compiler führen die Operationen in verschiedenen Reihenfolgen durch, da sie nicht explizit gezeigt werden, was zu "seltsamen" Ergebnissen führt, die Sie nicht erwarten können, wenn Sie Ihr Spiel spielen. Um dies zu vermeiden, verwenden Sie die (), um die Teile wie folgt zu trennen:

a = (b == c || d); //better
a = ((b == c) || d); //best

Hier sind einige abschließende Beispiele für die verschiedenen Ausdrücke:

{
x = 23;
colour = $FFAA00;
str = "hello world";
y += 5;
x *= y;
x = y << 2;
x = 23 * ((2 + 4) / sin(y));
str = "hello" + "world";
b = (x < 5) && !(x==2 || x==4);
}


Accessoren

Es gibt auch Ausdrücke "Abkürzungen", die Accessoren für die Verwendung mit bestimmten Datenstrukturen und Arrays genannt werden, die es Ihnen ermöglichen, Daten innerhalb dieser Formate schnell und einfach und ohne die Verwendung von Funktionsaufrufen hinzuzufügen oder zu ersetzen.
