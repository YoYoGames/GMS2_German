Skripte


Skripts sind im Wesentlichen Funktionen, die Sie selbst als kurze Codeschnipsel schreiben, die Ausdrücke, Rückgabewerte oder alles andere, was die Sprache erlaubt, auflösen können. Wenn Sie einen Codeblock haben, den Sie an mehr als einem Ort oder Objekt verwenden, ist es wahrscheinlich besser in einem Skript, da Sie es bei Bedarf einmal ändern können und die Änderung von jedem "abgeholt" wird Objekt, das einen Aufruf an das Skript hat. Sie können auch aus organisatorischer Sicht sehr praktisch sein, auch wenn der Code, den sie haben, nur einmal von einem einzigen Objekt aufgerufen wird, da Sie große Codeblöcke an logischeren Punkten in leichter zu verwaltende "Brocken" zerlegen können.

Wenn Sie ein Skript erstellen, werden die Werte, die Sie übergeben möchten, als Argumente bezeichnet, und Sie möchten auf diese Argumente zugreifen - entweder bei Verwendung der Skriptaktion oder beim Aufruf des Skripts als Funktion eines Programms oder eines anderen Skripts. Diese Argumente werden in den eingebauten Variablen gespeichert:

argument0, argument1, ..., etc... up to argument15

Es können also maximal 16 Argumente an ein Skript übergeben werden, wenn diese aus einem Code mit den Argumentvariablen aufgerufen werden, und wenn der argument0...15 Variablen für ein Skript müssen Sie sicherstellen, dass das Skript alle angegebenen Argumente verwendet. Wenn Sie zum Beispiel ein Skript haben, das zwei Argumente akzeptiert und Sie nur eins angeben, erhalten Sie einen Fehler für dieses Skript. Das Gleiche gilt, wenn Sie mehr Argumente angeben, als Sie benötigen.

Sie können jedoch eine variable Anzahl von Argumenten an ein Skript übergeben, das das integrierte Argumentarray verwendet, und Sie sind nicht auf maximal 16 Argumente beschränkt, wenn Sie dieses Array verwenden, sondern können so viele haben, wie Sie benötigen (auch wenn Sie müssen sicherstellen, dass alle Argumente innerhalb des Skripts referenziert werden):

argument[0 ... max_num]

Wenn Sie eine variable Anzahl von Argumenten als Array von Werten übergeben, können Sie die folgende Funktion verwenden:

  • argument_count

Dies kann verwendet werden, um herauszufinden, wie viele Argumente übergeben wurden, und das Skript so anzupassen, dass nur die angegebenen Argumente verwendet werden.

HINWEIS: Sie können die beiden Arten von Argumentvariablen beim Aufrufen eines Skripts nicht mischen. Sie müssen beides verwenden argument0...15 oder Argument [n].

Skripts können auch einen Wert zurückgeben, sodass sie in Ausdrücken verwendet werden können. Dafür würdest du das verwenden return Erklärung:

return <expression>

Es sollte beachtet werden, dass die Ausführung des Skripts bei der return-Anweisung endet, was bedeutet, dass kein Code ausgeführt wird, der nach dem Aufruf der Rückgabe aufgerufen wird. Hier ist ein kurzes Beispielskript mit dem Namen "scr_sqr", das das Quadrat von jedem Wert berechnet, der an es übergeben wird, und es enthält einen Fehlerfang, falls das Argument, dass es übergeben wird, keine reelle Zahl ist:

{
if !is_real(argument0)
   {
   return 0;
   }
else
   {
   return (argument0 * argument0);
   }
}

Um ein Skript aus einem Codeabschnitt heraus aufzurufen, gehen Sie genauso vor wie beim Aufrufen von Funktionen - schreiben Sie den Skriptnamen mit den Argumentwerten in Klammern. Also würde das obige Skript so heißen:

if keyboard_check_pressed(vk_enter)
   {
   val = scr_sqr(amount);
   }

Es ist erwähnenswert, dass Sie beim Schreiben Ihrer eigenen Skripts auch bestimmte Kommentare im JSDoc- Stil hinzufügen können, sodass sie bei der Verwendung in Ihrem Code zusammen mit ihren Argumenten und anderen Details automatisch vervollständigt werden. Weitere Informationen hierzu finden Sie im Abschnitt JSDoc-Skriptkommentare. Beachten Sie außerdem, dass Sie, wenn Sie Ihre Skripte im Code-Editor eingeben, auf klicken können oder benutze die mittlere Maustaste auf den Skriptnamen, um ihn zum direkten Bearbeiten zu öffnen.
