Funktionen


Die allgemeine Definition einer Funktion in GML ist etwa so:

Eine Funktion hat einen Eingang und einen Ausgang und der Ausgang ist irgendwie mit dem Eingang verbunden.

In GameMaker Studio 2 Sie Funktionen als Teil der Code-Struktur, um Dinge in Ihrem Spiel zu ermöglichen, und Ihnen steht eine große Anzahl von GML-Funktionen zur Verfügung, die im Abschnitt " Sprachreferenz " des Handbuchs erläutert werden.

In GML hat eine Funktion die Form eines Funktionsnamens, gefolgt von den Eingabeargumenten in Klammern und durch Kommas getrennt (wenn die Funktion keine Eingabeargumente hat, werden nur Klammern verwendet). Hier ist ein Überblick über die Struktur einer Funktion:

<function>(<arg0>, <arg1> ,... <arg15>);

Im Allgemeinen gibt es zwei Arten von Funktionen - erstens gibt es eine riesige Sammlung von eingebauten Funktionen, die verwendet werden, um alle Aspekte deines Spiels zu kontrollieren, und zweitens können alle Skripte, die du in deinem Spiel definierst, auch als Funktion verwendet werden ( aber nicht immer).

Einige Funktionen geben Werte zurück und können in Ausdrücken verwendet werden, während andere einfach Befehle ausführen, wie in den folgenden zwei Beispielen veranschaulicht:

instance_destroy();                              // Destroy the calling instance - this requires no arguments and returns nothing
dist = point_distance(x, y, mouse_x, mouse_y;    // Get the distance from the current instance position to the mouse position - takes four arguments and returns a real value

Sie sollten beachten, dass es unmöglich ist, eine Funktion selbst als linke Seite einer Zuweisung zu verwenden. Beispielsweise würde der folgende Code Ihnen einen Fehler geben:

instance_nearest(x, y, obj).speed = 0;

Der Rückgabewert für den Ausdruck in diesem Codebeispiel ist eine reelle Zahl (der eindeutige ID-Wert für die nächstgelegene Instanz). Daher muss er in eckigen Klammern eingeschlossen sein, um die erforderliche Instanz korrekt adressieren zu können (siehe Variablen in anderen adressieren) Instanzen für weitere Informationen). Der obige Code wäre korrekt geschrieben als:

(instance_nearest(x, y, obj)).speed = 0;
//or
var inst = instance_nearest(x, y, obj);
inst.speed = 0;
