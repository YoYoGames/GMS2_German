Variablen und variabler Umfang


Wie jede Programmiersprache verwendet GML Variablen als Basiseinheit für die meisten Programmiervorgänge. Variablen werden verwendet, um Informationen im Gerätespeicher für eine spätere (oder sofortige) Verwendung zu speichern, und sie erhalten einen Namen, so dass Sie in Funktionen und Skripten darauf verweisen können. Eine Variable in GML kann viele verschiedene Datentypen speichern, wie eine reelle Zahl (wie 100, 2.456575, -56 usw.), eine Zeichenkette (wie "Hallo Welt!"), Eine Ganzzahl (wie 1, 556, -7 ) oder ein Boolescher ( true oder false ).

Eine Variable ist etwas, das wir verwenden, um einen Wert für die Verwendung in einer oder mehreren Operationen oder Funktionen zu speichern. Denken Sie an "pi", zum Beispiel... es ist eine reale Weltvariable, die den Wert 3.14159265 (etc...) enthält. Warum? Nun, es ist viel einfacher, jemandem "Pi" zu sagen als "Dreipunkt eins vier eins fünf neun zwei sechs fünf"! Also, die Dinge zu benennen macht das Leben viel einfacher und es bedeutet auch, dass sollte der Wert dieser Variablen jemals ändern, wir müssen nicht überall die Nummer ändern, wie der Variablenname immer noch das gleiche ist. In GML hat eine Variable einen Namen, der mit einem Buchstaben beginnen muss und nur Buchstaben, Zahlen und das Unterstrichsymbol '_' mit einer maximalen Länge von 64 Zeichen enthalten darf. Also, gültige Variablen sind Dinge wie fish, foo_bar, num1 und ungültige Variablen wären 6fish, foo bar, oder *num.

Jetzt müssen Sie in vielen Programmiersprachen eine Variable "deklarieren", bevor Sie sie verwenden können. Das bedeutet im Grunde, dass Sie dem Computer den Namen geben, den Sie verwenden möchten, damit er einen Speicherplatz im Speicher zuweisen kann, um den Wert zu speichern, den Sie in dieser Variablen speichern möchten. Bei GML ist das nicht immer notwendig, da es vom Gültigkeitsbereich der Variablen abhängt. Es gibt vier Hauptvariablenkategorien, wenn Sie mit GameMaker Studio 2 programmieren, und jeder hat seinen eigenen Bereich (der als sein Bereich oder seine Reichweite angesehen werden kann). Diese Variablen und ihr Umfang sind nachstehend aufgeführt:

Eine Instanzvariable wird in einer Instanz eines Objekts erstellt und gilt als eindeutig für diese Instanz - dh viele Instanzen desselben Objekts können die gleiche Variable haben, aber jede Variable kann einen anderen Wert enthalten, da sie für jede Instanz eindeutig sind. Aber wie wird eine Instanzvariable erstellt? Nun, Sie erstellen neue Variablen, indem Sie ihnen einfach einen Wert zuweisen, wie in diesem kleinen Beispiel gezeigt:

potions = 12;
life = 100;
name = "Jock MacSweeney";
strength = 5.5;
armour = -2;

Wie Sie sehen können, müssen Sie nur den Namen und dann einen Wert (entweder numerisch oder eine Zeichenfolge) angeben, um diese Variable festzulegen und sie in einer Instanz des Objekts, für das Sie codieren, zu verwenden. Diese Variablen können dann auf verschiedene Arten innerhalb der Instanz verwendet und modifiziert werden, zum Beispiel könnte dieser Code in einem Kollisionsereignis sein und dazu verwendet werden, einen Betrag von der Variablen "life" zu nehmen:

life -= 5 + armour;

Wenn "life" auf 100 steht, hat es jetzt einen Wert von 97 (100 - (5 + -2) = 97). Nun, das ist ein einfaches Beispiel, und Sie könnten "Rüstung" für den tatsächlichen Wert von -2 ersetzen, aber was passiert, wenn Sie diesen Wert an mehreren Stellen verwenden und dann entscheiden, ihn zu ändern? Du müsstest ALLEN Code durchgehen und alle -2 zu dem neuen Wert ändern, was zeitaufwendig und sehr fehleranfällig ist! Aber wenn Sie eine Variable verwenden, müssen Sie ihr lediglich einen neuen Wert zuweisen, und der Code wird ab diesem Zeitpunkt automatisch diesen neuen Wert verwenden, wodurch die Dinge wesentlich flexibler und einfacher zu beheben sind, sollte ein Problem auftreten. Es sollte auch angemerkt werden, dass, selbst wenn sich ein Wert nicht ändert, es viel einfacher ist sich daran zu erinnern, was eine Variable namens "Leben" bedeutet, anstatt nur auf eine Zahl zu schauen.

GameMaker Studio 2 gibt es auch eine Sammlung von "eingebauten" Instanzvariablen, Sie sollten sich also bewusst sein, dass Sie eine Ihrer eigenen Instanzvariablen gleich nennen oder eine eigene globale Variable mit demselben Namen haben möchten und sich fragen, warum bekomme Fehler. Sie sind jedoch leicht zu erkennen, da sie im Code-Editor in einer anderen Farbe angezeigt werden und in der Leiste für die automatische Vervollständigung angezeigt werden.

Es gibt eine Reihe von Funktionen, die Ihnen beim Umgang mit Instanzvariablen helfen sollen (hauptsächlich für die Verwendung mit Compatibility Scripts für importierte Projekte und den Code für Drag & Drop-Aktionen, sie können jedoch auch anderswo verwendet werden):

  • variable_instance_exists
  • variable_instance_get
  • variable_instance_set
  • variable_instance_get_names

Eine lokale Variable ist eine, die wir für ein bestimmtes Ereignis erstellen nur dann verwerfen, wenn das Ereignis (die einzige Ausnahme ist in den Script - Ressourcen beendet hat, wo ein var deklarierte Variable ist lokal für das Skript und wird dann verworfen). Warum sollten wir sie brauchen? Nun, Variablen nehmen Platz im Speicher ein und es kann sein, dass wir sie nur für eine Operation oder Funktion verwenden. In diesem Fall müssen wir sie nur für die kurze Zeit im Speicher haben, in der sie verwendet wird. Dies hält Ihre Code-Basis sauber und ordentlich und sorgt dafür, dass Speicherplatz für die Dinge, die ihn wirklich brauchen, optimiert wird. Um eine lokale Variable zu deklarieren, verwenden wir die Funktion var so was:

var i, num, str;
i = 0;
num = 24.5;
str = "Hi there!";

Alle oben erstellten Variablen werden am Ende des Ereignisses (oder Skripts), in dem sie erstellt wurden, "vergessen" (dh aus dem Speicher entfernt). Sie müssen vorsichtig sein, dass der Name Sie geben var deklarierte Variablen stimmen nicht mit einer anderen Instanzvariablen innerhalb des Objekts überein, das den Code ausführt, und stellen auch sicher, dass Sie nicht beabsichtigen, den in dieser Variablen gespeicherten Wert außerhalb des Ereignisses zu verwenden. Deklarieren Sie diese Variablen. insbesondere in Schleifen zum Zählen von Iterationen oder wenn Sie einen Wert mehrfach in einem Vorgang verwenden, der nicht noch einmal wiederholt wird. Hier sind noch ein paar Beispiele:

var i = 0;
repeat (10)
   {
   inventory[i] = 0;
   i+=1;
   }

Der obige Code erstellt eine lokale Variable namens "i" und setzt sie auf 0, alle in derselben Zeile. Beachten Sie, dass Sie in früheren Versionen von GameMaker zuerst Ihre lokalen Variablen deklarieren und ihnen dann Werte zuweisen mussten. In dieser Version können Sie jedoch gleichzeitig einen Wert deklarieren und zuweisen. Der obige Code verwendet dann diese Variable, um ein Array zu initialisieren. Da die Variable "i" für keine weiteren Funktionen in der anderen Instanz verwendet wird, kann sie lokal im Gültigkeitsbereich sein. Hier ist ein weiteres Beispiel:

var xx,yy;
xx = x - 32 +irandom(64);
yy = y - 32 +irandom(64);
instance_create_layer(xx, yy, "Effects", obj_blood);

Hier haben wir die lokalen Variablen verwendet, um einige zufällige Koordinaten zu speichern, die wir dann verwenden, um eine Instanz zu erstellen. In diesem Beispiel können Sie sehen, dass es nicht unbedingt notwendig ist, dass wir diese Variablen verwenden, aber aus Gründen der Lesbarkeit und Benutzerfreundlichkeit tun wir dies. Es ist VIEL klarer und offensichtlicher, was wir dort tun, als wenn wir diesen Code verwenden würden:

instance_create_layer(x - 32 + irandom(64), y - 32 + irandom(64), "Effects", obj_guts);

Eine andere Sache über var deklarierte Variablen sollten beachtet werden... Da sie für das Ereignis, das sie ausführt, einzigartig sind, können sie auch in anderen Instanzen durch Code verwendet werden! Dies bedeutet, dass wir diese Variablen verwenden können, um Dinge in anderen Instanzen mithilfe des Konstrukts "with ()" festzulegen und zu ändern (es gibt einen Abschnitt dazu im Abschnitt GML-Übersicht des Handbuchs). Der eigentliche Code selbst würde in etwa so aussehen:

var num = instance_number(obj_Enemy);
with (obj_Enemy)
   {
   if num>10 instance_destroy();
   }

Der obige Code funktioniert, weil der var Die deklarierte Variable ist lokal für das Ereignis (oder Skript), in dem sie enthalten ist, nicht für die Instanz oder die Spielwelt, und kann daher in jeder Funktion in jedem Objekt verwendet werden, solange sie sich im selben Codeblock befindet.


Eine grundlegende Beschreibung einer globalen Variablen ist eine, die, wenn sie einmal deklariert wurde, keiner bestimmten Instanz gehört und dennoch für alle zugänglich ist. Genau wie lokale Variablen müssen globale Variablen deklariert werden, aber im Gegensatz zu einer lokalen Variablen bleibt eine globale Variable bis zum Ende des Spiels im Speicher. Sie können also eine globale Variable erstellen, um zum Beispiel die Anzahl der Kugeln zu überwachen, die der Spieler hat, und diese Variable dann an verschiedenen Punkten im Spiel aktualisieren. Eine globale Variable gehört keiner bestimmten Instanz an und kann von allen Instanzen jederzeit aufgerufen, geändert und verwendet werden. Jede Änderung an der Variablen ist jedoch "global", da alle Instanzen, die die Variable verwenden, von der Änderung betroffen sind. Schauen wir uns ein Beispiel an:

global.food = 5;

Wir deklarieren die Variable "food", indem wir zunächst "global" und dann ein "." um GameMaker Studio 2 mitzuteilen, dass diese Variable jetzt globaler Geltungsbereich ist. Wir müssen diese Methode von nun an immer dann verwenden, wenn wir auf diese Variable zugreifen oder sie ändern müssen. Also haben wir eine neue Variable namens "food" erstellt und wir haben sie als global deklariert. Nun kann jede Instanz diese Variable verwenden und ändern, und alle anderen Instanzen "sehen" dies. Zum Beispiel könnten wir ein anderes Essensobjekt haben, mit dem der Spieler kollidiert und in dem Kollisionsereignis haben wir:

global.food +=1;

Wir haben auch ein anderes Objekt, das diesen Wert wie folgt zeichnet:

draw_text(32, 32, "food = " + string(global.food));

Mit globalen Variablen können wir Werte ändern und diese Änderungen in allen Instanzen der Objekte sehen, die auf diese Variable verweisen. Wie bei lokalen Variablen müssen Sie darauf achten, dass Ihre globalen Variablen nicht mit den Instanzvariablen identisch sind, da dies zu Problemen führt und aufgrund variabler Überlappungen zu Fehlern in Ihren Spielen führt, die manchmal schwierig zu debuggen sind. Im Allgemeinen sollten Sie ein einzelnes Skript oder Objekt haben, das alle Ihre globalen Variablen gleich zu Beginn des Spiels deklariert (zum Beispiel im Room Start Event), so dass sie initialisiert werden und für jede Instanz bereit sind, und dies auch gibt Ihnen einen praktischen Ort, um zurück zu gehen und Referenz, wenn Sie einen Variablennamen überprüfen müssen.

GameMaker Studio 2 hat auch eine Sammlung von "eingebauten" globalen Variablen, also sollten Sie sich dessen bewusst sein, da Sie eine Ihrer Instanzvariablen gleich nennen oder eine eigene globale Variable mit dem gleichen Namen haben möchten und sich fragen, warum Fehler bekommen! Sie sind jedoch leicht zu erkennen, da sie im Code-Editor in einer anderen Farbe angezeigt werden und in der Leiste für die automatische Vervollständigung angezeigt werden. Die meisten integrierten globalen Variablen sind sehr spezifisch und werden nur selten verwendet - und sind in den entsprechenden Abschnitten des Handbuchs aufgeführt. Es gibt jedoch einen wichtigen, der häufig verwendet wird und nicht an anderer Stelle aufgeführt ist:

  • async_load

Es gibt auch drei veraltete integrierte globale Variablen, die Sie beachten sollten (diese Variablen sind nur dazu gedacht, ältere Projekte aus früheren Versionen von GameMaker und sollten nicht in neuen Projekten verwendet werden ):

  • score
  • health
  • lives

Die folgende Methode kann auch zum Deklarieren von globalen Variablen verwendet werden, sie ist jedoch nur aus Gründen der Abwärtskompatibilität enthalten. Es wird nicht empfohlen, diese Methode für neue Projekte zu verwenden, da zukünftige Versionen von GameMaker Studio 2 dies möglicherweise nicht unterstützen.

Die zweite Methode zum Erstellen globaler Variablen besteht darin, sie als solche zu deklarieren globalvar Deklaration, ähnlich wie eine lokale Variable mit der var Erklärung.

WICHTIG! Das globalvar Die Deklaration ist veraltet und wird nur für ältere Zwecke unterstützt. Du solltest immer das verwenden global. Bezeichner zum Markieren globaler Variablen.


Diese (veraltete) Deklaration würde wie folgt verwendet:

globalvar food;
food = 5;

Einmal auf diese Weise deklariert, wird diese Variable "Nahrung" nun als global betrachtet und erfordert keine global. Präfix - was auch bedeutet, dass es viel schwieriger ist, globale Variablen in Ihrem Code zu identifizieren, und es ist auch viel einfacher, variable Überlappungen zu erhalten, wenn Sie denselben Variablennamen in verschiedenen Objekten oder von von Ihnen installierten Erweiterungen verwenden. Sobald auf diese Weise deklariert, wird auf die globale Variable wie folgt zugegriffen:

food += 2;

oder:

draw_text(32, 32, "food = " + string(food));

Wie Sie sehen können, ohne dass Sie nachweisen können, dass die Variable einen globalen Umfang hat, stellen Sie sich potentiell darauf ein, dass viele subtile Probleme in Ihrem Spiel auftauchen, weshalb diese Deklaration vermieden werden sollte.

  • variable_global_exists
  • variable_global_get
  • variable_global_set

Eingebaute Variablen sind spezielle Variablen, die in die Objekte und die Räume in der Spielwelt "eingebaut" werden und sie können entweder Instanz oder global im Bereich sein (aber niemals lokal). Einige globale Variablen sind in dem obigen Abschnitt aufgeführt, und die verschiedenen Abschnitte des Handbuchs für Sprites, Räume, Objekte usw. umreißen auch die verfügbaren eingebauten Variablen. Beispiele für solche integrierten Instanzvariablen wären:

  • sprite_index
  • path_scale
  • speed

Beispiele für integrierte globale Variablen wären:

  • view_xport
  • GM_version
  • room_speed

Integrierte Variablen können wie die meisten anderen Variablen geändert und gesetzt werden. Einige können sogar Arrays sein, nur müssen Sie sie nicht so setzen, dass sie wie normale Variablen erstellt werden, da sie bereits auf einen Standardwert initialisiert werden.


Obwohl sie nicht genau Variablen sind, ähneln Makros ihnen in ihrer Verwendung, dh sie sind benannte Werte, die Sie im gesamten Code verwenden können, um harte Werte zu ersetzen. Grundsätzlich ist ein Makro (auch als Konstante bezeichnet ) eine benannte Variable, die einen konstanten einzelnen Wert (oder String) enthält. Sie können Ihre eigenen Konstanten mit dem Skript-Editor definieren und sie dann in Ihrem Code und DnD verwenden, als wären sie reguläre Variablen, mit dem Unterschied, dass sie im Spiel nicht geändert werden können. Angenommen, Sie definieren das folgende Makro als Konstante (beachten Sie das vorangehende "#" und das Fehlen eines Doppelpunkts ";" am Ende):

#macro total_weapons 10

Sie würden dies dann in Ihrem Code so nennen:

if ++pos == total_weapons
   {
   pos = 0;
   }

Beachten Sie, dass Sie den konstanten Wert nicht ändern können. Code wie dieser führt zum Absturz des Spiels:

total_weapons = 11;

Sie können ein Makro irgendwo in Ihrem Code oder in Ihren Skripten definieren. Es wird vorkompiliert und in Ihr Spiel eingebunden, als wäre es von Anfang an vorhanden. Wir empfehlen jedoch, eine dedizierte Skript-Ressource zu erstellen und alle darin enthaltenen Makros zu definieren. Es wird einfacher zu organisieren und später zu debuggen!

Wenn Sie den Wert eines Makros zur Laufzeit ändern möchten, sollten Sie es wahrscheinlich zu einer globalen Variable machen, da diese während eines Spiels vom Code aus geändert werden können, es sei denn, Sie legen das Makro als Funktion fest. Wenn Sie das Makro auf eine Funktion setzen, bedeutet dies, dass diese Funktion jedes Mal aufgerufen wird, wenn Sie das Makro verwenden. Beispielsweise:

#macro col make_colour_hsv(irandom(255), 255, 255)

Sie würden dieses Makro dann so etwas nennen:

image_blend = col;

Wenn Sie diesen Code verwenden, wird das Bild bei jeder Verwendung des Makros mit einer anderen Farbe gemischt.
