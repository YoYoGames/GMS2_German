Kommentare im Code


Bei der Zusammenarbeit mit anderen Personen, beim Umgang mit einem großen Projekt oder auch nur für Ihre eigene Referenz und das Debuggen ist es sehr wichtig, Kommentare in Ihren Codes und Skripten zu hinterlassen. In GameMaker Studio 2 Ihnen verschiedene Mechanismen zur Verfügung, mit denen Sie Notizen und Kommentare zu Codeabschnitten und sogar ganzen Codeblöcken hinterlassen können, die in der Ereignisliste des aktuell bearbeiteten Objekts angezeigt werden. Auf diese Weise können Sie Notizen hinterlassen Sie selbst und zu Ihren Kollegen, oder um ein besonders kompliziertes Stück Code zu erklären, oder einfach eine Erinnerung darüber zu lassen, was etwas tut.

Das erste, was Sie tun können, ist einen kurzen Kommentar mit zu hinterlassen // vor dem Text. Beispielsweise:

//initialize variables
sr = 10;
hp = 100;

Sie können auch mehrzeilige Kommentare hinterlassen, um einen Kredit zu vergeben, einen vollständigen Codeabschnitt zum Debuggen wegzulassen oder sogar die Argumente eines Skripts zu erklären. Dafür können Sie verwenden /* ... */ so was:

/*
usage:
diff = angle_difference(angle1,angle2);

angle1 first direction in degrees, real
angle2 second direction in degrees, real

returns: difference of the given angles in degrees, -180 to 180

GMLscripts.com
*/
{
return ((((argument0 - argument1) mod 360) + 540) mod 360) - 180;
}

Wenn Sie schließlich benutzerdefinierte Skripts schreiben und eine gewisse Kontrolle darüber haben möchten, wie sie von der IDE behandelt werden (zB Auto-Vervollständigen oder Skript-Argumente anzeigen), sollten Sie Ihre Kommentare mit der JSDoc-Notation schreiben, wie hier erklärt.
