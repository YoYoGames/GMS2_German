Datentypen


In den vorherigen Abschnitten haben wir Variablen und ihre Scoping-Regeln behandelt, aber es wurde wenig über die verschiedenen Datentypen gesprochen, die eine Variable speichern kann. Daher werden in diesem Kapitel die verschiedenen Typen und ihre Verwendungsmöglichkeiten erläutert.

Bevor wir fortfahren, wollen wir kurz erklären, was wir unter "Datentypen" verstehen. Wenn Sie eine Variable erstellen, kann sie zum Speichern von Informationen verwendet werden. Wenn Sie eine GML- Funktion aufrufen, kann sie auch Informationen zurückgeben. Jedoch kann diese Information in verschiedenen "Geschmacksrichtungen" vorkommen, so wie sie (zum Beispiel) eine reelle Zahl sein kann oder eine Zeichenkette sein kann. Diese verschiedenen Arten von Werten werden als Datentypen bezeichnet und bei Verwendung von GML können sie eine der folgenden sein:

Eine Zeichenfolge ist einfach ein beliebiger Text, der in Anführungszeichen gesetzt wurde (entweder "..." oder "..."). Sie können bestimmte Operationen für Strings ausführen, z. B. zwei Strings zusammenfügen, um eine längere Zeichenfolge (Verkettung) zu erstellen, und Sie können auch die String-Eigenschaften ändern und sogar echte Zahlen daraus extrahieren. Weitere Informationen zu den String-Funktionen finden Sie unter: GameMaker Language Reference - Strings.


Reelle Zahlen sind beliebige Werte, die keine Zeichenfolge oder andere mögliche Datentypen sind. Also, 124, 45639, 566546456, 0, -45,5 etc... sind alle Beispiele für reelle Zahlen. Alle reellen Zahlen werden als 32-Bit-Fließkommawerte (gerade Ganzzahlwerte) gespeichert, so dass es zu leichten Rundungsfehlern kommen kann. Weitere Informationen zu dieser und anderen numberbezogenen Funktionen finden Sie unter: GML-Referenz - Reale Zahlen.

Beachten Sie, dass in GameMaker Studio 2 erstellte Variablen alle als Gleitkommazahlen mit doppelter Genauigkeit gespeichert werden. Sie können jedoch auch andere Formate verwenden, wenn Sie mit Erweiterungen arbeiten. Diese können von einer Erweiterung in GameMaker Studio 2 und dann mit dem entsprechenden Kontrollkästchen überprüft werden is_ Funktion, am Ende dieser Seite aufgeführt.


Ein Array ist ein spezieller Datentyp, der zum Speichern mehrerer Werte verwendet werden kann. Sie weisen das Array einer Variablen zu und "füllen" Sie dann verschiedene Indizes des Arrays mit Werten. Dieses Array kann dann passwortweise an Skripts und Funktionen übergeben werden. Wenn Sie jedoch das übergebene Array ändern, wird es zu einer Kopie des ursprünglichen Arrays und muss daher an die ursprüngliche Variable zurückgegeben werden damit das Array aktualisiert wird. Weitere Informationen zu Arrays finden Sie unter: GML-Übersicht - Arrays.


Ein boolescher Wert ist einfach ein Wert, der entweder sein kann true oder false. Beachten Sie, dass GameMaker Studio 2 derzeit keine "wahren" booleschen Werte unterstützt und tatsächlich jede reelle Zahl unter 0,5 als falschen Wert und jede reelle Zahl gleich (oder größer) als wahr annimmt. Dies bedeutet jedoch nicht, dass Sie 1 und 0 (oder eine andere reelle Zahl) für wahr und falsch überprüfen sollten, da Sie auch die Konstanten erhalten true und false Das sollte immer in Ihrem Code verwendet werden, um Probleme zu vermeiden, sollten echte boolesche Datentypen in einem zukünftigen Update hinzugefügt werden.


Ein Zeiger ist ein Datentyp, der auf einen Speicherort "zeigt". Sie können keine Operationen mit einem Zeiger ausführen und es wird nur für einige sehr spezifische Funktionen verwendet, wie zum Beispiel eine Textur oder Pufferadresse aus dem Speicher für eine andere Funktion zu holen. Beispiele für Funktionen, die einen Zeiger zurückgeben, den Sie sehen können buffer_get_address oder sprite_get_texture.

Es gibt auch eine Funktion, um zu überprüfen, ob ein Wert ein Zeiger ist (siehe " Überprüfen von Datentypen ", unten) und eine Funktion, um einen Wert in einen Zeiger zu konvertieren:

  • ptr

Sie können die folgenden integrierten Konstanten auch verwenden (und erhalten), wenn Sie Zeiger verwenden:

Konstante Beschreibung
pointer_null Diese Konstante zeigt an, dass der Zeiger nicht auf etwas Bedeutungsvolles zeigt (das Gleiche wie NULL in C ++ oder Null in C #)
pointer_invalid Diese Konstante bedeutet einfach, dass der Wert kein gültiger Zeiger ist

Eine Enumeration ist ein "Enumerator" und erlaubt Ihnen im Wesentlichen, einen eigenen begrenzten Datentyp mit einer Liste von konstanten Werten zu erstellen. Enums sind globale Bereichsvariablen (dies ist implizit und sie erfordern keine " global "Suffix" und sie haben folgende Struktur:

enum <variable>{<constant> [= <value>]}

Im folgenden Beispiel erstellen wir eine Aufzählung für die Farben des Regenbogens und weisen ihm verschiedene Konstanten und Standardwerte zu:

enum rainbowcolors {
   red,
   orange,
   yellow,
   green,
   blue,
   indigo,
   violet
   }

Die enum-Einträge können nur reelle Zahlen oder Ausdrücke mit früheren enums sein und sind standardmäßig von 0 aufwärts nummeriert, so dass unser oben angegebenes Beispiel standardmäßig verwendet wird red = 0, orange = 1, yellow = 2 usw. Sie können den enum-Variablen zum Zeitpunkt der Erstellung auch Werte zuweisen:

enum rainbowcolors {
   red = 5,
   orange = 5 * 2,
   yellow = 15,
   green = 20,
   blue = 25,
   indigo = 30,
   violet = 35 * enum_test.entry
   }

Beachten Sie, dass wir im obigen Beispiel eine andere Enumeration verwenden, um einen Ausdruck für "violett" zu erstellen. Dies funktioniert nur, wenn die referenzierte Enumeration vor der Enumeration erstellt wurde, die sie in einem Ausdruck verwendet, aber nicht für Variablen oder Funktionen, da der enum-Wert als Konstante beim Kompilieren ausgewertet werden kann. Beachten Sie außerdem, dass alle Aufzählungswerte zu ganzzahligen Werten ausgewertet werden, und wenn Sie eine eigene Aufzählung erstellen, sollten Sie beachten, dass nur Integer-Werte zulässig sind, damit Aufzählungen funktionieren.

Um später auf den Wert innerhalb eines bestimmten Aufzählungstyps zuzugreifen, können Sie den Punkt verwenden "." Methode, wie folgt:

var value = rainbowcolours.green;

Beachten Sie, dass Sie die Werte für eine Enum-Konstante nach dem Erstellen nicht ändern können.


Ein undefinierter Wert (auch als "Nullwert" bezeichnet) ist ein Wert, bei dem ein Ausdruck keinen korrekten Wert hat, obwohl er syntaktisch korrekt ist und daher etwas zurückgeben muss ds_map und benutze die Funktion ds_map_find_value(). Was passiert nun, wenn die Karte nicht den gesuchten Wert hat? Nun, da die Funktion korrekt formatiert ist und das Problem besteht, dass kein solcher Wert existiert, würde sie die Konstante zurückgeben undefined und Sie können nach dieser Konstante suchen, wie Sie es wünschen true oder irgendein anderer Wert.


Datentypen prüfen

  • is_string
  • ist echt
  • is_bool
  • is_array
  • is_ptr
  • is_int32
  • is_int64
  • is_vec3
  • is_vec4
  • is_matrix
  • ist nicht definiert
  • Art der
