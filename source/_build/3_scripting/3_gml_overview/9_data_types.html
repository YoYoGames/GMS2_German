Datentypen


In den vorangegangenen Abschnitten haben wir Variablen und ihre Bereichsregeln behandelt, aber zu den verschiedenen Datentypen, die eine Variable speichern kann, wurde wenig gesagt. Daher werden in diesem Kapitel die verschiedenen Typen erläutert und wofür sie verwendet werden können.

Bevor wir fortfahren, lassen Sie uns kurz erklären, was wir unter "Datentypen" verstehen. Wenn Sie eine Variable erstellen, kann sie zum Speichern von Informationen verwendet werden. Wenn Sie eine GML- Funktion aufrufen, können Sie auch Informationen zurückgeben. Diese Informationen können jedoch in verschiedenen "Geschmacksrichtungen" vorkommen, wie zum Beispiel eine reelle Zahl oder eine Zeichenfolge. Diese verschiedenen verwendeten Wertetypen werden Datentypen genannt. Wenn Sie GML verwenden, können Sie Folgendes annehmen:

Eine Zeichenfolge ist einfach ein Text, der in Anführungszeichen "..." gesetzt wurde. Sie können bestimmte Vorgänge für Zeichenfolgen ausführen, z. B. zwei Zeichenfolgen hinzufügen, um eine längere Zeichenfolge zu erstellen (Verkettung). Außerdem können Sie die Eigenschaften der Zeichenfolge ändern und sogar echte Zahlen daraus extrahieren. Weitere Informationen zu Strings und den String-Funktionen finden Sie in: GameMaker Referenzhandbuch - Strings.


Reelle Zahlen sind Werte, die keine Zeichenfolge sind, und auch keine der anderen möglichen Datentypen. 124, 45639.566546456, 0, -45.5 usw. sind Beispiele für reelle Zahlen. Alle reellen Zahlen werden als Gleitkommawerte mit doppelter Genauigkeit (sogar scheinbar ganzzahlige Werte) gespeichert, sodass bei diesen Fehlern möglicherweise Rundungsfehler auftreten. Weitere Informationen zu dieser und anderen nummerbezogenen Funktionen finden Sie unter: GML-Referenz - Reelle Zahlen.

Beachten Sie, dass in GameMaker Studio 2 erstellte Variablen zwar alle als Gleitkommazahlen mit doppelter Genauigkeit gespeichert werden, Sie können jedoch andere Formate verwenden, wenn Sie mit Erweiterungen arbeiten. Diese können von einer Erweiterung an GameMaker Studio 2 und dann mit der entsprechenden GameMaker Studio 2 überprüft werden is_ Funktion, die am Ende dieser Seite aufgeführt ist.


Ein Array ist ein spezieller Datentyp, der zum Speichern mehrerer Werte verwendet werden kann. Sie weisen das Array einer Variablen zu und "füllen" dann verschiedene Indizes des Arrays mit Werten. Dieses Array kann dann auf einer Referenz-für-Referenz-Basis an Skripts und Funktionen übergeben werden. Wenn Sie das übergebene Array jedoch ändern, wird es zu einer Kopie des ursprünglichen Arrays und muss daher wieder in die ursprüngliche Variable zurückgegeben werden für das zu aktualisierende Array. Weitere Informationen zu Arrays finden Sie unter: GML-Übersicht - Arrays.


GameMaker Studio 2 akzeptiert auch hexadezimale Literale als zulässige Werte. Hexadezimalwerte sind vor allem beim Arbeiten mit Farben üblich, können jedoch überall verwendet werden, wo ein positiver ganzzahliger Wert erforderlich ist. Beachten Sie, dass hexadezimale Werte auf eine der beiden folgenden Arten formatiert werden können:

$abcd
0xabcd

Zum Beispiel können die folgenden Werte wie gezeigt hexadezimal ausgedrückt werden:

11406 = $2c8e = 0x2c8e
16777215 = $ffffff = 0xffffff


Ein Boolean ist einfach ein Wert, der entweder sein kann true oder false. Beachten Sie, dass GameMaker Studio 2 derzeit eine reelle Zahl von 0,5 oder weniger als a interpretiert false Wert und jede reelle Zahl größer als 0,5 true. Dies bedeutet jedoch nicht, dass Sie 1 und 0 (oder eine andere reelle Zahl) auf true und false prüfen sollten, da Ihnen auch die Konstanten zur Verfügung stehen true und false Diese sollte immer in Ihrem Code verwendet werden, um Probleme zu vermeiden, falls in einem zukünftigen Update echte boolesche Datentypen hinzugefügt werden.

Sie können jede reelle Zahl mit der folgenden Funktion in einen implizit booleschen Wert konvertieren:

  • bool

Ein Zeiger ist ein Datentyp, der auf einen Speicherplatz "zeigt". Sie können keine Vorgänge an einem Zeiger ausführen. Er wird nur für einige sehr spezifische Funktionen verwendet, z. B. das Abrufen einer Textur- oder Pufferadresse aus dem Speicher für eine andere Funktion. Beispiele für Funktionen, die einen Zeiger zurückgeben, können Sie sehen buffer_get_address oder sprite_get_texture.

Es gibt auch eine Funktion, um zu überprüfen, ob ein Wert ein Zeiger ist (siehe " Datentypen überprüfen" unten) und eine Funktion, um einen Wert in einen Zeiger umzuwandeln:

  • ptr

Sie können auch die folgenden integrierten Konstanten verwenden (und zurückgeben), wenn Sie Zeiger verwenden:

Konstante Beschreibung
pointer_null Diese Konstante zeigt an, dass der Zeiger auf nichts Sinnvolles zeigt (das Gleiche wie NULL in C ++ oder null in C #)
pointer_invalid Diese Konstante bedeutet einfach, dass der Wert kein gültiger Zeiger ist

Eine Enumeration ist ein "Enumerator", mit dem Sie im Prinzip einen eigenen begrenzten Datentyp mit einer Liste von konstanten Werten erstellen können. Aufzählungen sind globale Gültigkeitsbereichsvariablen (dies ist implizit und erfordert kein " global "Präfix) und sie haben folgende Struktur:

enum <variable>{<constant> [= <value>]}

Im folgenden Beispiel erstellen wir eine Aufzählung für die Farben des Regenbogens und weisen ihm verschiedene Konstanten und Standardwerte zu:

enum rainbowcolors {
   red,
   orange,
   yellow,
   green,
   blue,
   indigo,
   violet
   }

Die Aufzählungseinträge können nur Ganzzahlen oder Ausdrücke sein, bei denen vorherige Aufzählungen eine Ganzzahl ergeben, die standardmäßig von 0 aufwärts nummeriert werden red = 0. orange = 1. yellow = 2, usw...

Sie können den Aufzählungsvariablen zum Zeitpunkt der Erstellung auch Werte zuweisen:

enum rainbowcolors {
   red = 5,
   orange = 5 * 2,
   yellow = 15,
   green = 20,
   blue = 25,
   indigo = 30,
   violet = 35 * enum_test.entry
   }

Beachten Sie im obigen Beispiel, dass wir ein anderes Enum verwenden, um einen Ausdruck für "violett" zu erstellen. Dies funktioniert nur, wenn die referenzierte Aufzählung vor der Aufzählung erstellt wurde, die sie in einem Ausdruck verwendet. Sie funktioniert jedoch nicht für Variablen oder Funktionen, da der Aufzählungswert als Konstante beim Kompilieren ausgewertet werden muss. Beachten Sie außerdem, dass alle Aufzählungswerte als Ganzzahlwerte ausgewertet werden. Wenn Sie eigene Werte erstellen, sollten Sie wissen, dass nur Ganzzahlwerte zulässig sind, damit Enumerationen funktionieren. Dieser Wert kann eine beliebige ganze Zahl sein, die eine Gleitkommazahl mit doppelter Genauigkeit darstellen kann.

Um später auf den Wert innerhalb eines bestimmten Aufzählungstyps zuzugreifen, können Sie den Punkt verwenden "." Methode wie folgt:

var value = rainbowcolours.green;

Beachten Sie, dass Sie die Werte für eine Enumenkonstante nicht ändern können, nachdem sie erstellt wurde.


Ein undefinierter Wert (auch als "Nullwert" bezeichnet) ist ein Wert, bei dem ein Ausdruck keinen korrekten Wert hat, obwohl er syntaktisch korrekt ist und daher etwas zurückgeben muss. Angenommen, Sie haben einen ds_map und benutze die Funktion ds_map_find_value(). Was passiert nun, wenn die Karte nicht den gesuchten Wert hat? Nun, da die Funktion korrekt formatiert ist und das Problem besteht, dass kein solcher Wert vorhanden ist, wird die Konstante zurückgegeben undefined und Sie können nach dieser Konstante suchen, wie Sie auch suchen würden true oder einen anderen Wert.


Datentypen überprüfen

GameMaker Studio 2 können Sie den Datentyp einer bestimmten Variablen mithilfe der folgenden Funktionen überprüfen:

  • is_string
  • ist echt
  • is_bool
  • is_array
  • is_ptr
  • is_int32
  • is_int64
  • is_vec3
  • is_vec4
  • is_matrix
  • ist nicht definiert
  • Art der
