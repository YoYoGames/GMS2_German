Schlüsselwörter


Um bestimmte Dinge in GameMaker Studio 2 vereinfachen, können Sie in Ihren Skripts und Aktionen eines von mehreren Schlüsselwörtern verwenden. Diese Schlüsselwörter werden hauptsächlich zum Identifizieren von Instanzen verwendet. Jedes wird im folgenden Text erläutert. Beachten Sie, dass alle Schlüsselwörter intern durch einen negativen ganzzahligen Wert dargestellt werden. Daher müssen Sie bei der Zuweisung von Werten zu Variablen Vorsicht walten lassen, da später unerwartete Ergebnisse auftreten können, da GameMaker Studio 2 den von Ihnen verwendeten Wert als etwas anderes interpretiert. Beachten Sie auch, dass die Verwendung der Werte anstelle der Schlüsselwörter in Ihrem Code überhaupt nicht empfohlen wird und später zu Problemen führen kann.

Stichwort Beschreibung Wert
self Die Instanz, die den aktuellen Codeblock ausführt. -1

self kann verwendet werden, um die aufrufende Instanz des aktuellen Codeblocks zu identifizieren, da sie immer die eindeutige ID der aktuell im Gültigkeitsbereich befindlichen Instanz ist. Zum Beispiel:

var val = 100;
with (instance_create_layer(x, y, "Instances", obj_Fire))
  {
  self.val = val;
  }

In diesem Beispiel können Sie sehen, dass eine lokale Variable aufgerufen wird val und wir wollen es die Instanz - Variable mit dem gleichen Namen in das neu erstellte Objektinstanz festgelegt. Um die Instanzvariable richtig zu identifizieren und GameMaker Studio 2 mitzuteilen, dass sie in der Instanz, die den Codeblock aufruft, festgelegt wird, verwenden Sie die self Stichwort. In den meisten Fällen können Sie auch die id Schlüsselwort statt self, aber self bietet bestimmte Vorteile. Zu Beginn ist es für den Compiler schneller, den Instanz-ID-Wert mithilfe von abzurufen self eher, als id, als die id value durchläuft die Instanznachschlagetabelle. Zweitens ist es für Personen, die Erweiterungen machen, sehr nützlich, den korrekten Bereich der Variablen zu gewährleisten, da es möglich ist, dass ein Projekt, das eine Erweiterung verwendet, eine globale Gültigkeitsbereichsvariable mit demselben Namen wie eine Variable in der Erweiterung hat.


Stichwort Beschreibung Wert
other Die andere an einem Kollisionsereignis beteiligte Instanz oder die andere Instanz von a with Funktion. -2

Das special- Schlüsselwort other kann auf zwei verschiedene Arten verwendet werden, um auf eine bestimmte Instanz zu verweisen: bei Verwendung in a with Funktion ( hier erklärt) oder bei Verwendung in einem Kollisionsereignis. Dies wird in diesem Abschnitt erläutert.

Ein Kollisionsereignis kann nur zwischen zwei Instanzen auftreten. Sie können mehrere Kollisionen zwischen mehreren Instanzen haben, die jedoch alle von GameMaker Studio 2 1-zu-1 gelöst werden, wobei die Instanz, die das Kollisionsereignis enthält, und die betreffende "andere" Instanz betroffen ist. Stellen Sie sich vor, Sie haben ein Spielerobjekt, mehrere Feindobjekte und mehrere Kugelobjekte, die der Feind auf Sie abfeuern kann. Sie können jedem Feind eine einzelne Aufzählungsinstanz zuweisen, der jedoch bei der Erstellung eine zufällige andere Schadensvariable zugewiesen wird. Beispiel:

var nnn;
nnn = instance_create_layer(x, y, "Bullets", obj_Bullet);
nnn.damage = 5 + irandom(5);
nnn.speed = 8;
nnn.direction = point_direction(x, y, obj_Player.x, obj_Player.y);

Sehen Sie, wie wir seine Variablen mithilfe der Punktmethode festlegen, die im Abschnitt zum Adressieren von Variablen beschrieben ist. Dies gibt den Aufzählungsobjekten einen anderen Schadenswert. Aber was ist mit dem Spielerobjekt? Wie erkennt es den Schaden, den es erleiden muss? Durch die Nutzung other im Kollisionsfall:

hp -= other.damage;
if hp <= 0 instance_destroy();

Der obige Code zieht den Betrag, der in der anderen Instanz in der Kollisionsvariablen "Schaden" gespeichert ist, von der Spielervariablen "hp" ab. Dann wird geprüft, ob "hp" kleiner oder gleich 0 ist. Wenn dies der Fall ist es wird das Spielerobjekt zerstören. Bitte beachten Sie, dass die auf diese Weise verwendeten anderen nur im Kollisionsereignis funktionieren und dass bei der anderen Instanz die Variable geprüft werden muss, andernfalls wird ein Fehler ausgegeben. Sie können jedoch Variablen Werte zuweisen oder sogar neue erstellen, indem Sie auch andere im Kollisionsereignis verwenden:

other.mana += 10; //add ten to the other instance "mana" variable
other.hit = true; //set the other instance variable "hit" to true, creating it if the variable doesn't already exist


Stichwort Beschreibung Wert
all Alle derzeit im Raum aktiven Instanzen. -3

Dieses Schlüsselwort wird verwendet, um GameMaker Studio 2 mitzuteilen, dass eine Funktion angewendet werden soll, oder um alle aktiven Instanzen innerhalb eines Raums zu überprüfen (deaktivierte Instanzen werden nicht geprüft oder nicht darauf zugegriffen). Sie können nicht verwenden all Um in anderen Instanzen mit der point-Methode auf Variablen zuzugreifen oder sie festzulegen (siehe hier ), können Sie sie beim Aufruf verwenden with(), zum Beispiel:

with (all)
   {
   speed = 0;
   }

Mit dem obigen Code wird die Geschwindigkeit aller Instanzen im Raum auf 0 gesetzt. Sie können auch verwenden all innerhalb von Funktionen, um alle Instanzen im Raum anzuvisieren oder zu überprüfen, zum Beispiel:

inst = instance_position(mouse_x, mouse_y, all);               //Check a point for any active instance in the room

if collision_line(x, y, mouse_x, mouse_y, all, false, true) {} //Check all instances for a collision along a line

mp_grid_add_instances(grid, all, false);                       //Add all instances in the room into a motion planning grid

all ist ein sehr nützliches Schlüsselwort und kann in zahlreichen Situationen in Ihrem Code und Ihren Aktionen verwendet werden, wobei häufig die Menge an Code reduziert wird, die Sie schreiben müssen, um einen gewünschten Effekt zu erzielen.


Stichwort Beschreibung Wert
noone Überhaupt keine Instanz. -4

Es mag seltsam erscheinen, aber während der Programmierung Ihrer Spiele müssen Sie oft feststellen, ob an einem Ort oder in einer Kollision keine Instanzen gefunden wurden. In diesen Fällen würden Sie dieses Schlüsselwort verwenden, um nichts zu suchen. etwas wie das:

if instance_nearest(x, y, obj_enemy) != noone
   {
   //do something as there is an enemy instance near
   }

In diesem Beispiel die Funktion instance_nearest() wird entweder zurückkehren noone oder die eindeutige ID der nächstgelegenen Instanz. Grundsätzlich können Sie jedes Mal, wenn Sie nach einer Instanz suchen müssen, davon ausgehen noone oder eine eindeutige zurückgegebene Instanz-ID.

Es ist erwähnenswert, dass das Stichwort struct ist ein reserviertes Schlüsselwort für mögliche zukünftige Erweiterungen von GML und kann daher nicht als Variablen- oder Skriptname verwendet werden und wird auch in der IDE nicht farblich gekennzeichnet.
