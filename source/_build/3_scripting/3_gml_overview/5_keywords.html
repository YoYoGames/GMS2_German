Schlüsselwörter


Um bestimmte Dinge in GameMaker Studio 2 einfacher zu machen, können Sie in Ihren Skripts und Aktionen eines von mehreren Schlüsselwörtern verwenden. Diese Schlüsselwörter werden hauptsächlich verwendet, um Instanzen zu identifizieren, und jede davon wird im folgenden Text erläutert. Beachten Sie, dass alle Schlüsselwörter intern durch einen negativen Integer-Wert dargestellt werden. Bei der Zuweisung von Werten an Variablen ist daher Vorsicht geboten, da es später zu unerwarteten Ergebnissen kommen kann, da GameMaker Studio 2 den von Ihnen verwendeten Wert interpretiert. Sie sollten auch beachten, dass die Verwendung der Werte anstelle der Schlüsselwörter in Ihrem Code überhaupt nicht empfohlen wird und später Probleme verursachen könnte.

Stichwort Beschreibung Wert
self Die Instanz, die den aktuellen Codeblock ausführt. -1

self kann verwendet werden, um die aufrufende Instanz des aktuellen Codeblocks zu identifizieren. Es gibt immer den Wert -1 zurück, den GameMaker Studio 2 als eindeutige ID für die Instanz interpretiert. Beispielsweise:

var val = 100;
with (instance_create_layer(x, y, "Instances", obj_Fire))
  {
  self.val = val;
  }

In diesem Beispiel können Sie sehen, dass wir eine lokale Variable namens haben val und wir wollen es die Instanz - Variable mit dem gleichen Namen in das neu erstellte Objektinstanz festgelegt. Um die Instanzvariable korrekt zu identifizieren und GameMaker Studio 2 mitzuteilen, dass sie in der Instanz gesetzt werden soll, die den Codeblock aufruft, verwenden wir die self Stichwort. Beachten Sie, dass wir die Verwendung des. Empfehlen id Stichwort in den meisten Situationen so über die self Stichwort, da es expliziter und offensichtlicher ist, was genau passiert.


Stichwort Beschreibung Wert
other Die andere Instanz, die an einem Kollisionsereignis beteiligt ist, oder die andere Instanz von a with Funktion. -2

Das spezielle Schlüsselwort " Andere" kann auf zwei verschiedene Arten verwendet werden, um eine bestimmte Instanz zu referenzieren: wenn es in einem with Funktion (wird hier erklärt) oder bei einem Kollisionsereignis verwendet, was in diesem Abschnitt erklärt wird.

Ein Kollisionsereignis kann nur zwischen zwei Instanzen auftreten. Sie können mehrere Kollisionen zwischen mehreren Instanzen haben, aber sie werden alle von GameMaker Studio 2 im 1: 1- GameMaker Studio 2 gelöst, wobei die Instanz das Kollisionsereignis hat und die "andere" Instanz beteiligt ist. Stellen Sie sich vor, Sie haben ein Spielerobjekt, mehrere feindliche Objekte und mehrere Kugelobjekte, die der Feind auf Sie schießen kann. Sie können jedem Feind eine einzelne Aufzählungsinstanz zuweisen, die ihm jedoch bei der Erstellung zufällig eine andere Schadensvariable zugewiesen bekommt, zum Beispiel:

var nnn;
nnn = instance_create_layer(x, y, "Bullets", obj_Bullet);
nnn.damage = 5 + irandom(5);
nnn.speed = 8;
nnn.direction = point_direction(x, y, obj_Player.x, obj_Player.y);

Sehen Sie, wie wir die Variablen mithilfe der Punktmethode im Abschnitt Adressierungsvariablen festlegen ? Dadurch erhalten die Geschossobjekte einen anderen Schadenswert. Aber was ist mit dem Spielerobjekt? Wie wird es den Schaden erkennen, den es zu nehmen hat? Durch die Nutzung other im Kollisionsereignis:

hp -= other.damage;
if hp <= 0 instance_destroy();

Der obige Code wird den Betrag, der in der anderen Instanz in der Variablen "damage" gespeichert ist, von der "hp" -Variable des Spielers abziehen, dann überprüft er, ob der "hp" kleiner oder gleich 0 ist. Wenn es dann ist Es wird das Spielerobjekt zerstören. Bitte beachten Sie, dass andere, die auf diese Weise verwendet werden, nur im Kollisionsereignis funktionieren und dass bei der anderen Instanz die Variable überprüft werden muss, da sonst ein Fehler ausgelöst wird. Sie können Variablen jedoch auch Werte zuweisen oder sogar neue erstellen, indem Sie andere auch im Kollisionsereignis verwenden:

other.mana += 10; //add ten to the other instance "mana" variable
other.hit = true; //set the other instance variable "hit" to true, creating it if the variable doesn't already exist


Stichwort Beschreibung Wert
all Alle momentan im Raum aktiven Instanzen. -3

Mit diesem Schlüsselwort wird GameMaker Studio 2 dass eine Funktion angewendet werden soll oder alle aktiven Instanzen innerhalb eines Raums überprüft werden sollen (deaktivierte Instanzen werden nicht geprüft oder nicht darauf zugegriffen). Sie können nicht verwenden all Um auf Variablen in anderen Instanzen zuzugreifen oder diese zu setzen, verwenden Sie die point-Methode (siehe hier ), aber Sie können sie beim Aufruf verwenden with(), beispielsweise:

with (all)
   {
   speed = 0;
   }

Der obige Code wird die Geschwindigkeit aller Instanzen im Raum auf 0 setzen. Sie können auch verwenden all innerhalb von Funktionen, um alle Instanzen im Raum anzuvisieren oder zu überprüfen, zum Beispiel:

inst = instance_position(mouse_x, mouse_y, all);               //Check a point for any active instance in the room

if collision_line(x, y, mouse_x, mouse_y, all, false, true) {} //Check all instances for a collision along a line

mp_grid_add_instances(grid, all, false);                       //Add all instances in the room into a motion planning grid

all ist ein sehr nützliches Schlüsselwort und kann in zahlreichen Situationen in Ihrem Code und Ihren Aktionen verwendet werden. Dabei wird oft die Menge an Code reduziert, die Sie zum Schreiben eines gewünschten Effekts benötigen.


Stichwort Beschreibung Wert
noone Keine Instanz. -4

Es mag seltsam erscheinen, aber bei der Programmierung Ihrer Spiele müssen Sie oft prüfen, ob an einem Ort, an einer Kollision oder an einer anderen Stelle keine Instanzen gefunden wurden. In diesen Fällen würden Sie dieses Schlüsselwort verwenden, um nach nichts zu suchen. etwas wie das:

if instance_nearest(x, y, obj_enemy) != noone
   {
   //do something as there is an enemy instance near
   }

In diesem Beispiel die Funktion instance_nearest() wird entweder zurückkehren noone oder die eindeutige ID der nächsten gefundenen Instanz. Grundsätzlich können Sie jederzeit, wenn Sie nach einer Instanz suchen müssen, beides erwarten noone oder eine eindeutige Instanz-ID zurückgegeben.
