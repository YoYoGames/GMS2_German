part_type_orientation

Legt den Winkel des Partikeltyps fest.


Syntax:

part_type_orientation( ind, ang_min, ang_max, ang_incr, ang_wiggle, ang_relative );


Streit Beschreibung
ind Der Index des zu ändernden Partikeltyps.
ang_min Der minimale Startwinkel der Partikel.
ang_max Der maximale Startwinkel der Partikel.
ang_incr Die Erhöhung (oder Verringerung, wenn negativ eingestellt) des Winkels in Grad pro Schritt.
ang_wiggle Der Betrag, um den der Winkel pro Schritt im Uhrzeigersinn / gegen den Uhrzeigersinn wackelt.
ang_relative Gibt an, ob der Winkel relativ zur Richtung der Bewegung des Partikels eingestellt werden soll (wahr) oder nicht (falsch).


Rückkehr: N / A


Beschreibung

Diese Funktion wird verwendet, um die Orientierung des Partikel-Sprites zu bestimmen, wenn er erzeugt wird, und kann auch verwendet werden, um die Partikelorientierung während ihrer Lebensdauer zu erhöhen oder zu verringern. Die minimalen und maximalen Orientierungswerte sind standardmäßig auf 0 eingestellt, aber diese Werte können geändert werden, um die Ausrichtung nach den standardmäßigen GameMaker Studio 2 Richtungen von 0 Grad nach rechts, 90 Grad nach oben, 180 Grad nach links und 270 Grad nach unten zu randomisieren. Wenn Sie sie auf denselben Wert setzen, werden die Partikel alle mit derselben Ausrichtung erstellt.

Sie können auch einen Inkrementwert festlegen, der der Ausrichtung über ihre Lebensdauer hinweg eine Gradzahl hinzufügt (wenn eine positive Zahl vorhanden ist) oder subtrahiert (wenn sie eine negative Zahl ist). Dieser Wert kann mindestens (+/-) 0,01 betragen.

Sie können auch den "Wiggle" -Faktor einstellen, genauso wie andere Partikelfunktionen. Dies ist ein Wert, der in jedem Schritt der Lebensdauer der Partikel zufällig zur Orientierung addiert oder subtrahiert wird. Offensichtlich sind größere Werte ausgeprägter als kleinere, und dieser Wert kann sogar negativ sein, wobei der maximale Bereich zwischen -20 und 20 liegt.

Schließlich können Sie wählen, ob die Ausrichtung relativ ist oder nicht, was bedeutet, dass das Partikel-Sprite um diesen Vektor herum ausgerichtet ist, während das Partikel eine Richtung (und eine Geschwindigkeit) hat. Wenn die Partikelgeschwindigkeit auf "Reduzieren" eingestellt ist und 0 erreicht wird, wird die Richtung durch den Mangel an Geschwindigkeit auf den Standardwert von 0 ° festgelegt. Daher bewirkt eine relative Ausrichtung, dass das Partikelsprite in einen anderen Winkel "springt".


Beispiel:

part_type_shape(particle2, pt_shape_spark);
part_type_size(particle2, 0.10, 0.50, 0.01, 0);
part_type_scale(particle2, 0.30, 0.30);
part_type_colour1(particle2, 8454143);
part_type_alpha1(particle2, 0.50);
part_type_speed(particle2, 4, 4, -0.07, 1);
part_type_direction(particle2, 0, 359, 0, 20);
part_type_orientation(particle2, 0, 359, 0, 20, 1);
part_type_blend(particle2, 1);
part_type_life(particle2, 1, 5);

Der obige Code wird verschiedene Partikelwerte einschließlich der Orientierung einstellen, die ein zufälliger Wert zwischen 0 und 359 sein wird. Es wird auch eine zufällige Menge von irgendwo zwischen 0 und 20 jedem Schritt hinzugefügt, und die Orientierung ist relativ zu der Bewegungsrichtung.
