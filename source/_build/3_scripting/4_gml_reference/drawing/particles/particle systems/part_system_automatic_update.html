part_system_automatic_update

Legt fest, ob die Partikel im Partikelsystem automatisch aktualisiert werden sollen oder nicht.

Syntax:

part_system_automatic_update(ind, automatic);


Streit Beschreibung
ind Der Index des Partikelsystems soll sich ändern.
automatic Ob die automatische Aktualisierung aktiviert ist ( true ) oder nicht (falsch).


Rückkehr: N / A


Beschreibung

Diese Funktion steuert, ob GameMaker Studio 2 das Partikelsystem automatisch aktualisieren soll oder nicht, und normalerweise müssen Sie diese Funktion nicht als Standardwert verwenden true (automatische Aktualisierung ist aktiviert), was Sie wünschen. Für spezielle Effekte oder zum Pausieren des Spiels können Sie dies jedoch auf einstellen false und es wird verhindert, dass Aktualisierungen an dem gegebenen Partikelsystem ausgeführt werden, es sei denn, dies wird explizit durch die Verwendung der Funktion befohlen part_system_update oder Sie verwenden diese Funktion erneut, um automatische Updates zu aktivieren. Keine Aktualisierung bedeutet, dass Partikel, die in diesem System erzeugt werden, nicht länger die Position, die Farbe oder irgendeinen anderen Parameter ändern und dass die Emitter nicht mehr funktionieren und das System genau an dem Punkt "einfriert", an dem das automatische Update eingestellt wurde false.

Eine Sache zu beachten ist, dass wenn Sie die einfachen Effekte verwenden, die von den Funktionen erstellt werden effect_create_above oder effect_create_below dann können Sie die Werte von 0 (für die unteren Effekte) oder 1 (für die oben genannten Effekte) als Partikelsystemindex verwenden und so auch das automatische Update für diese aktivieren (dies funktioniert auch, um die Aktualisierung für die DnD™ Partikeleffekte umzuschalten).


Beispiel:

if global.Pause
   {
   part_system_automatic_update(global.Sname, false);
   }

Der obige Code schaltet die Partikelaktualisierungen aus, wenn die globale Variable "Pausiert" wahr ist.
