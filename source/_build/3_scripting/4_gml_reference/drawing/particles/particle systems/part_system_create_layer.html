part_system_create_layer


Beschreibung

Diese Funktion erstellt ein neues Partikelsystem auf einer bestimmten Ebene. Sie geben die eindeutige Layer-ID an, die von der Funktion zurückgegeben wird layer_create() oder der Name des Layers, der als String verwendet werden soll - zum Beispiel "instance_layer" - und dann das System als persistent kennzeichnen oder nicht. Wenn das System als persistent markiert ist, wird es nach der Erstellung in allen Räumen beibehalten (und eine verwaltete Schicht zugewiesen, wenn die anfängliche Schicht in nachfolgenden Räumen nicht vorhanden ist), aber wenn die Persistenz nicht markiert ist, ist das System automatisch am Ende des Raums zerstört, in dem es erstellt wurde (dies ist das gleiche, als ob Sie die Funktion aufgerufen hätten part_system_destroy und wird auch alle mit dem System verbundenen Emitter zerstören. Die Funktion gibt einen eindeutigen ID-Wert für das Partikelsystem zurück, der in allen weiteren Funktionsaufrufen verwendet werden soll, wenn Sie eine System-ID angeben müssen.

WICHTIG! Wenn Sie das Partikelsystem als persistent markieren, muss es (und alle ihm zugeordneten Emitter) manuell bereinigt werden, indem die entsprechenden Zerstörungsfunktionen verwendet werden, andernfalls riskieren Sie ein Speicherleck, das sich negativ auf Ihr endgültiges Spiel auswirkt.


Syntax:

part_system_create_layer(layer_id, persistent);

Streit Beschreibung
layer Der Layer-ID-Wert oder die Zeichenfolge, der das Partikelsystem zugewiesen werden soll (kann ein beliebiger Layertyp sein)
persistent Kennzeichnen Sie das Partikelsystem als dauerhaft (auf true ) oder nicht (auf false )


Kehrt zurück:

Real (unique particle system ID value)


Beispiel:

global.p_sys = part_system_create_layer("effects_layer", true);

Der obige Code erstellt ein neues Partikelsystem auf der gegebenen Ebene und markiert ihn so, dass er in nachfolgenden Räumen bestehen bleibt. Die ID für das Partikelsystem wird in einer globalen Bereichsvariablen für zukünftige Referenz gespeichert.
