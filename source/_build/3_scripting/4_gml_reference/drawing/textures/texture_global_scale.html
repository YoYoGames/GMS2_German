texture_global_scale

Wird verwendet, um die Skalierung der Texturseiten beim Laden aus der WAD-Datei zu steuern.

Syntax:

texture_global_scale(pow2integer);


Streit Beschreibung
pow2integer Der zu verwendende Maßstabsfaktor (1, keine Skala, 2, halbe Skala, 4, Viertel Skala, etc...)


Rückkehr: N / A


Beschreibung

Mit dieser Funktion können Sie die Skalierung der Texturseiten beim Laden aus der WAD-Datei steuern, die Teil Ihrer endgültigen ausführbaren Datei ist. Der Eingabewert muss eine Zweierpotenz sein und funktioniert so, dass:

  • Um die Textur-Seiten auf 1: 1 einzustellen, verwenden Sie texture_global_scale(1)
  • Zu den halben Texturen die ganze Zeit benutzen texture_global_scale(2)
  • Um sie zu viertel benutzen texture_global_scale(4), etc...

Auf diese Weise können Sie die Nutzung des Texturseitenspeichers auf Plattformen mit geringen Speicherproblemen besser steuern. Beachten Sie, dass Texturen-Seiten bei Bedarf von der WAD erstellt werden. Sie können diese also entweder zu Beginn des Spiels aufrufen, um alle Texturen skaliert zu laden, oder Sie können sie zu bestimmten Zeiten aufrufen, um bestimmte Texturen zu laden. In diesem Fall würden Sie das tun wahrscheinlich anrufen wollen draw_texture_flush Bevor Sie diese Funktion aufrufen, verwenden Sie die Pre-Fetch-Funktionen, um die richtigen Texturseiten in den Speicher zu bekommen.


Beispiel:

draw_texture_flush();
texture_global_scale(2);
sprite_prefetch(spr_Trees);

Der obige Code löscht alle Texturen aus dem Speicher, setzt dann die Textur-Skalierung auf 2 (also Textur-Seiten sind halbe Größe) und holen dann schließlich die Textur-Seite, in der das Sprite "spr_Trees" enthalten ist.
