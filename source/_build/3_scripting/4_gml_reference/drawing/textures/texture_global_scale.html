texture_global_scale


Beschreibung

Mit dieser Funktion können Sie die Skalierung der Textur-Seiten beim Laden von der WAD-Datei aus steuern, die Teil Ihrer endgültigen Spieldatei ist. Der Eingabewert muss eine Potenz von zwei sein und funktioniert folgendermaßen:

  • Um die Textur-Seiten auf 1: 1 einzustellen, verwenden Sie texture_global_scale(1)
  • Zur halben Texturen die ganze Zeit verwenden texture_global_scale(2)
  • Vierteln sie verwenden texture_global_scale(4), usw...

Auf diese Weise können Sie die Verwendung des Texturspeichers auf Plattformen mit wenig Speicherproblemen besser steuern. Beachten Sie, dass Texturseiten auf Anforderung vom WAD erstellt werden. Sie können dies also zu Beginn des Spiels aufrufen, um alle skalierten Texturen zu laden, oder Sie können es zu bestimmten Zeiten aufrufen, um bestimmte Texturen zu laden. In diesem Fall würden Sie dies tun möchte wahrscheinlich anrufen draw_texture_flush Bevor Sie diese Funktion aufrufen, verwenden Sie die Pre-Fetch-Funktionen, um die richtigen Textur-Seiten in den Speicher zu bekommen.


Syntax:

texture_global_scale(pow2integer);

Streit Beschreibung
pow2integer Der zu verwendende Skalierungsfaktor (1, keine Skala, 2, halbe Skala, 4, Viertelskala usw.)


Kehrt zurück:

N/A


Beispiel:

draw_texture_flush();
texture_global_scale(2);
sprite_prefetch(spr_Trees);

Der obige Code löscht alle Texturen aus dem Speicher, setzt dann die Skalierung der Textur auf 2 (dh die Textur-Seiten sind halb so groß) und ruft schließlich die Textur-Seite ab, in der das Sprite "spr_Trees" enthalten ist.
