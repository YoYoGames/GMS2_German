Primitives Gebäude


GameMaker Studio 2 können Sie Ihre eigenen benutzerdefinierten Grundelemente gemäß den Spezifikationen des von Ihnen gewählten Scheitelpunktformats definieren (siehe Scheitelpunktformate ). Diese können dann in allen von Ihnen erstellten Shadern verwendet werden.

Alle von Ihnen erstellten Grundelemente werden in einem Scheitelpunktpuffer gehalten. Dies muss zuvor erstellt und dann von den Funktionen referenziert werden, mit denen Ihr Grundelement erstellt wird. Der Vertex-Puffer kann so oft wie nötig wiederverwendet werden, um verschiedene Primitive zu erstellen, oder er kann eingefroren werden, um einen bestimmten Primitivtyp für die Dauer Ihres Spiels oder Ihres Levels beizubehalten (was der schnellste Ansatz ist, wenn Sie wissen.) dass ein von Ihnen erstelltes Grundelement sich nicht ändert, sollten Sie diese Option immer verwenden).

Für den Vertex-Puffer stehen folgende Funktionen zur Verfügung:

  • vertex_create_buffer
  • vertex_create_buffer_ext
  • vertex_create_buffer_from_buffer
  • vertex_create_buffer_from_buffer_ext
  • vertex_get_buffer_size
  • vertex_get_number
  • vertex_delete_buffer

Die Primitiven, die Sie bauen sollen das Format folgen, dass Sie die eingestellte mit Vertex - Formaten Funktionen, wenn Sie also ein Vertex - Format mit nur Positionsdaten definiert haben, gibt es keinen Punkt Aufbau Ihre primitiven mit Farbdaten. Beachten Sie, dass die Reihenfolge, in der Sie dem Grundelement, das Sie erstellen, Eigenschaften hinzufügen, durch die Reihenfolge definiert wird, in der Sie diese Eigenschaften beim Erstellen des Scheitelpunktformats hinzugefügt haben. Wenn Sie also das Vektorformat mit der Reihenfolge, Farbe und Textur der Reihenfolge definiert haben Koordinate müssen Sie diese Eigenschaften in derselben Reihenfolge zu dem zu erstellenden Grundelement hinzufügen, andernfalls erhalten Sie Fehler.

Ein Beispiel für ein einzelnes Dreieck-Grundelement wird im folgenden Code dargestellt:

v_buff = vertex_create_buffer();
vertex_begin(v_buff, global.my_format);
vertex_position(v_buff, 10, 10);
vertex_colour(v_buff, c_white, 1);
vertex_texcoord(v_buff, 0, 0);
vertex_position(v_buff, 110, 10);
vertex_colour(v_buff, c_white, 1);
vertex_texcoord(v_buff, 1, 0);
vertex_position(v_buff, 110, 110);
vertex_colour(v_buff, c_white, 1);
vertex_texcoord(v_buff, 1, 1);
vertex_end(v_buff);
var tex = sprite_get_texture(spr_Background);
shader_set(shader_prim);
vertex_submit(v_buff, pr_trianglelist, tex);
shader_reset();

Grundelemente können Punkte, Linienlisten oder -streifen oder Dreiecklisten oder -streifen sein, aber Sie dürfen keine Dreieckfächer sein, da die meisten mobilen Geräte diesen Grundtyp nicht akzeptieren. Die Konstanten zur Definition dieser verschiedenen Grundtypen finden Sie hier.

HINWEIS: Sie können das Grundelement erstellen und in jedem Schritt des Spiels im Scheitelpunktpuffer speichern. Um jedoch zeichnen zu können, müssen Sie es während des Draw-Ereignisses übergeben.

Die folgenden Funktionen stehen für das Erstellen von Grundelementen zur Verfügung:

  • vertex_begin
  • vertex_colour
  • vertex_normal
  • vertex_position
  • vertex_position_3d
  • vertex_argb
  • vertex_texcoord
  • vertex_float1
  • vertex_float2
  • vertex_float3
  • vertex_float4
  • vertex_ubyte4
  • vertex_end
  • vertex_freeze
  • vertex_submit
