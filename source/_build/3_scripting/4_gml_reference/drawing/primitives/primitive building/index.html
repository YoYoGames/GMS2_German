Primitives Gebäude


GameMaker Studio 2 können Sie eigene benutzerdefinierte Primitive definieren, die den Spezifikationen des von Ihnen gewählten Vertex-Formats folgen (siehe Vertex-Formate ). Diese können dann in allen von Ihnen erstellten Shadern verwendet werden.

Alle von Ihnen erstellten Grundelemente werden in einem Vertexpuffer gespeichert. Dies muss zuvor erstellt und dann von den Funktionen referenziert werden, die zum Erstellen Ihres Primitivs verwendet werden. Der Vertex-Puffer kann so oft wie nötig verwendet werden, um verschiedene Primitive zu erstellen, oder er kann "eingefroren" werden, um einen spezifischen Primitivtyp für die Dauer Ihres Spiels oder Levels beizubehalten (was der schnellste Ansatz ist, wenn Sie es wissen) dass ein primitives Build nicht geändert wird, sollten Sie immer diese Option verwenden).

Folgende Funktionen stehen für den Vertex-Puffer zur Verfügung:

  • vertex_create_buffer
  • vertex_create_buffer_ext
  • vertex_create_buffer_from_buffer
  • vertex_create_buffer_from_buffer_ext
  • vertex_get_buffer_size
  • vertex_get_number
  • vertex_delete_buffer

Die von Ihnen erstellten Grundelemente sollten dem Format entsprechen, das Sie mit den Vertex- Format-Funktionen festgelegt haben. Wenn Sie also ein Vertexformat mit Positionsdaten definiert haben, ist es nicht sinnvoll, Ihr Grundelement mit Farbdaten zu erstellen. Sie sollten beachten, dass die Reihenfolge, in der Sie dem zu erstellenden Grundelement Eigenschaften hinzufügen, durch die Reihenfolge definiert wird, in der Sie diese Eigenschaften beim Erstellen des Vertexformats hinzugefügt haben, also wenn Sie das Vektorformat mit der Auftragsposition, Farbe und Textur definiert haben koordinieren, müssen Sie diese Eigenschaften zum Grundelement hinzufügen, das in der gleichen Reihenfolge gebaut wird, andernfalls erhalten Sie Fehler.

Ein Beispiel für ein einzelnes zu erstellendes Dreiecks-Primitiv wird im folgenden Code gezeigt:

v_buff = vertex_create_buffer();
vertex_begin(v_buff, global.my_format);
vertex_position(v_buff, 10, 10);
vertex_colour(v_buff, c_white, 1);
vertex_texcoord(v_buff, 0, 0);
vertex_position(v_buff, 110, 10);
vertex_colour(v_buff, c_white, 1);
vertex_texcoord(v_buff, 1, 0);
vertex_position(v_buff, 110, 110);
vertex_colour(v_buff, c_white, 1);
vertex_texcoord(v_buff, 1, 1);
vertex_end(v_buff);
var tex = background_get_texture(background1);
shader_set(shader_prim);
vertex_submit(v_buff, pr_trianglelist, tex);
shader_reset();

Primitive können Punkte, Linienlisten oder Streifen oder Dreieckslisten oder -streifen sein, aber Sie sind nicht erlaubt Dreieck Fans, da die meisten mobilen Hardware diesen primitiven Typ nicht akzeptieren. Die zur Definition dieser verschiedenen Grundtypen verwendeten Konstanten finden Sie hier.

HINWEIS: Sie können das Primitiv erstellen und es in jedem Schritt Ihres Spiels im Vertex-Puffer speichern. Um es jedoch zu zeichnen, müssen Sie es während des Draw-Events einreichen.

Die folgenden Funktionen sind zum Erstellen von Primitiven verfügbar:

  • vertex_begin
  • Vertexfarbe
  • Vertex_normal
  • Vertexposition
  • vertex_position_3d
  • Vertex_argb
  • Vertex_texcoord
  • vertex_float1
  • Vertex_float2
  • vertex_float3
  • vertex_float4
  • vertex_ubyte4
  • vertex_end
  • vertex_freeze
  • vertex_submit
