vertex_submit

Fixiere den angegebenen Vertex-Puffer.

Syntax:

vertex_submit(buffer, primitive, texture);


Streit Beschreibung
buffer Der zu verwendende Puffer
primitive Der primitive Basistyp.
texture Die zu verwendende Textur (oder -1 für keine Textur).


Rückkehr: N / A


Beschreibung

Sie können diese Funktion verwenden, um den Inhalt eines Vertexpuffers an die Grafikpipeline zur Verwendung mit einem Shader zu senden. Sie geben den zu verwendenden Pufferindex, den zu verwendenden Basisprimitivtyp (siehe die folgenden Konstanten) und die zu verwendende Textur an. Der Basis-Primitivtyp wird nur zum Zuweisen der Reihenfolge verwendet, in der die im Puffer gespeicherten Vertexe gezeichnet und verbunden werden, aber die tatsächlichen Daten für jeden der Vertexpunkte sind diejenigen, die Sie beim Erstellen des Vertex-Puffers definiert haben.

HINWEIS: Diese Funktion kann nur im Draw-Event verwendet werden.


Ein bildliches Beispiel für die verschiedenen verfügbaren Basis-Grundelemente finden Sie im folgenden Bild:

Konstante Beschreibung
pr_pointlist Ein Grundelement, das aus einer Liste von Punkten besteht.
pr_linelist Ein Grundelement, das aus einzelnen Zeilen in einer Liste besteht.
pr_linestrip Ein Grundelement, das aus einem fortlaufenden Streifen von Linien besteht.
pr_trianglelist Ein Grundelement, das aus einzelnen Dreiecken in einer Liste besteht.
pr_triangle_strip Ein Grundelement, das aus einem fortlaufenden Streifen von Dreiecken besteht.


Beispiel:

shader_set(shader_prim);
vertex_submit(buff, pr_trianglelist, sprite_get_texture(sprite_index));
shader_reset();

Der obige Code wird den in der Variablen "buff" indizierten Vertexpuffer zum Zeichnen mit dem Shader "shader_prim" übergeben, wobei er eine Dreiecksliste als Basisgrundelement und die Textur des Sprites für die Instanz verwendet, die den Code ausführt.
