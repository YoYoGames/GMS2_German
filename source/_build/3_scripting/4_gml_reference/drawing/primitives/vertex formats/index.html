Eckpunktformate

Wenn Sie mit der Arbeit mit 3D, Spezialeffekten, komplexen Zeichenprozessen oder Shadern beginnen, müssen Sie sich nicht allzu viele Gedanken über das verwendete Vertex-Format machen, da GameMaker Studio 2 automatisch die Vertex-Daten für Sie einrichten und übergeben wird. Manchmal ist es jedoch notwendig, eigene Vertex-Daten zu erstellen und sie entsprechend zu formatieren, insbesondere wenn Sie die Geschwindigkeit erhöhen oder zusätzliche Informationen eingeben möchten. Zum Beispiel enthält das Standard-Vertex-Format eine x, y, z 3D-Position, Farbe (mit Alpha) und UV-Texturkoordinaten, die, wenn Sie es selbst erstellen würden, ungefähr so ​​aussehen würden:

vertex_format_begin();
vertex_format_add_position_3d();
vertex_format_add_colour();
vertex_format_add_textcoord();
my_format = vertex_format_end();

Wenn Sie jedoch zum Beispiel nur den Shader verwenden, um die Position des Scheitelpunkts zu manipulieren, müssen Sie keine Farb- oder Texturdaten durchlaufen. In diesem Fall würden Sie Ihr eigenes Format wie folgt erstellen:

vertex_format_begin();
vertex_format_add_position_3d();
my_format = vertex_format_end();

Sie sollten beachten, dass die Reihenfolge, in der Sie die Eckpunkteigenschaften definiert haben, beim Erstellen Ihrer Eckpunkte berücksichtigt werden muss, nachdem Sie Ihr Eckpunktformat erstellt haben. Wenn Sie also ein Vertexformat als Position, Farbe, Texturkoordinate definiert haben, müssen Sie diese Eigenschaften dem Grundelement in der gleichen Reihenfolge hinzufügen, sonst erhalten Sie einen Fehler. Beachten Sie auch, dass ein Vertexformat wie jede andere dynamische Ressource Speicher benötigt und daher entfernt werden sollte, wenn die Funktion nicht benötigt wird vertex_format_delete().

Die folgenden Funktionen sind zum Zeichnen von Shadern verfügbar:

  • vertex_format_begin
  • vertex_format_add_colour
  • vertex_format_add_position
  • vertex_format_add_position_3d
  • vertex_format_add_textcoord
  • vertex_format_add_normal
  • vertex_format_add_custom
  • vertex_format_end
  • vertex_format_delete

Ein letzter wichtiger Punkt, der bei der Verwendung eigener Vertex-Puffer auf diese Weise beachtet werden muss, ist die Auswirkung auf die Vertex-Stapel, die an die GPU gesendet werden. Wenn Sie einen Vertex-Puffer erstellen, erstellen Sie die niedrigste Grafikdatenebene. Wenn Sie also alles zeichnen, senden wir Ihren Puffer direkt an die Grafikkarte. Aus diesem Grund müssen Sie, wenn Sie eine bessere Stapelverarbeitung durchführen möchten, diese selbst ausarbeiten und die gewünschten Stapel im selben Puffer speichern. Sie können also nicht 3 Sprites einfügen, dann viele davon zeichnen und erwarten, dass Vertex-Puffer zusammengeführt werden - Sie sind an diesem Punkt für Vertex-Pufferdaten zuständig. Wenn Sie das Auto-Batching durchführen möchten, müssen Sie der Engine höhere Level-Primitives geben - wie ein Sprite - an diesem Punkt kann GameMaker Studio 2 herausfinden, was Sie tun, und Puffer richtig füllen.
