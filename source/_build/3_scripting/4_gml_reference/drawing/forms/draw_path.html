draw_path

Zeichnet eine Pfadressource.

Syntax:

draw_path(path, x, y, absolute);


Streit Beschreibung
path Der Weg zum Zeichnen
x Die x-Koordinate, an der der Pfad gezeichnet wird
y Die y-Koordinate, an der der Pfad gezeichnet wird
absolute Ob der Pfad an der absoluten Position gezeichnet wird ( true ) oder die relative Position ( false )


Rückgabe: N / A


Beschreibung

Mit dieser Funktion können Sie GameMaker Studio 2 dazu bringen, einen Pfad zum Bildschirm zu zeichnen. Der Pfad wird als einfache Linie gezeichnet und kann sich entweder auf die aufrufende Instanz oder auf die absolute Position beziehen, an der sie im Pfad-Editor erstellt wurde, oder durch Code. Diese Funktion ist äußerst hilfreich beim Debuggen dynamischer Pfade (z. B. für Instanzen mit der mp_grid_path Funktion).

HINWEIS: Wenn Sie eine Form mit einem Shader zeichnen möchten, sollten Sie wissen, dass die meisten Shader die folgenden Eingaben erwarten: Scheitelpunkt, Textur, Farbe. Bei Verwendung dieser Funktion werden jedoch nur Vertex- und Farbdaten übergeben, sodass der Shader möglicherweise nichts zeichnet (oder etwas zeichnet, aber nicht korrekt). Wenn Sie auf diese Weise Formen zeichnen müssen, sollte der Shader entsprechend angepasst werden.



Beispiel:

if mp_grid_path(grid, path, x, y, obj_Player.x, obj_Player.y, 1)
   {
   draw_path(path, x, y, false);
   }

Der obige Code verwendet die mp_grid_path Funktion, um einen Pfad zu erzeugen und in der Variablen "path" zu speichern. Wenn der Pfad erfolgreich erstellt wurde, wird er auf dem Bildschirm relativ zu der Instanz angezeigt, die den Code ausführt.
