Oberflächen


In den normalen Zeichenereignissen zeichnet GameMaker Studio 2 nicht direkt auf den Bildschirm, sondern auf eine Oberfläche, die so genannte Anwendungsoberfläche. Diese Oberfläche ist im Grunde eine leere "Leinwand", die später bearbeitet werden kann, bevor sie bei Bedarf auf den Bildschirm gezogen wird. In den meisten Fällen übernimmt GameMaker Studio 2 dies für Sie (Sie können sie jedoch auch selbst im Code für Shader, Skalierung und viele andere Dinge - weitere Details sind unten angegeben).

Abgesehen von dieser Anwendungsoberfläche können Sie jedoch auch eigene Oberflächen erstellen und diese verwenden, um atemberaubende oder subtile Spezialeffekte in Ihrem Spiel zu erstellen. Zum Beispiel können Sie Oberflächen verwenden, um Instanzen zu "fangen", die dann zerstört werden können. Auf diese Weise können Sie einen Decal-Effekt erzeugen, bei dem das Sprite für die Instanz auf der Oberfläche angezeigt wird, als ob es noch existierte Trümmer, Blut, etc... ohne wirklichen Verarbeitungsaufwand. Eine andere Sache, die Sie mit Oberflächen machen können, ist, sie als Texturen zu verwenden, die manipuliert werden sollen, oder Sprites "on the fly" zu erstellen oder komplexe Overlays zu erstellen. In der Tat sind die Anwendungen für Oberflächen endlos!

Normale Oberflächen sind recht einfach zu verwenden, aber es gibt einige grundlegende Regeln, die bei der Verwendung beachtet werden müssen:

  • Zuerst sollten Sie feststellen, dass Oberflächen ( außer der Anwendungsoberfläche) "flüchtig" sind. Dies bedeutet, dass, wenn das Gerät oder das Fenster den Fokus verliert oder minimiert wird (gute Beispiele sind ein Bildschirmschoner in Windows oder auf einem Android Gerät, wenn die App aufgrund eines Anrufs den Fokus verliert), die Oberfläche möglicherweise zerstört wird. Dies liegt daran, dass es im Texturspeicher gespeichert ist und überschrieben werden kann, wenn die Zielplattform diesen Speicher für etwas anderes benötigt, was bedeutet, dass Sie immer eine Art von fehlersicherem Code haben sollten, normalerweise mit dem surface_exists Funktion.

  • Zweitens sollten Sie beachten, dass Oberflächen große Mengen an Texturspeicher benötigen, und Sie sollten versuchen, sie so klein wie möglich zu halten. Normalerweise würden Sie versuchen, sie nicht größer als die Größe der Ansicht oder des Anzeigefensters zu halten.

  • Drittens sollten Sie nur Flächen im Zeichenereignis erstellen. Wenn Sie eine Oberfläche im Create-Ereignis einer Instanz erstellen, können Sie möglicherweise den gleichen Index wie die application_surface. Dies kann dann zu vielen Problemen und Verwirrung führen, wenn Sie denken, dass Sie Ihre eigene Oberfläche verwenden, aber Sie verwenden tatsächlich das aktuelle Renderziel. Sie sollen auch immer versuchen, Zeichnung in der Auslosung Ereignisse auf eine Oberfläche zu begrenzen, zu, da aufgrund der optimierten Art und Weise, in der GameMaker Studio 2 auf den Bildschirm zieht, wird empfohlen, dass Sie alle Funktionen innerhalb der Auslosung Veranstaltung zeichnen halten - dazu gehört Clearing eine Oberfläche, wenn sie zum ersten Mal erstellt wird, etc... Das Zeichnen auf eine Oberfläche außerhalb des Zeichenereignisses ist möglich und kann sogar für einige Effekte notwendig sein, aber es ist nicht so, wie es gemacht werden sollte.

  • Viertens befindet sich die Oberfläche beim manuellen Zeichnen auf einer Oberfläche immer in der Position von (0,0). Dies bedeutet, dass Sie möglicherweise absolute Koordinaten in lokale Koordinaten für die Oberfläche konvertieren müssen. Wenn Sie z. B. eine Ansichtsfläche haben und etwas zeichnen möchten, das derzeit auf dieser Fläche angezeigt wird, sollten Sie die x- und y-Koordinaten der Kameraansicht von den tatsächlichen x- und y-Koordinaten subtrahieren, um eine relative Position zur Fläche zu erhalten (0,0) Position. Der Code würde also ungefähr so ​​aussehen:

    if view_current = 0
       {
       surface_set_target(surf);
       with (obj_Effect)
          {
          var _vx = camera_get_view_x(view_camera[1]);
          var _vy = camera_get_view_y(view_camera[1]);
          draw_sprite(sprite_index, image_index, x - _vx, y - _vy);
          }
       surface_reset_target();
       }
    else
       {
       draw_surface(surf, 0, 0);
       }

Die grundlegende Verwendung einer Oberfläche wäre wie folgt: Sie erstellen zuerst eine Oberfläche und weisen ihren Index einer Variablen zu. Sie würden dann das Zeichnungsziel auf die Oberfläche anstatt auf die Anzeige setzen und die Dinge zeichnen, die Sie wünschen, sowie andere Manipulationen durchführen. Sobald Sie fertig sind, setzen Sie das Zeichenziel zurück, so dass alle weiteren Zeichen wieder auf dem Bildschirm erscheinen. Beachten Sie, dass Sie, wenn Sie das gesamte Display auf eine Fläche zeichnen müssen (einschließlich Kacheln, Hintergründe usw.), einfach auf die Anwendungsoberfläche selbst zugreifen können (siehe unten für weitere Details) oder Sie können einer Ansicht eine Fläche zuweisen Port, der die Variable verwendet view_surface_id[0..7] da damit alles sichtbar ist, was in dieser Ansicht sichtbar ist, wird Port auf die entsprechende Oberfläche gezeichnet.

Oberflächen sind ein unglaublich leistungsstarkes Grafikwerkzeug, mit dem Sie mit GameMaker Studio 2 Spezialeffekte und viele andere Dinge erstellen können. Sie sind relativ einfach zu verwenden, da sie im Grunde eine Leinwand sind, auf die Sie anstelle des Bildschirms zeichnen, und diese Leinwand kann dann manipuliert und geändert werden, bevor sie mit den Funktionen in diesem Abschnitt gezeichnet wird.

HINWEIS: Beim Arbeiten mit Oberflächen besteht die Möglichkeit, dass sie aufgrund ihrer Speicherung im Texturspeicher jederzeit nicht mehr existieren können. Sie sollten IMMER überprüfen, ob eine Oberfläche vorhanden ist surface_exists bevor Sie direkt auf sie verweisen.
