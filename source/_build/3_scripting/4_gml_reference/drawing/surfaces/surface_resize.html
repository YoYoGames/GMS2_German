Oberflächengröße


Syntax:

surface_resize(surface_id, w, h);

Streit Beschreibung
surface_id Die ID der zu ändernden Oberfläche.
w Die Breite der neuen Oberfläche.
h Die Höhe der neuen Oberfläche.


Kehrt zurück:

N/A


Beschreibung

Mit dieser Funktion wird eine Fläche auf die angegebenen Abmessungen (in Pixel) skaliert. Die Oberflächen-ID ist diejenige einer zuvor erstellten Oberfläche oder die application_suface und die Funktion ändert die Größe dieser Oberfläche. Beachten Sie, dass dadurch weder der Inhalt der Oberfläche beschnitten noch gedehnt wird, sondern die aktuelle Oberfläche zerstört und mit demselben Handle (Oberfläche_ID) mit den neuen Dimensionen neu erstellt wird, was bedeutet, dass sie gelöscht und erneut gezeichnet werden muss (es sei denn es ist der application_surface In diesem Fall wird GameMaker Studio 2 dies automatisch tun.

HINWEIS: Wenn Sie die Größe der Anwendungsoberfläche ändern, werden diese Änderungen erst beim Start des nächsten Zeichnungsrahmens registriert, was bedeutet, dass der Aufruf des surface_get_width() oder surface_get_height() Funktionen in demselben Ereignis oder Schritt werden die vorherigen Werte zurückgeben.


Beispiel:

if view_wport[0] != surface_get_width(application_surface) || view_hport[0] != surface_get_height(application_surface)
   {
   surface_resize(application_surface, view_wport[0],view_hport[0]);
   }

Der obige Code überprüft die Größe des Ansichtsanschlusses mit der der Oberfläche "aplication_surface", und wenn es anders ist, wird die Größe der Oberfläche angepasst.
