Anwendungsoberfläche

Dieser Abschnitt behandelt die Anwendungsoberfläche und die spezifischen Funktionen.

GameMaker Studio 2 die meisten GameMaker Studio 2 nicht direkt auf dem Bildschirm wiedergegeben, sondern stattdessen auf der Anwendungsoberfläche gerendert. Dies ist im Wesentlichen eine Oberfläche, genau wie jede andere, die Sie selbst mit den Oberflächenfunktionen machen können und als solche mit den Oberflächenfunktionen manipuliert werden können, gezeichnet, an Shader gesendet werden und alles andere, was Sie normalerweise mit der Oberfläche tun würden, sind auch anwendbar Dies.

HINWEIS: Das einzige, was Sie mit der Anwendungsoberfläche nicht tun können, ist es freizugeben. Es existiert immer, obwohl sich der Indexwert ändern kann.

Wenn Sie Ihr Spiel ausführen, wird diese Oberfläche zum ersten Mal erstellt, wenn das Zeichenereignis in jedem neuen Raum aufgerufen wird, was bedeutet, dass bis zu diesem Zeitpunkt nichts gezeichnet wird. Sie können jedoch immer noch die Position der Anwendungsoberfläche abrufen und die Größe in CreateEvent oder einem anderen Ereignis ändern, ohne dass dabei Fehler auftreten. Die verwendeten Werte sind für die Oberfläche relevant, wenn sie erstellt wird. Die tatsächliche Abfolge der Ereignisse ist wie folgt:

  • Ereignis vor dem Zeichnen

  • ---> Anwendungsoberfläche erstellt und Zielsatz gerendert

  • Für jede sichtbare Ansicht oder, wenn keine Ansichten aktiv sind, einmal
    Start Ereignis zeichnen
    Ereignis zeichnen
    Ende-Ereignis zeichnen
  • ---> Das Ziel der Anwendungsoberfläche wird hier zurückgesetzt

  • Ereignis nachzeichnen

  • ---> die Anwendungsoberfläche wird jetzt standardmäßig zum Back-Buffer gezogen (obwohl Sie diese Option ausschalten können)

  • Draw GUI begin event
  • Draw GUI-Ereignis
  • Zeichnen Sie ein GUI-Endereignis

Die Verwendung dieser Oberfläche bedeutet, dass Sie mithilfe von Shadern mühelos unglaubliche Übergänge erstellen oder den Bildschirm um eine 3D-Form wickeln oder ein Low-Res-Spiel auf einen beliebigen Auflösungsbildschirm skalieren können... Die Möglichkeiten sind endlos!

Um auf diese Oberfläche zugreifen zu können, müssen Sie die integrierte globale Variable verwenden application_surface was auf der folgenden Seite erklärt wird:

  • Anwendungsoberfläche

Sie haben auch ein paar spezielle Funktionen, die nur für die Verwendung mit der Anwendungsoberfläche bestimmt sind:

  • application_surface_enable
  • application_surface_is_enabled
  • application_get_position
  • application_surface_draw_enable
