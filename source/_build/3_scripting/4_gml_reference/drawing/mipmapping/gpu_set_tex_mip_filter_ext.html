gpu_set_tex_mip_filter_ext


Beschreibung

Mit dieser Funktion können Sie den Mip-Filter-Modus für einen bestimmten Shader-Sampler einstellen. Sie geben den Indexwert für den Shader-Sampler an (wie von der Funktion zurückgegeben) shader_get_sampler_index() und dann eine der unten aufgeführten Moduswertkonstanten.

Konstante Beschreibung
tf_point Dies bedeutet, dass die Überblendung zwischen Mipmap-Ebenen deaktiviert ist, was zu sichtbaren Texturübergängen führen kann, aber die beste Leistung bietet.
tf_linear Dies bedeutet, dass die Überblendung zwischen Mipmap-Ebenen aktiviert ist (dies wird auch als trilineare Filterung bezeichnet ), wodurch die Texturübergänge geglättet werden, die Leistung jedoch geringfügig beeinträchtigt wird.
tf_anisotropic Dies bedeutet, dass die anisotrope Filterung aktiviert ist, die die Qualität der Texturübergänge erheblich verbessert und die bei anderen Filtermodi sichtbare Unschärfe verringern kann, aber die Leistung wird am stärksten beeinträchtigt.


Syntax:

gpu_set_tex_mip_filter_ext(sampler_index, filter);


Streit Beschreibung
sampler_index Der Index des Shader Samplers wird abgerufen.
filter Der zu verwendende Mip-Filter-Modus (eine Konstante, Standard: tf_point ).


Kehrt zurück:

N/A


Beispiel:

var _sampleIndex = shader_get_sampler_index(shd_Glass, "s_Background");
var _spriteTex = sprite_get_texture(sprite_index, 0);
shader_set(shd_Glass);
if gpu_get_tex_mip_filter_ext(_sampleIndex) != tf_point
    {
    gpu_set_tex_mip_filter_ext(_sampleIndex, tf_point);
    }
texture_set_stage(_sampleIndex , _spriteTex);
shader_reset();

Der obige Code setzt den Mip-Filter-Modus auf tf_point (Deaktivieren von Mipmapping) für den angegebenen Shader-Textur-Sampler, wenn dieser noch nicht festgelegt wurde.
