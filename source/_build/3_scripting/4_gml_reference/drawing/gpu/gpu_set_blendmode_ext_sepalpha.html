gpu_set_blendmode_ext_sepalpha


Beschreibung

Wenn GameMaker Studio 2 ein Pixel zeichnet, gibt es eine Quellfarbe (die Farbe des Pixels, die wir zeichnen werden) und eine Zielfarbe (die Farbe, die bereits in dem Pixel ist, zu dem wir zeichnen) Neue Farbe des Pixels, die Quell- und Zielfarben werden entsprechend dem ausgewählten Mischmodus berechnet. Jede Komponente der Farben wird als Gleitkommawert zwischen 0 und 1 gespeichert, und die neue Farbe wird berechnet, indem jede Komponente der Quellfarbe um einen Faktor multipliziert wird und indem jede Komponente der Zielfarbe mit einem anderen Faktor multipliziert und dann addiert wird ergibt zusammen Komponente für Komponente.

Mit dieser Funktion können Sie die verschiedenen Komponenten festlegen, die zusammengerechnet werden sollen, um einen benutzerdefinierten Mischmodus zu erstellen, indem Sie verschiedene Teile verschiedener Mischmodusfaktoren für die RGB-Komponenten und die Alpha-Komponente auswählen. Das Argument "src" und "dest" verwendet also die ersten drei Mischmoduswerte (RGB), und die Argumente "alphasrc" und "alphadest" nehmen den letzten Teil des Mischmodusfaktors. Zum Beispiel, wenn Sie die RGB-Quelle als haben bm_zero, dann wird (0, 0, 0) verwendet. Wenn Sie dann die Alpha-Quelle als bm_one festlegen, wird (1) verwendet, was einen endgültigen Quellmischmodus von (0, 0, 0, 1) ergibt.

Das folgende Diagramm zeigt die RGBA-Werte für jeden Mischmodus, wobei (Rs, Gs, Bs, As) als die Quellwerte und (Rd, Gd, Bd, Ad) die Zielwerte sind. Die elf verfügbaren Konstanten können Quelle oder Ziel (oder beides) sein, wenn sie in dieser Funktion verwendet werden.

Konstante Mischfaktor (Rot, Grün, Blau, Alpha)
bm_zero (0, 0, 0, 0)
bm_one (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-Rs, 1-Gs, 1-Bs, 1-As)
bm_src_alpha (Wie, als, wie, als)
bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As)
bm_dest_alpha (Anzeige, Anzeige, Anzeige, Anzeige)
bm_inv_dest_alpha (1-Ad, 1-Ad, 1-Ad, 1-Ad)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) mit f = min (As, 1-Ad)

Beachten Sie, dass Sie entweder vier einzelne Argumente für diese Funktion angeben können oder ein Array von Argumenten (wie von der Funktion zurückgegeben) bereitstellen können gpu_get_blendmode_ext_sepalpha beispielsweise). Wenn ein Array bereitgestellt wird, sollte es die folgenden vier Elemente enthalten:

  • [0] = Quellmischmodus minus Alphakomponente (Standard ist bm_src_alpha (Wie, als, wie))
  • [1] = Ziel-Mischmodus minus Alphakomponente (Standard ist bm_inv_src_alpha (1-As, 1-As, 1-As))
  • [2] = Quell-Alpha-Komponente (Standard ist bm_src_alpha (Wie))
  • [3] = Ziel-Alpha-Komponente (Standard ist bm_inv_src_alpha (1-As))
WARNUNG! Diese Funktion ist derzeit nur auf der Windows- Zielplattform verfügbar.


Syntax:

gpu_set_blendmode(src, dest, alphasrc, alphadest);


Argumente:

Streit Beschreibung
src Quell-Mischmodus-Faktor für RGB (siehe obige Konstanten).
dest Ziel-Mischmodus-Faktor für RGB (siehe obige Konstanten)
alphasrc Quellmischmodusfaktor Alpha (siehe obenstehende Konstanten).
alphadest Zielmischmodusfaktor Alpha (siehe obenstehende Konstanten)


Kehrt zurück:

N/A


Beispiel:

var bm = gpu_get_blendmode_ext_sepalpha();
bm[0] = bm_src_alpha;
bm[1] = bm_inv_src_alpha;
gpu_set_blendmode_ext_sepalpha(bm);

Der obige Code erstellt ein lokales Array und ruft den aktuellen erweiterten Mischmodus mit separaten Alpha-Werten ab. Dieses Array wird dann manipuliert und zurückgegeben, um den Mischmodus einzustellen.
