gpu_set_zwriteenable


Beschreibung

Mit dieser Funktion können Sie einschalten ( true ) und aus ( false ) Tiefenreihenfolge mit dem Z-Puffer. Wenn eine Instanz in GameMaker Studio 2 gerendert wird, wird die Tiefe eines erzeugten Pixels (seine z-Koordinate) im Z-Puffer gespeichert. Wenn eine andere Instanz der Szene mit demselben Pixel gerendert werden muss, werden die beiden Tiefen verglichen (z-testing - siehe gpu_set_ztestenable ) und derjenige, der dem Beobachter näher ist, wird dann im Z-Puffer gespeichert und ersetzt den alten. Dies ist jedoch nicht immer das, was Sie möchten, und es gibt Momente, in denen Sie möchten, dass etwas über alles andere gezogen wird, unabhängig davon, welchen z-Wert dies hat. In diesen Fällen können Sie diese Funktion verwenden, um das Schreiben in den Z-Puffer auszuschalten und zu zwingen, das Gezeichnete über alles andere zu ziehen. Beachten Sie, dass die hier gegebene Beschreibung für den Standardvergleichsmodus für Z-Tests gilt, der jedoch geändert werden kann

Der Standardwert ist, dass die Z-Pufferung aktiviert ist ( true ).


Syntax:

gpu_set_zwriteenable(enable);


Argumente:

Streit Beschreibung
enable Der zu verwendende Vergleichsmodus (siehe Liste oben)


Kehrt zurück:

N/A


Beispiel:

gpu_set_zwriteenable(false);
with (obj_Menu_Containers)
   {
   draw_self();
   }
gpu_set_zwriteenable(true);

Der obige Code deaktiviert das Schreiben von Z-Puffern, zeichnet eine Anzahl von Objekten und schaltet sie wieder ein, um mit dem Zeichnen fortzufahren.
