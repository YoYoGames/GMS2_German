gpu_set_blendmode_ext


Beschreibung

Wenn GameMaker Studio 2 ein Pixel zeichnet, gibt es eine Quellfarbe (die Farbe des Pixels, die wir zeichnen werden) und eine Zielfarbe (die Farbe, die bereits in dem Pixel ist, zu dem wir zeichnen) Neue Farbe des Pixels, die Quell- und Zielfarben werden entsprechend dem ausgewählten Mischmodus berechnet. Jede Komponente der Farben wird als Gleitkommawert zwischen 0 und 1 gespeichert, und die neue Farbe wird berechnet, indem jede Komponente der Quellfarbe um einen Faktor multipliziert wird und indem jede Komponente der Zielfarbe mit einem anderen Faktor multipliziert und dann addiert wird ergibt zusammen Komponente für Komponente.

Mit dieser Funktion können Sie die verschiedenen Komponenten festlegen, die zusammengerechnet werden sollen, um einen benutzerdefinierten Mischmodus zu erstellen. Die Quelle und das Ziel haben beide eine Rot-, Grün-, Blau- und Alpha-Komponente und in der folgenden Tabelle werden die RGBA der Quelle (Rs, Gs, Bs, As) und die Destination (Rd, Gd, Bd, Ad) berücksichtigt. Die elf verfügbaren Konstanten können Quelle oder Ziel (oder beides) sein, wenn sie in dieser Funktion verwendet werden.

Konstante Mischfaktor (Rot, Grün, Blau, Alpha)
bm_zero (0, 0, 0, 0)
bm_one (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-Rs, 1-Gs, 1-Bs, 1-As)
bm_src_alpha (Wie, als, wie, als)
bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As)
bm_dest_alpha (Anzeige, Anzeige, Anzeige, Anzeige)
bm_inv_dest_alpha (1-Ad, 1-Ad, 1-Ad, 1-Ad)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) mit f = min (As, 1-Ad)

Beachten Sie, dass Sie entweder zwei einzelne Argumente für diese Funktion angeben können oder ein Array von Argumenten angeben können (wie von der Funktion zurückgegeben) gpu_get_blendmode_ext beispielsweise). Wenn ein Array bereitgestellt wird, sollte es die folgenden zwei Elemente enthalten:

  • [0] = Quellmischmodus (Standard ist bm_src_alpha )
  • [1] = Zielmischmodus (Standard ist bm_inv_src_alpha )

Es kann ziemlich schwierig sein zu visualisieren, wie diese verschiedenen Mischmodi funktionieren. Im Folgenden finden Sie ein einfaches Diagramm, das die grundlegende Interaktion eines einzelnen Sprites zeigt, das mit verschiedenen Kombinationen von Mischmodi über einen einzelnen Hintergrund gezeichnet wird. Bitte beachten Sie, dass die tatsächlichen Ergebnisse des von Ihnen verwendeten Mischmodus sehr von der Farbe im Sprite, dem Alpha-Wert des Sprites und der Farbe und dem Alpha des Hintergrunds abhängen.

WARNUNG!: Nicht alle Mischmodi sind auf allen Plattformen verfügbar. Android, Tizen, iOS und HTML5 ohne WebGL aktiviert werden folgende Modi korrekt angezeigt werden nicht in der Lage sein:

  • bm_src_farbe
  • bm_inv_src_colour
  • bm_dest_farbe
  • bm_inv_dest_colour
  • bm_src_alpha_sat


Syntax:

gpu_set_blendmode(src, dest);


Argumente:

Streit Beschreibung
src Quellmischmodusfaktor (siehe obenstehende Konstanten).
dest Ziel-Mischmodus-Faktor (siehe obenstehende Konstanten)


Kehrt zurück:

N/A


Beispiel:

gpu_set_blendmode_ext(bm_src_alpha, bm_one);
draw_circle_colour(100, 100, 50, c_white, c_black, 0);
gpu_set_blendmode(bm_normal);

Dies wird das Schwarz in Transparenz verwandeln und einen "Leucht" -Effekt erzeugen, da das Weiß außen stark ist und vom Kreismittelpunkt aus immer schwächer wird. Der Mischmodus wird nach dem Zeichnen des Kreises zurückgesetzt, um sicherzustellen, dass die additive Mischung nicht auch auf alle anderen Elemente angewendet wird.
