gpu_set_blendmode_ext


Beschreibung

Wenn GameMaker Studio 2 ein Pixel zeichnet, gibt es eine Quellfarbe (die Farbe des Pixels, die wir zeichnen werden) und eine Zielfarbe (die Farbe, die sich bereits in dem Pixel befindet, in das wir zeichnen) Wenn die Farbe des Pixels neu ist, werden die Quell- und die Zielfarbe entsprechend dem ausgewählten Mischmodus berechnet. Jede Komponente der Farben wird als Gleitkommawert zwischen 0 und 1 gespeichert, und die neue Farbe wird berechnet, indem jede Komponente der Quellfarbe mit einem Faktor multipliziert wird und indem jede Komponente der Zielfarbe mit einem anderen Faktor multipliziert und dann der Wert addiert wird ergibt zusammen komponente für komponente.

Mit dieser Funktion können Sie die verschiedenen Komponententeile festlegen, die zusammengefügt werden sollten, um einen benutzerdefinierten Überblendungsmodus zu erstellen. Quelle und Ziel haben beide eine Rot-, Grün-, Blau- und Alphakomponente. In der folgenden Tabelle werden die RGBA-Werte der Quelle berücksichtigt (Rs, Gs, Bs, As), während die Zielorte (Rd, Gd, Bd, Ad) sind. Die elf Konstanten, die zur Verwendung verfügbar sind, können entweder Quelle oder Ziel (oder beides) sein, wenn sie in dieser Funktion verwendet werden.

Konstante Mischungsfaktor (Rot, Grün, Blau, Alpha)
bm_zero (0, 0, 0, 0)
bm_one (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-Rs, 1-Gs, 1-Bs, 1-As)
bm_src_alpha (As, As, As, As)
bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As)
bm_dest_alpha (Anzeige, Anzeige, Anzeige, Anzeige)
bm_inv_dest_alpha (1-Ad, 1-Ad, 1-Ad, 1-Ad)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) wobei f = min (As, 1-Ad)

Beachten Sie, dass Sie dieser Funktion entweder zwei einzelne Argumente oder ein Array von Argumenten (wie von der Funktion zurückgegeben) zur Verfügung stellen können gpu_get_blendmode_ext zum Beispiel). Wenn Sie ein Array angeben, sollte es die folgenden zwei Elemente enthalten:

  • [0] = Quell-Mischmodus (Standardeinstellung ist bm_src_alpha )
  • [1] = Ziel-Mischmodus (Standardeinstellung ist bm_inv_src_alpha )

Es kann ziemlich schwierig sein, sich vorzustellen, wie diese verschiedenen Überblendungsmodi funktionieren. Im Folgenden finden Sie ein einfaches Diagramm, das die grundlegende Interaktion eines einzelnen Sprites zeigt, das mit verschiedenen Kombinationen von Überblendungsmodi über einem einzigen Hintergrund gezeichnet wird. Beachten Sie, dass die tatsächlichen Ergebnisse des Mischmodus, den Sie verwenden, sehr stark von der Farbe des Sprites, dem Alpha-Wert des Sprites und der Farbe und dem Alpha des Hintergrunds abhängen.

ACHTUNG!: Nicht alle Mischmodi sind auf allen Plattformen verfügbar. Android, iOS und HTML5 ohne WebGL aktiviert werden folgende Modi korrekt angezeigt werden nicht in der Lage sein:

  • bm_src_colour
  • bm_inv_src_colour
  • bm_dest_colour
  • bm_inv_dest_colour
  • bm_src_alpha_sat


Syntax:

gpu_set_blendmode(src, dest);


Argumente:

Streit Beschreibung
src Quellmodus für den Mischmodus (siehe Konstanten oben).
dest Ziel-Mischmodusfaktor (siehe Konstanten oben)


Kehrt zurück:

N/A


Beispiel:

gpu_set_blendmode_ext(bm_src_alpha, bm_one);
draw_circle_colour(100, 100, 50, c_white, c_black, 0);
gpu_set_blendmode(bm_normal);

Dadurch wird Schwarz in Transparenz umgewandelt, wodurch ein "Glühen" -Effekt entsteht, da das Weiß außen stark ist und sich vom Kreismittelpunkt aus weiter abschwächt. Der Mischmodus wird nach dem Zeichnen des Kreises zurückgesetzt, um sicherzustellen, dass das additive Mischen nicht auch auf alles andere angewendet wird.
