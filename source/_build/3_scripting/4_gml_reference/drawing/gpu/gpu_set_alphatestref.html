gpu_set_alphadestref


Beschreibung

Mit dieser Funktion wird der Referenzwert für den Alphatest festgelegt (wenn er mit der Taste aktiviert wurde gpu_set_alphatestenable ). Dies ist die "Cut-Off" -Schwelle, bei der Pixel mit Alpha nicht gezeichnet werden.

Wenn Sie diesen Wert auf 0 setzen und die lineare Interpolation eingeschaltet haben (entweder in den Spieloptionen oder mit gpu_set_texfilter ), können Sie immer noch visuelle Fehler an den Rändern überlappender Objekte bekommen. Dies liegt daran, dass Sie fast vollständig transparente Bereiche haben können, die gezeichnete Objekte blockieren, aber wenn Sie diesen Referenzwert beispielsweise auf 254 setzen (dh einen Wert unterhalb des Maximums), wird garantiert, dass nur 100% deckende Pixel gezeichnet werden. Sie können auch die Textur-Interpolation deaktivieren, wodurch Sie normalerweise die Standardwerte ohne Probleme verwenden könnten und diese Funktion gar nicht benötigen würde, jedoch hängt vieles von der Quellgrafik ab.

HINWEIS: Dies ist eine erweiterte Zeichnungsfunktion. Wenn Sie also nicht sicher sind, ob Sie sie benötigen oder nicht genau wissen, was sie tut, sollten Sie sie wahrscheinlich nicht verwenden, da sie die Darstellung Ihres Spiels ernsthaft beschädigen kann.


Syntax:

gpu_set_alphatestref(val);


Argumente:

Streit Beschreibung
val Legt den Referenzwert für den Alphatest von 0 bis 255 fest (Standardwert ist 0)


Kehrt zurück:

N/A


Beispiel:

if !gpu_get_alphatestenable()
   {
   gpu_set_alphatestenable(true);
   gpu_set_alphatestref(128);
   }

Der obige Code wird prüfen, ob der Alphatest aktiviert ist, und wenn nicht, schaltet er den Alphatest ein und setzt den Testschwellenwert auf 128 (nur Pixel mit einem Alpha über 0,5 werden gezeichnet).
