font_add_sprite_ext


Beschreibung

Mit dieser Funktion können Sie einen "Sprite-Strip" verwenden (das Sprite selbst muss ein Sprite-Asset aus dem Ressourcenbaum sein oder ein Sprite, das Sie den Game-Ressourcen hinzugefügt haben) sprite_add ), um ein neues Schriftarten-Asset zu erstellen, wobei jedes Teilbild ein einzelnes Symbol oder ein einzelner Buchstabe wäre. Anders als normal font_add_sprite Diese Funktion weist eine spezifische Reihenfolge für die Teilbilder des Sprites auf. Diese Funktion ordnet die Sprite-Teilbilder basierend auf dem Argument "string_map" der Funktion zu. Dieses Argument ist eine Zeichenfolge, mit der Sie GameMaker Studio 2 mitteilen können, in welcher Reihenfolge die Teilbilder der Sprite-Schriftart angeordnet sind, und diese automatisch beim Schreiben von Text zuordnen. So zum Beispiel, wenn Sie einen String-Karte „AaBbCcDdEeFfGgHh“ haben, Ihre Sprite Schriftart auf diese Weise haben die Teilbilder bestellt müssen. Sie können "space" als ein beliebiges beliebiges Zeichen definieren, z. B. eine einzelne Zeile mit der Größe, die der Raum haben soll, und alle Leerzeichen im Text werden mit dieser Breite gerendert (aber das ausgewählte Bild wird nie gerendert), oder wenn Sie kein Sprite für Leerzeichen angeben, wird stattdessen die Breite des breitesten Zeichens in der Sprite-Schriftart verwendet.

Sie können den Abstand für die Schriftart auch proportional festlegen ( true ) oder nicht ( false ), wobei eine proportionale Schriftart basierend auf der tatsächlichen Breite der Buchstaben beabstandet ist (so nimmt beispielsweise der Buchstabe "i" weniger Platz ein als der Buchstabe "w"), während eine nichtproportionale Schriftart basierend auf der Unterteilung beabstandet ist. Bildbreite, wie eine monospaced Schriftart. Schließlich können Sie den Abstand definieren, der beim Schreiben zwischen den einzelnen Buchstaben verbleibt. Dies kann ein beliebiger ganzzahliger Wert sein, wobei 0 kein Leerzeichen ist (die Buchstaben werden "berühren", wenn sie proportional sind). Die Funktion gibt einen Indexwert zurück, der in einer Variablen gespeichert werden sollte, da dies in allen weiteren Codes, die sich auf diese Schriftart beziehen, benötigt wird.

Es ist erwähnenswert, dass die Funktionen zur Schriftausrichtung funktionieren (wie draw_set_halign() ) Dann sollte der Ursprung des Font-Sprites auf die obere linke Ecke gesetzt werden. Wenn Sie andere Werte verwenden, müssen Sie den Ursprungsoffset berücksichtigen, wenn Sie Text mit der Schriftart zeichnen.

HINWEIS: Wenn Sie eine Schriftart in GameMaker Studio 2 laden, müssen Sie daran denken, sie erneut zu entfernen (mit font_delete ) wenn es nicht mehr benötigt wird, besteht sonst die Gefahr eines Speicherlecks, das das Spiel verlangsamt und schließlich zum Absturz bringt.
WICHTIG! Diese Funktion ist mit der Testlizenz des Produkts nicht verfügbar.


Syntax:

font_add_sprite_ext(spr, string_map, prop, sep);


Streit Beschreibung
spr Das Sprite zum Hinzufügen einer Schriftart basierend auf.
string_map Zeichenfolge, aus der die Sprite-Teilbildreihenfolge ausgewählt wird.
prop Als proportionale Schriftart festlegen oder nicht.
sep Der Platz zwischen jedem Buchstaben zu verlassen.


Kehrt zurück:

Real


Beispiel:

global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2);

Der obige Code erstellt ein neues Font-Asset aus dem Sprite, das in der Variablen "spr_CalcFont" indiziert ist, und speichert den Index der neuen Schriftart in der Variable "global.Font". Die Teilbilder der Sprite-Schriftart werden der angegebenen Zeichenfolge zugeordnet.
