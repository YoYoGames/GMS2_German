font_texture_page_size


Beschreibung

Diese integrierte Variable kann verwendet werden, um bei Verwendung der Funktion entweder die Texturseitengröße zu erhalten oder festzulegen font_add. Beim Hinzufügen einer Schriftart mit dieser Funktion erstellt GameMaker Studio 2 einen Texturseiten-Cache mit den erforderlichen Symbolen bis zu der Größe, die durch diese Variable definiert wird. Der Prozess ist wie folgt:

  • Jedes von Ihnen verwendete Zeichen wird von der Schriftart in eine Texturseite zwischengespeichert
  • Wenn die Textur-Seite voll ist (weil Sie große Zeichen oder viele verschiedene Zeichen verwenden), entfernt GameMaker Studio 2 Zeichen aus dem Cache und ersetzt sie durch neue, die Sie benötigen
  • GameMaker Studio 2 versucht, die Textur nicht zu groß zu machen, da nicht alle Geräte mit großen Texturen umgehen können, und versucht, die Textur nicht zu klein zu machen, da das Entfernen und Hinzufügen von Zeichen zur Textur CPU-Zeit beansprucht und das Spiel zum Laufen bringt langsam

Wenn Sie der Meinung sind, dass Sie eine größere oder eine kleinere Seitengröße benötigen als die von GameMaker Studio 2 standardmäßig verwendeten 1024 Pixel, können Sie den Wert mithilfe dieser integrierten Variable festlegen. Wenn Sie nicht die font_add() Diese Einstellung hat keine Auswirkungen auf die Leistung Ihres Spiels, da in der IDE hinzugefügte Fonts auf den regulären Texturseiten (die Sie über den Font-Editor und die Spieloptionen für die Zielplattform steuern können) zwischengespeichert werden.


Syntax:

font_texture_page_size;


Kehrt zurück:

Real (maximum width/height in pixels)


Beispiel:

font_texture_page_size = 512;
newfont = font_add( 'Arial', 24, true, true, 32, 128);

Der obige Code setzt die Schriftart-Cache-Textur-Seite auf eine maximale Größe von 512px und fügt dann dem Spiel eine neue Schriftart hinzu.
