font_get_first


Beschreibung

Beim Definieren einer Schriftart in GameMaker Studio 2 können Sie eine Reihe von Zeichen definieren, die eingeschlossen werden sollen. Dies liegt daran, dass die Schriftart selbst aus rechtlichen Gründen nicht in Ihrem Spiel enthalten ist, aber ein Bild der Schriftart auf einer Textur-Seite enthalten ist und das wird Ihr Spiel verwenden (genau wie jedes andere Grafik-Asset). Das bedeutet, dass Sie die Anzahl der verwendeten Zeichen auf ein Minimum beschränken und nur die Anzahl der Zeichen angeben müssen, die Ihr Spiel benötigt, um den Texturspeicher so optimal wie möglich zu halten. Diese Funktion kann verwendet werden, um das Anfangszeichen (als ASCII-Wert) zu finden, das beim Hinzufügen Ihres Schriftarten-Assets zu Ihrem Spiel verwendet wurde.

HINWEIS: Aufgrund von Änderungen in der Art und Weise, wie Zeichensätze gehandhabt werden, gibt diese Funktion immer 32 zurück, da GameMaker Studio 2 das Leerzeichen (32) in der Schriftart erzwingt und Zeichen mit Codes unter 32 ebenfalls nicht erlaubt. Neue Funktionen für ähnliche Aufgaben werden in einem zukünftigen Update hinzugefügt.


Syntax:

font_get_first(ind);


Streit Beschreibung
ind Index der zu überprüfenden Schriftart.


Kehrt zurück:

Real


Beispiel:

num = font_get_first(fnt_Main);

Der obige Code speichert den ASCII-Wert des ersten Buchstabens des Zeichensatzbereichs für die Schriftart, die in "fnt_Main" indiziert ist.
