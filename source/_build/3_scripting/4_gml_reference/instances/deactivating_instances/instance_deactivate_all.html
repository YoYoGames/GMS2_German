instance_deactivate_all


Syntax:

instance_deactivate_all(notme);


Streit Beschreibung
notme Ob die aufrufende Instanz aktiviert bleiben soll (true) oder nicht (false).


Rückkehr: N / A


Beschreibung

Diese Funktion deaktiviert alle Instanzen, die sich in dem Moment befinden, in dem der Code ausgeführt wird. Dies kann die Instanz enthalten, auf der der Code ausgeführt wird, wenn das Flag "notme" auf gesetzt ist false Normalerweise möchten Sie jedoch, dass diese Instanz aktiv ist. In diesem Fall sollte das "notme" -Flag auf gesetzt werden true.

HINWEIS: Wenn Sie eine Instanz beim Raumstart deaktivieren (dh aus dem Raumerstellungscode oder aus einem Instanzenereignis einer Instanz im Raum), führen alle Instanzen, die vom Raumeditor aus im Raum platziert sind, ihr create-Ereignis noch aus bevor sie deaktiviert werden.

WARNUNG: Das Deaktivieren von Instanzen, bei denen die Physik aktiviert ist, verhindert NICHT, dass ihre Fixtures in der Physiksimulation interagieren. Dafür sollten Sie ihre einstellen phy_active Variable zu true oder false wenn Sie die Instanzen aktivieren / deaktivieren.


Beispiel:

instance_deactivate_all(true);
var _vx = camera_get_view_x(view_camera[0]);
var _vy = camera_get_view_y(view_camera[0]);
var _vw = camera_get_view_width(view_camera[0]);
var _vh = camera_get_view_height(view_camera[0]);
instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false);

Der obige Code deaktiviert alle Instanzen mit Ausnahme derjenigen, die den Code ausführen und aktiviert dann eine Region innerhalb des Raums.
