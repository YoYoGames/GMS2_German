instance_activate_region


Syntax:

instance_activate_region(left, top, width, height, inside);


Streit Beschreibung
left Die X-Koordinate der linken Seite des zu aktivierenden rechteckigen Bereichs.
top Die y-Koordinate des oberen Bereichs der zu aktivierenden rechteckigen Region.
width Die Breite der zu aktivierenden Region.
height Die Höhe der zu aktivierenden Region.
inside Ob Instanzen innerhalb der Region (true) oder extern (false) aktiviert werden sollen.


Rückkehr: N / A


Beschreibung

Mit dieser Funktion können Sie eine Region innerhalb des Raums definieren, um Instanzen zu aktivieren, die zuvor deaktiviert wurden. Diese Region kann entweder als "innen" oder "außen" gekennzeichnet werden, wie im folgenden Bild gezeigt:

Sie können im obigen Bild sehen, dass die Instanz "apple" immer aktiv ist, auch wenn das Sprite selbst die Region nicht überlappt, die Bounding Box. Es wird also davon ausgegangen, dass sich Instanzen innerhalb der angegebenen Region befinden, wenn ihre Begrenzungsbox mit ihr überlappt, und der Status der Kollisionsmaske (präzise oder nicht) wird nicht berücksichtigt.


Beispiel:

instance_deactivate_all(true);
var _vx = camera_get_view_x(view_camera[0]);
var _vy = camera_get_view_y(view_camera[0]);
var _vw = camera_get_view_width(view_camera[0]);
var _vh = camera_get_view_height(view_camera[0]);
instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false);

Der obige Code deaktiviert alle Instanzen mit Ausnahme derjenigen, die den Code ausführen und aktiviert dann eine Region innerhalb des Raums.
