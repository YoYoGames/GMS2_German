Instanzen deaktivieren


GameMaker Studio 2 bietet Ihnen die Möglichkeit, Instanzen "auszuschalten", so dass sie in keiner Weise mehr bearbeitet werden. Technisch existieren sie eigentlich nicht mehr, außer als Zeiger innerhalb des Deaktivierungsprozesses selbst, was bedeutet, dass eine deaktivierte Instanz nicht manipuliert oder in irgendeiner Weise verändert werden kann, bis sie wieder aktiviert wird. Daher sollten diese Funktionen mit großer Vorsicht verwendet werden, da sie Probleme verursachen können, wenn sie nicht ordnungsgemäß verwendet werden, insbesondere bei persistenten Objekten, da ein dauerhaftes Objekt, das deaktiviert wurde, nicht in den nächsten Raum verschoben wird, es sei denn, es wird zuerst reaktiviert Löschen aus dem Spiel).

Es ist normalerweise nicht notwendig, Instanzen in jedem Schritt Ihres Spiels zu deaktivieren. Dies kann dazu führen, dass Ihr Spiel verzögert wird und langsam läuft. Stattdessen wird empfohlen, diese Funktionen nur alle paar Schritte in einem Alarm auszuführen (z. B.), oder wenn die Ansicht ihre Position geändert hat, und es besonders wichtig ist, dass Sie diese Funktionen nicht im Zeichenereignis verwenden, da dies zu Fehlern führen kann ernsthafte Fehler in Ihrem Spiel.

Eine deaktivierte Instanz hört tatsächlich auf, im Spiel zu existieren, aber einzelne Instanzen können immer noch auf ihre Variablen zugreifen. Sie können nicht verwenden with(object) oder with(instance) Sie müssen jedoch direkt auf die Instanz zugreifen, indem Sie ihre eindeutige ID in der folgenden Form verwenden:

val = inst.variable;

Im obigen Beispiel wäre "inst" eine Variable, die die ID der deaktivierten Instanz enthält, und dies ist die einzige Möglichkeit, auf die eine deaktivierte Instanz referenziert werden kann, ohne dass sie zuvor aktiviert wurde.

HINWEIS: Wenn Sie eine Instanz beim Raumstart deaktivieren (dh aus dem Raumerstellungscode oder aus einem Instanzerstellungs-Ereignis einer Instanz im Raum), führen alle Instanzen, die vom Raumeditor aus im Raum platziert sind, ihr create-Ereignis noch aus bevor sie deaktiviert werden.
WARNUNG: Das Deaktivieren von Instanzen, bei denen die Physik aktiviert ist, verhindert NICHT, dass ihre Fixtures in der Physiksimulation interagieren. Dazu sollten Sie ihre phy_active-Variable beim Aktivieren / Deaktivieren der Instanzen auf true oder false setzen.

Zum Deaktivieren von Instanzen existieren folgende Funktionen:

  1. instance_activate_all
  2. instance_activate_object
  3. instance_activate_region
  4. instance_activate_layer
  5. instance_deactivate_all
  6. instance_deactivate_object
  7. instance_deactivate_region
  8. instance_deactivate_layer
