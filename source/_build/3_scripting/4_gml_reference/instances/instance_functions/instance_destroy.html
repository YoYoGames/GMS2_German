instance_destroy


Beschreibung

Sie rufen diese Funktion immer dann auf, wenn Sie eine Instanz "zerstören" möchten, indem Sie normalerweise ein Destroy-Ereignis und auch ein Cleanup-Ereignis auslösen. Dadurch wird es aus dem Raum entfernt, bis der Raum neu gestartet wird (es sei denn, der Raum ist persistent). Wenn Sie die Funktion ohne Argumente aufrufen, wird nur die aktuelle Instanz gelöscht und der Code wird ausgeführt. Sie können jedoch ein optionales "id" -Argument angeben und eine bestimmte Instanz mithilfe des Instanz-ID-Werts anvisieren, oder Sie können alle Instanzen von ein bestimmtes Objekt mit einem object_index. Beispielsweise:

instance_destroy(other); // destroy the other instance in a Collision Event
instance_destroy(obj_Bullet); // destroy ALL instances of the object "obj_Bullet"

Das zweite optionale Flag ermöglicht es Ihnen, das Destroy Event für die zu zerstörende Instanz "abzuschalten". Standardmäßig wird das Destroy Event immer ausgeführt, aber wenn Sie dieses Flag setzen false Dann können Sie die Instanz dazu bringen, sich selbst zu zerstören und die Ausführung dieses Ereignisses zu überspringen.

WICHTIG! Dadurch wird das Destroy-Ereignis übersprungen, aber das Cleanup-Ereignis wird weiterhin aufgerufen.


Es ist erwähnenswert, dass wenn Sie eine Instanz zerstören, das Destroy-Ereignis sofort nach dem Code oder der Aktion aufgerufen wird, die das Destroy aufruft. Zweitens, obwohl das Zerstörungsereignis ausgeführt wird, wird die Instanz nicht sofort aus dem Spiel entfernt, und sie wird weiterhin den Code ausführen, der in dem aktuellen Ereignis enthalten ist. Nur wenn das aktuelle Ereignis vorbei ist, wird es aus dem Spiel entfernt.

Also, wenn Sie zum Beispiel diesen Code haben:

if hp <= 0 instance_destroy();
score += 10;

Die Variable "score" wird erhöht , obwohl die instance_destroy Funktion wurde aufgerufen, und die Instanz wird am Ende des Events endgültig aus dem Spiel entfernt. Seien Sie vorsichtig, wenn Sie beispielsweise eine dynamische Ressource für die Instanz erstellt haben (z. B. eine Datenstruktur) und diese Ressource dann im Ereignis destroy zerstört haben, wenn nach der Funktion oder Aktion des Destroy Verweise darauf vorhanden sind wurde ausgeführt, dann erhalten Sie eine "unbekannte Ressource" -Fehler, da das Zerstörungsereignis es aus dem Spiel entfernt hat.


Syntax:

instance_destroy([id, execute_event_flag]);


Argumente:

Streit Beschreibung
id Die zu löschende Instanz-ID oder object_index (optional, Standard ist die aufrufende Instanz)
execute_event_flag Einstellen true oder false um das Zerstörungsereignis auszuführen oder nicht (optional, Standard ist true )


Kehrt zurück:

N/A


Beispiel:

if bbox_right < 0 || bbox_left > room_width || bbox_bottom < 0 || bbox_top > room_height
    {
    instance_destroy(id, false);
    }

Der obige Code zerstört die Instanz, in der der Code ausgeführt wird, ohne das Destroy-Ereignis aufzurufen, wenn sich die Instanz außerhalb der Raumgrenzen befindet.
