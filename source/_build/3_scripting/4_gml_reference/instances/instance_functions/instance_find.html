instance_find


Beschreibung

Alle Instanzen haben eine eindeutige Kennung ( id ), die verwendet werden können, um sie zu modifizieren und zu manipulieren, während ein Spiel läuft, aber Sie wissen vielleicht nicht immer, wie die ID für eine bestimmte Instanz ist und so kann diese Funktion helfen, indem Sie alle durchlaufen, um was zu finden du brauchst. Sie geben das Objekt an, nach dem Sie die Instanz von und eine Zahl suchen möchten, und wenn sich an dieser Position in der Instanzliste eine Instanz befindet, gibt die Funktion die ID dieser Instanz zurück, andernfalls gibt sie das spezielle Schlüsselwort noone zurück. Sie können auch das Schlüsselwort all verwenden, um alle Instanzen in einem Raum zu durchlaufen, sowie ein übergeordnetes Objekt, um alle Instanzen dieser übergeordneten / untergeordneten Hierarchie zu durchlaufen, und Sie können sogar eine Instanz selbst angeben (falls dies der Fall ist) seine ID), um zu prüfen, ob es tatsächlich im aktuellen Raum existiert. Bitte beachten Sie, dass bei Instanzen, die auf eine beliebige Art und Weise sortiert sind, keine bestimmte Reihenfolge für die Überprüfung der Instanzen durch diese Funktion vorhanden ist und dass jede Instanz an einer beliebigen Position stehen kann.

Der maximale Wert für "n" in dieser Funktion wäre

  • Für das Schlüsselwort all: instance_count - 1
  • Für einen Objektindex: instance_number(OBJ) - 1


Syntax:

instance_find(obj, n);


Argumente:

Streit Beschreibung
obj Das Objekt, um die n-te Instanz von
n Die Nummer der zu suchenden Instanz.


Kehrt zurück:

Real (either an instance ID value or the keyword noone)


Beispiel:

var i;
for (i = 0; i < instance_number(obj_Enemy); i += 1)
   {
   enemy[i] = instance_find(obj_Enemy,i);
   }

Der obige Code verwendet a for Schleife, um alle Instanzen von "obj_Enemy" zu durchlaufen und ihre ID im Array "enemy []" zu speichern.
