Instanzwechsel


Beschreibung

Sie können diese Funktion verwenden, um eine Instanz eines Objekts in eine andere Instanz eines anderen Objekts zu ändern, und dabei entscheiden, ob die ersten Instanzen Destroy und Cleanup Events und die neuen Instanzen Create Event ausgeführt werden sollen. Auf diese Weise können Sie zum Beispiel eine Bombe in eine Explosion verwandeln - in diesem Fall würde das Argument "perf" wahrscheinlich wahr sein, da Sie möchten, dass die Bombe ihr Destroy Event und Clean Up Event sowie die Explosion, um ihr Create Event durchzuführen - oder Sie könnten Ihren Playercharakter in einen anderen ändern - in diesem Fall wäre das Argument "perf" wahrscheinlich falsch, da Sie nicht möchten, dass die Instanzen ihre Create and Destroy / Clean Up Events ausführen.

Es ist erwähnenswert, dass das Ändern der Instanz bedeutet, dass Sie bis zum nächsten Schritt keine weiteren Aktionen mit dieser Instanz durchführen sollten, insbesondere nicht versuchen, auf Variablen usw. zuzugreifen, da dies zu einem Fehler führen wird. Grundsätzlich ändern Sie die Instanz, aber sie ist erst am Ende des aktuellen Schritts verfügbar, sodass der Zugriff auf eine der Variablen, die sie direkt enthält (z. B. Aufruf von obj_Changed.x ), nicht funktioniert.

WARNUNG! Wenn Sie eine physikaktivierte Instanz ändern, werden die physikalischen Eigenschaften nicht auf die neue Instanz übertragen, in die sie geändert wird. Daher sollten Sie Code verwenden, um den physikalischen Zustand der aktuellen Instanz auf die neue Instanz zu übertragen oder die neuen physischen Eigenschaften der Instanzen in ihrem Create Event oder im Objekteditor definiert zu haben. Aus diesem Grund wird empfohlen, dass Sie diese Funktion nicht bei physikfähigen Instanzen verwenden, sondern eine Kombination aus verwenden instance_destroy() und instance_create_layer().


Syntax:

instance_change(obj, perf);


Argumente:

Streit Beschreibung
obj Das neue Objekt, in das sich das aufrufende Objekt ändert.
perf Gibt an, ob das Erstellen und Zerstören von Ereignissen dieses neuen Objekts ausgeführt werden soll (true) oder nicht (false).


Kehrt zurück:

N/A


Beispiel:

if keyboard_check(vk_enter)
   {
   instance_change(obj_Player_Swimming, false);
   exit;
   }

Der obige Code ändert die aufrufende Instanz in eine Instanz von "obj_Player_Swimming", ohne das ursprüngliche Ereignis destroy zu erstellen, oder die neuen Instanzen erzeugen ein Ereignis, wenn der Benutzer die "Enter" -Taste drückt und den Code-Block beendet.
