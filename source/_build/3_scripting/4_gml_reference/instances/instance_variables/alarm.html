Alarm


Beschreibung

Dieses 1-dimensionale Array wird verwendet, um den aktuellen Wert für alle Alarme zu erhalten, die die Instanz möglicherweise hat, oder es kann verwendet werden, um diese Alarme einzustellen. In jede Instanz eines Objekts sind zwölf Alarme eingebaut, und jeder hat sein eigenes Ereignis, das ausgeführt wird, wenn diese Variable 0 erreicht. Es sollte beachtet werden, dass der Alarm zu diesem Zeitpunkt noch nicht beendet ist (obwohl das Ereignis ausgelöst wurde) Im nächsten Schritt wird es auf -1 heruntergesetzt. Wenn Sie einen Alarm aus irgendeinem Grund stoppen müssen, sollten Sie den Array-Wert auf -1 nicht 0 setzen. Die Alarmzeiten werden in Spielschritten berechnet, wobei der Wert 30 30 Schritte beträgt etc...

HINWEIS: Ein Alarm ohne Aktionen oder Code wird nicht ausgeführt. Aber auch mit nur einem Kommentar und ohne Code wird der Alarm heruntergezählt.

Sie können das Alarmarray direkt einstellen (und den aktuellen Alarmwert auch direkt über das Array abrufen), aber unter bestimmten Umständen ist dies nicht die am besten geeignete Methode, daher können Sie auch die Funktion verwenden alarm_set() um einen Alarm und die Funktion einzustellen alarm_get() um den Wert eines Alarms zu erhalten.


Syntax:

alarm[val];


Kehrt zurück:

Real (steps in the alarm, -1 if the alarm isn't running)


Beispiel:

if canshoot
   {
   if keyboard_check_pressed(vk_space)
      {
      canshoot = false;
      alarm[0] = room_speed;
      instance_create_layer(x, y, "Bullets", obj_Bullet);
      }
   }

Der obige Code prüft eine Variable und prüft dann, ob sie wahr ist, nach einem Tastendruck und wenn das wahr ist, setzt sie schließlich die Variable auf falsch, setzt einen Alarm (wo sie wieder auf wahr gesetzt wird) und erzeugt eine Instanz.
