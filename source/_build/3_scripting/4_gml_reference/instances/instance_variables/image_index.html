Bildindex


Beschreibung

Ein Sprite besteht aus einem oder mehreren Sub-Bildern, die das Sprite animiert erscheinen lassen, wenn sie von einem zum anderen wechseln, oder können zwischen Codes umgeschaltet werden, um verschiedene "Zustände" zu geben, ähnlich wie ein Knopf Fenster. Wenn das Sprite animiert ist, können Sie den aktuellen Frame der Animation abrufen, indem Sie das Kontrollkästchen aktivieren image_index Variable oder wenn Sie den Status eines statischen Sprites ändern möchten, können Sie ein neues Teilbild auswählen, indem Sie diese Variable auf das gewünschte Teilbild für das Sprite setzen. Bitte beachten Sie, dass für Änderungen in dieser Variablen, die sichtbar sein sollen, die Instanz entweder kein Zeichenereignis haben sollte (und deshalb GameMaker Studio 2 standardmäßig das Sprite zeichnet) oder mit einer der erweiterten Zeichenfunktionen wie gezeichnet wird draw_self() oder draw_sprite_ext().

Bitte beachten Sie, dass Sie bei Verwendung von Skeleton - Animations - Sprites immer noch die image_index Werte - siehe Funktion skeleton_animation_get_duration Beispiele dafür, wie dies zu tun ist.


Syntax:

image_index;


Kehrt zurück:

Real


Beispiel:

if image_speed > 0
   {
   if image_index > image_number - 1 instance_destroy();
   }

Der obige Code überprüft, ob das Sprite animiert, und wenn dies der Fall ist, prüft es, ob der aktuelle image_index größer ist als die Anzahl der Sub-Images und ob es die Instanz zerstört.
