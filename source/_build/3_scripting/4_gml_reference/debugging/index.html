Debuggen


Während eines Spiels ist es oft sehr wichtig, dass Sie bestimmte Details der Spielmechanik zu bestimmten Zeitpunkten kennen oder bestimmte Schlüsselwerte für einige Funktionen eingeben können, um verschiedene Methoden und Ideen für Ihr Spiel zu testen. Zu diesem Zweck stellt Ihnen GameMaker Studio 2 eine Reihe von Debug-Funktionen zur Verfügung, die im Debug-Modus (oder auf einem Windows-Ziel) Meldungen anzeigen und nach Eingabe fragen, aber ignoriert werden, wenn sie für andere Zielmodule kompiliert werden Funktionen setzen das Spiel in eine geschlossene Schleife, bis die Eingabe aufgelöst ist.

Hinweis: Obwohl sie auf bestimmten Plattformen funktionieren, dürfen sie nicht für andere Zwecke als zum Debuggen verwendet werden.
  • Debug-Modus
  • debug_event
  • get_integer
  • get_string
  • show_error
  • zeige Nachricht
  • show_question
  • show_debug_message
  • show_debug_overlay
  • code_is_compiliert
  • fps
  • fps_real

Abgesehen von diesen speziellen Funktionen bietet GameMaker Studio 2 auch eine lange Liste von Fehlermeldungen, die Sie entweder beim Kompilieren Ihres Spiels oder beim Ausführen des Spiels anzeigen können. Diese Nachrichten verweisen auf den betreffenden spezifischen Fehler und können verwendet werden, um ein Problem innerhalb eines bestimmten Objekts oder Skripts zu lokalisieren, was sehr hilfreich beim Debugging Ihres Spiels ist. Es ist jedoch nicht immer offensichtlich, was genau die Fehlermeldung bedeutet.
