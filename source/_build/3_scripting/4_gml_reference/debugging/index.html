Debuggen


Bei der Erstellung eines Spiels ist es oft sehr wichtig, dass Sie bestimmte Details der Spielmechanik zu bestimmten Zeitpunkten kennen oder in der Lage sind, bestimmte Schlüsselwerte für einige Funktionen einzugeben, um verschiedene Methoden und Ideen für Ihr Spiel zu testen. Zu diesem Zweck bietet GameMaker Studio 2 eine Reihe von Debug-Funktionen, die im Debug-Modus (oder auf einem Windows-Ziel) Meldungen anzeigen und Eingaben anfordern, die jedoch bei der Kompilierung für andere Zielmodule ignoriert werden Funktionen setzen das Spiel in eine geschlossene Schleife, bis die Eingabe aufgelöst ist.

HINWEIS: Obwohl sie auf bestimmten Plattformen funktionieren, dürfen sie nicht für andere Zwecke als das Debuggen verwendet werden.
  • Debug-Modus
  • debug_event
  • debug_get_callstack
  • get_integer
  • get_string
  • show_error
  • zeige Nachricht
  • show_question
  • show_debug_message
  • show_debug_overlay
  • code_is_compiled
  • fps
  • fps_real

Abgesehen von diesen speziellen Funktionen bietet GameMaker Studio 2 eine umfangreiche Liste von Fehlermeldungen, die Sie beim Erstellen Ihres Spiels unterstützen können. GameMaker Studio 2 kann entweder beim Kompilieren des Spiels oder beim Ausführen des Spiels angezeigt werden. Diese Meldungen weisen auf den jeweiligen Fehler hin und können verwendet werden, um ein Problem in einem bestimmten Objekt oder Skript zu lokalisieren und so den Debugging-Prozess Ihres Spiels zu unterstützen. Es kann jedoch nicht immer offensichtlich sein, was genau die Fehlermeldung bedeutet. Daher werden im folgenden Abschnitt alle möglichen Meldungen aufgelistet und erläutert, was sie bedeuten, um Ihnen das Leben zu erleichtern:

  1. Kompilierungs- und Syntaxfehler
