fps_real


Beschreibung

In GameMaker Studio 2 können Sie auf zwei Arten feststellen, wie schnell Ihr Spiel abläuft. Das game speed (wie in den Spieloptionen angegeben) und die fps (Bilder pro Sekunde). Diese Werte sind oft verwirrend, aber im Grunde ist einer die Anzahl der Spielschritte, die GameMaker Studio 2 in einer Sekunde GameMaker Studio 2 soll (Spielgeschwindigkeit), während der andere die Anzahl der CPU-Schritte ist, die GameMaker Studio 2 tatsächlich in einem GameMaker Studio 2 Sekunde (die echten fps), und dieser Wert ist im Allgemeinen viel höher als die Spielgeschwindigkeit, sinkt jedoch, wenn das Spiel komplexer wird und mehr Rechenleistung benötigt, um die eingestellte Raumgeschwindigkeit aufrechtzuerhalten.

Diese schreibgeschützte Variable gibt die aktuellen fps als Ganzzahlwert zurück. Bitte beachten Sie, dass die Funktion nur einmal in jedem Schritt des Spiels aktualisiert wird und so aussieht, als ob sie von einem Wert zum anderen "springen" würde. Dies ist jedoch normal.

HINWEIS: Auf dem HTML5-Ziel gibt diese Variable stattdessen ein Vielfaches der Monitoraktualisierungsrate zurück, da GameMaker Studio 2 sich für das Timing und den Versand auf den Browser verlassen muss.


Syntax:

fps_real


Kehrt zurück:

Real


Beispiel:

if debug_mode
   {
   draw_text(32, 32, "FPS = " + string(fps_real));
   }

Der obige Code prüft, ob sich das Spiel im Debug-Modus befindet. Wenn dies der Fall ist, werden die aktuellen echten Bilder pro Sekunde auf dem Bildschirm angezeigt.
