physics_joint_pulley_create

Erstellen Sie ein Seilrollengelenk, das zwei Instanzen so verbindet, dass die Bewegung eines einen direkten Einfluss auf die Bewegung des anderen hat.

Syntax:

physics_joint_pulley_create(inst1, inst2, w_anchor1_x, w_anchor1_y, w_anchor2_x, w_anchor2_y, l_anchor1_x, l_anchor1_y, l_anchor2_x, l_anchor2_y, ratio, max_len1, max_len2, col)


Streit Beschreibung
inst1 Die erste Instanz zum Verbinden mit dem Gelenk
inst2 Die zweite Instanz, um sich mit dem Gelenk zu verbinden
w_anchor1_x Die x-Koordinate, an der der erste Teil des Gelenks innerhalb der Spielwelt verankert ist
w_anchor1_y Die y-Koordinate, an der der erste Teil des Gelenks innerhalb der Spielwelt verankert ist
w_anchor2_x Die x-Koordinate, an der der zweite Teil des Gelenks innerhalb der Spielwelt verankert ist
w_anchor2_y Die y-Koordinate, an der der zweite Teil des Gelenks innerhalb der Spielwelt verankert ist
l_anchor1_x Die lokale x-Koordinate, in der die erste Verbindung mit der ersten Instanz verankert ist
l_anchor1_y Die lokale y-Koordinate, in der die Verbindung 1 mit der ersten Instanz verankert ist
l_anchor2_x Die lokale x-Koordinate, in der das Gelenk 2 an der zweiten Instanz verankert ist
l_anchor2_y Die lokale y-Koordinate, bei der das Gelenk 2 an der zweiten Instanz verankert ist
ratio Stellen Sie das Geschwindigkeitsverhältnis zwischen zwei Instanzen ein
col Ob die beiden Instanzen kollidieren können (wahr) oder nicht (falsch)


Rückgabe: Index des Gelenks


Beschreibung

Das Flaschenzuggelenk wird verwendet, um zwei Instanzen innerhalb der physikalischen Welt so zu verbinden, dass eine Bewegung direkt die Bewegung der anderen beeinflusst. Diese Gelenke sind zuerst an zwei Punkten im Welt-Raum verankert, von denen jeder mit einer Instanz in seinem Massenzentrum verbunden ist. Dies kann geändert werden, indem die lokalen x / y-Koordinaten des Ankers relativ zum Ursprung der Instanz gesetzt werden, was bedeutet, dass die tatsächliche Verbindung von der Instanz zum Ankerpunkt der physikalischen Welt an einer anderen Stelle als ihrem Ursprung versetzt werden kann. Das folgende Bild zeigt dies:

Wenn Sie inst2 betrachten, können Sie sehen, dass sein lokaler Ankerpunkt um 20 Pixel gegenüber dem "linken" seines Ursprungs versetzt wurde. Denken Sie daran, dass die Weltankerpunkte mit den Raumkoordinaten definiert werden, während die lokalen Ankerpunkte relativ zu den Ursprung der Instanz). Sie können auch ein Verhältnis für das Scheibengelenk angeben, das der einen oder anderen Seite sagt, dass sie sich schneller bewegen soll, mit einem Wert unter 1, der die erste Instanz schneller bewegt (EG 0,5 wird sich doppelt so schnell bewegen) und einem Wert über 1 machen die zweite Instanz bewegt sich schneller (EG 2 wird sich doppelt so schnell bewegen).

Wie bei allen Verbindungen, wenn Sie den "col" -Wert auf stellen true dann können die beiden Instanzen interagieren und miteinander kollidieren, aber nur wenn sie Kollisionsereignisse haben, wenn sie jedoch auf gesetzt sind false sie werden nicht kollidieren, egal was passiert.


Beispiel:

var t_fix, inst1, inst2;
t_fix = physics_fixture_create();
physics_fixture_set_circle_shape(t_fix, 20);
inst1 = instance_create_layer(150, room_height - 90, "Instances", obj_Block);
inst2 = instance_create_layer(300, room_height - 90, "Instances", obj_block);
physics_fixture_bind(t_fix, inst1);
physics_fixture_bind(t_fix, inst2);
physics_joint_pulley_create(inst1, inst2, 150, room_height - 140, 300, room_height - 140, 0, 0, 0, 0, 2, 0);
physics_fixture_delete(t_fix);

Der obige Code erstellt und definiert ein neues Fixture, erstellt zwei Instanzen von "obj_Block" und bindet dann dieses Fixture an zwei. Es wird dann eine Riemenscheibenverbindung zwischen diesen Beispielen ohne Versatz für die Gelenke definiert, ein Verhältnis von 2: 1 (was bedeutet, dass sich inst2 schneller bewegt). Zwischen den beiden Fällen in der Riemenscheibe treten keine Kollisionen auf.
