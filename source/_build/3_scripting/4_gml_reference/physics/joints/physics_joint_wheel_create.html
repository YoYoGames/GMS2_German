physics_joint_wheel_create

Erstellen Sie ein Radgelenk, das zwei Instanzen an einem gemeinsamen Punkt verbindet.

Syntax:

physics_joint_wheel_create(inst1, inst2, anchor_x, anchor_y, axis_x, axis_y, enableMotor, max_motor_torque, motor_speed, freq_hz, damping_ratio, col)


Streit Beschreibung
inst1 Die erste Instanz zum Verbinden mit dem Gelenk
inst2 Die zweite Instanz, um sich mit dem Gelenk zu verbinden
anchor_x Die x-Koordinate, an der das Gelenk verankert ist, innerhalb der Spielwelt
anchor_y Die y-Koordinate, an der das Gelenk innerhalb der Spielwelt verankert ist
axis_x Die x-Komponente des Radachsenvektors
axis_y Die y-Komponente des Radachsenvektors
enableMotor Ob der Motor aktiv sein soll (wahr) oder nicht (falsch)
max_motor_torque Stellt das maximale Motordrehmoment ein, das zum Erreichen der gewünschten Motordrehzahl verwendet wird (in Newton pro Meter)
motor_speed Dies ist die Geschwindigkeit, mit der sich der Motor drehen sollte
freq_hz Dies ist die Schwingungsfrequenz für das Gelenk, in Hertz
damping_ratio Dieses Dämpfungsverhältnis für die Verbindung
col Ob die beiden Instanzen kollidieren können (wahr) oder nicht (falsch)


Rückgabe: Index des Gelenks


Beschreibung

Ein Radgelenk kombiniert einfach einen Kolben und ein Drehgelenk, wie ein Rad, das am Stoßdämpfer eines Autos montiert ist. Sie geben den Ankerpunkt für den Gelenkbrunnen an, wenn die beiden physikalisch aktivierten Instanzen sich verbinden, und die erste Instanz wird als "Körper" für das Gelenk fungieren, während der zweite das "Rad" ist und sich frei bewegen darf die gemeinsame Achse. Sie können auch einen Achsenvektor einstellen, der die imaginäre Linie ist, entlang der das Gelenk als "Feder" wirkt, die es dem Rad ermöglicht, auf Hindernisse zu springen (ähnlich wie beim Kolbengelenk). Das folgende Bild zeigt dies:

Wenn Sie einen Motor aktivieren, dreht sich die zweite Instanz um die Ankerposition und Sie können das maximale Motordrehmoment für die gewünschte Motordrehzahl (N / m) sowie die Drehzahl für den Motor einstellen. Da das Radgelenk auch eine Achse hat, entlang der es sich bewegen kann, können Sie die Schwingungsfrequenz (in Hz) sowie das Dämpfungsverhältnis für das Gelenk einstellen - Sie müssen möglicherweise mit diesen Werten spielen, um sie fein abzustimmen, und es wird empfohlen Sie beginnen mit kleineren Werten und erhöhen sie, bis Sie den gewünschten Effekt erhalten.

Wie bei allen Verbindungen, wenn Sie den "col" -Wert auf stellen true dann können die beiden Instanzen interagieren und miteinander kollidieren, aber nur wenn sie Kollisionsereignisse haben, wenn sie jedoch auf gesetzt sind false sie werden nicht kollidieren, egal was passiert.


Beispiel:

var mainFixture = physics_fixture_create();
physics_fixture_set_box_shape(mainFixture, 128, 32);
var wheelFixture = physics_fixture_create();
physics_fixture_set_circle_shape(wheelFixture, 32);
var o_id = instance_create_layer(x+25, y + 100, "Instances", obj_Wheel);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(wheelFixture, o_id);
physics_joint_wheel_create(id, o_id, o_id.x, o_id.y, 0, 1, true, 10, 1, 15, 6, false);
physics_fixture_delete(mainFixture);
physics_fixture_delete(wheelFixture);

Der obige Code erstellt und definiert zwei neue Fixtures. Diese werden dann an die aufrufende Instanz und an eine neue Instanz gebunden, die erstellt wird, bevor ein Radgelenk angewendet wird, um sie zu verbinden, woraufhin die Vorrichtungen aus dem Speicher gelöscht werden.
