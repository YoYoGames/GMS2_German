physics_joint_revolute_create

Erstellen Sie ein Drehgelenk, das zwei Instanzen an einem gemeinsamen Punkt verbindet.

Syntax:

physics_joint_revolute_create(inst1, inst2, w_anchor_x, w_anchor_y, ang_min_limt, ang_max_limit, ang_limit, max_motor_torque, motor_speed, motor, col)


Streit Beschreibung
inst1 Die erste Instanz zum Verbinden mit dem Gelenk
inst2 Die zweite Instanz, um sich mit dem Gelenk zu verbinden
w_anchor_x Die x-Koordinate für das Gelenk innerhalb der Spielwelt
w_anchor_y Die y-Koordinate für das Gelenk innerhalb der Spielwelt
ang_min_limit Die untere zulässige Grenze für den Gelenkwinkel
ang_max_limit Die obere zulässige Grenze für den Gelenkwinkel
ang_limit Ob der Winkel der Verbindung zu begrenzen (wahr) oder nicht (falsch)
max_motor_torque Stellt das maximale Motordrehmoment ein, das zum Erreichen der gewünschten Motordrehzahl verwendet wird (in Newton pro Meter)
motor_speed Dies ist die Geschwindigkeit, mit der sich der Motor drehen sollte
motor Ob der Motor aktiv sein soll (wahr) oder nicht (falsch)
col Ob die beiden Instanzen kollidieren können (wahr) oder nicht (falsch)


Rückgabe: Index des Gelenks


Beschreibung

Ein Drehgelenk zwingt zwei Körper, einen gemeinsamen Ankerpunkt zu teilen (oft Gelenkpunkt genannt) und das Gelenk hat einen einzigen Freiheitsgrad - die relative Rotation der beiden Körper um diesen Punkt. Um eine Drehung zu spezifizieren, müssen Sie zwei Instanzen und einen einzelnen Ankerpunkt im Raum bereitstellen, wie Sie auf dem Bild sehen können:

Wenn Sie sich das Bild ansehen, können Sie sehen, dass die beiden Instanzen zum Überlappen erstellt wurden und an dem Punkt, an dem sie sich berühren, haben wir ein Drehgelenk definiert. Jetzt kann dieses Gelenk durch die Werte "ang_min_limit" und "ang_max_limit" in seiner Rotationsfreiheit eingeschränkt werden. Wie funktioniert das? Nun, schauen wir uns ein anderes Bild an:

Wie du sehen kannst, sind die Winkel in der Physikwelt nicht die gleichen wie in den Standardwinkeln von GameMaker Studio 2 wo rechts 0 Grad ist und dann gegen den Uhrzeigersinn, also oben 90, links 180 und unten 270 ist. Nein, Bei der Drehung des Drehgelenks verläuft die 0 ° -Achse von der Gelenkposition zum Ursprung der zweiten durch die Funktion definierten Instanz und die Winkel werden dann im Uhrzeigersinn berechnet. Wenn Sie die Winkelbegrenzung einschalten, werden die Grenzen als relativ zu dieser 0-Grad-Achse definiert und der Grenzbereich sollte Null umfassen, da andernfalls die Verbindung beim Start des Raums schlingern wird. Schließlich können Sie definieren, ob das Gelenk einen Motor hat oder nicht. Dies bedeutet, dass sich das Gelenk bei unbeeinflusster Kollision in eine Richtung bewegt, die durch die Motordrehzahl definiert ist, wobei eine positive Zahl im Uhrzeigersinn und eine negative Zahl im Gegenuhrzeigersinn steht. Das Argument "max_motor_torque" dient dazu, die Drehzahl so zu begrenzen, dass Sie keinen ständig beschleunigenden Motor bekommen und den Einfluss einer Kollision auf die Drehung begrenzen. Auf diese Weise können Sie mit einem Gelenkmotor die Gelenkreibung simulieren, indem Sie die Gelenkgeschwindigkeit auf Null und das maximale Drehmoment auf einen kleinen, aber signifikanten Wert einstellen. Der Motor wird versuchen, zu verhindern, dass sich die Verbindung dreht, wird jedoch zu einer signifikanten Last führen.

Wie bei allen Verbindungen, wenn Sie den "col" -Wert auf stellen true dann können die beiden Instanzen interagieren und miteinander kollidieren, aber nur wenn sie Kollisionsereignisse haben, wenn sie jedoch auf gesetzt sind false sie werden nicht kollidieren, egal was passiert.


Beispiel:

var mainFixture, o_id;
mainFixture = physics_fixture_create();
physics_fixture_set_circle_shape(mainFixture, sprite_get_width(sprite_index) / 2);
o_id=instance_create_layer(x+25, y, "Instances", obj_Door);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(mainFixture, o_id);
physics_joint_revolute_create(id, o_id, x+25, y, -90, 90, 1, 0, 0, 0, 0);
physics_fixture_delete(mainFixture);

Der obige Code erzeugt und definiert ein neues Fixture und erzeugt dann eine Instanz von "obj_Door", wobei das erzeugte Fixture an die zwei neuen Objekte gebunden wird. Sie werden dann durch ein Drehgelenk ohne Motor verbunden und die Winkel auf +/- 90 Grad begrenzt. Schließlich wird das Fixture gelöscht, da es nicht mehr benötigt wird.
