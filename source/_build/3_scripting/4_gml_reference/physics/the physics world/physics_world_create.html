physik_welt_erschaffen

Erstellt das Physiksystem in dem angegebenen Raum.

Syntax:

physics_world_create(pixeltometrescale);


Streit Beschreibung
pixeltometrescale definiert das Verhältnis von Pixeln auf dem Bildschirm zu Metern in der realen Welt. Ein Verhältnis von 32: 1 wird als 1/32 (oder 0,03125) angegeben.


Rückkehr: N / A


Beschreibung

Selbst wenn Sie die Physik für Ihre Objekte definiert haben, ohne eine Welt, in der sie platziert werden können, wird standardmäßig das traditionelle GameMaker Studio 2 Kollisionssystem verwendet. Sie benötigen diese Funktion, um das Physiksystem für den angegebenen Raum in einem Objektereignis zu erstellen. oder der Erstellungscode des Raums selbst (er kann auch ohne Verwendung von Code im Raumeditor definiert werden). Diese Funktion assoziiert eine physikalische "Welt" mit dem Raum und alle im Raum erzeugten Instanzen verhalten sich mit der Physik, die für sie und die Welt selbst definiert wurde.

Sie werden feststellen, dass wir ein Argument haben, das ein Verhältnis zwischen Pixelgröße und Metern definiert. Dies liegt daran, dass die Physikfunktionen mit realen Messungen arbeiten und daher müssen Pixelkoordinaten und Pixelgrößen in diese realen Messungen unter Verwendung einer "Pixel-zu-Meter-Skala" übersetzt werden. Sie sollten die Einstellung für die Pixelmaßskala anpassen, bis die durchschnittliche Pixelgröße der Objekte, die Sie verwenden, grob in simulierte Physikobjekte mit einer angemessenen Größe übersetzt wird. Darüber können Sie sehen, dass wir 32 Pixel als Äquivalent von 1 Meter eingestellt haben. Es sollte beachtet werden, dass das Physiksystem mit größeren und kleineren Verhältnissen arbeitet, aber dass die beste Leistung ist, wenn die realen Messungen Ihrer Objekte nicht weniger als 0,1 Meter und nicht mehr als 50 Meter sind.


Beispiel:

physics_world_create(1/32)

Der obige Code erzeugt eine physikalische Welt im aktuellen Raum mit einem Maßstab von 32 Pixeln, was einem Meter entspricht.
