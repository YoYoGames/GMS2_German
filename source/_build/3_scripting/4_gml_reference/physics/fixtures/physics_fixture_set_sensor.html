physics_fixture_set_sensor

Diese Funktion weist GameMaker Studio 2 dass das Gerät nur als Sensor behandelt werden soll.

Syntax:

physics_fixture_set_sensor(fixture, state)


Streit Beschreibung
fixture der Index der Vorrichtung
state ob eine Fixture ein Sensor ist (true) oder nicht (false)


Rückkehr: N / A


Beschreibung

Manchmal möchten Sie, dass Ihr Spiel erkennt und erkennt, wenn zwei Instanzen mit Fixtures kollidieren, aber keine physische Reaktion auf die Kollision haben. Dies kann erreicht werden, indem die Vorrichtung in einen Sensor umgewandelt wird, was im Grunde bedeutet, dass sie ein Kollisionsereignis ohne physische Reaktion erzeugen werden, so dass Sie diese Instanzen als "Auslöser" für andere Ereignisse im Spielraum verwenden können. Jede Fixture kann als Sensor markiert werden, und es macht keinen Unterschied, ob die Instanz, an die sie gebunden ist, statisch oder in Bewegung ist.

HINWEIS: Eine Sensorvorrichtung löst das Kollisionsereignis nur aus, wenn die Kollision zuerst auftritt, dh Sie erhalten keinen Strom von Kollisionsereignissen, da sich die beiden Körper weiterhin überlappen (was normalerweise der Fall ist). Wenn sie sich nicht mehr überlappen und sich überlappen, wird ein weiteres Kollisionsereignis ausgelöst.


Beispiel:

physics_fixture_set_sensor(fix_Cloud, 1);

Der obige Code wird den Sensorstatus des Fixtures, der in "fix_Cloud" indiziert ist, auf "true" setzen.
