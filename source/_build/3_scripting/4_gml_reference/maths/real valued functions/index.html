Nummer Funktionen

Reale Zahlen in GameMaker Studio 2 werden als Fließkommazahlen mit doppelter Genauigkeit betrachtet, dh sie haben einen Dezimalpunkt in ihnen ohne eine feste Anzahl von Ziffern vor oder nach dem Punkt. 2 ist beispielsweise eine binäre Ganzzahl, aber 2.0 ist ein Gleitkommawert. Diese Unterscheidung zwischen Ganzzahlen und Gleitkommazahlen ist sehr wichtig, wenn es um plattformübergreifende Entwicklung geht, da die Genauigkeit von Berechnungen auf einem Windows-PC nicht mit der Präzision derselben Berechnungen auf einem mobilen Gerät übereinstimmt. Dies bedeutet, dass Sie besonders auf Vergleiche achten sollten, zum Beispiel:

if image_index == 1 {do something...}

Nun, das scheint zu funktionieren, aber aufgrund der Art und Weise, wie Fließkomma-Mathematik funktioniert, kann es bedeuten, dass image_index tatsächlich gleich 1,00000000001 ist. In diesem Fall wird der Vergleich nie zu einem Ergebnis führen true Ergebnis und der Code läuft. Diese Art von Fehlern kann ziemlich schwer zu debuggen sein. Daher ist es immer gut, sie zu kennen und vorausschauend zu planen und andere Mittel zur Überprüfung von Werten zu verwenden oder die entsprechenden Boden- oder Rundungsfunktionen (unten aufgeführt) zu verwenden in eine ganze Zahl einchecken. Zum Beispiel könnte der obige Code geschrieben werden als:

if floor(image_index) == 1 {do something...}

Es ist auch erwähnenswert, dass bei der Verwendung der YoYo Compiler- Ziele alle Ausdrücke und Funktionen von links nach rechts ausgewertet werden, während sie auf allen anderen Zielplattformen von rechts nach links ausgewertet werden, was zum Beispiel zu unterschiedlichen Ergebnissen führt auf dem Bahnsteig:

val = max(num, ++num, num++);

Sie können auch eine spezielle Funktion verwenden, die in GameMaker Studio 2 zur Verfügung steht, um den Epsilon- Wert für Fließkomma-Berechnungen festzulegen. Wenn eine reelle Zahl auf die nächste Gleitkommazahl gerundet wird, bildet das Epsilon (auch als "Maschinen-Epsilon" bezeichnet) eine Obergrenze für den relativen Fehler, und Sie können den Epsilon-Wert mithilfe dieser speziellen Funktion festlegen:

  • math_set_epsilon
  • math_get_epsilon

Wir können die Funktionen, die GameMaker Studio 2 für den Umgang mit reellen Zahlen hat, in die folgenden Kategorien aufteilen:

Diese Funktionen behandeln alle mit Zufallszahlen und -werten:

  • wählen
  • zufällig
  • Zufallsbereich
  • irandom
  • irandom_range
  • random_set_seed
  • random_get_seed
  • Randomisieren
HINWEIS: Wenn Sie die Zufallsfunktionen verwenden, behält GameMaker Studio 2 jedem Start des Spiels den gleichen Zufallswert bei. Dies macht das Debuggen sehr viel einfacher, da garantiert ist, dass die Zufallsfunktionen immer den gleichen Wert zurückgeben. Wenn Sie dies nicht wünschen, müssen Sie zu Beginn des Spiels zuerst einen neuen Zufallssatz setzen, entweder mit randomise oder random_set_seed.
