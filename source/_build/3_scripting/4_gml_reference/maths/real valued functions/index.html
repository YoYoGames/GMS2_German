Anzahl Funktionen


Reelle Zahlen in GameMaker Studio 2 werden als Gleitkommazahlen mit doppelter Genauigkeit betrachtet, das heißt, sie enthalten ein Dezimalzeichen ohne feste Anzahl von Ziffern vor oder nach dem Punkt. 2 ist zum Beispiel eine binäre ganze Zahl, aber 2.0 ist eine Fließkommazahl. Diese Unterscheidung zwischen Ganzzahlen und Gleitkommazahlen ist bei der plattformübergreifenden Entwicklung sehr wichtig, da die Genauigkeit der auf einem Windows-PC durchgeführten Berechnungen nicht mit der Genauigkeit der gleichen Berechnungen auf einem mobilen Gerät identisch ist. Das bedeutet, dass Sie bei Vergleichen besonders aufpassen sollten, zum Beispiel:

if image_index == 1 {do something...}

Nun scheint das kann zu arbeiten, aber aufgrund der Art und Weise, die Punkt Mathematik funktioniert schwimmen, kann es bedeuten, dass image_index tatsächlich gleich 1,00000000001, wobei in diesem Fall der Vergleich nie in einem Ergebnis true Ergebnis und der Code läuft. Diese Art von Fehlern kann sehr schwer zu debuggen sein. Daher ist es immer gut, sich dessen bewusst zu sein und im Voraus zu planen und andere Mittel zur Überprüfung von Werten zu verwenden oder die entsprechenden Boden- oder Rundungsfunktionen (unten aufgeführt) zu verwenden, um die Anzahl in umzuwandeln check in eine ganze Zahl. Zum Beispiel könnte der obige Code geschrieben werden als:

if floor(image_index) == 1 {do something...}

Es ist auch erwähnenswert, dass bei Verwendung der YoYo Compiler- Ziele alle Ausdrücke und Funktionen von links nach rechts ausgewertet werden, während sie auf allen anderen Zielplattformen von rechts nach links ausgewertet werden. Dies bedeutet, dass diese Ergebnisse beispielsweise unterschiedliche Ergebnisse liefern auf dem Bahnsteig:

val = max(num, ++num, num++);

Sie können auch eine spezielle Funktion in GameMaker Studio 2 verwenden, um den Epsilon- Wert für Fließkomma-Mathematik festzulegen. Wenn eine reelle Zahl auf die nächste Gleitkommazahl gerundet wird, bildet das Epsilon (auch als "Maschinen-Epsilon" bezeichnet) eine Obergrenze für den relativen Fehler, und Sie können den Epsilon-Wert mithilfe der folgenden Funktionen abrufen und einstellen:

  • math_set_epsilon
  • math_get_epsilon

HINWEIS: Wenn Sie die Zufallsfunktionen verwenden, behält GameMaker Studio 2 jedem Start des Spiels den gleichen Zufallsstartwert bei. Dies erleichtert das Debuggen erheblich, da garantiert wird, dass die Zufallsfunktionen immer den gleichen Wert zurückgeben. Wenn Sie dies jedoch nicht wünschen, müssen Sie zu Beginn des Spiels einen neuen Zufallsstartpunkt festlegen, entweder mit randomise oder random_set_seed.
