Saiten


Irgendwann bei der Erstellung Ihres Spiels müssen Sie Text verwenden. Text in Spielen wird mit den String- Funktionen behandelt (eine Zeichenfolge ist nur eine andere Art, eine Textzeile auszusprechen) und GameMaker Studio 2 verfügt über einen vollständigen Satz von Funktionen, mit denen Sie Strings auf vielfältige Weise bearbeiten können, einschließlich der Einfügung eines Strings String in einem anderen, das Kopieren von Strings und die Fähigkeit, Strings für die Ziffern oder die Buchstaben, die sie enthalten, zu analysieren. Im Allgemeinen kann eine Zeichenfolge nur durch Hinzufügen von Text in Anführungszeichen "" erstellt werden. Einzelne Anführungszeichenfolgen werden nicht akzeptiert. Sie können die Zeichenfolge auch nicht über mehrere Zeilen aufteilen und erwarten, dass GameMaker Studio 2 Zeilenumbrüche als Zeilenumbrüche wiedergibt Ein Zeichenfolgenliteral @ Bezeichner wird verwendet, wie unten erläutert.

Es ist erwähnenswert, dass es bestimmte Konventionen gibt, die Sie beim Erstellen von Zeichenfolgen verwenden können, die hauptsächlich die Verwendung von Escape-Zeichen betreffen. Dies sind Zeichen, denen ein "\" Symbol vorangestellt ist. Wenn Sie beispielsweise Anführungszeichen in eine Zeichenfolge einfügen möchten, haben Sie beispielsweise Folgendes:

str = "Hello\"World\"";

Sie können das "\" verwenden, um jedem Unicode-Literal voranzukommen - 4 Ziffern von hex nach einem "u", zum Beispiel "\ u0000", wobei die 4 Ziffern die Nummer des Unicode-Zeichens - oder jedes hexadezimale Literals - 2 Ziffern von sind hex nach einer "0x", zum Beispiel "\ xff", wobei die 2 Ziffern die Nummer des zu verwendenden Zeichens sind. Diese und andere vordefinierte Escape-Zeichen sind in der folgenden Tabelle aufgeführt:

Konstante Beschreibung
\n Neue Zeile
\r Wagenrückgabe
\b Rücktaste (0x08)
\f Formular-Feed (0x0c)
\t Horizontale Registerkarte (0x09)
\v Vertikale Registerkarte (0x0b)
\\ Backslash selbst (0x5c)
\a Warnung (0x07)
\u[Hex Digits] Hex-Unicode-Zeichen einfügen (max. 4 Ziffern beachten)
\x[Hex Digits] Hex Unicode Zeichen einfügen
\[Octal Digits] Fügen Sie ein oktales Unicode-Zeichen ein
HINWEIS: Wenn Sie bei der Verwendung der oktalen oder hexadezimalen Unicode-Werte Zeichen benötigen (z. B. "\ x1234567" und Sie nur das Zeichen 0x123 und 4567 als tatsächliche Ziffern haben wollten), beenden Sie es mit "\" und es wird buchstäblich das nächste Zeichen zitieren (zB "\ x123 \ 4567")
HINWEIS: Zeichenfolgen unterstützen den Seitenvorschub, die vertikale Registerkarte usw. Dies bedeutet jedoch nicht, dass das Rendering dies tut, und beim Zeichnen von Zeichenfolgen können diese Zeichen ignoriert werden.

Sie können auch wortgetreue Zeichenfolgenliterale erstellen, indem Sie der ganzen Zeichenfolge das Zeichen @ voranstellen:

var test = @"
Line breaks
over multiple
lines
";

Der obige Code wird die Zeichenfolge über mehrere Zeilen rendern, als ob ein Zeilenumbruch-Escape-Zeichen enthalten wäre. Ein wortgetreues String-Literal ähnelt früheren String-Literalen der GameMaker Version, verwendet aber auch doppelte oder einfache Anführungszeichen und muss ein @ -Zeichen als Präfix haben. Sie können über mehrere Zeilen in der Code-Datei aufgeteilt werden und unterstützen KEINE Escape-Zeichen, z. Hello \ World "wird NICHT versuchen dem W on World zu entfliehen und wird wörtlich gespeichert. Beachten Sie jedoch, dass Sie bei Verwendung von Zeichenfolgenliteralen wie dieser die Zeichenfolge unterbrechen müssen, wenn Sie Anführungszeichen als Teil der Zeichenfolge einfügen möchten, dh:

var test = @"Hello " + "\"" + @"World" + "\""

Eine andere Sache zu beachten ist, dass das Unicode-Zeichen 9647 (▯) verwendet wird, um fehlende Glyphen zu ersetzen, die Sie möglicherweise in der angegebenen Schriftart haben, wenn Sie es im Zeichenereignis rendern. Wenn Ihre Schriftart also beispielsweise nicht das Symbol ° hat, ergibt das Schreiben von 90 ° tatsächlich 90 °.
