iap_purchase_details

Fügen Sie einer ds_map Kaufdetails eines einzelnen Produkts hinzu.

Syntax:

iap_purchase_details(product_id, ds_map);


Streit Beschreibung
product_id Die Produkt-ID für den Kauf.
ds_map Die ds_map, in der die Kaufinformationen gespeichert werden.


Rückgabe: N / A


Beschreibung

Mit dieser Funktion können Sie ein zuvor erstelltes füllen ds_map mit einer Reihe von Schlüssel / Wert-Paaren, die die Kaufdetails des jeweiligen Produkts beschreiben. Die folgenden Daten werden in der Karte zurückgegeben:

  • " Produkt " - Die Produkt-ID-Zeichenfolge.

  • " order " - Die eindeutige Bestell-ID, sofern verfügbar (nicht alle Geschäfte unterstützen dies).

  • " token " - Die Kauf-Token-Zeichenfolge, sofern verfügbar (nicht alle Stores unterstützen dies).

  • " payload " - Die vom Zielspeicher zurückgegebene Nutzlast, sofern verfügbar (nicht alle Einkäufe erfordern dies und nicht alle Geschäfte unterstützen dies).

  • " Quittung " - Die Quittungszeichenfolge, sofern verfügbar (nicht alle Geschäfte unterstützen dies).

  • „Antwort“ - Ein Antwortwert (als reelle Zahl), soweit verfügbar, auf der Grundlage der Antwortwerte von Google Play (siehe hier für weitere Details).

Die Karte enthält außerdem einen weiteren Schlüssel, " status ", mit dem der aktuelle Status des Kaufs überprüft werden kann. Dieser Schlüssel hat eine der folgenden Konstanten als Wert:

Konstante Beschreibung
iap_available Produkt ist verfügbar oder das Spiel wartet auf das Ergebnis eines Kaufs.
iap_failed Der Kaufversuch ist fehlgeschlagen.
iap_purchased Das Produkt wurde erfolgreich gekauft.
iap_canceled Der Benutzer hat den Kauf abgebrochen.
iap_refunded Der Kauf wurde zurückgenommen.



Die folgenden Hinweise beziehen sich auf die Schlüssel "Token" und "Quittung" der Karte, die in Bezug auf den verwendeten Zielspeicher zurückgegeben werden:

  1. Für den Amazon Store werden die Kaufeinträge "Token" und "Quittung" als ein und dasselbe behandelt.

  2. Für Google Play gibt es keine "Beleg" -Daten, und es wird von Ihnen erwartet, dass Sie Einkäufe über die zurückgegebene "Nutzlast" bestätigen (falls Sie über eine Nutzlast prüfen, wird empfohlen, keine fest codierten Werte zu verwenden). GameMaker Studio 2 wird die von Google Play bei einem Kauf zurückgegebene Datensignatur weiterhin anhand des öffentlichen Schlüssels überprüfen, der auf der Registerkarte IAP der Spieloptionen angegeben ist.

  3. Bei Google Play ist das mit einem Kauf gelieferte "Token" für die Verwendung mit umfangreichen Käufen vorgesehen.

  4. Für iOS- und MacOS- Zielspeicher stehen "Kauf" -Daten für Einkäufe zur Verfügung, das zugehörige "Token" wird jedoch intern für Entwicklerzwecke generiert.



HINWEIS: Die ds_map used wird nicht von der Funktion erstellt, was bedeutet, dass Sie sie zuvor erstellen und aus dem Speicher entfernen müssen, wenn sie nicht mit der entsprechenden Funktion verwendet wird.


Beispiel:

var p_map = ds_map_create();
var p_index = ds_map_find_value(iap_data, "index");
iap_purchase_details(p_index, p_map);
if ds_map_find_value(p_map, "status") == iap_purchased
   {
   global.Gold += 100000;
   }
ds_map_destroy(p_map);

Der obige Code erstellt eine ds_map und füllt sie mit den Schlüssel / Wert-Paaren für den Kauf mit der Produkt-ID, die in der Variablen "p_index" gespeichert ist, aus, die wir aus dem Special erhalten iap_data ds_map, das ausschließlich im IAP-Ereignis erstellt wird. Wenn die "Status" -Taste die Konstante zurückgibt iap_purchased wird dann eine globale Variable gesetzt.
