iap_purchase_details

Hinzufügen von Einkaufsdetails eines einzelnen Produkts zu einer ds_map

Syntax:

iap_purchase_details(product_id, ds_map);


Streit Beschreibung
product_id Die Produkt-ID-Zeichenfolge für den Kauf.
ds_map Die ds_map, die die Kaufinformationen speichert.


Rückkehr: N / A


Beschreibung

Mit dieser Funktion können Sie ein zuvor erstelltes Objekt auffüllen ds_map mit einer Anzahl von Schlüssel / Wert-Paaren, die die Kaufdetails des gegebenen Produkts beschreiben. Die folgenden Daten werden in der Karte zurückgegeben:

  • " Produkt " - Die Produkt-ID-Zeichenfolge.

  • " Bestellung " - Die eindeutige Bestell-ID, sofern verfügbar (nicht alle Shops unterstützen dies).

  • " Token " - Die Einkaufstokenkette, sofern verfügbar (nicht alle Filialen unterstützen dies).

  • " Payload " - Die vom Zielspeicher zurückgegebene Payload, sofern verfügbar (nicht alle Käufe erfordern dies, und nicht alle Shops unterstützen dies).

  • " Quittung " - Die Quittungszeichenfolge, sofern verfügbar (nicht alle Filialen unterstützen dies).

  • „Antwort“ - Ein Antwortwert (als reelle Zahl), soweit verfügbar, auf der Grundlage der Antwortwerte von Google Play (siehe hier für weitere Details).

Die Karte enthält außerdem einen weiteren Schlüssel, " Status ", mit dem der aktuelle Status des Kaufs überprüft werden kann. Dieser Schlüssel hat eine der folgenden Konstanten als Wert:

Konstante Beschreibung
iap_available Das Produkt ist verfügbar oder das Spiel wartet auf das Ergebnis eines Kaufs.
iap_failed Der Kaufversuch ist fehlgeschlagen.
iap_purchased Das Produkt wurde erfolgreich erworben.
iap_canceled Der Nutzer hat den Kauf storniert.
iap_refunded Der Kauf wurde storniert.



Die folgenden Hinweise beziehen sich auf die Schlüssel "token" und "receipt" der Map, die in Bezug auf den verwendeten Zielspeicher zurückgegeben werden:

  1. Für den Amazon Store werden die Kauf- "Token" und "Quittungs" -Einträge als ein und dasselbe behandelt.

  2. Für Google Play gibt es keine "Quittungs" -Daten und es wird von Ihnen erwartet, dass Sie die Käufe über die zurückgegebenen "Payloads" verifizieren (sollten Sie über eine Payload verifizieren, wird empfohlen, dass Sie keine hartcodierten Werte verwenden). GameMaker Studio 2 überprüft weiterhin die bei einem Kauf durch Google Play zurückgegebene Datensignatur mit dem öffentlichen Schlüssel, der auf der Registerkarte " IAP" der Spieloptionen angegeben ist.

  3. Bei Google Play ist das mit einem Kauf versehene "Token" zur Verwendung mit konsumierenden Käufen gedacht.

  4. Für iOS-, MacOS-, Windows 8 (N und JS) - und Windows Phone Zielspeicher gibt es "Empfangsdaten" für Einkäufe, aber das zugehörige "Token" wird intern für Entwicklerzwecke generiert.



HINWEIS: Die ds_map used wird nicht von der Funktion erstellt, dh Sie müssen es zuvor erstellen und es aus dem Speicher entfernen, wenn es nicht mit der entsprechenden Funktion verwendet wird.


Beispiel:

var p_map = ds_map_create();
var p_index = ds_map_find_value(iap_data, "index");
iap_purchase_details(p_index, p_map);
if ds_map_find_value(p_map, "status") == iap_purchased
   {
   global.Gold += 100000;
   }
ds_map_destroy(p_map);

Der obige Code erstellt eine ds_map und füllt sie dann mit den Schlüssel / Wert-Paaren für den Kauf mit der Produkt-ID, die in der Variablen "p_index" gespeichert ist, die wir von der speziellen erhalten iap_data ds_map, die ausschließlich im IAP-Event erstellt wird. Wenn die Taste "Status" die Konstante zurückgibt iap_purchased Dann wird eine globale Variable festgelegt.
