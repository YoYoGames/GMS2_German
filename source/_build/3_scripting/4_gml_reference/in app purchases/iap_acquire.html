iap_acquire

Dies erfordert den Erwerb des gegebenen Produkts.

Syntax:

iap_acquire(purchase_id, payload);


Streit Beschreibung
purchase_id Die Produkt-ID-Zeichenfolge für den Kauf.
payload Dies ist eine Zeichenfolge, die für die Kaufbestätigung bei Google Play verwendet werden kann (verwenden Sie eine leere Zeichenfolge "" auf allen anderen Plattformen).


Rückgabe: Echt


Beschreibung

Mit dieser Funktion erwerben Sie einen Kauf über den Zielspeicher für das Gerät, auf dem Ihr Spiel ausgeführt wird. Diese Produkt-ID-Zeichenfolge ist diejenige, die Sie beim Aktivieren des IAP mit angeben iap_activate und sollte identisch mit dem sein, mit dem Sie das Produkt im Zielspeicher identifiziert haben. Diese Funktion gibt einen Wert als "Index" zurück, so dass sie identifiziert werden kann, wenn die Kaufdaten durchkommen und das Aufrufen der Funktion ein IAP-Ereignis vom "Typ" auslöst. iap_ev_purchase, die den folgenden zusätzlichen Schlüssel enthalten wird:

  • " index " - Die Kauf-ID für das aktivierte Produkt.

Mit dieser Einkaufs-ID kann dann geprüft werden, welcher Einkauf das Ereignis ausgelöst hat (Vergleich mit dem von Ihnen gespeicherten Indexwert), damit Sie entsprechend damit umgehen können. Wenn Sie für Google Play (auf dem Android Ziel) entwickeln, können Sie auch die Zeichenfolge "Payload" festlegen. Dies wird von Google zur Überprüfung des Produkts verwendet und zusammen mit der Kaufanfrage an Google Play gesendet. Wenn ein Kauf erfolgreich abgeschlossen wurde, erhält das Spiel die Nutzlast, die als Teil der Kaufinformationen verwendet wird. Indem Sie bei jedem Kauf eine Payload generieren, können Sie damit eine einfache Form der Kaufbestätigung bereitstellen.


Beispiel:

if mouse_check_button_pressed(mb_left)
   {
   iap_acquire("noads", "")
   }

Der obige Code sendet eine Kaufanfrage, wenn die Maustaste gedrückt wird.
