iap_activate

Ermöglicht die Unterstützung für Einkäufe in Apps.

Syntax:

iap_activate(product_index);


Streit Beschreibung
product_index Der Index der ds_list das enthält die verfügbaren Einkäufe.


Rückgabe: N / A


Beschreibung

Diese Funktion ermöglicht den Kauf von In-App-Käufen und bereitet GameMaker Studio 2 durch Bereitstellung des Produkts vor ds_list das hält eine reihe von ds_maps mit den Informationen zu jedem verfügbaren Kauf. Das bedeutet, dass Sie GameMaker Studio 2 alle verfügbaren Kaufoptionen mitteilen müssen, indem Sie sie individuell festlegen ds_maps (eine für jeden verfügbaren Kauf), die dann in einem gespeichert werden ds_list, das dann mit dieser Funktion "aktiviert" wird.

Die einzelnen Kaufkarten, die im gespeichert werden sollen ds_list sollte das folgende Format von Schlüssel-Wert-Paaren haben:

  • " id " - Die Produkt-ID für den Kauf, z. B. "LevelPack".

  • " title " - Der Name des Kaufs, zB: "Level Pack 1".

  • " description " - Die Beschreibung des Kaufs, zB: "Level Pack 1 für Catch The Clown".

  • " price " - Der Preis des Kaufs, zB: "1,00 $1 ".

Es ist erwähnenswert, dass der einzige wesentliche Schlüssel für die Einrichtung eines Zielspeichers (pro Produkt-ID) die "ID" ist.

Wenn Sie Einkäufe aktivieren, wird auch ein IAP-Ereignis ausgelöst, das ein besonderes Ereignis erzeugt iap_data ds_map des Ereignistyps iap_ev_product. Diese ds_map hat den folgenden zusätzlichen Schlüssel:

  • " index " - Die Produkt-ID für das aktivierte Produkt.

Wenn Sie mehrere Produkte aktivieren, löst jedes Produkt ein eigenes IAP-Ereignis des Typs aus iap_ev_product wo Sie dann die Produkt-ID erhalten können. Es ist erwähnenswert, dass der Google Play Store (für Android ) nur Details für Produkte 20 gleichzeitig verarbeiten kann, was zu recht langen Ladezeiten für Anwendungen mit einer großen Anzahl von Produkten führen kann.



HINWEIS: Alle Schlüssel / Wert-Paare, aus denen eine Kaufkarte besteht, sind Zeichenfolgen!


Beispiel:

var purchaseList, purchase1;
purchaseList = ds_list_create();
purchase1 = ds_map_create();
ds_map_add(purchase1, "id", "LevelPack");
ds_map_add(purchase1, "title", "ExtraLevels1");
ds_map_add(purchase1, "description", "Level Pack 1 for Catch The Clown");
ds_map_add(purchase1, "price", "$1.00");
ds_list_add(purchaseList, purchase1);
iap_activate(purchaseList);
ds_map_destroy(purchase1);
ds_list_destroy(purchaseList);

Der obige Code erstellt eine ds_list und ein ds_map, die dann mit den Informationen für einen Kauf gefüllt wird. Diese Karte wird der Liste hinzugefügt und dann als verfügbarer Kauf für das Spiel aktiviert. Endlich, das ds_list und ds_map werden entfernt, um Speicherlecks zu vermeiden, da sie nicht mehr benötigt werden.
