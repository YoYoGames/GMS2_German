iap_activate

Ermöglicht die Unterstützung für In-App-Käufe.

Syntax:

iap_activate(product_index);


Streit Beschreibung
product_index Der Index der ds_list das enthält die verfügbaren Käufe.


Rückkehr: N / A


Beschreibung

Diese Funktion ermöglicht die Unterstützung von In-App-Käufen und bereitet GameMaker Studio 2 durch Bereitstellung des Produkts vor ds_list das hält eine Reihe von ds_maps mit den Informationen zu jedem verfügbaren Kauf. Das bedeutet, dass Sie GameMaker Studio 2 alle verfügbaren Kaufoptionen mitteilen müssen, indem Sie sie einzeln festlegen ds_maps (eine für jeden verfügbaren Kauf), die dann in einem gespeichert werden ds_list, die dann mit dieser Funktion "aktiviert" wird.

Die einzelnen Einkaufskarten, die in der. Gespeichert werden sollen ds_list sollte das folgende Format von Schlüssel / Wert-Paaren haben:

  • " id " - Die Produkt-ID für den Kauf, zB: "LevelPack".

  • " Titel " - Der Name des Kaufs, zB: "Level Pack 1".

  • " Beschreibung " - Die Beschreibung des Kaufs, zB: "Level Pack 1 für Catch The Clown".

  • " Preis " - Der Preis des Kaufs, zB: " $1 1,00".

  • " Typ " - nur Windows 8 und Windows Phone! Die Art des Kaufs, die entweder "Verbrauchbar" oder "Langlebig" sein sollte. "Dauerhaft" ist der Standardwert (nicht konsumierbar).

Es ist erwähnenswert, dass die einzigen wesentlichen Schlüssel für jedes Zielspeicher-Setup (pro Produkt-ID) die "ID" ist, mit Ausnahme der Windows 8 und Windows Phone Ziele. In diesem Fall benötigen Sie auch den Schlüssel "type". Beachten Sie, dass für diese Zielplattformen, wenn Sie den Speicher wünschen sinnvoll (korrekt) Daten zu melden, wenn im Sandbox - Modus, dann den „Titel“, „Beschreibung“, etc... sind notwendig.

Durch das Aktivieren von Käufen wird auch ein IAP-Ereignis ausgelöst, wodurch eine spezielle Aktion erstellt wird iap_data ds_map des Ereignistyps iap_ev_product. Diese ds_map wird den folgenden zusätzlichen Schlüssel haben:

  • " index " - Die Produkt-ID für das aktivierte Produkt.

Wenn Sie mehrere Produkte aktivieren, löst jedes Produkt ein eigenes IAP-Ereignis des Typs aus iap_ev_product wo Sie dann die Produkt ID bekommen können. Es ist erwähnenswert, dass der Google Play Store (für Android ) nur Details für Produkte 20 gleichzeitig verarbeiten kann, was zu recht langen Ladezeiten für Anwendungen mit einer erheblichen Anzahl von Produkten führen kann.



HINWEIS: Alle Schlüssel / Wert-Paare, aus denen eine Einkaufszuordnung besteht, sind Zeichenfolgen!


Beispiel:

var purchaseList, purchase1;
purchaseList = ds_list_create();
purchase1 = ds_map_create();
ds_map_add(purchase1, "id", "LevelPack");
ds_map_add(purchase1, "title", "ExtraLevels1");
ds_map_add(purchase1, "description", "Level Pack 1 for Catch The Clown");
ds_map_add(purchase1, "price", "$1.00");
ds_list_add(purchaseList, purchase1);
iap_activate(purchaseList);
ds_map_destroy(purchase1);
ds_list_destroy(purchaseList);

Der obige Code erstellt ein ds_list und ein ds_map, die dann mit den Informationen zum Kauf gefüllt wird. Diese Karte wird zur Liste hinzugefügt und dann als verfügbarer Kauf für das Spiel aktiviert. Endlich, das ds_list und ds_map werden entfernt, um Speicherlecks zu verhindern, da sie nicht mehr benötigt werden.
