In-App Käufe


In-App-Käufe sind ein wichtiger Bestandteil moderner, plattformübergreifender Spiele, die Ihnen die Möglichkeit geben, "Freemium" -Inhalte zu haben, bei denen Ihr Spiel kostenlos ist. Bestimmte Inhalte sind jedoch nur durch Kauf innerhalb des Spiels selbst verfügbar. Zum Beispiel ist Ihr Puzzle-Spiel kostenlos, aber die zusätzlichen Level kosten den Benutzer 0,99 $. Dieses App-Modell ist sehr verbreitet und kann mit minimalem Arbeitsaufwand in GameMaker Studio 2 werden. In diesem Abschnitt des Handbuchs finden Sie Details zu allen verfügbaren Funktionen, mit denen Sie genau dies tun können, sowie einige Ratschläge für Einrichtung und Informationen zum Testen auf den verschiedenen Plattformen.

Das Einrichten von App-Käufen innerhalb Ihres Spiels ist mit GameMaker Studio 2 relativ einfach, erfordert jedoch einige Kenntnisse des Zielplattform-Setups sowie eine sorgfältige Planung, bevor Sie mit der Erstellung des Spiels beginnen. Es wird daher nicht empfohlen, dies zu verwenden Funktionalität, wenn Sie gerade mit GameMaker Studio 2 oder wenn Sie nicht sicher sind, wie die verschiedenen Stores für jede Zielplattform eingerichtet sind.

Bevor Sie irgendetwas anderes tun, müssen Sie sicherstellen, dass Sie die Kaufdetails im Zielspeicher für jede Plattform, die Sie unterstützen möchten, eingerichtet haben. Beachten Sie dabei genau die von Ihnen definierten Produkt-IDs, da diese ID-Zeichenfolgen wie angegeben sind GameMaker Studio 2 identifiziert jedes Produkt und seinen Kauf. In den meisten Stores müssen Sie auch eine Basisdatei hochgeladen haben, auch wenn das Spiel noch keine In-App-Käufe enthält.

HINWEIS: In GameMaker Studio 2 derzeit bis zu 128 verschiedene Kaufoptionen pro Titel zulässig, es sei denn, Sie verwenden den Amazon Store (für Android ), wo er auf 100 beschränkt ist.


Nachfolgend sind die Funktionen aufgeführt, die in GameMaker Studio 2 integriert sind, um App-Käufe auf den verschiedenen Zielplattformen zu ermöglichen (damit können Sie zusätzliche Credits, Gegenstände, Level und andere Dinge innerhalb des Spiels selbst erwerben). Viele dieser Funktionen lösen ein IAP Asynchronous Event aus, das ein spezielles Ereignis enthält ds_map namens iap_data mit einem Anfangsschlüssel "type", der abgefragt werden kann, um den Typ des IAP-Ereignisses zu ermitteln, das ausgelöst wurde. Sobald Sie wissen, um welche Art von IAP-Ereignis es sich handelt, können Sie weitere Daten aus der für das Ereignis relevanten Karte abrufen.

Die in der App verfügbaren Funktionen sind:

  • iap_activate
  • iap_status
  • iap_enumerate_products
  • iap_restore_all
  • iap_acquire
  • iap_consume
  • iap_product_details
  • iap_purchase_details

Sie sollten sich auch darüber im Klaren sein, dass GameMaker Studio 2 aktiven GameMaker Studio 2 nicht für mehrere Spiele des Spiels (und somit auch für das Spiel) beibehält iap_is_purchased() ist unzuverlässig, wenn ein Geschäft nicht erfolgreich geladen wurde). Das bedeutet, dass Sie diese Informationen selbst speichern und bei jedem Start Ihres Spiels erneut abrufen müssen. Zu diesem Zweck können Sie die speziellen "sicheren" Speicher- und Ladefunktionen für ds_maps verwenden:

  • ds_map_secure_save
  • ds_map_secure_load

GameMaker Studio 2 nicht für mehrere Spiele des Spiels bei, da es für uns nicht angemessen ist, eine sichere Methode zur Überprüfung der Gültigkeit eines Einzelkaufs bereitzustellen, und daher wäre jeder automatisch gespeicherte Kaufzustand übermaßig. Auf den meisten Plattformen sind "Empfangsdaten" über verfügbar iap_purchase_details() welche Benutzer zur Bestätigung eines Kaufs verwenden können, obwohl Google Play stattdessen "Token" -Daten verwendet.

GameMaker Studio 2 erfordert zur einfachen Überprüfung des lokalen Kaufs den öffentlichen Google Play-Schlüssel. Sie können überprüfen, ob die mit einem Kaufergebnis zurückgegebenen "Payload" -Daten denen des übereinstimmenden Anrufs entsprechen iap_acquire() (Weitere Informationen zum Google Public Key finden Sie im Abschnitt " Packaging" in den Android Spieloptionen.)

In-App-Käufe sind für Android, Amazon Fire, iOS- und MacOS-Ziele verfügbar. Im YoYo Games Helpdesk (und in einem Tutorial für Amazon Fire ) sind Artikel verfügbar, mit denen Sie die einzelnen Komponenten richtig einrichten können:

  • iOS: IAPs einrichten
