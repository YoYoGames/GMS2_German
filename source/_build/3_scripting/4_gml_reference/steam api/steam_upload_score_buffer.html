steam _upload_score_buffer

Syntax:

steam_upload_score_buffer(lb_name, score, buffer);


Streit Beschreibung
lb_name Der Name der Bestenliste, in die Sie die Bewertungen hochladen (eine Zeichenfolge).
score Die Punktzahl zum Hochladen (eine echte Zahl).
buffer Die ID des Puffers zum Anhängen.


Rückgabe: Echt


Beschreibung

Diese Funktion sendet zusammen mit einem aus einem Puffer erstellten Datenpaket eine Punktzahl an die angegebene Bestenliste. Der Puffer sollte nicht mehr als 256 Bytes groß sein - alles darüber hinaus gehackte - und kann alle Daten enthalten, die Sie benötigen. Die zu ladende Punktzahl sollte eine reelle Zahl sein, und der Name der Bestenliste ist eine Zeichenfolge, die beim Erstellen der Bestenliste mithilfe der Funktion definiert wurde steam_create_leaderboard.

Wenn die Funktion fehlschlägt, wird ein Wert von -1 zurückgegeben, während ein erfolgreicher Post eine eindeutige ID für den Upload zurückgibt und ein asynchrones Steam Ereignis auslöst. Dieses Ereignis erstellt ein ds_map in der eingebauten Variable async_load mit den folgenden Schlüssel / Wert-Paaren:

  • " event_type " - Dieser Schlüssel enthält den Wert " leaderboard_upload ", wenn er von dieser Funktion ausgelöst wird

  • " lb_name " - Dieser Schlüssel enthält den Namen der Bestenliste, auf die gepostet wurde

  • " Erfolg " - Wird 1, wenn der Beitrag erfolgreich war, 0 fehlgeschlagen

  • " aktualisiert " - Wird 1, wenn der Ranglistenpunkt tatsächlich aktualisiert wurde (dh der neue Punktestand war besser) oder 0, andernfalls. Beachten Sie, dass die hochgeladenen Pufferdaten auch nicht hinzugefügt werden, wenn der Score aktualisiert wird.

  • " post_id " - Dieser Schlüssel sollte mit dem vom Upload-Aufruf zurückgegebenen ID-Wert übereinstimmen

  • " score " - Dieser Schlüssel enthält den Punktestand, der gepostet wurde

Beachten Sie, dass, wenn die Funktion beim Herunterladen der Bestenliste die "Einträge" -Taste der async_load Die Karte hat nun einen "Daten" -Schlüssel, so dass Sie den hochgeladenen Datenpuffer abrufen können (weitere Informationen finden Sie im erweiterten Steam Event- Codebeispiel).


Erweitertes Beispiel:

In diesem Beispiel laden wir zunächst eine Partitur hoch und analysieren dann die Partitur async_load Karte zurückgegeben, wenn erfolgreich. Der folgende Code zeigt ein typisches Beispiel für das Hochladen, wobei ein Puffer erstellt wird, der eine Zeichenfolge enthält, die uns sagt, aus welcher Ebene der Score hochgeladen wurde:

if (hp <= 0)
   {
   var buff = buffer_create(256, buffer_fixed, 1 );
   buffer_write(buff, buffer_string, "Uploaded on level " + string(global.Level));
   upload_ID = steam_upload_score("Game Scores", score, buff);
   if (!upload_ID)
      {
      alarm[0] = room_speed;
      }
   buffer_delete(buff);
   }

Beachten Sie, dass wir einen Alarm gesetzt haben, wenn der Anruf fehlschlägt. Dies würde verwendet werden, um den Upload zu einem späteren Zeitpunkt erneut zu versuchen, und Sie können dort zusätzlichen Code hinzufügen, um den Upload erneut zu versuchen oder den Score in eine Textdatei zu speichern, sollte es weiterhin fehlschlagen, etc... Beachten Sie auch, dass wir sofort löschen Puffer, da er für die Funktion nicht mehr benötigt wird. Wir fügen nun Folgendes in das asynchrone Steam Ereignis für die Instanz ein, die die Scores steuert:

var type = ds_map_find_value(async_load, "event_type");
if (type == "leaderboard_upload")
   {
   var lb_ID = ds_map_find_value(async_load, "post_id");
   if lb_ID == upload_ID
      {
      var lb_name = ds_map_find_value(async_load, "lb_name");
      var lb_done = ds_map_find_value(async_load, "success");
      var lb_score = ds_map_find_value(async_load, "score");
      var lb_updated = ds_map_find_value(async_load, "updated");
      show_debug_message("leaderboard post id:" + string(lb_ID) + " to lb:" + string(lb_name) + " with score:" + string(lb_score) + " updated=" + string(lb_updated));
      if (lb_done)
         {
         show_debug_message("- Succeeded");
         }
      else
         {
         show_debug_message("- Failed");
         }
      }
   }

Im Beispiel geben wir einfach die Rückgabewerte als Debug-Meldungen an das Compiler-Fenster aus, aber Sie können dieses Ereignis verwenden, um mit den Informationen in beliebiger Weise umzugehen.
