steam _ugc_unsubscribe_item

Syntax:

steam_ugc_unsubscribe_item(published_file_id);


Streit Beschreibung
published_file_id Die eindeutige Datei-ID, mit der sich der UGC abmelden soll.


Rückgabe : Async-ID


Beschreibung

Diese Funktion kann verwendet werden, um sich von einem UGC-Element abzumelden. Sie geben die veröffentlichte Datei-ID an und die Funktion gibt dann einen eindeutigen Async-ID-Wert zurück, mit dem überprüft werden kann, ob das Element im Steam Async-Ereignis korrekt abgemeldet wurde. Diese Veranstaltung wird die async_load DS-Karte, die mit den folgenden Schlüssel / Wert-Paaren gefüllt wird:

  1. " id " - Die asynchrone ID, die von der Funktion zurückgegeben wurde, die das Ereignis ausgelöst hat

  3. " Ergebnis " - Das Ergebnis der Operation (ein realer Wert). Dies ist entweder die GML-Konstante ugc_result_success oder eine andere reelle Zahl. Sie sollten also nach dieser Konstante suchen, um sicherzustellen, dass der Aufruf erfolgreich war, und wenn sonst etwas nicht korrekt funktioniert hat. Der Rest der möglichen Werte wird als Ergebnis des Steam "EResult" -Werts angezeigt und Sie sollten sehen steamclientpublic.h in den SDK-Headern für alle 89 möglichen Werte.

  5. " event_type " - Die Zeichenfolge " ugc_unsubscribe_item ".

  7. " published_file_id " - Die eindeutige ID für den veröffentlichten Artikel.


Beispiel:

steam_sub = steam_ugc_subscribe_item(global.pubFileID);

Der obige Code wird das Element mit der Datei-ID, die in der globalen Variable "pubFileID" gespeichert ist, abonnieren (und herunterladen).
