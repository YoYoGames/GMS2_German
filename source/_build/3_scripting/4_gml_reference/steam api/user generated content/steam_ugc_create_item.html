steam _ugc_create_item

Syntax:

steam_ugc_create_item(consumer_app_id, file_type);


Streit Beschreibung
consumer_app_id Die einzigartige App ID für dein Spiel auf Steam.
file_type Eine der verfügbaren Dateitypkonstanten (unten aufgeführt).


Rückgabe : Async-ID


Beschreibung

Diese Funktion wird verwendet, um die Workshop-API vorzubereiten und eine veröffentlichte Datei-ID für den hinzuzufügenden Artikel zu generieren. Die Funktion muss aufgerufen werden, bevor Sie etwas anderes mit dem zu ladenden Objekt ausführen, da Sie den eindeutigen veröffentlichten ID-Wert verwenden müssen, den er im Steam Async-Ereignis zur Aktualisierung zurückgibt. Um diese Funktion nutzen zu können, müssen Sie die Steam App - ID für Ihr Spiel angeben und eine der folgenden Konstanten für das Spiel verwenden file_type Streit:

Konstante Beschreibung
ugc_filetype_community Dies wird verwendet, um Dateien zu erstellen, die hochgeladen und für jeden in der Community verfügbar gemacht werden.
ugc_filetype_microtrans Dies wird verwendet, um Dateien zu beschreiben, die hochgeladen werden, aber nur für das Spiel gedacht sind, um sie als offiziellen Inhalt hinzuzufügen.



Bei Verwendung dieser Funktion wird ein Async-ID-Wert zurückgegeben, der dann analysiert werden kann, wenn das Steam Asynchronous- Ereignis ausgelöst wird, um die Erstellung des Elements zu melden. Das Ereignis enthält die folgenden Schlüssel / Map-Werte in der async_load ds_map:

  1. " id " - Die von der aufrufenden Funktion zurückgegebene asynchrone ID

  3. " Ergebnis " - Das Ergebnis der Operation (ein realer Wert). Dies ist entweder die GML-Konstante ugc_result_success oder eine andere reelle Zahl. Sie sollten also nach dieser Konstante suchen, um sicherzustellen, dass der Aufruf erfolgreich war, und wenn sonst etwas nicht korrekt funktioniert hat. Der Rest der möglichen Werte wird als Ergebnis des Steam "EResult" -Werts angezeigt und Sie sollten sehen steamclientpublic.h in den SDK-Headern für alle 89 möglichen Werte.

  5. " event_type " - Dieser Schlüssel enthält den Wert " ugc_create_item "

  7. " legal_agreement_required " - Wird sein true oder false (Weitere Informationen finden Sie in den Steam Dokumenten )

  9. " published_file_id " - Dieser Schlüssel enthält die eindeutige veröffentlichte ID für das Element




Erweitertes Beispiel:

In diesem Beispiel rufen wir zuerst die Funktion auf und speichern den asynchronen ID-Wert in einer Variablen:

var app_id = steam_get_app_id();
new_item = steam_ugc_create_item(app_id, ugc_filetype_community);

Dies würde dann eine Anfrage an die Steam API senden, um das neue Worksop-Element zu erstellen und ein asynchrones Ereignis zu generieren, mit dem wir uns wie folgt befassen würden:

var event_id = async_load[? "id"];
if event_id == new_item
   {
   var type = async_load[? "event_type"];
   if type == "ugc_create_item"
      {
      global.Publish_ID = async_load[? "published_file_id"];
      }
   }

Der obige Code prüft den Ereignistyp und wenn er "ugc_create_item" ist, ruft er die veröffentlichte Datei-ID ab und speichert sie in einer globalen Variablen für zukünftige Referenz.
