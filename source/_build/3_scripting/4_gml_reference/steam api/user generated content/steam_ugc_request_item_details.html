steam _ugc_request_item_details

Syntax:

steam_ugc_request_item_details(published_file_id, max_age_seconds);


Streit Beschreibung
published_file_id Die eindeutige Datei-ID für den zu überprüfenden UGC.
max_age_seconds Das "Alter" der zu überprüfenden Daten (empfohlen 30 - 60 Sekunden).


Rückgabe : Async-ID


Beschreibung

Diese Funktion kann verwendet werden, um Informationen zu einer bestimmten Datei-ID abzurufen. Sie geben die Datei-ID an und geben ein Höchstalter für die Überprüfung an (weitere Informationen finden Sie in den Steam Dokumenten). Die Funktion gibt dann einen eindeutigen Async-ID-Wert zurück, mit dem die Details im Steam Async-Ereignis korrekt überprüft werden können. Diese Veranstaltung wird die async_load DS-Karte, die mit den folgenden Schlüssel / Wert-Paaren gefüllt wird:

  1. " id " - Die asynchrone ID, die von der Funktion zurückgegeben wurde, die das Ereignis ausgelöst hat

  3. " Ergebnis " - Das Ergebnis der Operation (ein realer Wert). Dies ist entweder die GML-Konstante ugc_result_success oder eine andere reelle Zahl. Sie sollten also nach dieser Konstante suchen, um sicherzustellen, dass der Aufruf erfolgreich war, und wenn sonst etwas nicht korrekt funktioniert hat. Der Rest der möglichen Werte wird als Ergebnis des Steam "EResult" -Werts angezeigt und Sie sollten sehen steamclientpublic.h in den SDK-Headern für alle 89 möglichen Werte.

  5. " event_type " - Die Zeichenkette " ugc_item_details "

  7. " cached_data " - Wird sein true wenn die zurückgegebenen Details aus dem lokalen Cache stammen oder false wenn sie vom Server genommen werden

  9. " published_file_id" "-Hält die eindeutige veröffentlichte Datei-ID für den Artikel

  11. " file_type " - Der Typ der verwendeten Datei

  13. " creator_app_id " - Die Steam ID des Artikelerstellers

  15. " consumer_app_id " - Die Steam ID des Artikelverbrauchers

  17. " Titel " - Der Titel des Artikels

  19. " Beschreibung " - Die Beschreibung des Artikels

  21. „ steam _id_owner“ - Der Steam ID des Element Eigentümer

  23. " time_created " - Die Zeit, zu der der Artikel zum ersten Mal erstellt wurde

  25. " time_updated " - Das letzte Mal, als der Artikel aktualisiert wurde

  27. " time_added_to_user_list " - Die Zeit, zu der der Artikel abonniert wurde

  29. " Sichtbarkeit " - Die Sichtbarkeit des Artikels (siehe steam_ugc_set_item_visibility() für die zurückgegebenen Konstanten)

  31. " verboten " - Ob der Gegenstand gesperrt wurde ( true ) oder nicht ( false )

  33. " accepted_for_use " - Ob der Artikel zur Verwendung akzeptiert wurde ( true ) oder nicht ( false )

  35. " tags_truncated " - Kurze Version der Tags als Array

  37. " Tags " - Ein Array der Tags für das Element

  39. " handle_file " - Das eindeutige Dateihandle für das Element

  41. " handle_preview_file " - Das eindeutige Handle für die Bildvorschau für das Objekt (kann als Argument mit verwendet werden) steam_ugc_download() um ein Vorschaubild herunterzuladen)

  43. " Dateiname " - Der Name der Artikeldatei

  45. " file_size " - Die Größe der Artikeldatei

  47. " preview_file_size " - Die Größe des Vorschaubildes

  49. " URL " - Die vollständige URL für den Artikel

  51. " votes_up " - Die Anzahl der erhaltenen Up-Stimmen

  53. " votes_down " - Die Anzahl der erhaltenen Down- Votes

  55. " score " - Die Gesamtpunktzahl des Artikels

  57. " account_id_owner " - Die Konto-ID vom Steam ID-Besitzer (dies kann in der Funktion verwendet werden) steam_ugc_create_query_user_ex() )


Erweitertes Beispiel:

In diesem Beispiel senden wir eine Detailanfrage für ein Objekt und analysieren dann das Ergebnis async_load DS-Karte, um einige Variablen zu setzen. Zuerst senden wir die Anfrage:

steam_details = steam_ugc_request_item_details(global.fileID, 60);

Der obige Code fordert Details zu dem Element mit der in der globalen Variablen gespeicherten Datei-ID an und löst ein Steam Async-Ereignis mit den zurückgegebenen Informationen aus. In diesem Fall können wir die Karte analysieren und einige der Werte in Variablen speichern, die dann zur Anzeige der Informationen für den Benutzer verwendet werden können:

var map_id = async_load[? "id"];
var result = async_load[? "result"];
if (map_id == steam_details) && (result == ugc_result_success)
   {
   mTitle = async_load[? "title"];
   mDesc = async_load[? "description"];
   mTags = async_load[? "tags"];
   m_hPreviewFile = async_load[? "handle_preview_file"];
   m_hOwnerSteamId = async_load[? "steam_id_owner"];
   mOwnerAccountId = async_load[? "account_id_owner"];
   mPubFileId = async_load[? "published_file_id"];
   mScore = async_load[? "score"];
   }
