steam _ugc_send_query

Syntax:

steam_ugc_send_query(ugc_query_handle);


Streit Beschreibung
ugc_query_handle Das Abfragehandle zum Senden.


Rückgabe: Boolean


Beschreibung

Diese Funktion kann verwendet werden, um eine Abfrageanforderung zu senden. Sie definieren zuerst die Abfrage mit den entsprechenden steam_ugc_create_query_*() Funktionen, die eine "Abfrage Handle" zurückgeben. Dieses Handle wird dann verwendet, um Filter usw. einzustellen, bevor es in dieser Funktion zum Senden der Anfrage verwendet wird. Die Funktion gibt einen eindeutigen Async-ID-Wert zurück, mit dem die Details im Steam Async-Ereignis korrekt überprüft werden können. Diese Veranstaltung wird die async_load DS-Karte, die mit den folgenden Schlüssel / Wert-Paaren gefüllt wird:

  1. " id " - Die asynchrone ID, die von der Funktion zurückgegeben wurde, die das Ereignis ausgelöst hat

  3. " Ergebnis " - Das Ergebnis der Operation (ein realer Wert). Dies ist entweder die GML-Konstante ugc_result_success oder eine andere reelle Zahl. Sie sollten also nach dieser Konstante suchen, um sicherzustellen, dass der Aufruf erfolgreich war, und wenn sonst etwas nicht korrekt funktioniert hat. Der Rest der möglichen Werte wird als Ergebnis des Steam "EResult" -Werts angezeigt und Sie sollten sehen steamclientpublic.h in den SDK-Headern für alle 89 möglichen Werte.

  5. " event_type " - Die Zeichenfolge " ugc_query "

  7. " num_results " - Die Anzahl der zurückgegebenen Ergebnisse (max. 50)

  9. " total_matching " - Die Gesamtzahl übereinstimmender Ergebnisse

  11. " cached_data " - Gibt an, ob diese Daten vom lokalen Cache auf der Festplatte abgerufen wurden oder nicht true oder false )

  13. " results_list " - Ein DS- Listenindexindex, wobei jeder Listeneintrag ein DS-Map-Index ist, der Details des bestimmten Elements enthält

Wenn Sie den asynchronen Ereignistyp "ugc_query" erhalten, können Sie die Ergebnisliste analysieren und die folgenden Informationen aus jeder der DS-Karten extrahieren (eine Karte pro Element in der Liste):

  1. " published_file_id" "-Hält die eindeutige veröffentlichte Datei-ID für den Artikel

  3. " file_type " - Der Typ der verwendeten Datei

  5. " creator_app_id " - Die Steam ID des Artikelerstellers

  7. " consumer_app_id " - Die Steam ID des Artikelverbrauchers

  9. " Titel " - Der Titel des Artikels

  11. " Beschreibung " - Die Beschreibung des Artikels

  13. „ steam _id_owner“ - Der Steam ID des Element Eigentümer

  15. " time_created " - Die Zeit, zu der der Artikel zum ersten Mal erstellt wurde

  17. " time_updated " - Das letzte Mal, als der Artikel aktualisiert wurde

  19. " time_added_to_user_list " - Die Zeit, zu der der Artikel abonniert wurde

  21. " Sichtbarkeit " - Die Sichtbarkeit des Artikels (siehe steam_ugc_set_item_visibility() für die zurückgegebenen Konstanten)

  23. " verboten " - Ob der Gegenstand gesperrt wurde ( true ) oder nicht ( false )

  25. " accepted_for_use " - Ob der Artikel zur Verwendung akzeptiert wurde ( true ) oder nicht ( false )

  27. " tags_truncated " - Kurze Version der Tags als Array

  29. " Tags " - Eine Zeichenfolge, die die Tags für das Element enthält, jeweils getrennt durch ein Komma

  31. " handle_file " - Das eindeutige Dateihandle für das Element

  33. " handle_preview_file " - Das eindeutige Handle für die Bildvorschau für das Objekt (kann als Argument mit verwendet werden) steam_ugc_download() um ein Vorschaubild herunterzuladen)

  35. " Dateiname " - Der Name der Artikeldatei

  37. " file_size " - Die Größe der Artikeldatei

  39. " preview_file_size " - Die Größe des Vorschaubildes

  41. " URL " - Die vollständige URL für den Artikel

  43. " votes_up " - Die Anzahl der erhaltenen Up-Stimmen

  45. " votes_down " - Die Anzahl der erhaltenen Down- Votes

  47. " score " - Die Gesamtpunktzahl des Artikels

  49. " account_id_owner " - Die Konto-ID vom Steam ID-Besitzer (dies kann in der Funktion verwendet werden) steam_ugc_create_query_user_ex() )


Beispiel:

var query_handle = steam_ugc_create_query_all(ugc_query_RankedByTrend, ugc_match_Items, 1);
steam_ugc_query_add_required_tag(query_handle, "RPG");
steam_ugc_query_set_return_long_description(query_handle, true);
steam_ugc_query_set_allow_cached_response(query_handle, true);
query_ID = steam_ugc_send_query(query_handle);

Der obige Code erstellt eine Abfrageanforderung und speichert sie in einer lokalen Variablen zur zukünftigen Verwendung in den übrigen Funktionen, die die Abfrageanforderung vor dem Senden der Abfrage weiter definieren.
