steam _upload_score

Syntax:

steam_upload_score(lb_name, score);


Streit Beschreibung
lb_name Der Name der Bestenliste, in die Sie die Bewertungen hochladen (eine Zeichenfolge).
score Die Punktzahl zum Hochladen (eine echte Zahl).


Rückgabe: Echt


Beschreibung

Diese Funktion sendet eine Punktzahl an die angegebene Bestenliste. Die Punktzahl, die hochgeladen werden soll, ist eine echte Zahl, und der Name der Bestenliste ist eine Zeichenfolge, die beim Erstellen der Bestenliste mithilfe der Funktion definiert wurde steam_create_leaderboard. Wenn die Funktion fehlschlägt, wird ein Wert von -1 zurückgegeben, während ein erfolgreicher Post eine eindeutige ID für den Upload zurückgibt und ein asynchrones Steam Ereignis auslöst. Dieses Ereignis erstellt ein ds_map in der eingebauten Variable async_load mit den folgenden Schlüssel / Wert-Paaren:

  • " event_type " - Dieser Schlüssel enthält den Wert " leaderboard_upload ", wenn er von dieser Funktion ausgelöst wird

  • " lb_name " - Dieser Schlüssel enthält den Namen der Bestenliste, auf die gepostet wurde

  • " Erfolg " - Wird 1, wenn der Beitrag erfolgreich war, 0 fehlgeschlagen

  • " aktualisiert " - Wird 1, wenn der Ranglistenpunkt tatsächlich aktualisiert wurde (dh der neue Punktestand war besser) oder 0, andernfalls

  • " post_id " - Dieser Schlüssel sollte mit dem vom Upload-Aufruf zurückgegebenen ID-Wert übereinstimmen

  • " score " - Dieser Schlüssel enthält den Punktestand, der gepostet wurde


Erweitertes Beispiel:

In diesem Beispiel laden wir zunächst eine Partitur hoch und analysieren dann die Partitur async_load Karte zurückgegeben, wenn erfolgreich. Der folgende Code zeigt ein typisches Beispiel für das Hochladen:

if (hp <= 0)
   {
   upload_ID = steam_upload_score("Game Scores", score);
   if (!upload_ID)
      {
      alarm[0] = room_speed;
      }
   }

Beachten Sie, dass wir einen Alarm gesetzt haben, wenn der Anruf fehlschlägt. Dies würde verwendet werden, um den Upload zu einem späteren Zeitpunkt erneut zu versuchen, und Sie können dort zusätzlichen Code hinzufügen, um den Upload erneut zu versuchen oder den Score in eine Textdatei zu speichern, falls es weiterhin fehlschlägt, etc... Asynchrones Steam Ereignis für die Instanz, die die Scores steuert:

var type = ds_map_find_value(async_load, "event_type");
if (type == "leaderboard_upload")
   {
   var lb_ID = ds_map_find_value(async_load, "post_id");
   if lb_ID == upload_ID
      {
      var lb_name = ds_map_find_value(async_load, "lb_name");
      var lb_done = ds_map_find_value(async_load, "success");
      var lb_score = ds_map_find_value(async_load, "score");
      var lb_updated = ds_map_find_value(async_load, "updated");
      show_debug_message("leaderboard post id:" + string(lb_ID) + " to lb:" + string(lb_name) + " with score:" + string(lb_score) + " updated=" + string(lb_updated));
      if (lb_done)
         {
         show_debug_message("- Succeeded");
         }
      else
         {
         show_debug_message("- Failed");
         }
      }
   }

Im Beispiel geben wir die Rückgabewerte einfach als Debug-Meldungen an das Compiler-Fenster aus, aber Sie können dieses Ereignis verwenden, um mit den Informationen in beliebiger Weise umzugehen.
