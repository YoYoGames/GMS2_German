object_is_ancestor

Gibt zurück, ob ein Objekt das übergeordnete Objekt eines anderen Objekts ist.

Syntax:

object_is_ancestor(obj, par);


Streit Beschreibung
obj Das Objekt, das als Kind überprüft wird.
par Das Objekt, das als Vorgänger (Eltern) überprüft wird.


Rückgabe: Boolean


Beschreibung

Diese Funktion kann verwendet werden, um ein Objekt zu prüfen und festzustellen, ob es ein Vorgänger eines anderen Objekts ist oder nicht. Also müssen beide Argumente Objekt-Indizes sein und nicht die Instanz-IDs, und die erste ist immer das Objekt, das Sie herausfinden wollen, wenn es ein Kind des zweiten Arguments ist, das immer das Objekt ist, das als das überprüfen will Vorfahre (Eltern).


Beispiel:

if object_is_ancestor(object_index, obj_Enemy)
   {
   instance_destroy();
   }

Der obige Code überprüft, ob die Instanz, die den Code ausführt, ein Kind des Objekts "obj_Enemy" ist, und wenn dies der Fall ist, wird es zerstört.
