Objekte

In GameMaker Studio 2 Sie Objekte und Sie haben Instanzen. Objekte sind im Grunde die Basisvorlage für eine Instanz und sind daher niemals direkt in einem Raum vorhanden. Es werden nur Instanzen des Objekts im Raum platziert. Wenn Sie also für alle Instanzen, die Sie erstellen möchten, etwas ändern möchten, können Sie dies tun, indem Sie das Objekt ändern.

WICHTIG! Wenn Sie etwas an einem Objekt ändern, werden keine Instanzen im Raum geändert, sondern nur diejenigen, die nach der Änderung erstellt wurden

Sie können auch auf Informationen zu einem Basisobjekt zugreifen und diese verwenden, um Verhalten oder Änderungen im Spiel zu diktieren. Sie können zum Beispiel nach einer übergeordneten Objekt-ID suchen, und im folgenden Code, wenn eine Instanz mit diesem übergeordneten Objekt gefunden wurde, können Sie das überprüfen object_index zu entscheiden, was getan werden soll.

GameMaker Studio 2 bietet Ihnen eine Reihe von Funktionen, mit denen Sie verschiedene Details zu einem Objekt erhalten können. Beachten Sie, dass ein Objekt keine Instanz (eine Ingame-Entity) ist, sondern eine Ressource, aus der alle Instanzen erstellt werden. Dies bedeutet, dass die Rückgabewerte dieser Funktionen sich von den tatsächlichen Werten unterscheiden können, die von Instanzen im Raum angezeigt werden. Sobald eine Instanz erstellt wurde, kann sie durch Code- und DnD™ -Aktionen geändert werden.

  • object_exists
  • Objekt_index
  • object_get_mask
  • object_get_name
  • object_get_parent
  • object_get_persistent
  • object_get_solid
  • object_get_sprite
  • object_get_visible
  • object_get_physics
  • object_is_ancestor

Es gibt auch eine Reihe von Funktionen, mit denen Sie die Eigenschaften für ein Objekt festlegen können. Es sollte beachtet werden, dass alle Instanzen dieses Objekts, die bereits im Raum vorhanden sind, nicht von diesen Funktionen betroffen sein können, aber alle neuen Instanzen dieses Objekts im Raum erstellt werden. Daher wird empfohlen, dass Sie niemals eine Objekteigenschaft ändern Instanzen dieses Objekts sind im aktuellen Raum vorhanden.

  • object_set_mask
  • object_set_persistent
  • object_set_solid
  • object_set_sprite
  • object_set_visible
