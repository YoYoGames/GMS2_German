Ereignisse generieren

GameMaker Studio 2 ist vollständig ereignisgesteuert und alle Aktionen werden als Ergebnis bestimmter Ereignisse ausgeführt, die in einer bestimmten Reihenfolge oder durch ein bestimmtes Ereignis innerhalb des Spiels ausgelöst werden. Es gibt eine Reihe von verschiedenen Ereignissen, jedes mit einer bestimmten Aufgabe oder einem bestimmten Timing und einige von ihnen sind sogar in separate "Sub" -Ereignisse aufgeteilt. Normalerweise werden diese Ereignisse von GameMaker Studio 2 wenn ein Trigger im Spiel oder jedes Spiel-Tick erkannt wird (wie bei den Step- und Draw-Events), aber Sie können auch ein Event auf eine Instanz im aktuellen Raum anwenden in einem Stück Code.

Weitere Informationen zu Veranstaltungen finden Sie im Abschnitt Veranstaltungen.

Es gibt folgende Funktionen, die sich mit dem Generieren von Ereignissen befassen:

  • event_inherited
  • Ereignis_perform
  • event_perform_object
  • Ereignisbenutzer

Es gibt auch eine Anzahl von Nur-Lese-Variablen, die mit den oben aufgeführten Funktionen verwendet werden können, die für das Debuggen besonders nützlich sind:

  • Ereignis_Aktion
  • Ereignisnummer
  • Ereignis_Objekt
  • Ereignistyp
