event_inherited

Führt das geerbte Ereignis aus.

Syntax:

event_inherited();


Rückgabe: N / A.


Beschreibung

Diese Funktion ruft das aktuelle Ereignis des übergeordneten Objekts der Instanz auf. Wenn eine Instanz über ein übergeordnetes Objekt verfügt, erbt sie normalerweise automatisch dieselben Ereignisse wie das übergeordnete Objekt. Wenn jedoch Ihr übergeordnetes Objekt ein Erstellungsereignis hat und Sie Ihrem untergeordneten Objekt ein Ereignis hinzufügen, werden alle Instanzen des untergeordneten Objekts übernommen Führen Sie das neue Erstellungsereignis aus, das Sie hinzugefügt haben, und nicht das, was sich im übergeordneten Objekt befindet. Wenn Sie sowohl das übergeordnete Objektereignis als auch das untergeordnete Objektereignis desselben Typs verwenden müssen, sollten Sie diese Funktion verwenden, da das übergeordnete Objektereignis ausgeführt wird, bevor Sie mit dem restlichen Code oder den Aktionen fortfahren, die das untergeordnete Ereignis enthält.


Beispiel:

event_inherited();
switch (sprite_index)
   {
   case spr_Enemy_1: dmg += 2; break;
   case spr_Enemy_4: dmg -= 1; break;
   case spr_Enemy_10: dmg +=10; break;
   }

Der obige Code ruft das geerbte übergeordnete Ereignis (in dem wir die Variable "dmg" und den Sprite-Index festlegen) auf und ändert anschließend die Variable "dmg". Wenn für die Instanz, die diesen Code ausführt, kein übergeordnetes Element angegeben ist, wird der Fehler "unbekannte Variable" angezeigt, da dmg nicht definiert wurde.
