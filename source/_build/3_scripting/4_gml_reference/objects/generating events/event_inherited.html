event_inherited

Führt das geerbte Ereignis aus.

Syntax:

event_inherited();


Rückkehr: N / A.


Beschreibung

Diese Funktion ruft das aktuelle Ereignis des übergeordneten Objekts der Instanz auf. Wenn eine Instanz ein übergeordnetes Objekt besitzt, erbt sie normalerweise automatisch alle gleichen Ereignisse wie das übergeordnete Objekt. Wenn jedoch beispielsweise Ihr übergeordnetes Objekt ein create-Ereignis hat und Sie ein untergeordnetes Objekt hinzufügen, werden alle Instanzen des untergeordneten Objekts ausgeführt Führen Sie das neue create-Ereignis aus, das Sie hinzugefügt haben, und nicht das, was sich im übergeordneten Objekt befindet. Wenn Sie sowohl das übergeordnete Objektereignis als auch das untergeordnete Objektereignis desselben Typs verwenden müssen, sollten Sie diese Funktion verwenden, da das übergeordnete Objektereignis ausgeführt wird, bevor Sie mit dem restlichen Code oder den Aktionen des untergeordneten Ereignisses fortfahren


Beispiel:

event_inherited();
switch (sprite_index)
   {
   case spr_Enemy_1: dmg += 2; break;
   case spr_Enemy_4: dmg -= 1; break;
   case spr_Enemy_10: dmg +=10; break;
   }

Der obige Code ruft das geerbte Elternereignis auf (in dem wir die Variable "dmg" und den Sprite-Index setzen) und ändert dann die Variable "dmg". Wenn für die Instanz, die diesen Code ausführt, kein Parent angegeben ist, erhalten wir den Fehler "unbekannte Variable", da dmg nicht definiert wurde.
