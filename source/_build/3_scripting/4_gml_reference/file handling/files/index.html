Textdateien

Textdateien sind eine nützliche Methode, um große Datenmengen extern von Ihrem Spiel zu speichern. Im Gegensatz zu Ini-Dateien können Sie bis zu 32 Textdateien gleichzeitig öffnen, und Sie können auch große Datenblöcke darauf lesen und schreiben. Es liegt jedoch an Ihnen zu entscheiden, wie diese Daten strukturiert werden sollen, da es keine festgelegte "section" - oder "key" -Struktur gibt *.ini Dateien.

Bitte beachten Sie, dass es bei Spielen mit Lokalisierung sehr wichtig ist, dass Buchstaben mit Akzent aus externen Dateien gelesen werden können. Das bedeutet, dass Sie die txt - Datei im UTF8 - Format erstellen sollte zuerst und dann in hinzufügen GameMaker Studio 2 als EPS - Datei enthalten, so dass sie das Spiel auf Laufen und verwendet anstelle des Standard - ANSI - Format TXT - Datei exportiert wird, die durch die erstellt wird, GameMaker Studio 2 Datei funktioniert, wenn keine Datei zuvor geliefert wurde. Auf diese Weise können Sie korrekt lesen und schreiben, wobei alle Akzente und nicht-lateinischen Buchstaben beibehalten werden.

Die folgenden Funktionen existieren, die mit Dateien umgehen:

  • file_text_open_read
  • file_text_open_write
  • file_text_open_append
  • file_text_open_from_string
  • file_text_read_real
  • file_text_read_string
  • file_text_readln
  • file_text_write_real
  • file_text_write_string
  • file_text_writeln
  • file_text_eoln
  • file_text_eof
  • file_text_close
WARNUNG! Diese Funktion funktioniert möglicherweise nicht wie erwartet, weil GameMaker Studio 2 Sandboxing ist! Weitere Informationen finden Sie im Abschnitt zu Dateisystemgrenzen.
