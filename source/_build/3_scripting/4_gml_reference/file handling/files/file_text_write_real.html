file_text_write_real

Syntax:

file_text_write_real(fileid, val);


Streit Beschreibung
fileid Die ID der zu bearbeitenden Datei.
val Der tatsächliche Wert, der in die Datei geschrieben werden soll.


Rückgabe: Real


Beschreibung

Mit dieser Funktion können Sie eine Nummer in die zuvor geöffnete Textdatei schreiben. Da der zu schreibende Wert eine reelle Zahl sein kann, werden alle Dezimalzahlen mit einem "." Punkt als Trennzeichen. Wenn die Datei bereits Informationen enthält, werden diese Informationen gelöscht und die Zeichenfolge wird am Anfang der Datei geschrieben, es sei denn, Sie haben die Datei mit der Datei geöffnet file_text_open_append. Sie können dies auch vermeiden, indem Sie die file_text_readln zusammen mit der file_text_eof Funktion, um den Inhalt der Datei zu durchlaufen, bis Sie zum Ende kommen und dann mit dem Schreiben beginnen.

Beachten Sie, dass beim Schreiben sehr großer Zahlen in eine Textdatei mit dieser Funktion ein vereinfachtes Standardformat wie "6.6624e + 003" übersetzt werden kann, das nicht ordnungsgemäß in GameMaker Studio 2 eingelesen werden kann. Um solche Probleme zu vermeiden, sollten Sie stattdessen den Wert in eine Zeichenfolge konvertieren und die Funktion verwenden file_text_write_string() stattdessen.


Beispiel:

var i, file;
file = file_text_open_write(working_directory + "hiscore.txt");
for (i = 0; i < 10; i += 1)
   {
   file_text_write_real(file, scr[i]);
   file_text_writeln(file);
   file_text_write_string(file, scr_name[i]);
   file_text_writeln(file);
   }
file_text_close(file);

Der obige Code öffnet eine Datei zum Schreiben und durchläuft dann zwei Arrays, wobei jeder Arraywert in eine neue Zeile der Datei geschrieben wird. Die Datei wird dann geschlossen, wenn die Schleife beendet ist.
