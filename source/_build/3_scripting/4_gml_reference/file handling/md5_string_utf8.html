md5_string_utf8

Syntax:

md5_string_utf8(string)


Streit Beschreibung
string Die zu haschende Zeichenfolge.


Rückgabe: Zeichenfolge


Beschreibung

In der Kryptographie ist MD5 (Message-Digest-Algorithmus 5) eine weit verbreitete kryptographische Hash-Funktion mit einem 128-Bit-Hash-Wert und wurde in einer Vielzahl von Sicherheitsanwendungen eingesetzt. Es wird auch häufig verwendet, um die Integrität von Dateien und Zeichenfolgen zu überprüfen. Diese Funktion nimmt eine Eingabe-UTF-8-Zeichenfolge (die eine variable Anzahl von Bytes pro Zeichen hat) und gibt den 32-stelligen hexadezimalen MD5-Hash zurück, der für diese Zeichenfolge eindeutig ist. Auf diese Weise können Sie einen sicheren Schlüssel generieren, der gespeichert und verwendet werden kann, um die Integrität der Informationen zu prüfen, die zum Beispiel an einen externen Server gesendet (oder von diesem empfangen) werden.

HINWEIS: Es gibt zwei Formate für die MD5-Codierung, UTF-8 und Unicode. Beide werden bereitgestellt, um die Kommunikation mit verschiedenen Server-Setups zu erleichtern, aber am häufigsten wird Unicode verwendet.


Beispiel:

var hash, str;
str = json_encode(hiscore_map);
hash = md5_string_utf8(str);
ini_open("local.ini");
ini_write_string("info", "0", hash);
ini_close();
get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str)

Der obige Code wird eine ds_map in eine JSON-Zeichenfolge codieren. Ein MD5-Hash wird dann erzeugt und in einer Ini-Datei gespeichert, so dass diese später verwendet werden kann, um die Integrität des JSON zu überprüfen, sollte die gleiche Information später vom Server empfangen werden. Der JSON wird dann gesendet.
