md5_string_utf8


Beschreibung

In der Kryptographie ist MD5 (Message-Digest-Algorithmus 5) eine weit verbreitete kryptographische Hashfunktion mit einem 128-Bit-Hashwert und wurde in einer Vielzahl von Sicherheitsanwendungen eingesetzt. Es wird auch häufig zur Überprüfung der Integrität von Dateien und Strings verwendet. Diese Funktion nimmt einen eingegebenen UTF-8-String (mit einer variablen Anzahl von Bytes pro Zeichen) und gibt den 32-stelligen hexadezimalen MD5-Hash zurück, der für diesen String eindeutig ist. Auf diese Weise können Sie einen sicheren Schlüssel generieren, der gespeichert und zur Überprüfung der Integrität der Informationen verwendet werden kann, die an einen externen Server gesendet (oder von diesem empfangen) werden (z. B.).

HINWEIS: Es gibt zwei Formate für die MD5-Kodierung, UTF-8 und Unicode. Beide werden bereitgestellt, um die Kommunikation mit verschiedenen Server-Setups zu erleichtern, am häufigsten wird jedoch Unicode verwendet.


Syntax:

md5_string_utf8(string)


Streit Beschreibung
string Die Zeichenfolge zum Hashing


Kehrt zurück:

String


Beispiel:

var hash, str;
str = json_encode(hiscore_map);
hash = md5_string_utf8(str);
ini_open("local.ini");
ini_write_string("info", "0", hash);
ini_close();
get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str)

Der obige Code kodiert eine ds_map in eine JSON-Zeichenfolge. Ein MD5-Hash wird dann generiert und in einer Ini-Datei gespeichert, sodass später die Integrität des JSON überprüft werden kann, falls die gleichen Informationen später vom Server empfangen werden. Die JSON wird dann gesendet.
