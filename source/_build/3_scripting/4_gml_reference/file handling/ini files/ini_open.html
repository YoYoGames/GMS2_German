ini_open


Beschreibung

Dies öffnet eine ini_file zum Lesen und / und Schreiben. Wenn die ini_file an dem von Ihnen überprüften Ort nicht vorhanden ist, kann GameMaker Studio 2 eine erstellen, jedoch nur, wenn Sie Daten in die Datei schreiben. Wenn Sie nur Informationen aus der Ini-Datei gelesen haben, werden die Standardwerte für die Lesefunktion zurückgegeben, die Ini-Datei wird jedoch nicht erstellt.

Bitte beachten Sie, dass Sie immer nur eine ini-Datei geöffnet haben können, und denken Sie daran, sie zu verwenden ini_close Sobald Sie mit dem Lesen / Schreiben aus der INI-Datei fertig sind, werden die Informationen erst dann tatsächlich auf der Festplatte gespeichert (sie werden auch im Speicher abgelegt, bis die Datei geschlossen wird).

WARNUNG! Diese Funktion funktioniert möglicherweise nicht wie erwartet, da GameMaker Studio 2 einer Sandbox ausgeführt wird! Weitere Informationen finden Sie im Abschnitt Dateisystemlimits.


Syntax:

ini_open(name);


Streit Beschreibung
name Der Dateiname für die INI-Datei.


Kehrt zurück:

N/A


Beispiel:

ini_open("Settings/savedata.ini");
score = ini_read_real("save1", "score", 0);
ini_close();

Dadurch wird 'savedata.ini' geöffnet und der Score-Wert unter dem Abschnitt "save1" mit dem Schlüssel "score" darin gelesen. Dann wird die.ini wieder geschlossen. Wenn unter "save1", "score" kein Wert vorhanden ist oder keine Datei "Savedata.ini" vorhanden ist, wird der Score auf 0 gesetzt (Standardwert). Beachten Sie, dass die INI-Datei im Unterverzeichnis "Settings" abgelegt wurde. Hierbei handelt es sich um den Ordner, der die INI-Datei in den Ressourcenbaumdateien enthält.
