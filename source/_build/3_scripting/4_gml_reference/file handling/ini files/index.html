Ini-Dateien

INI-Dateien sind kleine, leichte Dateien, die mit den meisten Plattformen kompatibel sind. Sie sind ideal für die Speicherung von kleinen Informationen wie Schnittstellenpräferenzen, lokalen Highscores, Level-Daten usw. und sind somit ein großartiger und einfacher Weg, um Informationen aus Ihren Spielen zu speichern. Es ist jedoch erwähnenswert, dass sie eine Begrenzung für die Daten haben, die sie speichern können, was maximal 64 KB ist. Wenn Sie beispielsweise Datenstrukturen speichern, können Sie sich einige der anderen Dateifunktionen wie Text ansehen Dateien.

Bitte beachten Sie, dass es bei Spielen mit Lokalisierung sehr wichtig ist, dass Buchstaben mit Akzent aus externen Dateien gelesen werden können. Dies bedeutet, dass Sie die INI - Datei im UTF8 - Format erstellen sollte zuerst und dann in hinzufügen GameMaker Studio 2 als EPS - Datei enthalten, so dass sie das Spiel auf Laufen und verwendet anstelle des Standard - ANSI - Format INI - Datei exportiert wird, die durch die ini erstellt Funktionen, wenn keine zuvor geliefert wurde. Auf diese Weise können Sie korrekt lesen und schreiben, wobei alle Akzente und nicht-lateinischen Buchstaben beibehalten werden.

Folgende Funktionen existieren für Ini-Dateien:

  • ini_open
  • ini_close
  • ini_write_real
  • ini_write_string
  • ini_read_real
  • ini_read_string
  • ini_key_exists
  • ini_section_exists
  • ini_key_delete
  • ini_section_delete
  • ini_open_from_string
WARNUNG! Diese Funktionen funktionieren möglicherweise nicht wie erwartet, weil GameMaker Studio 2 Sandboxed ist! Weitere Informationen finden Sie im Abschnitt zu Dateisystemgrenzen.
