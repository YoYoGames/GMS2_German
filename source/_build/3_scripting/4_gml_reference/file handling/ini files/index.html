Ini-Dateien


INI-Dateien sind kleine, leichte Dateien, die mit den meisten Plattformen kompatibel sind. Sie eignen sich ideal zum Speichern kleiner Informationen, wie Benutzeroberflächenpräferenzen, lokale Highscores, Pegeldaten usw., und sind somit eine großartige und einfache Möglichkeit, Informationen aus Ihren Spielen zu speichern. Es ist jedoch erwähnenswert, dass sie über eine begrenzte Anzahl von Daten verfügen, die gespeichert werden können, und zwar maximal 64 KB. Wenn Sie beispielsweise Datenstrukturen speichern, möchten Sie möglicherweise einige andere Dateifunktionen wie Text betrachten Dateien

Bitte beachten Sie, dass es für Spiele mit Lokalisierungen sehr wichtig ist, dass Akzentbuchstaben aus externen Dateien gelesen werden können. Dies bedeutet, dass Sie die INI - Datei im UTF8 - Format erstellen sollte zuerst und dann in hinzufügen GameMaker Studio 2 als EPS - Datei enthalten, so dass sie das Spiel auf Laufen und verwendet anstelle des Standard - ANSI - Format INI - Datei exportiert wird, die durch die ini erstellt funktioniert, wenn zuvor keine geliefert wurde. Auf diese Weise können Sie es korrekt lesen und beschreiben, wobei alle Akzente und nicht römischen Buchstaben beibehalten werden.

Für ini-Dateien gibt es folgende Funktionen:

  • ini_open
  • ini_close
  • ini_write_real
  • ini_write_string
  • ini_read_real
  • ini_read_string
  • ini_key_exists
  • ini_section_exists
  • ini_key_delete
  • ini_section_delete
  • ini_open_from_string
WARNUNG! Diese Funktionen funktionieren möglicherweise nicht wie erwartet, da GameMaker Studio 2 eine Sandbox hat! Weitere Informationen finden Sie im Abschnitt Dateisystemlimits.
