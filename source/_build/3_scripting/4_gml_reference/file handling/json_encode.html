json_encode

Syntax:

json_encode(map)


Streit Beschreibung
map eine ds_map mit der zu kodierenden Information


Rückgabe: Zeichenfolge


Beschreibung

JSON (JavaScript Object Notation) ist ein leichtgewichtiges Datenaustauschformat, das sowohl für Personen als auch für Maschinen einfach zu lesen und zu schreiben ist. Es basiert auf zwei Grundstrukturen:

  • Eine Sammlung von Name / Wert-Paaren, genannt a ds_map in GameMaker Studio 2 aber auch als "Wörterbuch" oder "Objekt" bezeichnet.
  • Eine geordnete Liste von Werten, genannt a ds_list in GameMaker Studio 2 aber dies kann auch als "Array" oder "Sequenz" bezeichnet werden.

json_encode nimmt eine ds_map, die Sie zuvor erstellt und als JSON-String codiert haben, die Sie dann als (zB) Teil von einem verwenden können http_post_string() Anruf, oder so kann es extern gespeichert werden, in eine Datei geschrieben.

HINWEIS: Die hierarchische Funktionalität von JSON ist durch spezielle ds_map- und ds_list-Funktionen verfügbar, sodass Sie Sub-Listen und Maps codieren können.


Beispiel:

var hiscore_map, i, str;
hiscore_map = ds_map_create();
for (i = 0; i < 10; i ++;)
   {
   ds_map_add(name[i], score[i]);
   }
str = json_encode(hiscore_map);
get[0] = http_post_string("http://www.angusgames.com/game?game_id=" + string(global.game_id), str)
ds_map_destroy(hiscore_map);

Der obige Code erstellt eine ds_map und durchläuft dann die Arrays name und score, wobei jedes Schlüssel / Wert-Paar zu der neuen Karte hinzugefügt wird. Als nächstes wird diese Karte unter Verwendung von codiert json_encode und als String in der Variablen "str" ​​gespeichert. Diese Zeichenfolge wird dann mit einem Webserver gesendet http_post_string und die ds_map wird zerstört, um ein Speicherleck zu verhindern, da es nicht mehr benötigt wird.
