json_encode


Beschreibung

JSON (JavaScript Object Notation) ist ein leichtes Format für den Datenaustausch, das sowohl für Personen als auch für Maschinen einfach zu lesen und zu schreiben ist. Es ist auf zwei Grundstrukturen aufgebaut:

  • Eine Sammlung von Name / Wert-Paaren, genannt a ds_map in GameMaker Studio 2 aber auch "Wörterbuch" oder "Objekt" genannt.

  • Eine geordnete Liste von Werten, genannt a ds_list In GameMaker Studio 2 kann dies jedoch auch als "Array" oder "Sequenz" bezeichnet werden.

json_encode nimmt eine ds_map, die Sie zuvor erstellt und als JSON-String codiert haben, die Sie dann als (beispielsweise) Teil eines verwenden können http_post_string() Aufruf, oder so kann es extern gespeichert, in eine Datei geschrieben werden.

HINWEIS: Die hierarchische Funktionalität von JSON ist über die speziellen Funktionen ds_map und ds_list verfügbar, sodass Sie Unterlisten und Maps kodieren können.


Syntax:

json_encode(map)


Streit Beschreibung
map eine ds_map mit den zu kodierenden Informationen


Kehrt zurück:

string


Beispiel:

var hiscore_map, i, str;
hiscore_map = ds_map_create();
for (i = 0; i < 10; i ++;)
   {
   ds_map_add(name[i], score[i]);
   }
str = json_encode(hiscore_map);
get[0] = http_post_string("http://www.angusgames.com/game?game_id=" + string(global.game_id), str)
ds_map_destroy(hiscore_map);

Der obige Code erstellt eine ds_map und durchläuft dann die Arrays Name und Score, wobei jedes Schlüssel / Wert-Paar zur neuen Map hinzugefügt wird. Als Nächstes wird diese Karte mit codiert json_encode und als String in der Variablen "str" ​​gespeichert. Diese Zeichenfolge wird dann mit an einen Webserver gesendet http_post_string und ds_map wird zerstört, um einen Speicherverlust zu verhindern, da er nicht mehr benötigt wird.
