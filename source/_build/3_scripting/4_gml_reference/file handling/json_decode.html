json_decode

Syntax:

json_decode(string)


Streit Beschreibung
string Die Zeichenfolge im JSON-Format, die Sie an die Decodierfunktion übergeben


Kehrt zurück:

ds_map id or -1 if it fails


Beschreibung

JSON (JavaScript Object Notation) ist ein leichtgewichtiges Datenaustauschformat, das sowohl für Personen als auch für Maschinen einfach zu lesen und zu schreiben ist. Es basiert auf zwei Grundstrukturen:

  • Eine Sammlung von Name / Wert-Paaren, genannt a ds_map in GameMaker Studio 2 aber auch als "Wörterbuch" oder "Objekt" bezeichnet.
  • Eine geordnete Liste von Werten, genannt a ds_list in GameMaker Studio 2 aber dies kann auch als "Array" oder "Sequenz" bezeichnet werden.

Mit dieser Funktion können Sie ein Stück JSON dekodieren und in ein ds_map, bereit für den Einsatz in GameMaker Studio 2. Wenn der zu dekodierende JSON eine Hierarchie von Listen und Maps in der zentralen ds_map benötigt, werden diese ebenfalls decodiert und auch für Sie erstellt, wobei die folgenden Regeln verwendet werden (beachten Sie, dass diese Regeln nur für die Struktur auf oberster Ebene gelten):

  • Json ist ein einzelner Wert - gibt eine ds_map mit einem einzelnen Eintrag "default" zurück, bei dem es sich um den Wert handelt
  • Json ist ein Array von Objekten oder Werten - gibt eine ds_map mit einem einzelnen Eintrag "default" zurück, der eine ds_list der Objekte oder Werte ist
  • Json ist ein Objekt - gibt eine ds_map zurück, in der die Objekteinträge enthalten sind
HINWEIS: Beim Dekodieren von Arrays gibt es NUR eine Karte mit dem Schlüssel "default", wenn ein Array die Struktur der obersten Ebene ist, und NUR für dieses Array der obersten Ebene. Interne Listen dekodieren direkt in ds_lists, ohne in einer ds_map eingeschlossen zu sein.

Normalerweise würdest du wissen, mit welchen Schlüsseln der JSON dekodiert, aber wenn nicht, kannst du den ds_map_size, ds_map_find_first und ds_map_find_next Funktionen, um die Karte zu analysieren und die notwendigen Informationen zu erhalten.



HINWEIS: GameMaker Studio 2 erstellt die erforderlichen ds_maps und Listen aus dem JSON, und zum Aufräumen müssen Sie nur die oberste Ebene oder Liste löschen und GameMaker Studio 2 wird automatisch alle darunter liegenden Maps und Listen aus dem Speicher löschen.


Beispiel:

var resultMap = json_decode(requestResult);
var list = ds_map_find_value(resultMap, "default");
var size = ds_list_size(list);
for (var n = 0; n < ds_list_size(list); n++;)
   {
   var map = ds_list_find_value(list, n);
   var curr = ds_map_find_first(map);
   while (is_string(curr))
      {
      global.Name[n] = ds_map_find_value(map, "name");
      curr = ds_map_find_next(map, curr);
      }
   }
ds_map_destroy(resultMap);

Der obige Code wird eine JSON-Zeichenfolge dekodieren und analysieren, um ein globales Array zu generieren.
