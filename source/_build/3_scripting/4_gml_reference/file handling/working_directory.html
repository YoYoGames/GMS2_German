Arbeitsverzeichnis

Syntax:

working_directory


Rückgabe: Zeichenfolge.


Beschreibung

Working_directory kann tatsächlich zwei verschiedene Werte zurückgeben, je nachdem, wofür Sie es verwenden. Wenn Sie eine Datei auf die Festplatte schreiben, working_directory verweist auf den Bereich des lokalen Speichers, der für dein Spiel auf dem Zielgerät reserviert ist (in Windows ist dies "% LOCALAPPDATA% / gamedir /", wobei "gamedir" das Verzeichnis mit dem Namen deines Spiels ist). Beim Lesen von der Festplatte working_directory Dies kann entweder der lokale Speicher oder der Bereich sein, in dem die enthaltenen Dateien gespeichert sind.

HINWEIS: die working_directory Variable wiil gibt den Pfad mit dem letzten Backslash zurück.

Zum Beispiel, wenn Sie einen Standard haben *.txt Datei in Ihrem Spiel enthalten und mit working_directory, dann wird diese Datei aus dem Bereich übernommen, in dem die enthaltenen Dateien gespeichert sind. Wenn Sie dann diese Informationen in eine Textdatei schreiben, verwenden Sie working_directory Dadurch wird die Datei in den lokalen Speicher der Zielplattform geschrieben, wo sie beim nächsten Mal gelesen wird, es sei denn, Sie weisen GameMaker Studio 2 an, die Datei in diesem Fall zu löschen working_directory zeigt erneut auf den Bereich, in dem die enthaltenen Dateien gespeichert sind.


WARNUNG! Diese Funktion funktioniert möglicherweise nicht wie erwartet, weil GameMaker Studio 2 Sandboxing ist! Weitere Informationen finden Sie im Abschnitt zu Dateisystemgrenzen.


Beispiel:

ini_open(working_directory + "temp_ini.ini");

Dies öffnet eine Ini-Datei aus dem Arbeitsverzeichnis des Spiels (erstellt es, wenn es nicht bereits existiert). Dies kann der lokale Speicher oder der Bereich sein, in dem die enthaltenen Dateien davon abhängen, ob die gesuchte Datei an beiden Stellen vorhanden ist.
