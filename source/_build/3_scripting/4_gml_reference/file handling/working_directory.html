Arbeitsverzeichnis


Beschreibung

Working_directory kann tatsächlich zwei verschiedene Werte zurückgeben, je nachdem, wofür Sie es verwenden. Wenn Sie eine Datei auf die Festplatte schreiben, working_directory verweist auf den Bereich des lokalen Speichers, der für Ihr Spiel auf dem Zielgerät reserviert ist (in Windows ist dies "% LOCALAPPDATA% / gamedir /", wobei "gamedir" das Verzeichnis mit dem Namen Ihres Spiels ist). Beim Lesen von der Festplatte working_directory kann entweder der lokale Speicher oder der Bereich sein, in dem die enthaltenen Dateien gespeichert werden.

HINWEIS: Die working_directory Die Variable gibt den Pfad einschließlich des letzten Backslash zurück.

Zum Beispiel, wenn Sie einen Standard haben *.txt Datei in Ihrem Spiel enthalten und lesen Sie es mit working_directory Dann wird diese Datei aus dem Bereich übernommen, in dem die enthaltenen Dateien gespeichert sind. Wenn Sie sich dann entscheiden, diese Informationen mit in eine Textdatei zu schreiben working_directory Dadurch wird die Datei in den lokalen Speicher der Zielplattform geschrieben. Dort wird sie beim nächsten Mal gelesen, sofern Sie nicht GameMaker Studio 2 anweisen, die Datei zu löschen working_directory zeigt noch einmal auf den Bereich, in dem die enthaltenen Dateien gespeichert sind.

WARNUNG! Diese Funktion funktioniert möglicherweise nicht wie erwartet, da GameMaker Studio 2 einer Sandbox ausgeführt wird! Weitere Informationen finden Sie im Abschnitt Dateisystemlimits.


Syntax:

working_directory


Kehrt zurück:

String


Beispiel:

ini_open(working_directory + "temp_ini.ini");

Dadurch wird eine Ini-Datei aus dem Arbeitsverzeichnis des Spiels geöffnet (wenn sie noch nicht vorhanden ist). Dies kann der lokale Speicher oder der Bereich sein, in dem sich die eingeschlossenen Dateien befinden, abhängig davon, ob die gesuchte Datei an einem Ort vorhanden ist.
