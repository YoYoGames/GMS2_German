Dateibehandlung


Während der Erstellung Ihres Spiels müssen Sie irgendwann Informationen extern speichern und in der Lage sein, diese Informationen abzurufen. Sie können beispielsweise Anmeldedaten oder Hi-Scores oder den Fortschritt des Players durch Ihre Stufen speichern. Was auch immer der Grund ist, Sie müssen wissen, wie Sie die verschiedenen Dateifunktionen verwenden, die Ihnen bei der Arbeit mit GameMaker Studio 2.

Es ist sehr wichtig zu wissen, dass GameMaker Studio 2 Sandbox verfügt. Dies bedeutet, dass bei der Verwendung der Dateisystemfunktionen eine Reihe von Einschränkungen und geringfügigen Komplikationen bestehen. Außerdem ist es erwähnenswert, dass das Dateisystem je nach verwendet wird die Plattform, auf die Sie abzielen. Weitere Informationen hierzu finden Sie im folgenden Abschnitt:

  1. Das Dateisystem

Wenn Sie von Steam die Erlaubnis erhalten haben, ihre dedizierte API mit Ihrem Spiel zu verwenden, werden Sie feststellen, dass Ihnen auch eine Reihe neuer Dateifunktionen zur Verfügung steht. Diese werden im Steam API- Abschnitt dieses Handbuchs erläutert.

Schließlich sollten Sie auch beachten, dass es bestimmte Unterschiede zwischen bestimmten Zielplattformen gibt, die bei der Verwendung der Dateifunktionen berücksichtigt werden sollten. Diese sind:

  • iOS: Unter iOS laden Sie eine eingeschlossene Datei aus einem Ordner, dh mit einem Pfad wie " Backgrounds/background1.png msgstr "Sie müssen den Ordner NICHT als Teil des Pfads angeben und es reicht aus, den Dateinamen anzugeben. Auf allen anderen Plattformen müssen Sie den Ordner als Teil des Pfads zur Datei angeben.

  • macOS und Ubuntu (Linux): Auf MacOS und Ubunutu sind auf diesen Plattformen Dateinamen und Pfade zwischen Groß- und Kleinschreibung zu unterscheiden. Um potenziellen Problemen vorzubeugen, werden in GameMaker Studio 2 automatisch alle Namen / Pfade standardmäßig in Kleinbuchstaben verwendet.

  • HTML5: Unter HTML5 funktioniert das Speichern und Laden in und aus dem lokalen Speicher wie in den verschiedenen Funktionsseiten beschrieben. Wenn Sie jedoch eine Datei von einem Server laden, müssen Sie nur die Funktion verwenden buffer_load_async() Da das Laden synchron ausgeführt wurde, wurde es in den meisten Browsern als veraltet eingestuft und wird schließlich veraltet.

Natürlich gibt es Momente, in denen Sie auf das Verzeichnis, in dem die Dateien gespeichert sind, verweisen müssen. GameMaker Studio 2 verfügt außerdem über einige spezielle Funktionen, die sich auf die verschiedenen möglichen Verzeichnisse beziehen, die Sie möglicherweise benötigen:

  • verzeichnis_existiert
  • verzeichnis_erstellen
  • directory_destroy
  • temp_directory
  • Arbeitsverzeichnis
  • Programmverzeichnis

Beim Umgang mit externen Dateien besteht immer die Möglichkeit, dass der Endbenutzer die darin enthaltenen Informationen öffnen und ändern und so Ihr Spiel ändern kann. Dies kann zum Beispiel dazu führen, dass Elemente des Spiels beschädigt werden oder betrügerische Ergebnisse erzielt werden. GameMaker Studio 2 enthält einige grundlegende Codierungsfunktionen sowie Funktionen zum Durchführen von MD5-Überprüfungen von Zeichenfolgen und Dateien, um sicherzustellen, dass sie vorhanden sind ihre Integrität beibehalten, bevor sie verwendet werden. Außerdem werden zwei Funktionen zum Codieren und Decodieren von JSON-Formatzeichenfolgen (JavaScript Object Notation) bereitgestellt.

  • base64_encode
  • base64_decode
  • json_encode
  • json_decode
  • md5_string_utf8
  • md5_string_unicode
  • md5_file
  • sha1_string_utf8
  • sha1_string_unicode
  • sha1_file
  • zip_unzip
  • load_csv
HINWEIS: Bei der Kodierung handelt es sich NICHT um eine Verschlüsselung! Eine base64-Kodierung macht die Datei für das bloße Auge unlesbar und erfordert im Auftrag des Benutzers einen Aufwand zur Dekodierung, sie ist jedoch nicht sicher vor Hacking. Es wird empfohlen, diese Funktionen mit Ihrer eigenen Verschlüsselung zu kombinieren (es gibt viele Arten der Verschlüsselung und Skripts sind im Internet verfügbar).

WICHTIG! Alle GameMaker Studio 2 Projekte haben eine Datei namens " opions.ini ", Sie können also keine Datei mit diesem Namen benennen (Sie erhalten einen UID-Fehler, wenn Sie versuchen, Ihr Spielprojekt zu kompilieren).
