Dateiverarbeitung

Irgendwann während der Erstellung Ihres Spiels wird es notwendig sein, Informationen extern zu speichern und diese Informationen abrufen zu können. Zum Beispiel möchten Sie möglicherweise Login-Details oder Hi-Scores oder vielleicht die Progression des Spielers durch Ihre Ebenen speichern. Was auch immer der Grund sein mag, Sie müssen wissen, wie Sie die verschiedenen Dateifunktionen verwenden, die Ihnen bei der Arbeit mit GameMaker Studio 2.

Es ist sehr wichtig zu wissen, dass GameMaker Studio 2 Sandboxed ist, was bedeutet, dass es eine Reihe von Limits und kleinere Komplikationen bei der Verwendung der verfügbaren Dateisystemfunktionen gibt, und es ist erwähnenswert, dass das Dateisystem je nach System unterschiedlich funktioniert die Plattform, auf die Sie ausgerichtet sind. Weitere Informationen hierzu finden Sie im folgenden Abschnitt:

  1. Dateisystemgrenzen

Wenn Sie von Steam die Erlaubnis erhalten haben, ihre dedizierte API mit Ihrem Spiel zu verwenden, werden Sie feststellen, dass Ihnen auch eine Reihe neuer Dateifunktionen zur Verfügung steht. Diese werden im Steam API- Abschnitt dieses Handbuchs erläutert.

WICHTIG! Alle GameMaker Studio 2 Projekte haben eine Datei namens " opions.ini ". Sie können also keine Datei mit diesem Namen benennen (Sie erhalten einen Fehler, wenn Sie versuchen, Ihr Spielprojekt zu kompilieren).

Offensichtlich wird es Momente geben, in denen Sie auf das Verzeichnis verweisen müssen, in dem Dateien gespeichert werden, und GameMaker Studio 2 hat auch einige spezielle Funktionen, die sich auf die verschiedenen möglichen Verzeichnisse beziehen, die Sie benötigen:

  • directory_exists
  • directory_create
  • directory_destroy
  • temp_verzeichnis
  • Arbeitsverzeichnis
  • Programmverzeichnis

Wenn Sie mit externen Dateien arbeiten, besteht immer die Möglichkeit, dass der Endbenutzer die enthaltenen Informationen öffnet und ändert und so Ihr Spiel ändert. Dies kann zum Beispiel zu kaputten Spielelementen oder betrügerischen Partituren führen. Daher bietet Ihnen GameMaker Studio 2 einige grundlegende Codierungsfunktionen sowie Funktionen zur Durchführung von MD5-Überprüfungen von Zeichenfolgen und Dateien, um sicherzustellen, dass sie dies haben behielt ihre Integrität vor der Verwendung. Es gibt auch zwei Funktionen zum Kodieren und Dekodieren von JSON (JavaScript Object Notation) Formatzeichenfolgen.

  • base64_encode
  • base64_decode
  • json_encode
  • json_decode
  • md5_string_utf8
  • md5_string_unicode
  • md5_datei
  • sha1_string_utf8
  • sha1_string_unicode
  • sha1_datei
  • zip_unzip
  • load_csv
Hinweis: Codierung ist KEINE Verschlüsselung! Eine base64-Codierung macht die Datei für das bloße Auge unlesbar und erfordert eine Anstrengung im Namen des Benutzers, um zu dekodieren, aber es ist nicht sicher vor Hacking. Es wird empfohlen, dass Sie diese Funktionen mit Ihrer eigenen Verschlüsselung kombinieren (es gibt viele Formen der Verschlüsselung und Skripte sind im Internet verfügbar).
