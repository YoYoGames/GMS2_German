get_save_filename_ext

Syntax:

get_save_filename_ext(filter, fname, directory, caption);


Streit Beschreibung
filter Der Dateityp Filter und / oder Name.
fname Der vorgeschlagene Dateiname.
directory Das Verzeichnis, von dem aus die Suche gestartet werden soll.
caption Die Bildunterschrift für das Fenster.


Rückgabe: String.


Beschreibung

Diese Funktion öffnet einen Dialog und fordert den Player zur Eingabe eines Dateinamens auf, unter dem der Filter gespeichert werden soll. Der Filter hat die Form " Name1 | Maske1 | Name2 | Maske2 |... ", wobei die Maske die verschiedenen Optionen mit einem Semikolon enthalten kann. Sie können auch "*" als Zeichenfolge verwenden. Zum Beispiel: " bitmaps | *.bmp; *. Wmf " würde den Benutzer auffordern, den Namen einer Bitmap-Datei aus dem System auszuwählen, jedoch nur diejenigen, die die beiden angegebenen Erweiterungen haben ( bmp und png ). Diese Funktion speichert nicht die Datei selbst! Es wird nur eine Zeichenfolge mit dem vollständigen Namen und Pfad der Datei zurückgegeben. Wenn der Benutzer " Cancel " drückt, wird eine leere Zeichenfolge "" zurückgegeben. Sie können auch ein Verzeichnis hinzufügen, von dem aus die Suche gestartet wird, und eine Beschriftung, die am oberen Rand des Fensters angezeigt werden soll, anstelle der Standardeinstellung.

Beachten Sie bei der Verwendung dieser Funktion, dass Sie bestimmte Berechtigungen für diese Datei für die Dauer Ihres Spiels erhalten. Sobald Sie den Dateipfad vom Benutzer abgerufen haben, können Sie immer wieder darauf zugreifen, ohne dass Sie dazu aufgefordert werden müssen. Die Berechtigung gilt für Lesen und Schreiben gleichermaßen.

HINWEIS: Diese Funktion ist nur für Windows- und MacOS-Ziele gültig.


Beispiel:

var file;
file = get_save_filename_ext("screenshot|*.png", "", working_directory, "Save a level here");
if file != ""
   {
   screen_save(file);
   }

Der obige Code fordert den Benutzer auf, eine Image-Dateinamensdatei zum Speichern im Arbeitsverzeichnis auszuwählen und dann den zurückgegebenen Dateinamen und den Pfad zu überprüfen. Wenn es sich nicht um eine leere Zeichenfolge handelt, wird ein Screenshot erstellt und unter Verwendung des angegebenen Namens gespeichert.
