get_open_filename_ext

Syntax:

get_open_filename_ext(filter, fname, directory, caption);


Streit Beschreibung
filter Der Dateityp Filter und / oder Name.
fname Der vorgeschlagene Dateiname.
directory Das Verzeichnis, von dem aus die Suche gestartet werden soll.
caption Die Bildunterschrift für das Fenster.


Rückgabe: String.


Beschreibung

Diese Funktion öffnet einen Dialog und fordert den Player auf, einen Dateinamen mit dem angegebenen Filter zu öffnen. Der Filter hat die Form " Name1 | Maske1 | Name2 | Maske2 |... ", wobei die Maske die verschiedenen Optionen mit einem Semikolon enthalten kann. Sie können auch "*" als Zeichenfolge verwenden. Zum Beispiel: " bitmaps | *.bmp; *. Wmf " würde den Benutzer auffordern, den Namen einer Bitmap-Datei aus dem System auszuwählen, jedoch nur diejenigen, die die beiden angegebenen Erweiterungen haben ( bmp und png ). Diese Funktion öffnet nicht die Datei selbst! Es wird nur eine Zeichenfolge mit dem vollständigen Namen und Pfad der Datei zurückgegeben. Wenn der Benutzer " Cancel " drückt, wird eine leere Zeichenfolge "" zurückgegeben. Sie können auch ein Verzeichnis hinzufügen, von dem aus die Suche gestartet wird, und eine Beschriftung, die am oberen Rand des Fensters angezeigt werden soll, anstelle der Standardeinstellung.

Beachten Sie bei der Verwendung dieser Funktion, dass Sie bestimmte Berechtigungen für diese Datei für die Dauer Ihres Spiels erhalten. Sobald Sie den Dateipfad vom Benutzer abgerufen haben, können Sie immer wieder darauf zugreifen, ohne dass Sie dazu aufgefordert werden müssen. Da diese Funktion jedoch zum Lesen einer Datei dient, werden die erteilten Berechtigungen nur zum Lesen und Speichern vom Betriebssystem nicht zugelassen. Wenn Sie die Datei lesen und in die Datei schreiben müssen, können Sie die Berechtigung mit der Funktion erhalten get_save_filename.

HINWEIS: Diese Funktion ist nur für Windows- und MacOS-Ziele gültig.


Beispiel:

var file;
file = get_open_filename_ext("text file|*.txt", "", working_directory, "Open a saved level");
if file != ""
   {
   file_text_open_read(file);
   }

Der obige Code fordert den Benutzer auf, eine Textdatei zum Öffnen aus dem Ordner auszuwählen working_directory Überprüfen Sie dann den Namen und den Pfad der zurückgegebenen Datei, und falls vorhanden, wird die Datei zum Lesen geöffnet.
