get_open_filename_ext

Syntax:

get_open_filename_ext(filter, fname, directory, caption);


Streit Beschreibung
filter Der Dateityp-Filter und / oder Name.
fname Der vorgeschlagene Dateiname, der verwendet werden soll.
directory Das Verzeichnis, von dem aus die Suche gestartet werden soll.
caption Die Beschriftung für das Fenster.


Rückgabe: Zeichenfolge.


Beschreibung

Diese Funktion öffnet einen Dialog und fordert den Player auf, einen Dateinamen mit dem angegebenen Filter zu öffnen. Der Filter hat die Form " Name1 | Maske1 | Name2 | Maske2 |... ", wobei die Maske die verschiedenen Optionen mit einem Semikolon zwischen ihnen enthalten kann und Sie können auch ein "*" verwenden, um eine beliebige Zeichenfolge zu bezeichnen. Zum Beispiel: " bitmaps | *.bmp; *. Wmf ", würde den Benutzer auffordern, den Namen einer Bitmap-Datei aus dem System auszuwählen, aber nur diejenigen, die die zwei angegebenen Erweiterungen haben ( bmp und png ). Diese Funktion öffnet nicht die Datei selbst! Es gibt nur eine Zeichenfolge mit dem vollständigen Namen und dem Pfad der Datei zurück. Wenn der Benutzer " Abbrechen " drückt, wird eine leere Zeichenfolge "" zurückgegeben. Sie können auch ein Verzeichnis hinzufügen, von dem die Suche gestartet wird, und eine Beschriftung, die anstelle des Standardfensters im oberen Bereich des Fensters angezeigt wird.

Bei der Verwendung dieser Funktion ist es wichtig, dass Sie für die Dauer Ihres Spiels bestimmte Berechtigungen für diese Datei erhalten. Sobald Sie also den Dateipfad vom Benutzer erhalten, können Sie immer wieder auf ihn zugreifen, ohne nachfragen zu müssen. Da diese Funktion jedoch zum Lesen einer Datei dient, sind die gewährten Berechtigungen nur zum Lesen und Speichern durch das Betriebssystem zulässig. Wenn Sie die Datei lesen und in die Datei schreiben müssen, können Sie mit der Funktion eine Berechtigung erhalten get_save_filename.

HINWEIS: Dies ist eine Windows-Funktion.


Beispiel:

var file;
file = get_open_filename_ext("text file|*.txt", "", working_directory, "Open a saved level");
if file != ""
   {
   file_text_open_read(file);
   }

Der obige Code fordert den Benutzer auf, eine Textdatei zum Öffnen aus der Liste auszuwählen working_directory und dann überprüfen Sie den zurückgegebenen Dateinamen und Pfad, und wenn es existiert, wird die Datei zum Lesen geöffnet.
