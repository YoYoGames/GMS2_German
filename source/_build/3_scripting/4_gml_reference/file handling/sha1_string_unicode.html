sha1_string_unicode


Beschreibung

In der Kryptographie ist SHA-1 eine kryptographische Hash-Funktion, die von der National Security Agency der Vereinigten Staaten entwickelt wurde. Sie wird in mehreren weit verbreiteten Anwendungen und Protokollen wie dem bekannten Git verwendet, wo es zur Überprüfung auf Dateiänderungen verwendet wird. Diese Funktion nimmt einen eingegebenen Unicode-String und gibt einen 160-Bit-Nachrichten-Digest im ASCII-Format zurück. Beispiel:

sha1 input string = "The quick brown fox jumps over the lazy dog"
sha1 output hash = "2fd4e1c6 7a2d28fc ed849ee1 bb76e739 1b93eb12"


Auf diese Weise können Sie einen sicheren Schlüssel generieren, der gespeichert und zur Überprüfung der Integrität der Informationen verwendet werden kann, die an einen externen Server gesendet (oder von diesem empfangen) werden (z. B.).

HINWEIS: Es gibt zwei Formate für die sha1-Kodierung, UTF-8 und Unicode. Beide werden bereitgestellt, um die Kommunikation mit verschiedenen Server-Setups zu erleichtern, am häufigsten wird jedoch Unicode verwendet.


Syntax:

sha1_string_unicode(string)


Streit Beschreibung
string Die Zeichenfolge zum Hashing


Kehrt zurück:

String


Beispiel:

var hash, str;
str = base64_encode(game_data);
hash = sha1_string_unicode(str);
http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?hash=" + hash); http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?data=" + str);

Der obige Code codiert base64 einen String und generiert dann einen sha1-Hash. Schließlich werden sowohl der Hash als auch die codierte Zeichenfolge an einen Server gesendet.
