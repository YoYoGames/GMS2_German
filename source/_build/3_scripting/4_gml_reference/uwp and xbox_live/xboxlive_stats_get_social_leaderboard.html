xboxlive_stats_get_social_leaderboard


Beschreibung

Diese Funktion kann verwendet werden, um eine soziale Rangliste für eine bestimmte Statistik abzurufen. Sie geben die Benutzer-ID an (z. B. von der Funktion zurückgegeben) xboxlive_get_user ), die stat-Zeichenfolge (wie sie bei ihrer Erstellung mit den Funktionen xboxlive_stats_set_stat_ * definiert wurde), und Sie geben dann eine Reihe von Details zu den Leaderboard-Informationen an, die Sie abrufen möchten. Beachten Sie, dass Sie nur eine soziale Bestenliste für int oder echte Statistiken abrufen können, nicht jedoch für Zeichenfolgenstatistiken. Wenn Sie das Argument "favourites_only" als "true" kennzeichnen, enthalten die Ergebnisse nur Daten für die vom Benutzer markierten Freunde als "Favoriten".

WICHTIG! In Social-Leaderboards verwendete Statistiken müssen nicht als "Featured Stats" im XDP / Windows Dev Center registriert werden.


Die Funktion generiert einen Rückruf, der ein asynchrones Systemereignis auslöst. Dieses Ereignis wird die spezielle DS-Karte haben async_load die dann für die folgenden Schlüssel analysiert werden:

  • " id " - Hält die Konstante achievement_stat_event

  • " eventname " - enthält die Zeichenfolge " GetLeaderboardComplete "

  • " userid " - Die der Anfrage zugeordnete Benutzer-ID

  • " error " - 0 wenn erfolgreich, ein anderer Wert, wenn ein Fehler aufgetreten ist. Folgendes ist der am häufigsten zurückgegebene Fehler:

    • -2145844848: Fehlerhafte Anforderung (die Statistik ist kein gültiger Leaderboard-Typ, dh eine Zeichenfolge)

  • " errormessage " - Eine Zeichenfolge mit einer Fehlermeldung, falls vorhanden

  • " display_name " - Die eindeutige ID für die Bestenliste, wie im Provider-Dashboard definiert.

  • " numentries " - Die Anzahl der Einträge in der Bestenliste, die Sie erhalten haben.

Der Rest der DS-Karte enthält auch die Leaderboard-Daten mit dem folgenden Format (wobei "N" die Position in den Leaderboard-Daten ist, von 0 bis "Nummern"):

  • " Player N " - Der Name des Spielers, wobei "N" ein ganzzahliger Wert ist, der seiner Position in der Ranglisteneintragsliste entspricht.

  • " Playerid N " - Die eindeutige Benutzer-ID des Players, "N".

  • " Rang N " - Der Rang des Spielers "N" in der Rangliste.

  • " Score N " - Die Punktzahl des Spielers "N".

WICHTIG! Diese Funktion ist nur beim Exportieren mit dem UWP-Ziel für die Xbox One gültig, und Sie müssen die Option XBox Live aktivieren in den UWP-Spieloptionen aktiviert haben.


Syntax:

xboxlive_stats_get_social_leaderboard(user_id, stat, num_entries, start_rank, start_at_user, favourites_only);

Streit Beschreibung
user_id Die Benutzer-ID des Benutzers, für den die Bestenliste abgerufen werden soll
stat Die Statistik (als String), aus der die globale Bestenliste erstellt werden soll
num_entries Die Anzahl der Einträge, die von der globalen Bestenliste abgerufen werden sollen
start_rank Der Rang in der Bestenliste, von dem aus begonnen werden soll (verwenden Sie 0, wenn das Argument "start_at_user" auf festgelegt ist true )
start_at_user Einstellen true um mit dem Benutzer-ID-Rang zu beginnen, false ansonsten (auf einstellen false wenn das Argument "start_rank" nicht 0 ist
favourites_only Einstellen true um nur Freunde anzuzeigen, die als "Favoriten" oder gekennzeichnet sind false Andernfalls


Kehrt zurück:

N/A


Erweitertes Beispiel:

Das folgende Beispiel zeigt, wie diese Funktion verwendet werden kann. Um mit zu beginnen, rufen Sie es bei einer Veranstaltung wie Room Start oder Create an:

xboxlive_stats_get_social_leaderboard(user_id, "GlobalTime", 20, 1, false, false);

Der obige Code würde aufgerufen, um eine Liste aller globalen Leaderboard-Positionen für das Spiel zu erhalten, und es wird ein Rückruf für soziales asynchrones Ereignis generiert, den wir auf folgende Weise behandeln würden:

if async_load[? "id"] == achievement_stat_event
    {
    if async_load[? "eventname"] == "GetLeaderboardComplete"
        {
        global.numentries = async_load[? "numentries"];
        for(var i = 0; i < numentries; i++;)
            {
            global.playername[i] = async_load[? "Player" + string(i)];
            global.playerid[i] = async_load[? "Playerid" + string(i)];
            global.playerrank[i] = async_load[? "Rank" + string(i)];
            global.playerscore[i] = async_load[? "Score" + string(i]);
            }
        }
    }

Der obige Code prüft die zurückgegebene ds_map im sozialasynchronen Ereignis. Wenn die "id" mit der überprüften Konstante übereinstimmt, wird geprüft, ob das Ereignis durch zurückgegebene Leaderboard-Daten ausgelöst wurde, bevor die Karte durchlaufen und alle verschiedenen Werte gespeichert werden in einer Reihe von globalen Arrays.
