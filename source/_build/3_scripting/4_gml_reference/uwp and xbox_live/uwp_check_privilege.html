uwp_check_privilege


Beschreibung

Mit dieser Funktion können Sie prüfen, ob ein bestimmter Benutzer ein ausgewähltes Privileg hat. Die Funktion gibt im asynchronen Systemereignis wahr oder falsch zurück, je nachdem, ob das Privileg aktiviert ist oder nicht. Wenn Sie das Argument try_resolution auf true setzen und das Privileg nicht aktiviert ist, wird auch ein Systemdialogfeld geöffnet (das Spiel wird angehalten) ) um den Benutzer aufzufordern, das Konto zu aktualisieren, oder was auch immer erforderlich ist, um das Privileg zu erhalten. Wenn der Benutzer die erforderliche Option aktiviert, gibt die Funktion true zurück.

Für die Funktion müssen Sie eine der folgenden Konstanten für die Berechtigungsanforderung verwenden:

Konstante Beschreibung
uwp_privilege_internet_browsing Prüfen Sie, ob das Surfen im Internet zulässig ist
uwp_privilege_social_network_sharing Prüfen Sie, ob das Teilen in sozialen Netzwerken zulässig ist
uwp_privilege_share_kinect_content Prüfen Sie, ob das Teilen des Kinect Controllers zulässig ist
uwp_privilege_video_communications Überprüfen Sie, ob die Videokommunikation zulässig ist
uwp_privilege_communications Prüfen Sie, ob die Internetkommunikation zulässig ist
uwp_privilege_user_created_content Überprüfen Sie, ob der Zugriff auf vom Benutzer erstellte Inhalte zulässig ist
uwp_privilege_multiplayer_sessions Überprüfen Sie, ob Online-Multiplayer-Sitzungen zulässig sind
uwp_privilege_sessions Prüfen Sie, ob Online-Sitzungen zulässig sind
uwp_privilege_fitness_upload Prüfen Sie, ob das Hochladen von Fitnessdaten zulässig ist


Sobald die Funktion aufgerufen wurde, löst sie ein asynchrones Systemereignis aus, wenn das Rückrufergebnis empfangen wurde, das über eine spezielle DS-Zuordnung verfügt async_load. Diese Map sollte dann für den folgenden Schlüssel analysiert werden:

  • " event_type " - sollte den Wert "check_privilege_result" enthalten, wenn das Ereignis von dieser Funktion ausgelöst wurde

Wenn sich der Ereignistyp auf diese Funktion bezieht, gibt es auch die folgenden zusätzlichen Tasten:

  • " result " - Wird eine oder mehrere der folgenden Konstanten (kombinierte Werte) sein:
    Konstante Wert Beschreibung
    uwp_privilege_result_no_issue 0 There are no privilege issues with the user
    uwp_privilege_result_purchase_required 1 The user must purchase something additional, usually a subscription, for access
    uwp_privilege_result_aborted 2 The check was aborted
    uwp_privilege_result_banned 4 The user is banned
    uwp_privilege_result_restricted 8 The user is restricted from access, usually through parental controls

  • " privileg " - ist eine der oben aufgeführten Konstanten für die Verwendung in der Funktion. Sie können also prüfen, welches Privileg Sie bei mehreren Prüfungen angefordert haben.


Syntax:

uwp_check_privilege(user_id, privilege, attempt_resolution);

Streit Beschreibung
user_id Der zu überprüfende Benutzer-ID-Zeiger.
privilege Die zu prüfende Berechtigungskonstante (siehe Beschreibung oben).
attempt_resolution Aktivieren Sie den Lösungsversuch, wenn auf festgelegt ist true oder ignorieren, wenn auf gesetzt false.

Real


Beispiel:

var user_one = xboxlive_get_user(0);
if (xboxlive_user_is_signed_in(user_one))
    {
    uwp_check_privilege(user_one, xboxlive_privilege_multiplayer_sessions, true);
    }

Das obige überprüft, ob der Benutzer angemeldet ist und ob es sich um die Berechtigung für Mehrspieler-Sitzungen handelt.
