xboxlive_matchmaking_join_session


Beschreibung

Diese Funktion füllt eine DS-Karte mit den Details der Benutzer in der angegebenen Sitzung oder -1, wenn ein Fehler auftritt. Sie erhalten den Sitzungs-ID-Zeiger aus der async_load-DS-Zuordnung, die im asynchronen sozialen Ereignis zurückgegeben wird, wenn Sie eine Sitzung gefunden haben. Die zurückgegebene DS-Map enthält die folgenden Schlüssel / Wert-Paare:

  • " num_results " - enthält die Anzahl der Benutzer in der Sitzung

  • " userId <index> " - enthält die ID des Benutzers (<index> ist ein Wert von 0 bis "num_results" - 1)

  • " userIsOwner <index>" - enthält 1, wenn der Benutzer der Host ist, sonst 0 (<index> ist ein Wert von 0 bis "num_results" - 1)

Wenn die Karte nicht mehr benötigt wird, sollte sie mit der Taste aus dem Speicher entfernt werden ds_map_destroy() Funktion andernfalls erhalten Sie ein Speicherleck.

Diese Funktion ist nur beim Exportieren mit dem UWP-Ziel für die Xbox One gültig, und Sie müssen die Option XBox Live aktivieren in den UWP-Spieloptionen aktiviert haben.


Syntax:

xboxlive_matchmaking_join_session(session_id);

Streit Beschreibung
session_id Der Sitzungs-ID-Zeiger der zu betrachtenden Sitzung.


Kehrt zurück:

DS Map


Beispiel:

if !xboxlive_user_is_signed_in()
    {
    if !xboxlive_user_is_signing_in()
        {
        xboxlive_show_account_picker();
        }
    }
else
    {
    global.GamerTag = xboxlive_gamertag_for_user();
    }

Der obige Code prüft, ob ein Benutzer bei XBox Live angemeldet ist. Wenn er angemeldet ist, erhält der Code den Gamertag für den Benutzer und speichert ihn zur späteren Verwendung in einer globalen Variablen. Wenn sie nicht angemeldet sind, prüft der Code, ob sie gerade angemeldet sind. In diesem Fall wird nichts weiter geschehen. Wenn dies nicht der Fall ist, wird der Kontenwähler geöffnet, und er wird aufgefordert, sich anzumelden -im.
