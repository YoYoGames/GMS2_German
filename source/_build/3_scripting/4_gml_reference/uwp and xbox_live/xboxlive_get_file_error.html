xboxlive_get_file_error


Beschreibung

Diese Funktion gibt eine Konstante zurück (siehe unten), um das Debugging zu erleichtern, warum eine Dateifunktion fehlgeschlagen ist, sodass Sie geeignete Maßnahmen in Ihrem Spielcode ergreifen können. Wenn eine der Dateifunktionen fehlschlägt, können Sie die Funktion aufrufen, um genau herauszufinden, was der Fehler war.

HINWEIS: Wenn Sie die asynchronen Speicher- / Ladefunktionen verwenden, wird die async_load map hat einen zusätzlichen Schlüssel, " error ", der auch eine der Konstanten enthält (siehe unten).
WICHTIG! Diese Funktion ist nur beim Exportieren mit dem UWP-Ziel für die Xbox One gültig, und Sie müssen die Option XBox Live aktivieren in den UWP-Spieloptionen aktiviert haben.


Syntax:

xboxlive_get_file_error();


Kehrt zurück:

Constant

Konstante Beschreibung
xboxlive_fileerror_outoflocalstorage Wenn dies beim Speichern geschieht, bedeutet dies, dass der gesamte verfügbare Speicherplatz für das Speichern von Spielen aufgebraucht ist und der Benutzer das vom Betriebssystem angebotene Angebot der Speicherplatzfreigabe abgelehnt hat. Das Spiel sollte mit dem Benutzer bestätigen, dass er das Spiel ohne Speichern fortsetzen möchte. Wenn sie noch speichern können möchten, sollte das Spiel den Speichervorgang erneut versuchen.
xboxlive_fileerror_quotaexceeded In diesem Fall sollte das Spiel den Benutzer darüber informieren, dass er keine weiteren Daten speichern kann, bis er Speicherplatz freigibt, und ihm dann eine Möglichkeit zur Verfügung stellen.
xboxlive_fileerror_noerror Die Funktion zum Speichern oder Laden von Dateien wurde ohne Probleme ausgeführt.


Beispiel:

buffer_save(buff, "SaveGame.bin"); if xboxlive_get_file_error() != xboxlive_fileerror_noerror
    {
    show_debug_message("Save failed. Error code = " + string(xboxlive_get_file_error()));
    }

Der obige Code prüft, ob beim Speichern ein Fehler aufgetreten ist, und wenn ja, wird der Fehlercode an die Konsole ausgegeben.
