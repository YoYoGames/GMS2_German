Shader Konstanten

In diesem Abschnitt werden die GameMaker Studio 2 spezifischen Konstanten aufgeführt, die zum Erstellen von Shadern verwendet werden.

Neben den Shader-Funktionen und -Konstanten, die auf den GLSL ES- Referenzseiten definiert wurden, stehen Ihnen auch eine Reihe von Shader-Konstanten zur Verfügung, die nur für GameMaker Studio 2 zur Verfügung stehen.

Matrizen

Die folgenden Anzeigematrixkonstanten können als Array-Indizes verwendet werden, wenn die Shader-Array-Konstante verwendet wird gm_Matrices:

Konstante Beschreibung
MATRIX_VIEW Diese Array-Indexkonstante hält den Index in der aktuellen Ansichtsmatrix. Der zurückgegebene Index wird beim Aufruf von gm_Matrices Konstante innerhalb des Shader-Codes.
MATRIX_PROJECTION Diese Array-Indexkonstante hält den Index auf der aktuellen Projektionsmatrix. Der zurückgegebene Index wird beim Aufruf von gm_Matrices Konstante innerhalb des Shader-Codes.
MATRIX_WORLD Diese Array-Indexkonstante hält den Index zur aktuellen Weltmatrix. Dies kann für Dinge wie Beleuchtung verwendet werden, wenn Sie Lichtinformationen im Welt-Raum haben. Der zurückgegebene Index wird beim Aufruf von gm_Matrices Konstante innerhalb des Shader-Codes.
MATRIX_WORLD_VIEW Diese Array-Index-Konstante enthält den Index für das Ergebnis der Welt- und View-Matrizen, die miteinander multipliziert werden. Dies wird oft für Dinge wie Nebel verwendet. Der zurückgegebene Index wird beim Aufruf von gm_Matrices Konstante innerhalb des Shader-Codes.
MATRIX_WORLD_VIEW_PROJECTION Diese Array-Index-Konstante enthält den Index für das Ergebnis der Welt-, Ansichts- und Projektionsmatrizen, die miteinander multipliziert werden. Dies ist die normale Transformationsmatrix, die für Vertexpositionen verwendet wird. Der zurückgegebene Index wird beim Aufruf von gm_Matrices Konstante innerhalb des Shader-Codes.
MATRIX_MAX Dies ist kein Array-Index, sondern gibt die Größe des Matrix-Arrays im Vertex-Shader zurück.
MAX_VS_LIGHTS Dies ist kein Array-Index, sondern gibt die Anzahl der Lichter im Vertex-Shader zurück.

Andere Konstanten

Die folgenden vordefinierten Matrix-Uniformen und -Konstanten können in Ihrem Shader verwendet werden, um auf GameMaker Studio 2 spezifische Werte zuzugreifen:

Konstante Beschreibung
gm_Matrices[matrix] Diese Array-Konstante gibt eine Transformationsmatrix zurück und ist eine der verfügbaren vordefinierten Uniformen, die GameMaker Studio 2 für Sie im Shader-Code-Editor erstellt. Der Array-Index wird aus einer der oben aufgeführten Konstanten ausgewählt.
gm_BaseTexture Dies ist eine 2D-Sampler-Konstante, die die von GameMaker Studio 2 Textur des aktuellen Objekts zurückgibt. Es wäre also das aktuelle Sprite, die Oberfläche oder die Textur, die normalerweise beim Zeichnen ohne den Namen des Shaders verwendet würde.
gm_LightingEnabled Dies kann verwendet werden, um die Beleuchtung von GameMaker Studio 2 zu erhalten oder einzustellen, wenn 3D verwendet wird.
gm_FogStart Dies kann verwendet werden, um den Abstand zu berechnen, an dem Polygone beginnen, mit der Nebelfarbe gemischt zu werden.
gm_RcpFogRange Dies kann verwendet werden, um die Entfernung zu erhalten, bei der Nebel maximal ist und nichts mehr zu sehen ist.
gm_PS_FogEnabled Dies wird zurückkehren true oder false wenn die GPU Pixelnebel aktiviert hat oder nicht.
gm_FogColour Dies kann verwendet werden, um die von GameMaker Studio 2 verwendete GameMaker.
gm_VS_FogEnabled Dies wird zurückkehren true oder false wenn die GPU Vertex-Nebel aktiviert hat oder nicht.
gm_AlphaTestEnabled Dies kann verwendet werden, um Alpha-Tests im Shader zu erhalten. Sehen gpu_set_alphatestenable Weitere Informationen zum Alphatest
gm_AlphaRefValue Dies kann verwendet werden, um den aktuellen Alpha-Test-Referenzwert zu erhalten. Sehen gpu_set_alphatestref für weitere Informationen über die Alpha-Test-Referenz.
