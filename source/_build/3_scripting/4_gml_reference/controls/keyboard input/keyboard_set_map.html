keyboard_set_map


Beschreibung

Manchmal, wenn Sie ein Spiel erstellen, möchten Sie, dass ein Schlüssel dasselbe wie ein anderer macht. Zum Beispiel benutzen viele Leute die Tasten WASD für die Bewegung, aber dann benutzen viele Leute auch die Pfeiltasten! Was soll ich tun? Nun, Sie könnten das Bewegungssystem zweimal codieren, aber das ist etwas kompliziert und dankenswerterweise überflüssig, da diese Funktion es Ihnen ermöglicht, eine Taste einer anderen zuzuordnen, sodass jede Eingabe von einer der Tasten als gleich interpretiert wird. Dazu wählen Sie einen Schlüssel, den Sie zuordnen möchten (key2 - das wird der Schlüssel sein, dass Sie den Code für schreiben) und einen Schlüssel, der Sie es haben wollen (key1) abgebildet werden. Danach werden Tastendrücke auf eine der Tasten von GameMaker Studio 2 als von key2 stammend interpretiert. Sie können diese Funktion auch verwenden, um ein System zu entwerfen, in dem der Benutzer seine eigenen Tasten für das Spielen definieren kann, indem Sie die Benutzereingabetaste einfach der von Ihnen im Spiel codierten Taste zuordnen.


Syntax:

keyboard_set_map(key1, key2);

Streit Beschreibung
key1 Dies ist der Schlüssel, dem key1 zugeordnet werden soll
key2 Dies ist der Schlüssel, der zugeordnet werden soll


Kehrt zurück:

Boolean


Beispiel:

keyboard_set_map(ord("A"), vk_left);

Der obige Beispielcode ordnet die "A" -Taste der linken Pfeiltaste zu. Dies bedeutet, dass der Spieler entweder die "A" - oder die linke Pfeiltaste verwenden kann und der gesamte für den linken Pfeil geschriebene Code auch auf die "A" -Taste reagiert, die stattdessen verwendet wird.
