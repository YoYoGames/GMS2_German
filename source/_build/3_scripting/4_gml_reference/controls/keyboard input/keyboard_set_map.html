keyboard_set_map

Ordnet eine Taste auf der Tastatur einer anderen zu.

Syntax:

keyboard_set_map(key1, key2);


Streit Beschreibung
key1 Dies ist der Schlüssel, dem key1 zugeordnet werden soll
key2 Dies ist der Schlüssel, der zugeordnet werden soll


Rückgabe: Boolean


Beschreibung

Wenn Sie ein Spiel machen, möchten Sie vielleicht, dass ein Schlüssel das gleiche tut wie der andere. Zum Beispiel benutzen viele Leute die Schlüssel WASD für Bewegung, aber dann benutzen viele Leute auch die Pfeiltasten! Also, was zu tun ist? Nun, Sie könnten das Bewegungssystem zweimal codieren, aber das ist ein wenig kompliziert und glücklicherweise redundant, da diese Funktion es Ihnen ermöglicht, einen Schlüssel einem anderen zuzuordnen, so dass jede Eingabe von jedem Schlüssel als derselbe interpretiert wird. Dazu wählen Sie einen Schlüssel, den Sie zuordnen möchten (key2 - das wird der Schlüssel sein, dass Sie den Code für schreiben) und einen Schlüssel, der Sie es haben wollen (key1) abgebildet werden. Danach werden Keypressen für einen der beiden Schlüssel von GameMaker Studio 2 so interpretiert, dass sie von key2 kommen. Sie können diese Funktion auch verwenden, um ein System zu entwerfen, in dem der Benutzer seine eigenen Tasten zum Spielen definieren kann, indem er einfach den Benutzereingabeschlüssel auf den Schlüssel abbildet, den Sie im Spiel programmiert haben.


Beispiel:

keyboard_set_map(ord("A"), vk_left);

Der obige Beispielcode bildet die "A" -Taste auf die linke Pfeiltaste ab. Dies bedeutet, dass der Spieler entweder die "A" - oder die linke Pfeiltaste verwenden kann und dass der gesamte für den linken Pfeil geschriebene Code auch auf die verwendete "A" -Taste reagiert.
