Tastatureingabe


Wenn Sie in GameMaker Studio 2 mit der Tastatur arbeiten, haben Sie eine Vielzahl von Funktionen, mit denen Sie unterschiedliche Tastaturzustände wie gedrückt oder loslassen erkennen können. Es gibt auch einige, die alle Tastenbetätigungen als String speichern oder Ihnen sagen können, was die letzte Taste gedrückt hat, sowie andere, mit denen Sie den Tastaturstatus vollständig löschen können.

HINWEIS: Diese Funktionen sind nur für Windows / Mac / Ubuntu-Desktop-Plattformen vorgesehen. Möglicherweise finden Sie einige der integrierten Variablen und Konstanten sind nicht gültig auf anderen Plattformen und viele der Funktionen werden nicht auf Handys funktionieren.

Nun wird jeder Schlüssel normalerweise durch eine Zahl definiert, die als ASCII-Code bezeichnet wird, und Sie können diese Zahl direkt in diese Funktionen eingeben und sie werden gut funktionieren... Aber da es ein bisschen schwierig ist, sich so viele Zahlen und die Beziehung zu merken Sie haben mit Ihrer Tastatur, hat GameMaker Studio 2 eine Reihe von Konstanten für die am häufigsten verwendeten Tastatur Sondertasten und eine spezielle Funktion ord() um die Zahl aus gewöhnlichen Buchstaben (entweder Buchstaben oder Zahlen) zurückzugeben.

Das Folgende ist ein kleines Anwendungsbeispiel ord():

if keyboard_check(ord("A"))
   {
   hspeed = -5;
   }

Also wird oben die "A" -Taste überprüft und wenn sie gedrückt wird, wird die horizontale Geschwindigkeit des Objekts auf -5 gesetzt. Beachten Sie, dass das "A" ein großes "A" ist, und das bei der Verwendung ord() Die zu überprüfende Tastaturtaste muss immer in Anführungszeichen und als Großbuchstaben geschrieben sein. Was nun, wenn du die Pfeiltasten benutzen willst? Oder wenn Sie eine Aktion mit der "Shift" -Taste ändern möchten? Nun, für dieses GameMaker Studio 2 gibt es eine Reihe von vk_ Konstanten ( vk_ steht für virtuelle Tastatur ), die Sie anstelle von ord oder dem ASCII-Code verwenden können.

Hier ist eine vollständige Liste der vk_ Konstanten:

Konstante Beschreibung
vk_nokey Schlüsselcode, der anzeigt, dass keine Taste gedrückt wurde
vk_anykey Schlüsselcode, der angibt, dass eine beliebige Taste gedrückt wurde
vk_left Schlüsselcode für die linke Pfeiltaste
vk_right Schlüsselcode für die rechte Pfeiltaste
vk_up Schlüsselcode für Pfeiltaste nach oben
vk_down Schlüsselcode für die Abwärtspfeiltaste
vk_enter Enter-Taste
vk_escape Escape-Taste
vk_space Leerzeichen
vk_shift eine der Umschalttasten
vk_control eine der Steuertasten
vk_alt alt" Taste
vk_backspace Rücktaste
vk_tab Tab-Taste
vk_home Home-Taste
vk_end Ende-Taste
vk_delete Löschtaste
vk_insert Schlüssel einfügen
vk_pageup Seitenaufschlüsselung
vk_pagedown Abwärtstaste
vk_pause Pause / Pause Taste
vk_printscreen printscreen / sysrq Schlüssel
vk_f1... vk_f12 Keycode für die Funktionstasten F1 bis F12
vk_numpad0... vk_numpad9 Zifferntasten auf dem Ziffernblock
vk_multiply Multiplizieren Sie die Taste auf der Zehnertastatur
vk_divide Teilen Sie die Taste auf der Zehnertastatur
vk_add Taste auf dem Ziffernblock drücken
vk_subtract subtrahieren Sie die Taste auf dem Ziffernblock
vk_decimal Dezimalpunkttasten auf dem Ziffernblock

Die folgenden Konstanten können nur mit verwendet werden keyboard_check_direct():

Konstante Beschreibung
vk_lshift linke Umschalttaste
vk_lcontrol linke Steuertaste
vk_lalt Linke Alt-Taste
vk_rshift rechte Umschalttaste
vk_rcontrol rechte Steuertaste
vk_ralt rechte Alt-Taste



Im Folgenden finden Sie ein kleines Beispiel für die Verwendung des vk_ Konstanten:

if keyboard_check_pressed(vk_tab)
   {
   instance_create_layer(x, y, "Controllers", obj_Menu);
   }

Der obige Code wird erkennen, ob die "Tab" -Taste gedrückt ist, und eine Instanz des Objekts "obj_Menu" erstellen, falls dies der Fall ist.
