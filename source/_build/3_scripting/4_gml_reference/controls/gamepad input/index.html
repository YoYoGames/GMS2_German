GamePad-Eingabe


GameMaker Studio 2 verfügt über eine Reihe spezieller Funktionen, mit denen sowohl analoge als auch digitale Steuerelemente von mehreren angeschlossenen Gamepads erkannt werden können. Diese Funktionen funktionieren ähnlich wie die Geräteeingänge. Sie können bis zu vier verschiedene XInput-Gamepads (und bis zu 8 DirectInput-Gamepads) erkennen und die Eingaben von jedem mit den gleichen Funktionen verarbeiten. Wenn ein Gamepad an Ihr Gerät angeschlossen ist (oder entfernt wird), wird ein asynchrones Systemereignis ausgelöst, bei dem Sie mit den entsprechenden Funktionen mit der Situation umgehen können.

Die Gamepads "Slots" werden von 0 indiziert, wobei die Slots 0 - 3 nur für Xinput-Gamepads gelten, dh: Xbox360-Controller und kompatible Geräte. Sie können jedoch auch die Steckplätze 4 - 11 für DirectInput- Gamepads überprüfen. Dies bedeutet, dass Sie viele andere Controller-Modelle erkennen können, wenn Sie über diese Steckplätze verbunden sind. Es ist erwähnenswert, dass bei der Verwendung von DirectInput-Gamepads die unten angegebenen Konstanten möglicherweise nicht genau den Schaltflächen entsprechen, die Sie erwarten, wenn sie gedrückt werden, aufgrund der fragmentierten und nicht standardisierten Art und Weise, in der die API von Controller-Herstellern implementiert wird. Aus diesem Grund wird empfohlen, dass Sie in Ihren Spielen eine Art Gamepad-Setup-Bildschirm haben, in dem Benutzer die Gamepad-Schaltflächen basierend auf den Eingaben von allen angeschlossenen Geräten neu definieren können, um Probleme zu verringern (es gibt Gamepad-Zuordnungsfunktionen, die helfen können Dies gilt für Windows Desktop-, Ubuntu-, MacOS- und Android Ziele. Bei allen anderen müssen Sie dies selbst mit Code tun. Beachten Sie auch, dass Direct Input-Gamepads im kooperativen Modus ausgeführt werden. Dies bedeutet, dass Ihr Spiel nur dann Zugriff auf sie hat, wenn es sich um die Vordergrundanwendung handelt. Dies bedeutet, dass Direct Input-Controller "verloren gehen", wenn das Spiel den Fokus verliert und "gefunden" wird "wieder, wenn es wieder in den Fokus gerät (dies kann im Systemereignis erkannt und behandelt werden).

Die folgenden Gamepad-Funktionen sind ebenfalls vorhanden und werden verwendet, um die eingebauten Konstanten den direkten physischen Eingaben eines bestimmten Gamepads zuzuordnen. Diese Funktionen gelten nur für Windows-Desktop-, Ubuntu-, MacOS- und Android Zielplattformen und unter Windows nur für Direct-Eingabegeräte. Während GameMaker Studio 2 Mappings für eine Reihe verschiedener Gamepads enthält, die auf SDL Gamepad Controller DB basieren, ist es aufgrund der großen Anzahl von Controller-Typen und Marken nicht möglich, die GML-Konstanten für jede Marke und für jeden Hersteller auf die korrekten Eingaben abzubilden Mit diesen Funktionen haben Sie die Möglichkeit, Ihre eigenen benutzerdefinierten Zuordnungen zu erstellen.

  1. gamepad_get_mapping
  2. gamepad_test_mapping
  3. gamepad_remove_mapping


Kompatibilität

Die folgende Liste zeigt die aktuelle Kompatibilität auf allen Plattformen (beachten Sie, dass sich dies bei zukünftigen Updates ändern wird):

  • Windows wird vollständig unterstützt, wobei maximal 12 verbundene Geräte gleichzeitig zugelassen sind (von 0 bis 11 nummeriert, wobei 0 - 3 XInput-Geräte und 4 - 11 DirectInput sind). Die Zuordnung der Controller-Konstanten ist ebenfalls zulässig.
  • Mac OS wird mit maximal 4 verbundenen Geräten gleichzeitig unterstützt. Diese Geräte können NUR vom Typ Playstation 3 oder Xbox 360 sein. Bitte beachten Sie, dass die Option " Für Mac AppStore erstellen" in den Mac-Spieloptionen aktiviert sein muss OFF, damit die Pad-Unterstützung funktioniert. Die Zuordnung der Controller-Konstanten ist ebenfalls zulässig.

  • Ubuntu unterstützt auch Gamepad-Eingabe, möglicherweise müssen Sie jedoch zusätzliche Bibliotheken aus dem Ubuntu-Repository installieren. Sie können dies leicht tun, indem Sie ein Befehlszeilenterminal öffnen und Folgendes eingeben:

    sudo apt-get install jstest-gtk
    sudo apt-get install joystick.

    Dadurch wird die GUI-Unterstützung für das Joystick / Gamepad sowie die Joystick / Gamepad-API selbst installiert. Die Zuordnung von Controller-Konstanten ist nicht zulässig.

  • HTML5-Spiele mit Gamepads werden von den meisten gängigen Browsern außer Safari unterstützt. Die Zuordnung von Controller-Konstanten ist nicht zulässig.

  • Die Gamepad-Unterstützung erstreckt sich auch auf iOS mit dem iCade- Gehäuse. Die linke Achse ist dem Stick-Controller zugeordnet (obwohl der Eingang digital und nicht analog ist), die vier "face" -Tasten sind den vorderen Tasten des Schranks zugeordnet, und die vier Schulter-Tasten entsprechen denen auf der Rückseite des Schranks.

  • Android Export unterstützt NYKO-Controller und generische Bluetooth-Controller (einschließlich des OUYA), jedoch nur, wenn sie aktiviert sind. Dies bedeutet, dass Sie die Option iCade / Bluetooth im Abschnitt " Allgemein " der Android Spieloptionen aktivieren müssen. Sie benötigen API-Level 12, damit sie vollständig funktionieren können, und es sollte beachtet werden, dass GameMaker Studio 2 alle Bluetooth-Geräte, mit denen Ihr Gerät gekoppelt ist, als verbunden registriert, unabhängig davon, ob es tatsächlich verbunden ist oder nicht. Daher sollte dies bei der Zuweisung und Überprüfung von "Slots" berücksichtigt werden. Beachten Sie, dass das Remapping von Controller-Konstanten ebenfalls zulässig ist.

  • Wenn Sie auf der PS4 das Touchpad-Tracking verwenden möchten, müssen Sie das verwenden device_mouse_* Tasten. Die Zuordnung von Controller-Konstanten ist nicht zulässig.

  • Bei Windows UWP UWP-, XBoxOne- und Nintendo-Switch-Zielen werden Gamepads vollständig unterstützt. Eine Zuordnung von Controller-Konstanten ist jedoch nicht zulässig.

Idealerweise möchten Sie auf allen Zielplattformen eine Liste der verfügbaren Gamepad-Slots auflisten und diese prüfen, um festzustellen, ob Geräte erkannt werden.

var maxpads = gamepad_get_device_count();for (var i = 0; i < maxpads; i++)
    {
    if (gamepad_is_connected(i))
        {
        // do stuff with pad "i"
        }
    }
