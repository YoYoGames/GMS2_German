gamepad_test_mapping


Beschreibung

Diese Funktion kann verwendet werden, um die Gamepad-Zuordnung für die Ziele festzulegen, die dies zulassen. Sie geben den "Slot" -Index für das Gamepad und dann die Map-Zeichenfolge an, die im SDL-Format mit den folgenden Feldern erstellt werden sollte:

<guid>,<description>,platform:<platform-name>,<bindings>

Die Details, die in jedem dieser Felder enthalten sein sollten, sind:

  • guid - Eine Zeichenfolge aus Ziffern und Buchstaben, die einen Gerätetyp eindeutig identifiziert (dieser kann mit der Taste abgerufen werden.) gamepad_get_guid() Funktion)

  • description - eine ASCII-Beschreibung des Geräts (diese kann mit Hilfe der gamepad_get_description() Funktion)

  • platform-name - wird entweder "Plattform: Android", "Plattform: Windows", "Linux" oder "Mac" sein (beachten Sie, dass dies automatisch an den String angehängt wird, wenn Sie diese Funktion aufrufen, sodass Sie sie nicht angeben müssen beim Erstellen eigener Mapping-Strings)

  • bindings - eine Gruppe von Einträgen, die durch ein ',' getrennt sind, um die tatsächliche Eingabe an bestimmte GML-Konstanten zu binden

Die durch Kommas getrennte Liste der Einträge hat für jeden Eintrag ein bestimmtes Format:

<abstract-gp-name>:<value-binding>

Hier <value-binding> (dh die direkte Gamepad-Eingabe) kann eine der folgenden sein:

  • a<N> - Dies stellt eine Achse des Gamepads dar, wobei <N> eine der Achsen von 0 bis ist gamepad_axis_count() -1
  • b<N> - Dies ist eine Schaltfläche des Gamepads, bei der das <N> eine der Tasten von 0 bis ist gamepad_button_count() -1
  • h<N>.<mask> - Dies stellt einen Hut des Gamepads dar, wobei das <N> einen Hut von 0 bis darstellt gamepad_hat_count() -1 und die <mask> ist entweder 1, 2, 4 oder 8.

Jeder Eingabewert sollte an einen SDL-Namen gebunden sein, der als ausgedrückt wird <>abstract-gp-name>, die wiederum an eine GML-Konstante gebunden ist. Die folgende Tabelle zeigt die Äquivalenz zwischen dem SDL-Namen und der GameMaker Studio 2 Konstante:

Wenn Sie Ihren Mapping-String erstellt haben, sollte er ungefähr so ​​aussehen:

"050000005e040000fd020000ffff3f00, Xbox Wireless Controller, a: b0, b: b1, Start: b4, dpdown: h0.4, dpleft: h0.8, dpright: h0.2, dpup: h0.1, Anleitung: b6, leftshoulder: b9 linker Stick: b7, linker Auslöser: a4, leftx: a0, lefty: a1, rechter Schulter: b10, rechter Stick: b8, rechter Auslöser: a5, rightx: a2, righty: a3, x: b2, y: b3, Plattform: android "


Syntax:

gamepad_test_mapping(index, mapping_string);

Streit Beschreibung
index Welchen Gamepad-Index "Slot" einstellen soll.
colour Die zu verwendende Map-Zeichenfolge (weitere Informationen finden Sie in der Beschreibung).

Beispiel:

var mapping = gamepad_get_guid(global.padIndex) + "," + gamepad_get_description(global.padIndex);
var len = array_length_1d(global.PadInstances);
for( i = 0; i < len; i += 2)
    {
    var left = global.PadInstances[i];
    var right = global.PadInstances[i+1];
    mapping += "," + left.sdlLabel + ":" + right.binding;
    }
gamepad_test_mapping(global.padIndex, mapping);

Der obige Code durchläuft eine Reihe von Instanzen und verwendet die Werte der verschiedenen Variablen, die sie enthalten, um eine Mapping-Zeichenfolge zu erstellen, die dann für die Verwendung auf dem Gamepad im angegebenen Slot-Index festgelegt wird.
