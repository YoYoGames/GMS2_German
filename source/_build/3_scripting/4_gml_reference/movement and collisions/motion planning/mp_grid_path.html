mp_grid_path

Syntax:

mp_grid_path(id, path, xstart, ystart, xgoal, ygoal, allowdiag);


Streit Beschreibung
id Index des mp_grid, das verwendet werden soll
path Index des Pfades, der von der Funktion verwendet werden soll
x start Start der X-Koordinate des neuen Pfads
y start Start der y-Koordinate des neuen Pfads
xgoal Beenden der X-Koordinate des neuen Pfads
ygoal Beenden der y-Koordinate des neuen Pfads
allowdiag Gibt an, ob diagonale Bewegungen statt nur horizontal oder vertikal zulässig sind


Rückgabe: Boolean.


Beschreibung


Mit dieser Funktion können Sie einen Pfad erstellen, der mit einem von einem Startpunkt zu einem Zielpunkt navigiert mp_grid die Sie bereits definiert haben, unter Vermeidung von Hindernissen, die bereits in das Netz aufgenommen wurden. Die Argumente xstart und ystart geben den Beginn des Pfades in Raumkoordinaten an, während xgoal, ygoal Argumente das Ziel angeben. Sie können auch entweder nur die horizontale / vertikale Bewegung auswählen oder volle diagonale Bewegungen zulassen, indem Sie im allowdiag-Argument true angeben. Die Funktion liefert entweder true (es ist erfolgreich einen Pfad gefunden) oder false (es ist fehlgeschlagen) und setzt den gewählten Pfad. Im folgenden Bild sehen Sie, wie das alles zusammen funktioniert:

Wie Sie sehen können, wurden die "Pipe" -Objekte in das Raster eingefügt, was bedeutet, dass jeder erstellte Pfad um sie herum verlaufen muss. Es werden zwei Pfade im Bild erstellt, eines (grün) mit Diagonalen erlaubt und das andere (rot) ohne. Der Unterschied zwischen den beiden ist ziemlich offensichtlich, mit dem grünen Pfad, der viel "eleganter" und direkter erscheint, aber es hängt alles davon ab, welchen Nutzen du ihnen geben wirst, welchen du bevorzugst. Beachten Sie, dass der Pfad unabhängig von der aktuellen Instanz ist - es ist ein Pfad durch das Raster, kein Pfad für eine bestimmte Instanz, auch wenn eine bestimmte Instanz möglicherweise die Variable enthält, die den Pfadindex speichert. Möglicherweise müssen Sie diese Pfade auch debuggen, um zu sehen, wie sie in der Spielumgebung erstellt werden und interagieren. In diesem Fall sollten Sie die draw_path Funktion.

HINWEIS: Der Pfad muss zuvor erstellt worden sein (entweder im Code mit path_add oder als Ressource) und wird durch den von dieser Funktion generierten Pfad ersetzt.


Beispiel:

global.grid = mp_grid_create(0, 0, room_width div 32, room_height div 32, 32, 32);
mp_grid_add_instances(global.grid, obj_wall, false);
with (obj_Enemy)
   {
   path = path_add();
   if mp_grid_path(global.grid, path, x, y, obj_Player.x, obj_Player.y, 1)
      {
      path_start(path, 0, 3, 0);
      }
   }

Der obige Code erstellt eine globale Variable "global.grid", generiert dann ein mp_grid und weist dieser Variablen seinen Index (id) zur Verwendung in allen weiteren mp_grid Funktionsaufrufen zu. Es fügt dann alle Instanzen von "obj_Wall" in das Raster ein, bevor alle Instanzen von "obj_Enemy" einen Pfad erstellen und dann verwenden mp_grid_path um einen Router von seiner Position zur Position von "obj_Player" zu berechnen. Wenn eine Route existiert, startet das Objekt sich selbst entlang des Pfades.
