mp_grid_add_instanzen

Syntax:

mp_grid_add_instances(id, obj, prec);


Streit Beschreibung
id Index des mp_grid, das verwendet werden soll
obj Objektindex oder Instanz-ID der Instanzen, die dem mp_grid hinzugefügt werden sollen
prec Ob die Überprüfung auf pixel-perfekten Kollisionen (true = langsam) oder auf seiner Begrenzungsbox im Allgemeinen basiert (false = fast).


Rückkehr: N / A


Beschreibung

Diese Funktion verwendet die Instanz-Kollisionsmaske (durch die Sprite-Eigenschaften oder den Maskenindex der aufrufenden Instanz entschieden), um Zellen als verboten zu markieren oder nicht, wobei verbotene Zellen von keiner der Pfadsuchfunktionen gekreuzt werden können. Sie können in der Funktion angeben, ob Sie präzise Kollisionen berücksichtigen wollen oder nicht, und die markierten verbotenen Zellen ändern sich abhängig von dieser Einstellung. Dieses Bild veranschaulicht dieses Verhalten:

Die beiden obigen Instanzen wurden mit der Funktion mp_grid_add_instances in mp_grid hinzugefügt, wobei precise als true gesetzt wird. Wie Sie sehen können, hat die grüne Instanz die Gitterquadrate, die sie "berührt", nur als verboten markiert, da ihre Spritemaske ebenfalls präzise eingestellt ist. Die zweite Instanz hat jedoch auch andere Zellen markiert, die sich nicht zu berühren scheinen. Dies liegt daran, dass die Spritemaske nicht präzise eingestellt ist. Dies bedeutet, dass selbst wenn Sie die genaue Einstellung in der Funktion verwenden, nur die Begrenzungsbox des Sprites berücksichtigt wird. Dies würde auch passieren, wenn Sie das genaue Argument der Funktion auf false setzen - selbst Instanzen mit einer präzisen Spritemaske werden basierend auf ihren Begrenzungsfeldern in das Raster eingefügt.


Beispiel:

mp_grid_add_instances(grid, obj_Wall, 1);

Der obige Code fügt alle Instanzen von "obj_Wall" in das mp_grid ein, das in der Variable "grid" indiziert ist, wobei die präzise Kollisionsmaske anstelle der Begrenzungsbox verwendet wird.
