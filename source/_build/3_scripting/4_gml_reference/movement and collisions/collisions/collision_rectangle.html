collision_rectangle


Beschreibung

Collision_rectangle verwendet die ersten vier Argumente (x1, y1, x2, y2), um einen Bereich im aktuellen Raum zu definieren, und prüft, ob ein Objekt, das durch das Argument "obj" definiert ist, mit diesem Bereich kollidiert. Diese Kollision kann als genau geprüft werden oder nicht. Sie können auch prüfen, ob die Instanz den Code selbst ausführt oder nicht. Betrachten Sie dieses Bild:

Hier in der Mitte wird ein Kollisionsrechteck verwendet, um nach Ballobjekten zu suchen. Nun versteht die blauen keine genauen Begrenzungsrahmen hat und wie Sie sehen können, auch wenn das Sprit nicht tatsächlich das Rechteck berührt, wird die Kollision noch passieren (auch wenn Sie die genaue Option in der Funktion auf true gesetzt), wie die Der Begrenzungsrahmen dieses Sprites überlappt das Kollisions-Rechteck. Auf der anderen Seite werden die grünen Kugeln nur dann als Kollision betrachtet, wenn das tatsächliche Sprite das Rechteck überlappt. Um präzise Kollisionen zu berücksichtigen, müssen sowohl das Objekt-Sprite als auch die Kollisionsfunktion genau als aktiviert gekennzeichnet sein. Es sollte auch beachtet werden, dass der Rückgabewert der Funktion die ID einer der Instanzen sein kann, die als in Kollision betrachtet werden.


Syntax:

collision_rectangle(x1, y1, x2, y2, obj, prec, notme);

Streit Beschreibung
x1 Die x-Koordinate der linken Seite des zu überprüfenden Rechtecks.
y1 Die y-Koordinate der oberen Seite des zu überprüfenden Rechtecks.
x2 Die x-Koordinate der rechten Seite des Rechtecks, die geprüft werden soll.
y2 Die y-Koordinate der unteren Seite des Rechtecks, die geprüft werden soll.
obj Das Objekt, das auf Kollisionen geprüft werden soll.
prec Ob die Prüfung auf genauen Kollisionen basiert ( true, der langsamer ist) oder dessen Begrenzungsrahmen im Allgemeinen ( false, schneller).
notme Ob die aufrufende Instanz ggf. ausgeschlossen werden soll ( true ) oder nicht ( false ).


Kehrt zurück:

Instance id or noone


Beispiel:

var inst;
inst = collision_rectangle(50, 50, 200, 100, obj_Ball, false, true);
if inst != noone
   {
   with (inst) instance_destroy();
   }

Dieser Kurzcode verwendet collision_rectangle, um einen Bereich im Raum von 50x, 50y (oben links des Rechtecks) bis 200x, 200y (unten rechts des Rechtecks) auf eine Instanz eines Objekts namens "obj_ball" zu überprüfen. Es speichert den Rückgabewert in einer temporären Variablen, die daraufhin geprüft wird, ob es sich bei diesem Wert um eine Instanz-ID oder um das Schlüsselwort noone handelt. Ist dies nicht der Fall, verwendet es die gespeicherte Instanz-ID, um das Objekt zu zerstören.
