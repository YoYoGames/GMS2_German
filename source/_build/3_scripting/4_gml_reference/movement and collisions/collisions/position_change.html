Positionsänderung

Syntax:

position_change(x, y, obj, perf);


Streit Beschreibung
x Die x-Koordinate, an der kollidierende Instanzen geändert werden sollen.
y Die y-Koordinate, an der kollidierende Instanzen geändert werden sollen.
obj Das neue Objekt, in das sich das aufrufende Objekt ändert.
perf Gibt an, ob das Create-Ereignis dieses neuen Objekts ausgeführt werden soll (true) oder nicht (false).


Rückkehr: N / A


Beschreibung

Diese Funktion prüft eine Position für eine Kollision mit beliebigen Instanzen an dem gegebenen Punkt, und falls es eine gibt, werden alle Instanzen in der Kollision geändert, um Instanzen des ausgewählten Objekts zu sein. Sie können das "perf" Argument auf setzen true GameMaker Studio 2 wird GameMaker Studio 2 gezwungen, die Destroy- und Cleanup-Ereignisse der gefundenen Instanz und des Create-Ereignisses der neuen Instanz auszuführen false keine dieser Ereignisse durchführen. Bitte beachten Sie, dass jede erstellte Instanz, die eine Variable verwendet, die normalerweise im create-Ereignis definiert ist, das Spiel zum Absturz bringt, da diese Variable nicht mehr existiert, wenn Sie die Ereignisse Destroy, Cleanup und Create nicht ausführen.


Beispiel:

position_change(200, 200, obj_Bird, true);

Dadurch werden alle Instanzen, die an Punkt (200,200) kollidieren, in eine Instanz von "obj_Bird" geändert, wodurch "obj_Bird" Create-Ereignis für jeden von ihnen im Prozess ausgeführt wird.
