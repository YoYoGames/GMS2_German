Kollisionslinie

Syntax:

collision_line( x1, y1, x2, y2, obj, prec, notme );


Streit Beschreibung
x1 Die x-Koordinate des Zeilenanfangs.
y1 Die y-Koordinate des Zeilenanfangs.
x2 Die x-Koordinate des Zeilenendes.
y2 Die y-Koordinate des Zeilenendes.
obj Das Objekt, das auf Kollisionen geprüft werden soll.
prec Ob die Überprüfung auf pixel-perfekten Kollisionen (true = langsam) oder auf seiner Begrenzungsbox im Allgemeinen basiert (false = fast).
notme Ob die aufrufende Instanz gegebenenfalls ausgeschlossen werden soll (true) oder nicht (false).


Rückgabe : Instanz-ID oder nicht


Beschreibung

Kollisionslinie prüft eine Linie von Punkt x1, y1 zu Punkt x2, y2 für eine Kollision mit einer beliebigen Instanz des Objekts, das durch das Argument "obj" spezifiziert ist. Diese Prüfung kann entweder präzise oder nicht, aber für eine genaue Kollisionen aktiviert sein werden, das Objekt oder die Instanz, die Sie für die Überprüfung müssen auch präzise Kollisionen für ihre Sprite aktiviert. Wenn nicht, basiert die Standardüberprüfung auf Begrenzungsrahmen. Das folgende Bild zeigt, wie dies funktioniert:

Denken Sie daran, dass für präzise Kollisionen sowohl das Objekt-Sprite als auch die Kollisionsfunktion genau markiert sein müssen. Es sollte auch beachtet werden, dass der Rückgabewert der Funktion die ID eines beliebigen der Instanzen sein kann, die entlang der Linie als kollidierend betrachtet werden. Wenn also drei Instanzen die definierte Linie überlappen, könnte jede ihrer IDs der Rückgabewert sein der Funktion.


Beispiel:

with (obj_Enemy)
   {
   if collision_line(100, 400, 100, 600, id, false, false) instance_destroy();
   }

Dieser Code ruft alle Instanzen von "obj_Enemy" auf, um eine Linie von 100x, 400y bis 100x, 600y für eine Kollision mit sich selbst zu überprüfen, und wenn es eine gibt, werden sie zerstört.
