collision_line


Beschreibung

Die Kollisionslinie prüft entlang einer Linie von Punkt x1, y1 bis Punkt x2, y2 auf eine Kollision mit einer beliebigen Instanz des Objekts, die durch das Argument "obj" angegeben ist. Diese Prüfung kann entweder präzise oder nicht, aber für eine genaue Kollisionen aktiviert sein werden, das Objekt oder die Instanz, die Sie für die Überprüfung müssen auch präzise Kollisionen für ihre Sprite aktiviert. Wenn nicht, basiert die Standardprüfung auf Begrenzungsrahmen. Das folgende Bild zeigt, wie das funktioniert:

Um präzise Kollisionen zu berücksichtigen, müssen sowohl das Objekt-Sprite als auch die Kollisionsfunktion genau als aktiviert gekennzeichnet sein. Es sollte auch beachtet werden, dass der Rückgabewert der Funktion die ID einer der Instanzen sein kann, von denen angenommen wird, dass sie entlang der Linie in Kollision sind. Wenn also drei Instanzen die definierte Zeile überlappen, könnte jede ihrer IDs der Rückgabewert sein der Funktion.


Syntax:

collision_line(x1, y1, x2, y2, obj, prec, notme);

Streit Beschreibung
x1 Die x-Koordinate des Zeilenanfangs.
y1 Die y-Koordinate des Zeilenanfangs.
x2 Die x-Koordinate des Endes der Linie.
y2 Die y-Koordinate des Endes der Linie.
obj Das Objekt, das auf Kollisionen geprüft werden soll.
prec Ob die Prüfung auf genauen Kollisionen basiert ( true, der langsamer ist) oder dessen Begrenzungsrahmen im Allgemeinen ( false, schneller).
notme Ob die aufrufende Instanz ggf. ausgeschlossen werden soll ( true ) oder nicht ( false ).


Kehrt zurück:

Instance id or noone


Beispiel:

with (obj_Enemy)
   {
   if collision_line(100, 400, 100, 600, id, false, false) instance_destroy();
   }

Dieser Code ruft alle Instanzen von "obj_Enemy" auf, um entlang einer Linie von 100x, 400y bis 100x, 600y auf eine Kollision mit sich selbst zu prüfen, und wenn es eine gibt, dann werden sie zerstört.
