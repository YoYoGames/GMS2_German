buffer_async_group_option

Syntax:

buffer_async_group_option(option, value);


Streit Beschreibung
option Die Option zum Festlegen.
value Der einzustellende Wert (kann abhängig von der Option string oder real sein).


Rückkehr: N / A


Beschreibung

Mit dieser Funktion können Sie einige plattformspezifische Optionen für die zu speichernde Puffergruppe festlegen. Die verfügbaren Optionen sind wie folgt:

  • " Untertitel " oder " slottitle " - Der Wert für diese Option wäre eine Zeichenfolge, die dem Benutzer bei der Verwaltung seiner Sicherungsdaten im Betriebssystem angezeigt wird. Dies ist nur beim Speichern von Daten wichtig, nicht beim erneuten Laden.

  • " showdialog " - Der Wert für diese Option ist ein boolescher Wert true oder false. Wenn auf eingestellt true Wenn Sie speichern oder laden, wird in GameMaker Studio 2 die Systembenutzeroberfläche angezeigt. Andernfalls wird ein Schnellspeicher / Schnellladevorgang ohne Benutzeroberfläche ausgeführt. Normalerweise benötigen Sie diese Funktion nur, wenn Sie mehrere Speicherplätze für Speicherplätze zur Verfügung haben, um dem Benutzer die Auswahl eines Steckplatzes zu ermöglichen. Wenn Sie jedoch nur einen Steckplatz pro Benutzer unterstützen, legen Sie Folgendes fest false.

  • " Savepadindex " - Für diese Option würden Sie den Pad-Index des Spielers angeben, der speichert oder lädt, und das System schreibt Daten in den Speicherordner dieses Players und liest Daten aus diesem Speicher.

  • " saveslotsize " - Diese Option setzt voraus, dass Sie die tatsächliche Größe in Byte angeben, die Sie speichern möchten (damit Sie a buffer_seek und buffer_tell um das zum Beispiel zu bekommen). Beachten Sie, dass es nicht obligatorisch ist, diesen Wert anzugeben, da allen Speichervorgängen ein minimaler Speicherplatz zugewiesen wird, der normalerweise ungefähr 10 MB oder so beträgt.

WICHTIG - Diese Funktion ist derzeit nur für die Zielmodule PSVita, PS4 und XBox One gültig. Bei allen anderen Zielen wird es nichts tun.


Beispiel:

buffer_async_group_begin("save_folder_name");
buffer_async_group_option("showdialog", false);
buffer_async_group_option("slottitle", "Catch The Haggis Save");
buffer_async_group_option("subtitle", "All your haggis are saved here!");
save = buffer_save_async(buff, "Player_Save.sav", 0, 16384);
buffer_async_group_end();

Der obige Code startet eine Puffergruppe und legt dann die Gruppenoptionen fest, bevor 4 Dateien asynchron gespeichert werden. Die Gruppendefinition wird beendet (an diesem Punkt beginnt das Speichern).
