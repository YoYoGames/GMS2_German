buffer_create_from_vertex_buffer_ext

Syntax:

buffer_create_from_vertex_buffer_ext(vertex_buffer, type, alignment, start_vertex, num_vertices)


Streit Beschreibung
vertex_buffer Der zu verwendende Vertexpufferindex.
type Der Typ des zu erstellenden Puffers (siehe Liste der Konstanten).
alignment Die Byteausrichtung für den Puffer
start_vertex Der Startknoten.
num_vertices Die Gesamtzahl der zu kopierenden Scheitelpunkte.


Rückgabe: Echt


Beschreibung

Sie verwenden diese Funktion, um einen Teil des Speichers als Puffer in Ihrem Spiel zu reservieren, der mit den Daten eines zuvor erstellten Vertexpuffers gefüllt ist. Die Funktion gibt eine eindeutige Puffer-ID zurück, die in einer Variablen gespeichert und für alle weiteren Funktionsaufrufe an den Puffer verwendet werden soll. Die Funktion übernimmt die ID, die der Vertexpuffer verwenden soll (wie von der Funktion zurückgegeben) vertex_create_buffer() zum Beispiel) mit den folgenden Konstanten, die verwendet werden, um den Puffertyp zu definieren:

Konstante Beschreibung
buffer_fixed Ein Puffer von fester Größe.
buffer_grow Ein Puffer, der dynamisch wächst, wenn Daten hinzugefügt werden
buffer_wrap Ein Puffer, in dem die Daten "umbrechen". Wenn die hinzugefügten Daten das Limit der Puffergröße erreichen, wird das Überschreiben an den Anfang des Puffers zurückgesetzt und das weitere Schreiben wird von diesem Punkt aus fortgesetzt.
buffer_fast Spezieller "abgestreifter" Puffer, der extrem schnell zu lesen / zu schreiben ist. Kann nur mit verwendet werden buffer_u8 Datentypen und muss 1 Byte ausgerichtet sein.
buffer_vbuffer



Neben dem Puffertyp müssen Sie auch die Byte-Ausrichtung für den Puffer festlegen. Dieser Wert hängt von den Daten ab, die Sie im Puffer speichern möchten. In den meisten Fällen ist der Wert 1 völlig in Ordnung. Beachten Sie jedoch, dass für bestimmte Operationen eine bestimmte Ausrichtung wichtig ist und eine falsche Ausrichtung zu Fehlern führen kann (weitere Informationen zur Ausrichtung finden Sie unter Pufferausrichtung ). Im Folgenden finden Sie eine allgemeine Anleitung, welche Werte für jeden Datentyp am besten geeignet sind (im Zweifelsfall verwenden Sie eine Ausrichtung von 1):

  • Zeichenfolgen sollten auf 1 Byte ausgerichtet sein.
  • Vorzeichenbehaftete oder vorzeichenlose 8-Bit-Ganzzahlen können auf jeden Wert ausgerichtet werden, aber beachten Sie, dass für einen schnellen Puffer (siehe buffer_write ) muss es auf 1 ausgerichtet sein.
  • Vorzeichenlose oder vorzeichenlose 16-Bit-Ganzzahlen sollten auf 2 Bytes ausgerichtet sein.
  • Signierte oder vorzeichenlose 32-Bit-Ganzzahlen sollten auf 4 Bytes ausgerichtet werden
  • Floats von bis zu 16 Bits sollten auf 2 Bytes ausgerichtet werden. (Derzeit nicht unterstützt!)
  • Floats von bis zu 32 Bit sollten auf 4 Byte ausgerichtet werden.
  • Floats von bis zu 64 Bits sollten auf 8 Bytes ausgerichtet werden.

Hinweis: Vertex-Puffer sind 1 Byte ausgerichtet, aber Sie können den Puffer mit jeder Ausrichtung erstellen, je nachdem, wie Sie die Daten behandeln möchten, da die Vertex-Daten einfach eine rohe Speicherkopie in den Puffer sind.

Die letzten zwei Argumente werden verwendet, um den Bereich der Eckpunktdaten anzugeben, den Sie in den neu erstellten Puffer kopieren möchten. Der Start-Vertex kann irgendwo zwischen 0 und der Anzahl der Vertices -1 liegen, und dann geben Sie die Anzahl der Vertices von diesem Punkt zum Kopieren an. Sie können die Funktion verwenden vertex_get_number() auf dem Vertex-Puffer, um die Gesamtzahl der gespeicherten Eckpunkte zu erhalten.

HINWEIS: Es ist wichtig, dass Sie alle dynamisch erstellten Ressourcen wie diese aus dem Speicher entfernen, wenn Sie sie nicht mehr benötigen, um Speicherlecks zu vermeiden. Wenn Sie den erstellten Puffer fertiggestellt haben, sollten Sie ihn wieder freigeben buffer_delete. Beachten Sie auch, dass der Vertexpuffer, der zum Erstellen des neuen Puffers verwendet wird, ebenfalls nicht aus dem Speicher entfernt wird, und Sie sollten die Funktion verwenden vertex_delete_buffer() wenn es nicht mehr benötigt wird.


Beispiel:

var v_num = vertex_get_number(model_buff); player_buffer = buffer_create_from_vertex_buffer(model_buffer, buffer_grow, 1, 0, v_num - 1);

Der obige Code ordnet einem Puffer Speicher zu und kopiert dann die Daten aller Scheitelpunkte aus dem gegebenen Scheitelpuffer in diesen, wobei der Index des neuen Puffers, der in der Variablen "player_buffer" gespeichert ist, zur zukünftigen Verwendung zurückgegeben wird.
