Puffer


Ein binärer Puffer ist ein Bereich des physischen Speichers, in dem Daten temporär gespeichert werden, während sie von einem Ort an einen anderen verschoben oder auf irgendeine Weise manipuliert werden. Beispielsweise können Sie einen Puffer zum Speichern von Paketdaten verwenden, die auf die Übertragung über ein Netzwerk warten oder einen Speicherplatz speichern Paket, das über ein Netzwerk empfangen wurde. GameMaker Studio 2 verfügt über eine Reihe spezifischer Funktionen zum Erstellen, Ändern und Laden / Speichern von Puffern. Bevor Sie diese verwenden, beachten Sie bitte die folgenden Punkte:

  • Wenn ein Puffer erstellt wird, wird er automatisch gelöscht und mit 0 gefüllt.

  • Das Argument "size" der Pufferfunktionen bezieht sich immer auf die Größe des Puffers in Byte.

  • Das Argument "Ausrichtung" bezieht sich darauf, wie Daten in einem Puffer gespeichert werden. Wenn Ihr Alignment auf "4" eingestellt ist und Sie ein einzelnes Datenstück schreiben, das 1 Byte groß ist, dann führen Sie a aus buffer_tell Sie erhalten einen Versatz von 1. Wenn Sie jedoch ein weiteres Datenelement schreiben, das ebenfalls 1 Byte groß ist, führen Sie a aus buffer_tell Sie erhalten einen Versatz von 5, da die Daten durch die Ausrichtung auf diese Größe "aufgefüllt" wurden. Ausrichtung beeinflusst nur, wo Dinge geschrieben werden, wenn Sie also eine buffer_tell Nachdem Sie etwas geschrieben haben, wird die aktuelle Schreibposition zurückgegeben, die unmittelbar auf die von Ihnen geschriebenen Daten folgt. Wenn Sie jedoch ein weiteres Datenstück schreiben, wird der Puffer intern die Schreibposition entlang des nächsten Vielfachen der Ausrichtungsgröße verschieben, bevor das Datenelement tatsächlich geschrieben wird.

  • Bei "Offset" ist dies der Wert in Bytes, um die Daten innerhalb des angegebenen Puffers zu verschieben. Wenn Ihr Puffer also 2 Byte ausgerichtet ist und Sie die ersten 4 ausgerichteten Stellen überspringen möchten, haben Sie einen Versatz von 2 * 4 bytes.

  • Einige Pufferfunktionen erstellen auch einen neuen Puffer (z. B. buffer_load zum Beispiel). Denken Sie daran, dass diese Puffer auch aus dem Speicher entfernt werden müssen, wenn sie nicht verwendet werden buffer_delete Funktion.

  • Der Speicher, der zum Erstellen von Puffern verwendet wird , ist der Systemspeicher. Selbst wenn das Spiel keinen Fokus hat, sollten alle in einem Puffer gespeicherten Daten sicher sein. Wenn jedoch die App geschlossen oder neu gestartet wird, gehen diese verloren.
