Puffer


Ein binärer Puffer ist ein Bereich des physischen Speichers, der verwendet wird, um Daten temporär zu halten, während sie von einem Ort zum anderen bewegt oder auf irgendeine Weise manipuliert wird. Beispielsweise können Sie einen Puffer verwenden, um Paketdaten für die Übertragung über ein Netzwerk oder für die Speicherung eines Paket, das über ein Netzwerk empfangen wurde. GameMaker Studio 2 verfügt über eine Reihe spezifischer Funktionen zum Erstellen, Ändern und Laden / Speichern von Puffern. Bitte beachten Sie vor der Verwendung die folgenden Punkte:

  • Wenn ein Puffer erstellt wird, wird er automatisch gelöscht und auf 0 gesetzt.

  • Das Argument "size" der Pufferfunktionen bezieht sich immer auf die Größe in Bytes des Puffers.

  • Das Argument "Alignment" bezieht sich darauf, wie Daten in einem Puffer gespeichert werden. Wenn Ihre Ausrichtung auf "4" gesetzt ist und Sie ein einzelnes Datenelement schreiben, das 1 Byte groß ist, dann führen Sie a buffer_tell, erhalten Sie einen Offset von 1. Wenn Sie jedoch ein weiteres Stück Daten schreiben, das ebenfalls 1 Byte groß ist, dann tun Sie ein buffer_tell, erhalten Sie einen Offset von 5, da die Ausrichtung die Daten auf diese Größe "gepolstert" hat. Die Ausrichtung wirkt sich nur darauf aus, wo etwas geschrieben wird. Wenn Sie also ein buffer_tell nachdem Sie etwas geschrieben haben, wird die aktuelle Schreibposition zurückgegeben, die unmittelbar auf die Daten folgt, die Sie geschrieben haben. Beachten Sie jedoch, dass der Puffer die Schreibposition intern auf das nächste Vielfache der Ausrichtungsgröße vor dem eigentlichen Schreiben des Datenelements verschiebt, wenn Sie dann ein weiteres Datenelement schreiben.

  • Bei "Offset" ist dies der Wert in Byte, um die Daten innerhalb des angegebenen Puffers zu versetzen. Wenn Ihr Puffer also 2 Byte ausgerichtet ist und Sie die ersten 4 ausgerichteten Stellen überspringen möchten, hätten Sie einen Offset von 2 * 4 bytes.

  • Einige der Pufferfunktionen erzeugen auch einen neuen Puffer (wie buffer_load beispielsweise). Denken Sie daran, dass diese Puffer auch aus dem Speicher entfernt werden müssen, wenn sie nicht verwendet werden buffer_delete Funktion.

  • Der Speicher, der zum Erstellen von Puffern verwendet wird , ist Systemspeicher. Selbst wenn das Spiel keinen Fokus hat, sollten alle in einem Puffer gespeicherten Daten sicher sein. Wenn die App jedoch geschlossen oder neu gestartet wird, geht sie verloren.
