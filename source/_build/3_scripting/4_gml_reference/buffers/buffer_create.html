buffer_create

Syntax:

buffer_create(size, type, alignment)


Streit Beschreibung
size Die Größe (in Byte) des Puffers.
type Der Typ des zu erstellenden Puffers (siehe Liste der Konstanten).
alignment Die Byteausrichtung für den Puffer


Rückgabe: Echt


Beschreibung

Sie verwenden diese Funktion, um einen Teil des Speichers als Puffer in Ihrem Spiel zu reservieren, wobei die Funktion die eindeutige Puffer-ID zurückgibt, die in einer Variablen gespeichert und für alle weiteren Funktionsaufrufe in den Puffer verwendet werden soll. Der Puffer kann dann verwendet werden, um verschiedene Arten von Daten zu speichern (angegeben beim Schreiben in den Puffer mit dem buffer_write Funktion, wobei die folgenden Konstanten verwendet werden, um den Puffertyp zu definieren:

Dieser Typ von Puffer soll nur als Vertex-Puffer verwendet werden.
Konstante Beschreibung
buffer_fixed Ein Puffer von fester Größe.
buffer_grow Ein Puffer, der dynamisch wächst, wenn Daten hinzugefügt werden
buffer_wrap Ein Puffer, in dem die Daten "umbrechen". Wenn die hinzugefügten Daten das Limit der Puffergröße erreichen, wird das Überschreiben an den Anfang des Puffers zurückgesetzt und das weitere Schreiben wird von diesem Punkt aus fortgesetzt.
buffer_fast Spezieller "abgestreifter" Puffer, der extrem schnell zu lesen / zu schreiben ist. Kann nur mit verwendet werden buffer_u8 Datentypen und muss 1 Byte ausgerichtet sein.
buffer_vbuffer



Neben dem Puffertyp müssen Sie auch die Byte-Ausrichtung für den Puffer festlegen. Dieser Wert hängt von den Daten ab, die Sie im Puffer speichern möchten. In den meisten Fällen ist der Wert 1 völlig in Ordnung. Jedoch darüber im Klaren sein, dass eine bestimmte Ausrichtung für einige Operationen ist von wesentlicher Bedeutung, und eine falsche Ausrichtung kann zu Fehlern führt (für weitere Einzelheiten über die Ausrichtung siehe Buffers ). Im Folgenden finden Sie eine allgemeine Anleitung, welche Werte für jeden Datentyp am besten geeignet sind (im Zweifelsfall verwenden Sie eine Ausrichtung von 1):

  • Zeichenfolgen sollten auf 1 Byte ausgerichtet sein.
  • Vorzeichenbehaftete oder vorzeichenlose 8-Bit-Ganzzahlen können auf jeden Wert ausgerichtet werden, aber beachten Sie, dass für einen schnellen Puffer (siehe buffer_write ) muss es auf 1 ausgerichtet sein.
  • Vorzeichenlose oder vorzeichenlose 16-Bit-Ganzzahlen sollten auf 2 Bytes ausgerichtet sein.
  • Signierte oder vorzeichenlose 32-Bit-Ganzzahlen sollten auf 4 Bytes ausgerichtet werden
  • Floats von bis zu 16 Bits sollten auf 2 Bytes ausgerichtet werden. (Derzeit nicht unterstützt!)
  • Floats von bis zu 32 Bit sollten auf 4 Byte ausgerichtet werden.
  • Floats von bis zu 64 Bits sollten auf 8 Bytes ausgerichtet werden.

HINWEIS: Die Byteausrichtung kann sehr wichtig sein, da die falsche Auswahl die Leistung beeinträchtigen kann.

HINWEIS: Es ist wichtig, dass Sie alle dynamisch erstellten Ressourcen wie diese aus dem Speicher entfernen, wenn Sie sie nicht mehr benötigen, um Speicherlecks zu vermeiden. Wenn Sie den erstellten Puffer fertiggestellt haben, sollten Sie ihn wieder freigeben buffer_delete.


Beispiel:

player_buffer = buffer_create(16384, buffer_fixed, 2);

Der obige Code ordnet einem Puffer 16384 Byte Speicher zu und gibt den Index dieses Puffers, der in der Variablen "player_buffer" gespeichert ist, zur zukünftigen Verwendung zurück. Der Puffer ist auf eine Zwei-Byte-Grenze ausgerichtet.
