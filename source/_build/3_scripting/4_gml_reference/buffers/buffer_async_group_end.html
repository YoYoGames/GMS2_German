buffer_async_group_end

Syntax:

buffer_async_group_end();


Rückgabe: Echt


Beschreibung

Diese Funktion beendet die Definition einer Pufferspeichergruppe. Sie müssen zuvor die Funktion aufgerufen haben buffer_async_group_begin() Um die Gruppe zu initiieren, rufen Sie die Funktion auf buffer_save_async() für jede Datei, die Sie speichern möchten. Schließlich rufen Sie diese Funktion auf, die das Speichern der Dateien startet. Die Funktion gibt einen eindeutigen ID-Wert für das Speichern zurück, der dann im Asynchronous Save-Ereignis verwendet werden kann


Beispiel:

Das buffer_async_group_end() Die Funktion kann von jedem beliebigen Ereignis aus aufgerufen werden, und da sie asynchron ist, kann der Rückruf fast sofort erfolgen oder mehrere Sekunden dauern. Der Aufruf der Funktion ist einfach und würde etwa so aussehen:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
save_id = buffer_async_group_end();

Der obige Code startet eine Puffergruppe und setzt sie so, dass 4 Dateien asynchron gespeichert werden. Die Gruppendefinition wird dann beendet (an diesem Punkt beginnt das Speichern), wobei die ID des Funktionsaufrufs in der Variablen " save_id " gespeichert wird. Wenn der Speichervorgang abgeschlossen ist, wird das asynchrone Save / Load-Ereignis ausgelöst und Sie können das Ereignis analysieren async_load map für die korrekte ID der Funktion, so:

if ds_map_find_value(async_load, "id") == saveid
   {
   if ds_map_find_value(async_load, "status") == false
      {
      show_debug_message("Save failed!");
      }
   }

Der obige Code überprüft zuerst die ID der erstellten ds_map, überprüft dann den Status des Callbacks und gibt eine Debug-Nachricht aus, falls Probleme aufgetreten sind.
