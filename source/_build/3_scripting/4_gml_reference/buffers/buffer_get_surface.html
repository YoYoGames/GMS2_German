buffer_get_surface

Syntax:

buffer_get_surface(buffer, surface, mode, offset, modulo);


Streit Beschreibung
buffer Der Index des zu verwendenden Puffers.
surface Der Index der zu verwendenden Oberfläche.
mode Die Daten, die von der Oberfläche abgerufen werden sollen (siehe Liste der Konstanten unten).
offset Der Datenversatzwert.
modulo Der Offset vom Ende der Zeile bis zum Anfang des nächsten


Rückkehr: N / A


Beschreibung

Mit dieser Funktion können Sie die Informationen von einer Oberfläche in einen bestimmten Puffer schreiben. Der Puffer muss zuvor erstellt worden sein und sollte ein 1 Byte großer Wachstumspuffer einer ungefähren Größe (in Bytes) der Oberfläche sein, die Sie schreiben wollen. Sie können den Kopier- "Modus" für die gespeicherten Daten wählen, der durch die unten aufgeführten Konstanten sowie die Offset- und Modulo-Werte eingestellt wird. Sowohl offset als auch modulo sind Bytewerte, wobei der Offset die Startposition innerhalb des Puffers definiert und modulo die Anzahl der am Ende jeder Zeile verbleibenden Bytes ist (zum Speichern zusätzlicher Daten). Beachten Sie, dass die Funktion jedes Pixel der Oberfläche mit einer BGRA- Formatierung auf dem Windows-Ziel in den Puffer schreibt, auf anderen Zielen jedoch je nach Betriebssystem oder sogar Gerät.

Konstante Beschreibung
0 Kopiert alle Daten von der Oberfläche.


Beispiel:

buffer_get_surface(buff, application_surface, 0, 0, 0);

Der obige Code kopiert die in der Anwendungsoberfläche gespeicherten Daten in den Puffer, der in der Variablen "buff" ohne Offset- oder Modulo-Werte indiziert ist.
