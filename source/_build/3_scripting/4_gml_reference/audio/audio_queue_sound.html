audio_queue_sound


Beschreibung

Diese Funktion fügt die Daten aus einem Puffer in die Audiowarteschlange ein, die Sie zuvor mit der Funktion erstellt haben audio_create_play_queue. Sie geben den hinzuzufügenden Warteschlangenindex und die Puffer-ID zum Abrufen der Daten sowie die Position (Offset) innerhalb des Puffers und die Anzahl der hinzuzufügenden Bytes an. Nachdem Sie Audio aus einem Puffer zu einer Warteschlange hinzugefügt haben, kann der Puffer nicht gelöscht werden, bis Sie die Warteschlange zuerst mithilfe der Funktion freigegeben haben audio_free_play_queue, und die Puffereigenschaften sollten mit denen der Warteschlange übereinstimmen, der Sie den Sound hinzufügen.

WICHTIG! Diese Funktion ist mit der Testlizenz des Produkts nicht verfügbar.


Syntax:

audio_queue_sound(queueIndex, bufferId, bufferOffset, bufferLength);


Streit Beschreibung
queueIndex Der Index der hinzuzufügenden Warteschlange.
bufferId Der Puffer, der zur Warteschlange hinzugefügt werden soll.
bufferOffset Der Offset innerhalb des Quellpuffers, von dem aus gestartet werden soll.
bufferLength Die Länge des Puffers (die Anzahl der Bytes im Puffer).


Kehrt zurück:

N/A


Beispiel:

len = async_load[? "data_len"];
audio_buff = buffer_create(len, buffer_fast, 1);
buffer_copy(async_load[? "buffer_id"], 0, len, buff, 0);
audio_queue_sound(audio_queue, audio_buff, 0, len);
audio_play_sound(audio_queue, 10, 0);

Der obige Code wird im asynchronen Audio Recording- Ereignis aufgerufen und weist einige aufgezeichnete Daten einem Puffer zu, der dann zu einer Audio-Warteschlange hinzugefügt wird. Dies wird dann gespielt.
