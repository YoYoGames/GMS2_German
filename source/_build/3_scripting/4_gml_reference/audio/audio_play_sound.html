audio_play_sound


Beschreibung

Mit dieser Funktion können Sie beliebige Klangressourcen in Ihrem Spiel spielen. Sie stellen den Klangindex zur Verfügung und weisen ihm eine Priorität zu, die dann verwendet wird, um zu bestimmen, wie mit Klängen umgegangen wird, wenn die Anzahl der zu spielenden Klänge über dem von der Funktion gesetzten Grenzwert liegt audio_channel_num. Töne mit niedrigerer Priorität werden zugunsten von Tönen mit höherer Priorität gestoppt, und der Prioritätswert kann eine beliebige reelle Zahl sein (der tatsächliche Wert ist beliebig und kann von 0 bis 1 oder 0 bis 100 sein, da GameMaker Studio 2 ihnen dieselbe Priorität einräumt ). Das letzte Argument besteht darin, die Sound-Loop zu erstellen und auf sie zu setzen true macht die Soundschleife bis zum Stoppen und Einstellen auf false spielt den Ton nur einmal ab.

Diese Funktion gibt auch eine eindeutige Indexnummer für den abgespielten Sound zurück, die dann in einer Variablen gespeichert werden kann, sodass Sie sie anhalten oder mit den entsprechenden Funktionen stoppen können. Dies bedeutet, dass Sie, wenn Sie mehrere Instanzen desselben Sounds gleichzeitig spielen, nur eine Instanz dieses Sounds verwenden können, um mit den Audiofunktionen zu arbeiten.


Spielt den angezeigten Sound ab.

Syntax:

audio_play_sound(index, priority, loop);


Streit Beschreibung
index Der Index des zu spielenden Sounds.
priority Stellen Sie die Kanalpriorität für den Sound ein.
loop Stellt den Sound auf Schleife ein oder nicht.


Kehrt zurück:

Index


Beispiel:

if health <= 0
   {
   lives -= 1;
   audio_play_sound(snd_PlayerDead, 10, false);
   }

Der obige Code überprüft die globale Variable "health" und wenn sie kleiner oder gleich 0 ist, wird 1 aus der globalen Variablen "lives" entfernt und ein Sound abgespielt.
