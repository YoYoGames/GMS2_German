audio_play_sound_on


Beschreibung

Mit dieser Funktion können Sie jede Tonquelle über einen Emitter spielen, wobei Änderungen an der Emitterverstärkung, Position, Tonhöhe oder Anschlagsgeschwindigkeit beeinflussen, wie der Benutzer den zuletzt gespielten Sound hört. Sie geben den zu verwendenden Emitter-Index an, den Sound-Index des abzuspielenden Sounds und legen dann fest, ob der Sound looping ist oder nicht. Schließlich können Sie eine Priorität zuweisen, die dann verwendet wird, um zu bestimmen, wie mit Klängen umgegangen wird, wenn die Anzahl der wiedergegebenen Klänge über dem von der Funktion gesetzten Grenzwert liegt audio_channel_num. Töne mit niedrigerer Priorität werden zugunsten von Tönen mit höherer Priorität gestoppt, und der Prioritätswert kann eine beliebige reelle Zahl sein (der tatsächliche Wert ist beliebig und kann von 0 bis 1 oder 0 bis 100 sein, da GameMaker Studio 2 ihnen dieselbe Priorität einräumt ).

Diese Funktion gibt auch eine eindeutige Indexnummer für den abgespielten Sound zurück, die dann in einer Variablen gespeichert werden kann, sodass Sie sie anhalten oder mit den entsprechenden Funktionen stoppen können. Dies bedeutet, dass Sie, wenn Sie mehrere Instanzen desselben Sounds gleichzeitig spielen, nur eine Instanz dieses Sounds verwenden können, um mit den Audiofunktionen zu arbeiten.


Syntax:

audio_play_sound_on(emitter, sound, loop, priority);


Streit Beschreibung
emitter Der Index des zu verwendenden Senders.
sound Der Index des zu verwendenden Sounds.
loop Kennzeichnet den Sound als Looping oder nicht.
priority Stellen Sie die Kanalpriorität für den Sound ein.


Kehrt zurück:

Index


Beispiel:

if global.SFX
   {
   audio_play_sound_on(s_emit, snd_Explode, false, 1);
   }

Der obige Code prüft die globale Variable "SFX" und ob sie zurückkommt true Dann wird der in der Variable "snd_Explode" indizierte Sound über den in der Variablen "s_emit" indizierten Emitter mit niedriger Priorität und ohne Looping abgespielt.
