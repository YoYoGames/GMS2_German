audio_sync_group_debug


Beschreibung

Mit dieser Funktion können Sie Debug-Informationen zu einer bestimmten Synchronisierungsgruppe anzeigen (der Gruppenindex ist der Wert, der zurückgegeben wurde, als die Gruppe mit der Funktion erstellt wurde) audio_create_sync_group() ). Sie können die Funktion mit dem Wert -1 aufrufen, um die Überlagerung auszuschalten.

Wenn Sie die Audio-Synchronisierungsgruppen debuggen, wird in der Überlagerung angezeigt, welche Synchronisierungsgruppen debuggen

  1. Wie viele Kanäle sind drin?
  2. Wie oft ist die Synchronisierungsgruppe ausgefallen (das bedeutet, dass versucht wurde, zu viel Arbeit zu erledigen)
  3. Eine Liste der Kanäle
    • Rot bedeutet, dass dieser Kanal derzeit keine Arbeit verrichtet
    • Gelb bedeutet, dass sich der Kanal im Anfangszustand befindet (noch nicht dekodiert oder abgespielt wird)
    • Grün bedeutet, dass der Titel gerade sucht / dekodiert
    • Lila bedeutet, dass dem Kanal die Daten ausgehen
  5. Eine Profilleiste, die zeigt, welche Arbeit der Decodierungsthread leistet
    • Blau bedeutet, dass eine Datei zum ersten Mal gelesen wird
    • Rot bedeutet, dass der Thread eine Datei liest, um herauszufinden, wo er lesen kann
    • Grün bedeutet, dass der Thread Daten decodiert
    • Lila bedeutet, dass der Thread im Ruhezustand ist, bis er funktionieren muss

Beachten Sie, dass Sie für das allgemeine Debugging von Audio auch die Funktion haben audio_debug().

HINWEIS: Diese Funktionalität ist für die HTML5-Zielplattform nicht verfügbar.


Syntax:

audio_sync_group_debug(group_index);


Streit Beschreibung
group_index Der zu debuggende Gruppenindex oder -1, um das Debugging auszuschalten.


Kehrt zurück:

N/A


Beispiel:

if (debug_mode)
   {
   audio_sync_group_debug(sg);
   }
else
   {
   audio_sync_group_debug(-1);
   }

Der obige Code schaltet das Debug-Overlay der Audio-Sync-Gruppe ein oder aus, das die in der Variablen "sg" indizierte Sync-Gruppe anzeigt, abhängig davon, ob das Spiel im Debug-Modus ausgeführt wird oder nicht.
