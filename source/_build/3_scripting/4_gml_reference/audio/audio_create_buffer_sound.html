audio_create_buffer_sound


Beschreibung

Mit dieser Funktion können Sie aus dem Inhalt eines Puffers einen neuen Sound erzeugen. Der Puffer wurde zuvor erstellt (Details hierzu finden Sie in den Pufferfunktionen ), und Daten wurden hinzugefügt oder geladen. Sie übergeben es dann an diese Funktion mit dem Datenformat (nur buffer_u8 oder buffer_s16 werden derzeit unterstützt), die Abtastrate (die zwischen 1000 Hz und 48000 Hz liegen kann) und ein Offset in den Puffer, um die Daten abzurufen. Sie müssen auch die Anzahl der Samples im Puffer und die Kanäle, die der Sound benötigt, angeben. Diese Kanäle sind durch eine der folgenden Konstanten definiert:

Konstante Beschreibung
audio_mono Mono (Einzelkanal) Audio.
audio_stereo Stereo (Zweikanal) -Audio.
audio_3D 3D (5.1) Audio.


Beachten Sie, dass Sie nach dem Erstellen eines Sounds den zugehörigen Zeigerindex freigeben sollten, wenn er nicht mehr benötigt wird audio_free_buffer_sound(). Wenn Sie dies nicht tun und dann die Variable neu zuweisen oder die Räume ändern usw., wird die Sound-ID verloren gehen und Sie werden ein Speicherleck haben. Beachten Sie auch, dass Sie den Puffer nicht löschen können, wenn ein Sound erstellt wurde und der Sound nicht zuerst freigegeben wurde. Sie würden zuerst den Sound (oder die Sounds) freigeben und dann den Puffer löschen. Es ist auch erwähnenswert, dass das Hinzufügen von irgendetwas zum Puffer oder das Ändern der Puffergröße, nachdem ein Sound erstellt wurde, zu unerwarteten Ergebnissen führt und nicht empfohlen wird - sobald Sie Sounds aus irgendeinem Puffer erstellt haben, sollten Sie nicht manipulieren in irgendeiner anderen Weise danach.

WICHTIG! Diese Funktion ist mit der Testlizenz des Produkts nicht verfügbar.


Syntax:

audio_create_buffer_sound(bufferId, bufferFormat, bufferRate, bufferOffset, bufferLength, bufferChannels);


Streit Beschreibung
bufferId Die ID des zu verwendenden Puffers.
bufferFormat Das Format der Daten im Puffer ( buffer_u8 oder buffer_s16 ).
bufferRate Die Abtastrate der Daten im Puffer.
bufferOffset Der Offset in den Puffer zum Lesen der Beispieldaten aus (in Bytes).
bufferLength Die Länge des Puffers (die Anzahl der Samples im Puffer).
bufferChannels Die Kanäle, die von einer der unten aufgeführten Konstanten verwendet werden sollen.


Kehrt zurück:

Index


Beispiel:

rate = 44100;
hertz = irandom_range(220, 880);
samples = 44100;
bufferId = buffer_create(rate, buffer_fast, 1);
buffer_seek(bufferId, buffer_seek_start, 0);
num_to_write = rate / hertz;
val_to_write = 1;
for (var i = 0; i < (samples / num_to_write) + 1; i++;)
   {
   for (var j = 0; j < num_to_write; j++;)
      {
      buffer_write(bufferId, buffer_u8, val_to_write * 255);
      }
   val_to_write = (1 - val_to_write);
   }
soundId = audio_create_buffer_sound(bufferId, buffer_u8, rate, 0, 44100, audio_mono);

Das obige erzeugt einen Puffer und füllt ihn dann prozedural mit Daten. Aus diesen Daten wird dann ein neuer Sound erzeugt, der in der Variable "soundID" gespeichert wird.
