audio_create_buffer_sound


Beschreibung

Mit dieser Funktion können Sie einen neuen Sound aus dem Inhalt eines Puffers erstellen. Der Puffer wurde zuvor erstellt (siehe dazu die Pufferfunktionen ) und es wurden Daten hinzugefügt oder geladen. Sie übergeben es dann mit dem Datenformat an diese Funktion (nur buffer_u8 oder buffer_s16 werden derzeit unterstützt), die Abtastrate (die zwischen 1000 Hz und 48000 Hz liegen kann) und einen Versatz in den Puffer, um die Daten abzurufen. Sie müssen auch die Anzahl der Samples im Puffer und die Kanäle angeben, die der Sound benötigt. Diese Kanäle werden durch eine der folgenden Konstanten definiert:

Konstante Beschreibung
audio_mono Mono (Einkanal) -Audio.
audio_stereo Stereo-Audio (Zweikanal)
audio_3D 3D (5.1) -Audio.


Beachten Sie, dass Sie nach dem Erstellen eines Sounds den zugehörigen Zeigerindex freigeben sollten, wenn er mit der Funktion nicht mehr benötigt wird audio_free_buffer_sound(). Wenn Sie dies nicht tun und dann die Variable erneut zuweisen oder Räume ändern usw., geht die Sound-ID verloren und Sie haben ein Speicherverlust. Beachten Sie auch, dass Sie den Puffer nicht löschen können, wenn zuvor ein Sound erstellt wurde und der Sound noch nicht freigegeben wurde. Sie geben also zuerst den Sound (oder die Sounds) frei und löschen dann den Puffer. Es ist auch erwähnenswert, dass das Hinzufügen von etwas zu dem Puffer oder das Ändern der Puffergröße, nachdem ein Sound daraus erzeugt wurde, unerwartete Ergebnisse liefert, und es wird nicht empfohlen - sobald Sie mit dem Erzeugen von Sounds aus einem Puffer begonnen haben, sollten Sie nicht manipulieren es auf andere Weise danach.

WICHTIG! Diese Funktion ist nicht mit der Testlizenz des Produkts verfügbar.


Syntax:

audio_create_buffer_sound(bufferId, bufferFormat, bufferRate, bufferOffset, bufferLength, bufferChannels);


Streit Beschreibung
bufferId Die ID des zu verwendenden Puffers.
bufferFormat Das Format der Daten im Puffer ( buffer_u8 oder buffer_s16 ).
bufferRate Die Abtastrate der Daten im Puffer.
bufferOffset Der Offset in den Puffer, von dem die Probedaten gelesen werden sollen (in Bytes).
bufferLength Die Länge des Puffers (Anzahl der Bytes der Audiodaten, ohne Kopfzeile).
bufferChannels Die zu verwendenden Kanäle von einer der unten aufgeführten Konstanten.


Kehrt zurück:

Index


Beispiel:

var rate = 44100;
var hertz = irandom_range(220, 880);
var samples = 44100;
var bufferId = buffer_create(rate, buffer_fast, 1);
var buffer_seek(bufferId, buffer_seek_start, 0);
var num_to_write = rate / hertz;
var length = buffer_get_size(bufferId) ;
var val_to_write = 1;
for (var i = 0; i < (samples / num_to_write) + 1; i++;)
   {
   for (var j = 0; j < num_to_write; j++;)
      {
      buffer_write(bufferId, buffer_u8, val_to_write * 255);
      }
   val_to_write = (1 - val_to_write);
   }
soundId = audio_create_buffer_sound(bufferId, buffer_u8, rate, 0, length, audio_stereo);

Das Obige erstellt einen Puffer und füllt ihn dann prozedural mit Daten. Diese Daten werden dann verwendet, um einen neuen Sound zu erzeugen, der in der Variablen "soundId" gespeichert wird.
