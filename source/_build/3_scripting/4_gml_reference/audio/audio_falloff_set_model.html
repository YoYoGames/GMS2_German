audio_falloff_set_model


Beschreibung

Um der Audio-Engine mehr Vielseitigkeit zu verleihen, können Sie in GameMaker Studio 2 das zu Ihrem Spiel passende Falloff-Modell auswählen. Dieses Modell wird für alle Audiofunktionen im Spiel oder in der App verwendet. Daher sollten Sie sicherstellen, dass das von Ihnen gewählte Modell das richtige ist, da jedes davon beeinflusst, wie der Zuhörer die Töne wahrnimmt, die Sie durch die Emitter oder mit dem Funktion audio_play_sound_at.

Wenn Sie das Spiel mit den entsprechenden Funktionen absenken, müssen Sie drei Argumente festlegen, die jeweils für ein bestimmtes Modell geeignet sind und die Art und Weise beeinflussen, in der der letzte Sound vom Spieler je nach Entfernung "gehört" wird dass der Hörer von der Quelle kommt. Die drei Argumente sind:

  • Referenzentfernung - das ist die Entfernung vom Zuhörer, die Entfernung, unter der die Lautstärke für den Klang normalerweise um die Hälfte abnimmt, bevor sie durch den Rolloff-Faktor oder die angegebene maximale Entfernung beeinflusst wird.

  • maximale Entfernung - hier wird die Entfernung eingestellt, bei der keine Dämpfung des Quellentons mehr auftritt. Dies kann der Punkt sein, an dem der Ton nicht mehr gehört wird oder der Punkt, an dem die Lautstärke nicht mehr unter den vom gewählten Modell definierten Mindestschwellenwert absinkt.

  • Falloff-Faktor - Der Falloff-Faktor wird in der Distanzdämpfung basierend auf dem Inversdistanzmodell verwendet und legt den endgültigen Mindestschwellenwert für einen Sound mit Falloff fest.


Die Falloff-Modelle, die von diesen Argumenten betroffen sind, werden in GameMaker Studio 2 durch die folgenden Konstanten dargestellt (die Tabelle zeigt auch die genauen Berechnungen):

Konstante Funktion
audio_falloff_exponent_distance
gain = (listener_distanz / reference_distance) ^ (-falloff_factor)

audio_falloff_exponent_distance_clamped
distance = clamp (listener_distanz, reference_distance, maximum_distance)
gain = (distance / reference_distance) ^ (-falloff_factor)

audio_falloff_inverse_distance
gain = reference_distance / (reference_distance + falloff_factor * (listener_distance - reference_distance))

audio_falloff_inverse_distance_clamped
distance = clamp (listener_distanz, reference_distance, maximum_distance)
gain = reference_distance / (reference_distance + falloff_factor * (Abstand - reference_distance))

audio_falloff_linear_distance
Abstand = min (Abstand, maximaler Abstand)
gain = (1 - falloff_factor * (distance - reference_distance) / (maximum_distance - reference_distance))

audio_falloff_linear_distance_clamped
distance = clamp (listener_distanz, reference_distance, maximum_distance)
gain = (1 - falloff_factor * (distance - reference_distance) / (maximum_distance - reference_distance))

audio_falloff_none
Gewinn = 1


Die folgenden Grafiken zeigen die Funktionsweise der oben genannten Konstanten und beeinflussen den gespielten Sound:


Syntax:

audio_falloff_set_model(model);


Streit Beschreibung
model Die Konstante, mit der das Falloff-Modell festgelegt wurde.


Kehrt zurück:

N/A


Beispiel:

audio_falloff_set_model(audio_falloff_exponent_distance_clamped);
audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1);

Der obige Code legt das Falloff-Modell fest und spielt dann den in der Variablen "snd_Waterfall" indexierten Sound ab, der an seiner Raumposition mit einer Abfallreferenz von 100, einer Falloff-Distanz von 300, einem Abfallfaktor von 1 und eine niedrige Priorität.
