audio_falloff_set_model


Beschreibung

Um die Audio-Engine noch vielseitiger zu machen, können Sie in GameMaker Studio 2 das für Ihr Spiel geeignete Falloff-Modell auswählen. Dieses Modell wird für alle Audiofunktionen im Spiel oder in der App verwendet. Sie sollten daher sicherstellen, dass das von Ihnen gewählte Modell das richtige ist, da jedes davon beeinflusst, wie der Hörer die Sounds wahrnimmt, die Sie durch Emitter oder mit dem Emulator spielen Funktion audio_play_sound_at.

Wenn Sie in Ihrem Spiel mit den entsprechenden Funktionen Absenkungen einstellen, gibt es drei Argumente, die Sie einstellen müssen. Jeder ist für ein bestimmtes Modell geeignet und beeinflusst die Art und Weise, wie der Endklang vom Spieler "gehört" wird, je nach Entfernung dass der Listener von der Quelle stammt. Die drei Argumente sind:

  • Referenzentfernung - Dies ist die Entfernung vom Hörer, um die die Lautstärke für den gespielten Sound normalerweise um die Hälfte abnehmen würde, bevor sie vom Rolloff-Faktor oder der angegebenen maximalen Entfernung beeinflusst wird.

  • Maximaler Abstand - Hiermit wird der Abstand festgelegt, bei dem keine Dämpfung des Quelltons mehr auftritt. Dies kann der Punkt sein, an dem der Ton nicht mehr gehört wird, oder der Punkt, an dem die Lautstärke nicht mehr unter den durch das gewählte Modell festgelegten Mindestschwellenwert absinkt.

  • Falloff-Faktor - Der Falloff-Faktor wird bei der Abstandsdämpfung auf der Grundlage des inversen Abstandsmodells verwendet und legt den endgültigen Mindestschwellenwert für einen Klang mit Abfall fest.


Die von diesen Argumenten betroffenen Falloff-Modelle werden in GameMaker Studio 2 durch die folgenden Konstanten dargestellt (die Tabelle zeigt auch die genauen Berechnungen):

Konstante Funktion
audio_falloff_exponent_distance
gain = (listener_distance / reference_distance) ^ (-falloff_factor)

audio_falloff_exponent_distance_clamped
Abstand = Klemme (Listenerabstand, Referenzabstand, Maximalabstand)
Gewinn = (Entfernung / Referenzabstand) ^ (-falloff_factor)

audio_falloff_inverse_distance
Gain = Referenzabstand / (Referenzabstand + Fallofffaktor * (Listenerabstand? Referenzabstand))

audio_falloff_inverse_distance_clamped
Abstand = Klemme (Listenerabstand, Referenzabstand, Maximalabstand)
Verstärkung = Referenzabstand / (Referenzabstand + Falloff-Faktor * (Abstand? Referenzabstand))

audio_falloff_linear_distance
Abstand = min (Abstand, maximaler Abstand)
Gewinn = (1 Falloff_Factor * (Abstand Referenzabstand) / (Maximalabstand Referenzwert))

audio_falloff_linear_distance_clamped
Abstand = Klemme (Listenerabstand, Referenzabstand, Maximalabstand)
Gewinn = (1 Falloff_Factor * (Abstand Referenzabstand) / (Maximalabstand Referenzwert))

audio_falloff_none
Gewinn = 1


Die folgenden Grafiken zeigen visuell, wie die obigen Konstanten funktionieren, und wirken sich auf den abgespielten Sound aus:


Syntax:

audio_falloff_set_model(model);


Streit Beschreibung
model Die Konstante, mit der das Falloff-Modell festgelegt wurde.


Kehrt zurück:

N/A


Beispiel:

audio_falloff_set_model(audio_falloff_exponent_distance_clamped);
audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1);

Der obige Code legt das Falloff-Modell fest und spielt dann den in der Variablen "snd_Waterfall" indizierten Sound ab, der an seiner Raumposition mit einem Abfallreferenzwert von 100, einem Abfallabstand von 300, einem Abfallfaktor von 1 und 1 durchlaufen wird eine niedrige Priorität.
