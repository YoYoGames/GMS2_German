audio_play_sound_at


Beschreibung

Mit dieser Funktion können Sie jede Tonquelle an einer bestimmten Position im Audioraum abspielen. Sie stellen den Sound-Index zur Verfügung und weisen ihm dann eine Position im 3D-Raum zu. Die Standard-Listener-Position ist (0, 0, 0), dh wenn der Listener nicht bewegt wurde und Sie wollen, dass der Sound von links kommt (zB), sollten Sie die x-Position auf einen negativen Wert setzen (für Weitere Informationen zum Einstellen der Hörerposition finden Sie unter audio_listener_position ).

Sie können auch einen Abfallabstand einstellen (0 wird den Ton stummschalten, der Standardwert ist 100), wodurch der Ton ausgeblendet wird, wenn er weiter von der Hörerposition entfernt ist. Wie die Überblendung selbst gehört wird, hängt von der Abnahmereferenz (der Abstand, unter dem die Lautstärke für die Quelle normalerweise um die Hälfte abnimmt) und dem Roll-Off-Faktor ab (der nur den Ton beeinflusst, der über die Referenzentfernung fällt). Der Standardfaktor ist normalerweise 1, und die Auswirkung der verschiedenen Falloff-Werte hängt vom gewählten Modell ab (für eine vollständige Anleitung zu den verschiedenen Falloff-Modellen und wie diese Werte verwendet werden, sehen Sie bitte die Funktion audio_falloff_set_model ).

Die letzten zwei Argumente dienen dazu, den Sound auf loop oder nicht zu setzen und schließlich dem Sound eine Priorität zuzuordnen. Diese Priorität wird dann verwendet, um zu bestimmen, wie mit Klängen umgegangen wird, wenn die Anzahl der wiedergegebenen Klänge über dem durch die Funktion gesetzten Grenzwert liegt audio_channel_num. Töne mit niedrigerer Priorität werden zugunsten von Tönen mit höherer Priorität gestoppt, und der Prioritätswert kann eine beliebige reelle Zahl sein (der tatsächliche Wert ist beliebig und kann von 0 bis 1 oder 0 bis 100 sein, da GameMaker Studio 2 ihnen dieselbe Priorität einräumt ).

Diese Funktion gibt eine eindeutige Indexnummer für den abgespielten Sound zurück, die dann in einer Variablen gespeichert werden kann, sodass Sie sie anhalten oder mit den entsprechenden Funktionen stoppen können. Dies bedeutet, dass Sie, wenn Sie mehrere Instanzen desselben Sounds gleichzeitig spielen, nur eine Instanz dieses Sounds verwenden können, um mit den Audiofunktionen zu arbeiten.


Syntax:

audio_play_sound_at(index, x, y, z, falloff_ref, falloff_max, falloff_factor, loop, priority);


Streit Beschreibung
index Der Index des zu spielenden Sounds.
x Die x-Position.
y Die y-Position.
z Die z-Position.
falloff_ref Die Abnahmereferenz relativ zum Zuhörer (Klemme).
falloff_max Die maximale Abfallentfernung relativ zum Hörer.
falloff_factor Der Falloff-Faktor (Standard 1).
loop Kennzeichnet den Sound als Looping oder nicht.
priority Stellen Sie die Kanalpriorität für den Sound ein.


Kehrt zurück:

Index


Beispiel:

if global.SFX
   {
   audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1);
   }

Der obige Code prüft die globale Variable "SFX" und ob sie zurückkommt true dann wird der in der Variablen "snd_Waterfall" indexierte Klang an seiner Raumposition mit einer Abfallreferenz von 100, einer Abfallentfernung von 300, einem Abfallfaktor von 1 und einer niedrigen Priorität wiederholt.
