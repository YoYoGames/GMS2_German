audio_start_recording


Beschreibung

Diese Funktion startet die Aufnahme von Audio von der indexierten Recorderquelle. Mit dieser Funktion können Sie die Anzahl der Recorderquellen ermitteln audio_get_recorder_count(), und sobald Sie mit der Aufnahme beginnen, wird das Audio in einem temporären Puffer gespeichert und startet ein Audio Recording Asynchronous Event. Dieses Ereignis wird bei jedem Schritt der Aufzeichnung ausgelöst und erstellt das Besondere ds_map in der Variable async_load mit den folgenden Schlüssel / Wert-Paaren:

  • "buffer_id" - die ID des temporären Puffers, mit dem Sie die Audiodaten abrufen können
  • "channel_index" - der Aufzeichnungskanalindex (wie von der Funktion zurückgegeben), von dem diese Daten stammen
  • "data_len" - die Länge der Daten (in Bytes), die Sie erhalten haben

Beachten Sie, dass nach dem Auslösen des asynchronen Ereignisses die Audiodaten im temporären Puffer gelöscht werden. Daher sollten Sie die Daten in einen benutzerdefinierten Puffer kopieren, den Sie zuvor erstellt haben.

HINWEIS: Die meisten Plattformen unterstützen die Aufnahme von Audio in irgendeiner Form, aber das bedeutet nicht, dass alle Geräte dies zulassen, selbst wenn die Plattform dies tut audio_get_recorder_count() Die Funktion gibt einen Wert größer als 0 zurück, um zu überprüfen, ob Aufzeichnungsgeräte verfügbar sind, bevor die übrigen Aufzeichnungsfunktionen verwendet werden.


Syntax:

audio_start_recording(recorder_index);


Streit Beschreibung
recorder_index Der Index der zu verwendenden Recorderquelle.


Kehrt zurück:

Index


Beispiel:

audio_record = audio_start_recording(0);

Der obige Code startet die Aufzeichnung von der Recorderquelle 0 und speichert den Kanalindex der Aufzeichnung in der Variablen "audio_record" zur Verwendung in dem asynchronen Audioaufnahmeereignis.
