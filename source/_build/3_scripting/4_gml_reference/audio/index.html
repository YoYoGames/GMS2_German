Audio


GameMaker Studio 2 verfügt über eine vollständige Audio-Engine, die auf der basiert *.ogg. *.mp3 und *.wav Tonformate. Sounds dieser Typen, die der IDE hinzugefügt wurden, können dann mit den unten aufgeführten Audiofunktionen in Ihrem Spiel verwendet werden. Für komplexere Dinge als grundlegende Soundeffekte oder das Abspielen eines einzelnen Musikstücks können Sie sich auf die erweiterten Audiofunktionen beziehen, mit denen Sie die Wiedergabe eines Sounds ändern können. Es gibt auch eine Reihe spezialisierter Funktionen für das Streaming von Audio, das Positionieren von Audio - für 3D-Sound - und für das Gruppieren von Audio.

WICHTIG! Wenn Sie Audio auf dem HTML5-Ziel verwenden, sollten Sie beachten, dass nicht alle Browser Web Audio unterstützen und daher bei der Ausführung möglicherweise kein Sound für Ihr Projekt abgespielt wird. Einen allgemeinen Überblick über die Unterstützung von Web Audio erhalten Sie unter dem folgenden Link: Kann ich Web Audio verwenden?.


Die folgenden Funktionen dienen dem einfachsten und einfachsten Umgang mit Audio, der Verwendung von Emittern und dem Benutzer, Sounds einfach zu erzeugen und Musik zu spielen, da diese Sounds immer an der Hörerposition erzeugt werden der Listener ) und sind daher im Allgemeinen nicht von Änderungen am Listener betroffen:

  • audio_exists
  • audio_get_name
  • audio_get_type
  • audio_play_sound
  • audio_play_sound_at
  • audio_pause_sound
  • audio_pause_all
  • audio_resume_sound
  • audio_resume_all
  • audio_stop_sound
  • audio_stop_all
  • audio_is_playing
  • audio_is_paused
  • audio_create_stream
  • audio_destroy_stream

Die folgenden Funktionen geben mehr Kontrolle darüber, wie die Audio-Engine funktioniert und wie die durch sie abgespielten Klänge vom Zuhörer "gehört" werden. Daher wird empfohlen, dass Sie gute Kenntnisse über die Funktionsweise des Rest der GameMaker Studio 2- Audio-Engine haben, bevor Sie eine der folgenden Funktionen verwenden:

  • audio_sound_set_track_position
  • audio_sound_get_track_position
  • audio_sound_length
  • audio_sound_pitch
  • audio_sound_get_pitch
  • audio_falloff_set_model
  • audio_sound_gain
  • audio_sound_get_gain
  • audio_master_gain
  • audio_set_master_gain
  • audio_get_master_gain
  • audio_channel_num
  • audio_debug

Weitere erweiterte Funktionen, wie das Einstellen der Hörer, das Aufnehmen von Audio oder das Synchronisieren mehrerer Audiospuren über einen längeren Zeitraum, finden Sie in den folgenden Unterabschnitten:

  1. Audio-Emitter
  2. Audio-Hörer
  3. Audiogruppen
  4. Audiopuffer
  5. Audiosynchronisation
