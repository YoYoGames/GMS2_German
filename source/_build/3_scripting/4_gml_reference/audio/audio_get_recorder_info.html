audio_get_recorder_info


Beschreibung

Diese Funktion gibt ein zurück ds_map mit Informationen zum angegebenen Rekorder-Quellindex. Sie können mit der Funktion herausfinden, wie viele Recorder-Quellen verfügbar sind audio_get_recorder_count und die zurückgegebene Karte enthält die folgenden Schlüssel / Wert-Paare:

  • "name" - ein Name, der das Gerät beschreibt
  • "index" - der für die Aufzeichnung verwendete Index
  • "data_format" - Die Formatdaten werden zurückgegeben (dies ist derzeit immer der Fall) buffer_s16 Andere Formate werden jedoch in der Zukunft möglicherweise unterstützt.
  • "sample_rate" - die Abtastrate (in Hz) der zurückgegebenen Daten (derzeit auf 16000 Hz geklemmt, dies kann sich jedoch in Zukunft ändern)
  • "channels" - die Konstante audio_mono (weitere Konstanten für Stereo und 3D werden möglicherweise in der Zukunft unterstützt)

Beachten Sie, dass während der Funktion ein erstellt wird ds_map Für Sie wird sie später nicht mehr entfernt. Sie sollten also die Karte selbst zerstören, wenn sie nicht mehr benötigt wird, um Speicherlecks zu vermeiden.

ANMERKUNG: Die meisten Plattformen mit Ausnahme von HTML5 unterstützen die Aufnahme von Audio in irgendeiner Form. Dies bedeutet jedoch nicht, dass alle Geräte dies zulassen, auch wenn die Plattform dies tut audio_get_recorder_count() function gibt einen Wert größer als 0 zurück, um zu überprüfen, ob Aufnahmegeräte verfügbar sind, bevor die restlichen Aufnahmefunktionen verwendet werden.


Syntax:

audio_get_recorder_info(recorder_index);


Streit Beschreibung
recorder_index Der Index der Recorderquelle, von der die Informationen abgerufen werden.


Kehrt zurück:

ds_map


Beispiel:

len = async_load[? "data_len"];
audio_buff = buffer_create(len, buffer_fast, 1);
buffer_copy(async_load[? "buffer_id"], 0, len, buff, 0);
audio_queue_sound(audio_queue, audio_buff, 0, len);
audio_play_sound(audio_queue, 10, 0);

Der obige Code würde im asynchronen Audio-Recording- Ereignis aufgerufen und weist einige aufgezeichnete Daten einem Puffer zu, der dann einer Audiowarteschlange hinzugefügt wird. Dies wird dann gespielt.
