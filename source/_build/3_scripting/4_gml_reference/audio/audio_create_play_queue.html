audio_create_play_queue


Beschreibung

Diese Funktion bereitet eine Pufferwarteschlange für Audio vor. Sie übergeben das Datenformat (nur buffer_u8 oder buffer_s16 werden derzeit unterstützt), die Abtastrate (die zwischen 1000 Hz und 48000 Hz liegen kann) und die Kanäle, die der Ton von einer der folgenden Konstanten benötigt:

Konstante Beschreibung
audio_mono Mono (Einzelkanal) Audio.
audio_stereo Stereo (Zweikanal) -Audio.
audio_3D 3D (5.1) Audio.


Beachten Sie, dass Sie alle mit dieser Funktion erstellten Audiowarteschlangen freigeben sollten, wenn sie nicht mehr benötigt werden, um Speicherlecks mithilfe der Funktion zu verhindern audio_free_play_queue().

Diese Funktion löst ein Asynchrones Audiowiedergabeereignis aus, wenn die Warteschlange das Ende erreicht hat, und in diesem Fall ein spezielles Ereignis ds_map wird in der Variable erstellt async_load mit den folgenden Schlüssel / Wert-Paaren:

  • "queue_id" - die Warteschlange, die die Datenwiedergabe beendet hat
  • "buffer_id" - der Puffer, von dem nicht mehr gespielt wird
  • "queue_shutdown" - wird während der normalen Wiedergabe auf 0 gesetzt und 1, wenn das Ereignis empfangen wird audio_free_play_queue genannt worden. Wenn dies auf 1 gesetzt ist, möchten Sie keine weiteren Daten in die Warteschlange stellen.
WICHTIG! Diese Funktion ist mit der Testlizenz des Produkts nicht verfügbar.


Syntax:

audio_create_play_queue(queueFormat, queueRate, queueChannels);


Streit Beschreibung
queueFormat Das Format der zu verwendenden Pufferdaten ( buffer_u8 oder buffer_s16 ).
queueRate Die Abtastrate der Daten in der Pufferwarteschlange.
queueChannels Die Kanäle, die von einer der unten aufgeführten Konstanten verwendet werden sollen.


Kehrt zurück:

Index


Beispiel:

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono);

Der obige Code erstellt eine neue Audio-Queue und ordnet ihren Index einer Variablen zur späteren Referenz zu.
