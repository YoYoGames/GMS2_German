audio_debug


Beschreibung

Diese Funktion kann verwendet werden, um Debug-Informationen über das Audiosystem mit anzuzeigen true es einschalten und false um es auszuschalten. Wenn Sie Audio auf diese Weise debuggen, zeigt das Overlay die folgenden Informationen an:

  1. Handle - Der OpenAL-Source-Handle (einzigartig für jeden gespielten Sound)
  2. Data Handle - Der OpenAL-Puffer-Handle (zeigt denselben Wert für mehrere Sounds mit demselben Quell-Handle)
  3. Verarbeitete Puffer - wie viele Puffer wurden gespielt
  4. In Warteschlange gestellte Puffer - wie viele Puffer wurden in die Warteschlange gestellt (und warten noch auf die Wiedergabe)
  5. Verstärkung - Der Verstärkungswert für den Ton (0 - 65536)
  6. Play Offset - die Probenposition (aktuelle Wiedergabeposition) in den aktuellen Puffer
  7. Queued Buffers Bars - ein kleines Boxdiagramm der verarbeiteten / eingereihten Puffer mit dem Wiedergabeversatz (aktuelle Position) als blaue Linie gezeichnet

Für synchronisierte Audiogruppen haben Sie auch die Funktion audio_sync_group_debug().

HINWEIS: Diese Funktion funktioniert nicht auf der HTML5-Zielplattform.


Syntax:

audio_debug(enable);


Streit Beschreibung
enable Aktivieren (auf festlegen true ) oder deaktivieren ( false ) Audio-Debugging.


Kehrt zurück:

N/A


Beispiel:

if (debug_mode)
   {
   audio_debug(true);
   }
else
   {
   audio_debug(false);
   }

Der obige Code schaltet das Audio-Debug-Overlay je nachdem, ob das Spiel im Debug-Modus läuft oder nicht, ein oder aus.
