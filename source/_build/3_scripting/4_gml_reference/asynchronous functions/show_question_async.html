show_question_async


Beschreibung

Diese Funktion öffnet ein Fenster und zeigt dem Benutzer die Frage an, die Sie in der Funktion definiert haben. Dies ist eine asynchrone Funktion, und so blockiert GameMaker Studio 2 nicht das Gerät, auf dem es ausgeführt wird, während auf Antwort gewartet wird, sondern läuft weiter wie gewohnt. Die Funktion verfügt über zwei Tasten, die mit „Ja“ zeigen und „Nein“, und sobald der Benutzer einer gedrückt, ein asynchrones Dialog Ereignis wird ausgelöst, die für die Dauer des Ereignisses nur eine ds_map in den Variablen gespeichert haben async_load.

Diese Karte enthält die zwei Schlüssel "id" und "status". "id" ist der Wert, der von der Funktion beim Aufruf zurückgegeben wurde, während der "status" entweder der Wert ist true oder false für "Ja" bzw. "Nein".


Syntax:

show_question_async(string);

Streit Beschreibung
String Die Frage an den Benutzer zu stellen.


Kehrt zurück:

Real


Erweitertes Beispiel:

Das linke Mausereignis (zum Beispiel) des Objekts, das die Nachricht anzeigt, würde den folgenden Code haben:

msg = show_question_async("Do you want to buy some armour for " + string(armour[0, 5]) + "coins?");

Das obige zeigt eine Frage mit der gegebenen Zeichenkette, die anfordert, dass der Benutzer entweder "Ja" oder "Nein" drückt. Die Funktions-ID ist in der Variablen "msg" gespeichert und wird im asynchronen Dialog " Dialoge " wie folgt verwendet:

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if i_d == msg
   {
   if ds_map_find_value(async_load, "status")
      {
      coins -= armour[0,5];
      global.protection += armour[0,0];
      }
   }

Der obige Code überprüft die "ID" -Taste des zurückgegebenen ds_map gegen den in der Variable "msg" gespeicherten Wert. Wenn sie gleich sind, prüft sie, ob eine der beiden Tasten gedrückt wurde und ob sie zurückkehrt true Es zieht dann einen Wert von einer Variablen ab und fügt einen Wert auch einer globalen Variablen hinzu.
