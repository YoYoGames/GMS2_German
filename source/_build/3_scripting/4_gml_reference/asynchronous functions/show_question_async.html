show_question_async


Beschreibung

Diese Funktion öffnet ein Fenster und zeigt dem Benutzer die in der Funktion definierte Frage an. Dies ist eine asynchrone Funktion. Daher blockiert GameMaker Studio 2 nicht das Gerät, auf dem es ausgeführt wird, während es auf Antwort wartet, sondern führt Ereignisse weiterhin normal aus. Die Funktion hat zwei Schaltflächen, die "Ja" und "Nein" anzeigen. Sobald der Benutzer eine gedrückt hat, wird ein asynchrones Dialogereignis ausgelöst, bei dem nur für die Dauer dieses Ereignisses eine ds_map in der Variablen gespeichert ist async_load.

Diese Map enthält die beiden Schlüssel " id " und " status ". "id" ist der Wert, der von der Funktion beim Aufruf zurückgegeben wurde. Der "Status" ist einer der beiden true oder false für "Ja" bzw. "Nein".


Syntax:

show_question_async(string);

Streit Beschreibung
String Die Frage an den Benutzer.


Kehrt zurück:

Real


Erweitertes Beispiel:

Das Ereignis mit der linken Maustaste (beispielsweise) des Objekts, das die Nachricht anzeigt, hätte folgenden Code:

msg = show_question_async("Do you want to buy some armour for " + string(armour[0, 5]) + "coins?");

Das Obige zeigt eine Frage mit der angegebenen Zeichenfolge und fordert den Benutzer auf, entweder "Ja" oder "Nein" zu drücken. Die Funktions-ID wird in der Variablen "msg" gespeichert und im Ereignis asynchronous Dialogs wie folgt verwendet:

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if i_d == msg
   {
   if ds_map_find_value(async_load, "status")
      {
      coins -= armour[0,5];
      global.protection += armour[0,0];
      }
   }

Der obige Code überprüft den "id" -Schlüssel der zurückgegebenen ds_map gegen den in der Variablen "msg" gespeicherten Wert. Wenn sie gleich sind, wird geprüft, ob eine der beiden Tasten gedrückt wurde und ob sie zurückkehrt true Es zieht dann einen Wert von einer Variablen ab und fügt auch einen Wert zu einer globalen Variablen hinzu.
