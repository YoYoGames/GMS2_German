http_request


Beschreibung

Mit dieser Funktion können Sie eine HTTP-Header-Anfrage erstellen, um die Betriebsparameter einer HTTP-Transaktion zu definieren, die für viele Dinge verwendet werden kann (zB Authentifizierung über HTTP-Header, wenn Sie RESTful-APIs verwenden). Die Funktion benötigt die vollständige IP-Adresse des Servers, von der angefordert werden soll, sowie die Art der Anforderung (als Zeichenfolge, siehe Hinweis unten): "GET", "HEAD", "POST", "PUT", " LÖSCHEN "," TRACE "," OPTIONEN "oder" CONNECT ". Sie müssen auch einen liefern ds_map von Schlüssel / Wert-Paaren (als Strings, wobei der Schlüssel das Header-Feld und der Wert die erforderlichen Daten für den Header sind), und das letzte Argument ist eine optionale Datenzeichenfolge, die Sie der Anfrage hinzufügen können, und wenn sie nicht benötigt wird dann kann es entweder 0 oder eine leere Zeichenfolge "" sein. Beachten Sie, dass Sie auch einen Puffer senden können (weitere Informationen finden Sie im Abschnitt Puffer ). In diesem Fall wäre das letzte Argument das "Handle" des zu sendenden Puffers.

Dieses Ereignis wird einen "Rückruf" generieren, der von irgendwelchen HTTP-Ereignissen empfangen wird, in welchem ​​Fall er ein erzeugt ds_map (besser bekannt als "Wörterbuch"), das exklusiv für dieses Ereignis ist und in der speziellen Variablen async_load gespeichert ist. Diese ds_map weist die folgenden zwei Schlüssel auf, die sich auf die Anforderungsfunktion beziehen:

  • ID: Die ID, die vom Befehl zurückgegeben wurde. Wenn Sie eine Reihe von abfeuern http_ Anfragen, dann müssen Sie wissen, auf welche Sie die Antwort bekommen, und so würden Sie diesen Wert verwenden, um mit dem Wert zu vergleichen, den Sie speicherten, als Sie ursprünglich die Anfrage schickten, um die richtige zu finden.

  • response_headers: Wenn dieser Wert größer oder gleich 0 ist, enthält er eine ds_map, die die HTTP-Header enthält, die mit der Antwort auf die HTTP-Anforderung zurückgegeben werden.

HINWEIS: HTTP-Header folgen normalerweise dem Format "Schlüssel: Wert", aber da GMS diese Paare für Sie aus der ds_map erstellt, müssen Sie den Doppelpunkt ":" nicht in Ihren Map-Schlüssel oder Wertstrings einfügen.
HINWEIS: Sie sollten wissen, dass aufgrund von XSS-Schutz in Browsern Anforderungen an und Versuche, Ressourcen aus verschiedenen Domänen zu laden, blockiert werden und möglicherweise leere Ergebnisse zurückgeben. Weitere Informationen finden Sie im Abschnitt Domänenübergreifende Probleme.


Syntax:

http_request(url, method, header_map, body);

Streit Beschreibung
url Die Webadresse des Servers, von dem Sie Informationen erhalten möchten
method Die Anfrage-Methode (normalerweise "POST" oder "GET", aber alle Methoden werden unterstützt)
header_map Eine ds_map mit den erforderlichen Header-Feldern
body Die Daten, die nach den Headern übertragen werden (kann ein binärer Buffer-Handle sein)
