get_string_async


Beschreibung

Diese Funktion öffnet ein Fenster und zeigt eine Meldung sowie einen Bereich an, in dem der Benutzer eine Zeichenfolge eingeben kann (die die angegebene Standardzeichenfolge enthält, mit der begonnen werden soll). Dies ist eine asynchrone Funktion. Daher blockiert GameMaker Studio 2 nicht das Gerät, auf dem es ausgeführt wird, während es auf Antwort wartet, sondern führt Ereignisse weiterhin normal aus. Nachdem der Benutzer die Zeichenfolge eingegeben und die Schaltfläche "Okay" gedrückt hat, wird ein asynchrones Dialogereignis ausgelöst, bei dem nur für die Dauer dieses Ereignisses eine ds_map in der Variablen gespeichert wird async_load.

Diese Map enthält die drei Schlüssel " id ", " status " und " result ". " id " ist der Wert, der von der Funktion beim Aufruf zurückgegeben wurde, der " status " ist einer der beiden true für das Drücken der Taste "OK" oder false wenn die Nachricht abgebrochen wurde (falls dies nicht bei allen Zielplattformen zulässig ist). Schließlich gibt " result " die Zeichenfolge zurück, die der Benutzer eingegeben hat (oder eine leere Zeichenfolge "", wenn keine angegeben wurde).


Syntax:

get_string_async(string, default);

Streit Beschreibung
String Die Nachricht, die im Dialog angezeigt werden soll.
default Die Standardzeichenfolge


Kehrt zurück:

Real


Erweitertes Beispiel:

Das Ereignis mit der linken Maustaste (beispielsweise) des Objekts, das die Nachricht anzeigt, hätte folgenden Code:

msg = get_string_async("What's your name?","Anon");

Oben wird eine Nachricht angezeigt, in der der Benutzer aufgefordert wird, eine Zeichenfolge einzugeben und "OK" zu drücken. Die Funktions-ID wird in der Variablen "msg" gespeichert und im Ereignis asynchronous Dialogs wie folgt verwendet:

var i_d = ds_map_find_value(async_load, "id");
if i_d == msg
   {
   if ds_map_find_value(async_load, "status")
      {
      if ds_map_find_value(async_load, "result") != ""
         {
         global.Name = ds_map_find_value(async_load, "result");
         }
      }
   }

Der obige Code überprüft den " id " -Schlüssel der zurückgegebenen ds_map gegen den in der Variablen " msg " gespeicherten Wert. Wenn sie gleich sind, wird geprüft, ob "OK" gedrückt wurde (anstatt das Fenster geschlossen / abgebrochen) und ob es zurückkehrt true Er prüft dann das "Ergebnis" einer Zeichenfolge, um sicherzustellen, dass keine leeren Zeichenfolgen zurückgegeben werden, bevor eine globale Variable festgelegt wird.
