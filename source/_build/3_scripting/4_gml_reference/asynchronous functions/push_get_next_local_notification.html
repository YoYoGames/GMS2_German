push_get_next_local_notification


Beschreibung

Diese Funktion füllt ein Pre-Made ds_map mit einer Reihe von Schlüssel / Wert-Paaren für die nächste lokale Push-Benachrichtigung in der Warteschlange, die angezeigt werden soll. Es gibt -1 zurück, wenn keine weiteren Benachrichtigungen in der Warteschlange sind, oder ein realer Wert, der ansonsten die ID für die Benachrichtigung darstellt. Diese ID kann dann verwendet werden, um die Benachrichtigung mit der Funktion abzubrechen push_cancel_local_notification(). Sie sollten die Funktion aufrufen push_get_first_local_notification um die erste Benachrichtigung in der Warteschlange zu erhalten und diese Funktion dann zu verwenden, um sie fortzusetzen.

Die ds_map enthält die folgenden Schlüssel / Wert-Paare:

  • " title " - Der Titel der Benachrichtigung (diese wird nur auf der Android Plattform Android, da iOS die Titelinformationen nicht speichert)
  • " Nachricht " - Der Nachrichtentext
  • " data " - Die Datenpaket-Zeichenfolge
HINWEIS: Diese Funktion ist auf die Zielmodule für iOS und Android beschränkt.


Syntax:

push_get_next_local_notification(map);

Streit Beschreibung
map Das ds_map ID zu verwenden.


Kehrt zurück:

Real


Beispiel:

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
   {
   var data = ds_map_find_value(map, "data");
   if data == "Daily_Reward"
      {
      push_cancel_local_notification(ntf);
      }
   ntf = push_get_next_local_notification(map);
   }
ds_map_destroy(map);

Der obige Code erstellt ein ds_map und fordert dann Daten für die erste verfügbare Benachrichtigung in der Warteschlange an. Wenn eine Benachrichtigung vorhanden ist, wird die Karte aufgefüllt und der Code überprüft dann den Schlüssel "data" für eine bestimmte Zeichenfolge. Wenn die Zeichenfolge gefunden wird, wird die Benachrichtigung abgebrochen, und die nächste Benachrichtigung wird überprüft, bis alle in der Warteschlange überprüft wurden.
