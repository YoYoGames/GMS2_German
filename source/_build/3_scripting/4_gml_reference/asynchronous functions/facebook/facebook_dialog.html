facebook_dialog


Syntax:

facebook_dialog(graph_path, ds_map_parameters, ds_map_return)

Streit Beschreibung
graph_path Der Teil des sozialen Graphen, mit dem man interagieren soll
ds_map_parameters Die ds_map mit den Informationen zu senden.
ds_map_return Die zu empfangende ds_map (-1 für keine).


Kehrt zurück:

N/A


Beschreibung

Ein Facebook-Dialog ist die Art und Weise, in der Facebook den Benutzer bittet, Kommentare und Informationen zu geben, und sie benötigen keine zusätzlichen Berechtigungen, da sie eine Benutzerinteraktion erfordern. Es kann beispielsweise verwendet werden, um eine Freundschaftsanfrage zu senden oder einen Kommentar an eine Wand zu senden. Mit dem Argument "graph_path" definieren Sie den Teil des Graphen, auf den Sie zugreifen möchten. Der genaue Pfad kann mithilfe der in der Übersicht der Facebook- Dialoge beschriebenen Begriffe definiert werden.

Das nächste Argument ist etwas komplexer, da Sie a. Erstellt und gefüllt haben müssen ds_map mit den korrekten Informationen, die GameMaker Studio 2 dann in json automatisch beim Senden an die Facebook API. Die Informationen, die Sie in diese Karte eingeben, hängen sehr davon ab, welchen Pfad Sie verwenden und eine vollständige Liste aller möglichen Werte finden Sie hier. Ein Beispiel dafür, wie diese Karte strukturiert ist, ist im unten angegebenen Codebeispiel zu sehen.

Schließlich haben wir ein Argument dafür, alle Informationen zu speichern, die Facebook von der Graphanfrage zurückgeschickt hat. Diese Information kommt in Form von json welches GameMaker Studio 2 in eine ds_map konvertiert. Damit dies korrekt funktioniert, müssen Sie zuvor eine ds_map erstellt haben, und wenn diese Karte an anderer Stelle verwendet wurde und bereits einige Schlüssel / Wert-Paare enthält, wird sie vor dem Hinzufügen der Facebook-Daten von dieser Funktion gelöscht. Es ist sehr wichtig, dass Sie die Facebook-Dokumentation über mögliche Rückgabewerte lesen, damit Sie genau wissen, was Sie erwarten können. Zum Glück gibt es auf Facebook eine Graph Api Test Seite, auf der Sie verschiedene Grafikpfade sehen und genau sehen können, welche Informationen sie zurückgeben werden.

HINWEIS: Der Benutzer muss nicht angemeldet sein, um diese Funktion zu verwenden, da der Anmeldedialog als Teil der Dialoganfrage angezeigt wird.


Beispiel:

var interactiveMap;
interactiveMap = ds_map_create();
ds_map_add(interactiveMap, "picture", "http://MacSweeneygames.com/Clown.jpg");
ds_map_add(interactiveMap, "link", "http://MacSweeneygames.com/");
ds_map_add(interactiveMap, "name", "Catch The Clown");
ds_map_add(interactiveMap, "caption", "MacSweeneygames.com");
ds_map_add(interactiveMap, "description", "Play Catch the clown now on MacSweeney Games!");
facebook_dialog("feed", interactiveMap, -1);
ds_map_destroy(interactiveMap);

Der obige Code erstellt eine ds_map und speichert ihren Index in der lokalen Variablen "interactiveMap". Diese Karte wird dann mit den notwendigen Schlüssel / Wert-Paaren gefüllt, um eine interaktive Wandpost zu erzeugen. Weiter die facebook_dialog öffnet ein Fenster, in dem der Benutzer aufgefordert wird, eine Nachricht hinzuzufügen, die zusammen mit dem Rest der Information gesendet wird, und dies wird an der Benutzerwand veröffentlicht. Schließlich wird die ds_map gelöscht, um Speicher freizugeben.
