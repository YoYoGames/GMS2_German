facebook_graph_antrag


Syntax:

facebook_graph_request(graph_path, httpMethod, ds_map_parameters, ds_map_return)

Streit Beschreibung
graph_path Der Teil des sozialen Diagramms, mit dem Sie interagieren können
httpMethod Die zu verwendende http-Methode ( POST, GET, DELETE ).
ds_map_parameters Die ds_map mit den Informationen zu senden.
ds_map_return Die zu empfangende ds_map (-1 für keine).


Kehrt zurück:

N/A


Beschreibung

Mit dieser Funktion können Sie den Benutzer dazu bringen, mit dem Facebook Social Graph zu interagieren. Mit dem Argument "graph_path" definieren Sie den Teil des Graphen, auf den Sie zugreifen möchten, z. B. die Liste der aktuellen Freunde, ein anderer Kommentar oder sogar ein Ereignis. Der genaue Pfad kann mithilfe der im Abschnitt " Publishing " der Facebook Graph Api- Seiten angegebenen Begriffe definiert werden. Sie definieren dann die zu verwendende http-Methode, die normalerweise POST oder GET aber Facebook akzeptiert auch die DELETE-Methode.

Das nächste Argument ist etwas komplexer, da Sie ein erstellt und gefüllt haben müssen ds_map mit den richtigen Informationen, in die GameMaker Studio 2 dann konvertiert json automatisch beim Senden an die Facebook-API. Die Informationen, die Sie in diese Karte einfügen, hängen stark davon ab, welchen Pfad Sie verwenden. Eine vollständige Liste aller möglichen Werte finden Sie hier. Ein Beispiel für die Struktur dieser Karte finden Sie im folgenden Codebeispiel.

Zum Schluss haben wir ein Argument für das Speichern von Informationen, die Facebook von der Diagrammanfrage zurückgesendet hat. Diese Informationen werden in Form von angezeigt json welches GameMaker Studio 2 in eine ds_map konvertiert. Damit dies ordnungsgemäß funktioniert, müssen Sie zuvor eine ds_map erstellt haben. Wenn diese Karte an anderer Stelle verwendet wurde und bereits einige Schlüssel / Wert-Paare enthält, wird sie von dieser Funktion gelöscht, bevor die Facebook-Daten hinzugefügt werden. Es ist sehr wichtig, dass Sie die Facebook-Dokumentation über mögliche Rückgabewerte lesen, um genau zu wissen, was Sie erwarten. Zum Glück hat Facebook eine Diagramm-Api-Testseite, auf der Sie verschiedene Diagrammpfade überprüfen und genau sehen können, welche Informationen sie zurückgeben.

HINWEIS: Der Benutzer muss angemeldet sein, damit die Diagrammanforderung funktioniert.


Beispiel:

var wallPostMap;
wallPostMap = ds_map_create();
ds_map_add(wallPostMap, "message", "I just got a hi-score playing Catch The Clown!");
ds_map_add(wallPostMap, "picture", "http://MacSweeneygames.com/Clown.jpg");
ds_map_add(wallPostMap, "link", "http://MacSweeneygames.com/");
ds_map_add(wallPostMap, "name", "Catch The Clown");
ds_map_add(wallPostMap, "caption", "MacSweeneygames.com");
ds_map_add(wallPostMap, "description", "Play Catch the clown now on MacSweeney Games!");
facebook_graph_request("me/feed", "POST", wallPostMap, -1);
ds_map_destroy(wallPostMap);

Der obige Code erstellt eine ds_map und speichert ihren Index in der lokalen Variablen "wallPostMap". Es füllt dann die ds_map mit den relevanten Schlüssel / Wert-Paaren, die erforderlich sind, um ein Bild und einen Text an die aktuell angemeldete Benutzerwand zu senden. Schließlich sendet er die Diagrammanforderung und löscht die ds_map aus dem Speicher.
