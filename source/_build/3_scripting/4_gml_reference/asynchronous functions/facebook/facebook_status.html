Facebook-Status


Syntax:

facebook_status()


Kehrt zurück:

String


Beschreibung

Mit dieser Funktion können Sie Facebook auf den aktuellen Status der Verbindung abfragen. Eine der folgenden Zeichenfolgen wird zurückgegeben (beachten Sie, dass sie immer in Großbuchstaben zurückgegeben werden):

  • AUTORISIERT: Dies bedeutet, dass sich der Benutzer korrekt angemeldet hat, aber möglicherweise nicht alle Berechtigungen erteilt wurden.
  • VERARBEITUNG: Dies wird zurückgegeben, wenn der Anmeldeprozess derzeit nicht aufgelöst wird.
  • IDLE: Dies wird zurückgegeben, wenn die Verbindung initialisiert wurde, aber keine weiteren Maßnahmen ergriffen wurden.
  • FAILED: Dies bedeutet, dass der Benutzer nicht richtig angemeldet ist, normalerweise aufgrund eines Verbindungsfehlers.


Beispiel:

switch (facebook_status())
   {
   case "AUTHORISED": global.Auth = true; instance_destroy(); break;
   case "FAILED": facebook_login(permissions); alarm[0] = 30; break;
   default: alarm[0] = 30;
   }

Der obige Code würde in einem Alarm ausgeführt werden und überprüft den Status von Facebook, um zu sehen, ob sich der Benutzer korrekt angemeldet hat. Wenn die Return-Zeichenfolge "AUTHORIZED" ist, wird eine globale Variable auf TRUE gesetzt und das Spiel wird fortgesetzt. Wenn die Return-Zeichenfolge "FAILED" lautet, fordert das Spiel den Benutzer erneut an und wiederholt den Alarm, um den Status erneut zu überprüfen.
