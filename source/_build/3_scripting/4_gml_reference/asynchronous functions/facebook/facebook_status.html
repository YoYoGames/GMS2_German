Facebook-Status


Syntax:

facebook_status()


Kehrt zurück:

String


Beschreibung

Mit dieser Funktion können Sie Facebook nach dem aktuellen Status der Benutzeranmeldung abfragen. Eine der folgenden Zeichenfolgen wird zurückgegeben (beachten Sie, dass sie immer in Großbuchstaben zurückgegeben werden):

  • AUTHORISED: Dies bedeutet, dass der Benutzer sich korrekt angemeldet hat, aber möglicherweise nicht alle Berechtigungen erteilt wurden.
  • PROCESSING: zurückgegeben, wenn der Anmeldeprozess derzeit nicht aufgelöst ist.
  • IDLE: Dies wird zurückgegeben, wenn die Verbindung zur Facebook-API initialisiert wurde, jedoch keine weiteren Maßnahmen ergriffen wurden (z. B. kein Benutzer ist angemeldet).
  • FAILED: Dies bedeutet, dass der Benutzer sich normalerweise nicht ordnungsgemäß angemeldet hat, normalerweise aufgrund eines Verbindungsfehlers.

Beachten Sie, dass diese Funktion die Facebook-API nicht ständig überprüft und nur dann aktualisiert wird, wenn eine Facebook-Aktion gelöst wurde (z. B. Anmelden, Abmelden oder Anfordern von Berechtigungen).


Beispiel:

switch (facebook_status())
   {
   case "AUTHORISED": global.Auth = true; instance_destroy(); break;
   case "FAILED": facebook_login(permissions); alarm[0] = 30; break;
   default: alarm[0] = 30;
   }

Der obige Code würde in einem Alarm ausgeführt und überprüft den Status von Facebook, um festzustellen, ob sich der Benutzer korrekt angemeldet hat. Wenn der Rücksprung "AUTHORISED", wird eine globale Variable auf "AUTHORISED" gesetzt, und das Spiel wird fortgesetzt. Wenn der Rücksprung "FAILED" ist, "FAILED" das Spiel den Benutzer auf, sich erneut "FAILED" und wiederholt den Alarm, um den Status erneut zu überprüfen.
