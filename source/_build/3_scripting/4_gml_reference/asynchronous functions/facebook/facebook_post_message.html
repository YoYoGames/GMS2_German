facebook_post_message


Syntax:

facebook_post_message(title, caption, message, picture_url, redirect_url, actionname, action_url)

Streit Beschreibung
title Der "Name" des Links.
caption Die Bildunterschrift des Links (erscheint unter dem Linknamen).
message Die Nachricht, die im Post enthalten sein soll.
picture_url Ein Link zu dem in diesem Beitrag enthaltenen Bild.
redirect_url Der Link zu diesem Beitrag.
actionname Eine für diesen Beitrag anzuwendende Aktion.
action_url Der Link für die obige Aktion.


Kehrt zurück:

N/A


Beschreibung

Diese Funktion sendet eine Nachricht direkt in den Benutzer-Feed (z. B. einen Beitrag / me / feed in die Facebook-Grafik-API). Sie geben den Namen und die Beschriftung sowie eine Nachricht und ein Bild zum Einfügen an und geben dann die URL an, zu der ein Link erstellt werden soll. Die letzten beiden Argumente "actionname" und "actionurl" sind optional und können entweder als leere Zeichenfolgen ("") gelassen werden oder sie können auf eine der zulässigen Facebook-Aktionen gesetzt werden, z. B. Kommentieren oder Liken Geben Sie eine entsprechende URL an (weitere Informationen finden Sie unter Facebook Developers Api Post ).

HINWEIS: Es wird kein Facebook-Dialog für diese Funktion angezeigt, da der Beitrag ohne Benutzereingriff "ruhig" gesendet wird. Aus diesem Grund müssen Sie die richtigen Berechtigungen für Login angefordert ( „publish_actions“).

HINWEIS: Der Benutzer muss angemeldet sein, damit der Nachrichtenbeitrag funktioniert.


Beispiel:

if mouse_check_button_pressed(mb_left)
   {
   facebook_post_message("Catch The Haggis", "MacSweeneygames.com", "I just got a hi-score playing Catch The Haggis!", "http://MacSweeneygames.com/Clown.jpg", "http://MacSweeneygames.com/CatchTheHaggis.html", "", "");
   }

Der obige Code wird im Hintergrund eine Nachricht an den Wall-Feed der Benutzer senden.
