Facebook Login


Syntax:

facebook_login(permissions, ios_login_type)

Streit Beschreibung
permissions Die Berechtigungen, die Sie vom angemeldeten Benutzer erhalten möchten.
ios_login_type Der nur für iOS angemeldete Typ (siehe Liste der Konstanten unten).


Kehrt zurück:

N/A


Beschreibung

Mit dieser Funktion stellt GameMaker Studio 2 eine Verbindung zu Facebook her und fordert den Benutzer auf, sich bei seinem Konto anzumelden. Sie können auch zusätzliche Berechtigungsanfragen in Form einer ds_list eingeben, die diese Funktion dann in eine konvertiert json Array, um sie an Facebook zu kommunizieren. Es gibt eine Vielzahl von Berechtigungen, die Sie anfordern können. Sie sind auf den Facebook-Entwicklerseiten dokumentiert, die Sie hier finden. Wenn Sie keine speziellen oder erweiterten Berechtigungen benötigen, müssen Sie trotzdem eine ds_list senden, aber dieses Mal ist sie leer.

Diese Funktion gilt auch als asynchron, da das Facebook-Login-Fenster auf dem Bildschirm geöffnet ist und das Spiel im Hintergrund weiterläuft. Selbst wenn der Benutzer den Anmeldevorgang abgeschlossen hat, funktioniert keine der anderen Facebook-Funktionen facebook_status() gibt "AUTORISIERT" zurück. Aufgrund der Notwendigkeit einer Internetverbindung und der unterschiedlichen Datenübertragungsraten bedeutet dies, dass es einige Sekunden dauern kann, bis diese Autorisierung eingeht und das Spiel immer noch läuft.

Hinweis: Sie können jetzt nur Berechtigungen für diese Funktion lesen. Wenn Sie Schreibrechte benötigen, müssen Sie anrufen facebook_request_publish_permissions() nach dem Login. Beachten Sie auch, dass wenn Ihr Spiel mehr als nur das public_profile, email und user_friends anfordert, es von Facebook überprüft werden muss, bevor es von anderen Personen als den Entwicklern des Spiels verwendet werden kann.


Für die iOS-Anmeldung ist ein zusätzlicher Parameter erforderlich (andere Plattformen können ihn einfach verwenden) fb_login_default für das Argument). Dies sollte eine der folgenden Konstanten sein:

Konstante Beschreibung
fb_login_default Verwende das Standard-Login-Verhalten von iOS (siehe hier für weitere Details).
fb_login_fallback_to_webview Versuchen Sie sich mit Facebook anzumelden und fragen Sie den Benutzer nach seinen Zugangsdaten.
fb_login_no_fallback_to_webview Facebook-Login versuchen, aber keine direkte Anfrage nach Zugangsdaten wird gemacht.
fb_login_forcing_webview Versuchen Sie nur die WebView-Anmeldung und fragen Sie den Benutzer nach Anmeldeinformationen
fb_login_use_system_account Versuchen Sie sich mit Facebook anzumelden, bevorzugen Sie den Systemaccount und wechseln Sie ggf. zu einem schnellen App-Switch.
fb_login_forcing_safari Versuchen Sie nur, sich mit Safari anzumelden.

Die oben aufgeführten Konstanten ändern das Login-Verhalten unter iOS. Sie können auswählen, welche Fallback-Methoden verwendet werden sollen (falls vorhanden), sollte die Anmeldung über den normalen Prozess fehlschlagen. Weitere Informationen finden Sie auf dieser Facebook-Entwickler-Seite.


Beispiel:

facebook_init();
var permissions = ds_list_create();
ds_list_add(permissions,"public_profile", "user_friends");
facebook_login(permissions, fb_login_default);
ds_list_destroy(permissions);

Der obige Code wird Facebook initialisieren und dann den aktuellen Benutzer anmelden und nach den erweiterten Leseberechtigungen fragen.
