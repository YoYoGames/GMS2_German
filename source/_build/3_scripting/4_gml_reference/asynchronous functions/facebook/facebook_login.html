Facebook Login


Syntax:

facebook_login(permissions, ios_login_type)

Streit Beschreibung
permissions Die Berechtigungen, die Sie von dem Benutzer haben möchten, der sich anmeldet.
ios_login_type Der Anmeldetyp nur für iOS (siehe Liste der Konstanten unten).


Kehrt zurück:

N/A


Beschreibung

Mit dieser Funktion stellt GameMaker Studio 2 eine Verbindung zu Facebook her und fordert den Benutzer auf, sich bei seinem Konto anzumelden. Sie können auch zusätzliche Berechtigungsanfragen in Form einer ds_list angeben, die diese Funktion dann in eine konvertiert json Array, um sie an Facebook zu kommunizieren. Sie können eine Vielzahl von Berechtigungen anfordern, die auf den hier aufgeführten Facebook-Entwicklerseiten dokumentiert sind. Wenn Sie nicht nach besonderen oder erweiterten Berechtigungen fragen möchten, müssen Sie dennoch eine ds_list senden. Diesmal ist sie jedoch leer.

Diese Funktion wird auch als asynchron angesehen, da das Facebook-Anmeldefenster auf dem Bildschirm geöffnet ist, das Spiel im Hintergrund weiterläuft und auch wenn der Benutzer den Anmeldevorgang abgeschlossen hat, funktionieren keine der anderen Facebook-Funktionen bis zum facebook_status() gibt "AUTORISIERT" zurück. Aufgrund der Notwendigkeit einer Internetverbindung und der unterschiedlichen Datenübertragungsraten kann es einige Sekunden dauern, bis diese Autorisierung eingeht, und das Spiel läuft währenddessen weiter.

Hinweis: Sie können nur jetzt die Standard - Leseberechtigungen für diese Funktion verwenden. Wenn Sie Schreibberechtigungen oder erweiterte Leseberechtigungen benötigen, müssen Sie anrufen facebook_request_publish_permissions() nach dem Einloggen. Beachten Sie auch, dass, wenn Ihr Spiel mehr als das public_profile, email und user_friends verlangt, eine Überprüfung durch Facebook erforderlich ist, bevor es von anderen Personen als den Entwicklern des Spiels verwendet werden kann.


Für die iOS-Anmeldung ist ein zusätzlicher Parameter erforderlich (andere Plattformen können einfach verwendet werden) fb_login_default für das Argument). Dies sollte eine der unten aufgeführten Konstanten sein:

Konstante Beschreibung
fb_login_default Verwenden Sie den Standard iOS Anmeldeverhalten (siehe hier für weitere Details).
fb_login_fallback_to_webview Versuchen Sie, sich bei Facebook anzumelden, und fragen Sie den Benutzer ggf. nach Anmeldeinformationen.
fb_login_no_fallback_to_webview Versuchen Sie, sich bei Facebook anzumelden, es wird jedoch keine direkte Anfrage nach Anmeldeinformationen gemacht.
fb_login_forcing_webview Versuchen Sie nur, sich mit WebView anzumelden und den Benutzer nach Anmeldeinformationen zu fragen
fb_login_use_system_account Versuchen Sie, sich bei Facebook anzumelden, ziehen Sie das Systemkonto vor und greifen Sie gegebenenfalls auf einen schnellen App-Wechsel zurück.
fb_login_forcing_safari Versuchen Sie nur, sich mit Safari anzumelden.

Die oben aufgelisteten Konstanten ändern das Anmeldeverhalten unter iOS, sodass Sie auswählen können, welche Fallback-Methoden (falls vorhanden) verwendet werden sollen, um die Anmeldung durch den normalen Prozess zu scheitern. Für weitere Informationen wenden Sie sich bitte an diese Facebook-Entwicklerseite.


Beispiel:

facebook_init();
var permissions = ds_list_create();
ds_list_add(permissions,"public_profile", "user_friends");
facebook_login(permissions, fb_login_default);
ds_list_destroy(permissions);

Der obige Code initialisiert Facebook und meldet den aktuellen Benutzer an, um nach den Leseberechtigungen zu fragen.
