cloud_synchronise


Beschreibung

Diese Funktion wird normalerweise zu Beginn eines neuen Spiels aufgerufen und dient dazu, den aktuellen Status des Cloud-Dienstes beim Spielstart abzurufen. Die Funktion gibt einen eindeutigen ID- Wert zurück, der dann im asynchronen Cloud-Ereignis verwendet wird, um die relevanten Informationen aus der erstellten ds_map abzurufen.

Diese Funktion sendet Daten an die Cloud, die dann das entsprechende asynchrone Ereignis auslösen. In diesem Fall können Sie die zurückgegebenen überprüfen async_load ds_map für die folgenden Werte:

  • "status": Enthält den Statuscode, wobei ein negativer Wert einen Fehler angibt, dessen Beschreibung im "errorString" enthalten sein wird. Ein Wert von 0 (oder ein positiver Wert) zeigt einen Erfolg an (genaue Werte siehe unten), und der "resultString" enthält die zurückgegebenen Daten oder eine Erfolgsmeldung.

  • "id": Die ID, die von der aufgerufenen Funktion zurückgegeben wurde. Wenn Sie eine Reihe von abfeuern cloud_ Anfragen, dann müssen Sie wissen, auf welche Sie die Antwort bekommen, und so würden Sie diesen Wert verwenden, um mit dem Wert zu vergleichen, den Sie speicherten, als Sie ursprünglich die Anfrage schickten, um die richtige zu finden.

  • "description": Die Beschreibung der zuletzt hochgeladenen Datei.

  • "resultString": Enthält eine Zeichenfolge, bei der es sich um das aus der Cloud zurückgegebene Datenblob handelt.

  • "errorString": gibt eine Error String für einen Fehler zurück.

Die genaue Bedeutung des zurückgegebenen Karteneintrags "Status" wird in der folgenden Tabelle erläutert:

Statuswert errorString / resultString Beschreibung
-1 errorString = "Nicht bei <SERVICE> angemeldet" You have not successfully logged in to the given Cloud Service
0 resultString = wiederhergestellte Daten New game data downloaded from the cloud (following a cloud_synchronise call)
1 resultString = "AlreadySynchronized" No new data since you last called cloud_synchronise
2 resultString = "ConflictDeferral" A conflict was encountered, but the gamer chose to ignore it
3 resultString = "GameUploadSuccess" data from cloud_string_save or cloud_file_save was successfully uploaded to the cloud
-n errorString = Beschreibung des Fehlers Any other negative number means a synchronisation failure


Syntax:

cloud_synchronise();

Erweitertes Beispiel:

Diese Funktion wird in einem Ereignis wie dem Spielstart- Ereignis oder in einem Objekt aufgerufen, das im ersten Raum Ihres Spiels platziert ist, mit der Idee, dass Sie das aktuelle Daten-Blob vom Cloud-Server überprüfen, um zu sehen, ob dies der Fall ist Datum oder nicht.

cloud_check = cloud_synchronise();

Sie möchten dann die zurückgegebenen überprüfen ds_map im asynchronen Cloud-Ereignis, um den Status und die zurückgegebene Zeichenfolge, falls vorhanden, mit dem folgenden Code zu erhalten:

if ds_map_find_value(async_load, "id") == cloud_check
   {
   if ds_map_find_value(async_load, "status") < 0
      {
      show_message_async("Cloud Services not available. Please check connectivity.");
      }
   else
      {
      if ds_map_find_value(async_load, "status") == 0
         {
         var file = file_text_open_write("Save.txt");
         file_text_write_string(file, ds_map_find_value(async_load, "resultString"));
         file_text_close(file);
         }
      }
   }

Der obige Code prüft, ob der korrekte asynchrone Funktionsaufruf überarbeitet wird und fährt fort, um den Status der zurückgegebenen Cloud ds_map zu erhalten. Wenn der Status eine negative Zahl ist, ist etwas schiefgelaufen und der Benutzer wird informiert, andernfalls wird der Code fortgesetzt und ruft die synchronisierten Daten ab und schreibt sie in eine Textdatei zur späteren Verwendung.
