cloud_synchronise


Beschreibung

Diese Funktion wird normalerweise beim Start eines neuen Spiels aufgerufen und wird verwendet, um den aktuellen Status des Cloud-Services beim Starten des Spiels abzurufen. Die Funktion gibt einen eindeutigen ID- Wert zurück, der dann im Asynchronous Cloud-Ereignis verwendet wird, um die relevanten Informationen aus der erstellten ds_map abzurufen.

Diese Funktion sendet Daten in die Cloud, die dann das entsprechende asynchrone Ereignis auslösen. In diesem Fall können Sie die zurückgegebenen prüfen async_load ds_map für die folgenden Werte:

  • " status ": Hier ist der Statuscode enthalten, wobei ein negativer Wert einen Fehler bezeichnet, dessen Beschreibung im " errorString " enthalten ist. Ein Wert von 0 (oder ein positiver Wert) zeigt einen Erfolg an (genaue Werte siehe unten), und " resultString " enthält die zurückgegebenen Daten oder eine Erfolgsmeldung.

  • " id ": Die ID, die von der aufgerufenen Funktion zurückgegeben wurde. Wenn Sie eine Reihe von feuern cloud_ Wenn Sie also Anforderungen eingeben, müssen Sie wissen, auf welche Sie die Antwort erhalten, und daher würden Sie diesen Wert verwenden, um den Wert zu vergleichen, den Sie gespeichert haben, als Sie die Anfrage ursprünglich gesendet hatten, um den richtigen zu finden.

  • " description ": Die Beschreibung der zuletzt hochgeladenen Datei.

  • " resultString ": Dies enthält einen String, der aus der Cloud zurückgegebene Datenblob ist.

  • " errorString ": errorString eine errorString für einen Fehler zurück.

Die genaue Bedeutung des zurückgegebenen "status" -Karteneintrags wird in der folgenden Tabelle erläutert:

Statuswert errorString / resultString Beschreibung
-1 errorString = "Not logged in to <SERVICE> errorString = "Not logged in to " You have not successfully logged in to the given Cloud Service
0 resultString = wiederhergestellte Daten New game data downloaded from the cloud (following a cloud_synchronise call)
1 resultString = "AlreadySynchronized" No new data since you last called cloud_synchronise
2 resultString = "ConflictDeferral" A conflict was encountered, but the gamer chose to ignore it
3 resultString = "GameUploadSuccess" data from cloud_string_save or cloud_file_save was successfully uploaded to the cloud
-n errorString = Fehlerbeschreibung Any other negative number means a synchronisation failure


Syntax:

cloud_synchronise();

Erweitertes Beispiel:

Diese Funktion würde in einem Ereignisse wie das heiße Spiel Startereignis oder in einem Objekt, das im ersten Raum des Spiels platziert ist, mit der Idee ist, dass Sie die aktuelle Datenblob aus dem Cloud - Server zu überprüfen, um zu sehen, ob es bis zu Datum oder nicht.

cloud_check = cloud_synchronise();

Sie möchten dann die zurückgegebenen überprüfen ds_map im asynchronen Cloud-Ereignis, um den Status und die zurückgegebene Zeichenfolge (sofern vorhanden) mit folgendem Code abzurufen:

if ds_map_find_value(async_load, "id") == cloud_check
   {
   if ds_map_find_value(async_load, "status") < 0
      {
      show_message_async("Cloud Services not available. Please check connectivity.");
      }
   else
      {
      if ds_map_find_value(async_load, "status") == 0
         {
         var file = file_text_open_write("Save.txt");
         file_text_write_string(file, ds_map_find_value(async_load, "resultString"));
         file_text_close(file);
         }
      }
   }

Der obige Code prüft, ob der korrekte asynchrone Funktionsaufruf überarbeitet wird, und ruft dann den Status der zurückgegebenen Cloud ds_map ab. Wenn der Status eine negative Zahl ist, ist ein Fehler aufgetreten, und der Benutzer wird informiert. Andernfalls wird der Code fortgesetzt und die synchronisierten Daten abgerufen und zur späteren Verwendung in eine Textdatei geschrieben.
