http_post_string


Beschreibung

Beim Computing wird eine Post- Anforderung verwendet, wenn der Client als Teil der Abrufanforderung Daten an den Server senden muss, z. B. beim Hochladen einer Datei oder Senden eines ausgefüllten Formulars. Dies gilt auch für diese Funktion in GameMaker Studio 2. Im Gegensatz zur http_get-Anforderungsmethode, bei der nur eine URL an den Server gesendet wird, http_post_string() Enthält auch eine Zeichenfolge, die an den Server gesendet wird und zur Erstellung einer neuen Ressource oder zur Aktualisierung einer vorhandenen Ressource oder von beiden führen kann. Es sollte beachtet werden, dass HTTP-Anforderungsparameter (die Bits, die beim Senden eines Formulars auf einer Webseite manchmal bis zum Ende einer URL "angeheftet" werden) durchaus akzeptabel sind, wenn diese Funktion ebenfalls verwendet wird.

Dieses Ereignis generiert einen "Rückruf", der von allen HTTP-Ereignissen abgerufen wird. In diesem Fall wird ein Ereignis generiert ds_map (häufiger als "Wörterbuch" bezeichnet), das exklusiv für dieses Ereignis verfügbar ist und in der speziellen Variablen async_load gespeichert wird. Diese ds_map enthält unterschiedliche Werte, je nachdem, ob Daten zurückgegeben werden oder nicht. Wenn Sie beispielsweise eine Datei angefordert haben, wird das Ereignis beim Empfang jedes Datenpakets mehrmals ausgelöst, sodass Sie beispielsweise eine Fortschrittsleiste anzeigen können. Die allgemeine Struktur für die ds_map sieht wie folgt aus:

  • id: Die ID, die vom Befehl zurückgegeben wurde. Wenn Sie eine Reihe von feuern http_ Wenn Sie also Anforderungen eingeben, müssen Sie wissen, auf welche Sie die Antwort erhalten, und daher würden Sie diesen Wert verwenden, um den Wert zu vergleichen, den Sie gespeichert haben, als Sie die Anfrage ursprünglich gesendet hatten, um den richtigen zu finden.

  • status: Gibt einen Wert unter 0 für einen Fehler, 0 für vollständig und 1 für das Empfangen von Paketen zurück (weitere Einzelheiten siehe unten).

  • result: Die empfangenen Daten (nur Zeichenfolge).

  • url: Die vollständige URL, die Sie angefordert haben.

  • http_status: Der http_status http-Statuscode (falls verfügbar). Dies gibt den Standard-Webstatuscode für die meisten Browser zurück, zB: 304 für "Not Modified" oder 204 für "No Content" usw.

Wenn mehrere Pakete an Ihr Spiel zurückgegeben werden, gibt die Rückruftaste " status " den Wert 1 zurück. In diesem Fall enthält die ds_map die folgenden zusätzlichen Tasten:

  1. " contentLength ": Dies ist die Größe der Datei, die der Web-Server erwartet hat, die Sie erwarten sollen (möglicherweise -1, wenn der Server diese Daten nicht contentLength ).

  3. " sizeDownloaded ": Die Größe der Daten, die bereits heruntergeladen wurden.
ANMERKUNG: Sie sollten wissen, dass aufgrund des XSS-Schutzes in Browsern Anforderungen an und Versuche zum Laden von Ressourcen aus verschiedenen Domänen blockiert werden und möglicherweise leere Ergebnisse zurückgeben. Weitere Informationen finden Sie im Abschnitt zu domänenübergreifenden Problemen.


Syntax:

http_post_string(url, string);

Streit Beschreibung
url Die Webadresse des Servers, von dem Sie Informationen erhalten möchten
string Die Zeichenfolge, die Sie an die angegebene URL senden möchten

Erweitertes Beispiel:

Das http_post_string Die Funktion kann von jedem Ereignis aus aufgerufen werden. Da sie asynchron ist, kann der Rückruf fast augenblicklich sein oder mehrere Sekunden dauern. Der Aufruf der Funktion ist einfach und würde ungefähr so ​​aussehen:

var str = "name=" + global.player_name + "&score=" + string(global.player_score);
post = http_post_string("http://www.angusgames.com/game?game_id=" + string(global.game_id), str);

Der obige Code sendet eine Abrufanfrage an die angegebene URL mit den angegebenen Parametern sowie das Senden der zusätzlichen Daten (Spielername und Punktzahl in URL-kodierter Form), die in der Variablen str gespeichert sind. Dadurch werden alle definierten asynchronen HTTP-Ereignisse ausgelöst, wenn ein Rückruf empfangen wird, und Sie können die zurückgegebene " id " mit der in der Variablen gespeicherten id post um sicherzustellen, dass Sie den richtigen Code ausführen, falls Sie verschiedene verwendet haben http_post_string() Funktionen. Der folgende Beispielcode zeigt, wie dies geschehen würde:

var r_str = "null";
if ds_map_find_value(async_load, "id") == post
   {
   if ds_map_find_value(async_load, "status") == 0
      {
      r_str = ds_map_find_value(async_load, "result");
      }
   }

Der obige Code überprüft zuerst die ID der erstellten ds_map und dann den Status des Rückrufs. Wenn der Wert gleich 0 ist (Erfolg der Signalisierung), wird das Ergebnis des Callbacks zur späteren Verwendung in einer Variablen gespeichert. Andernfalls enthält die Variable nur einen Standardwert (in diesem Fall " null ").
