http_post_string


Beschreibung

Bei der Berechnung wird eine Post - Anforderung verwendet, wenn die Client Daten an den Server als Teil der Abrufanforderung senden muss, wie beim Hochladen eine Datei oder ein ausgefülltes Formular einreichen, und das gleiche gilt für diese Funktion in GameMaker Studio 2. Im Gegensatz zur Request-Methode http_get, bei der nur eine URL an den Server gesendet wird, http_post_string() enthält auch eine Zeichenfolge, die an den Server gesendet wird und zur Erstellung einer neuen Ressource oder zur Aktualisierung einer vorhandenen Ressource oder beider führen kann. Es sollte beachtet werden, dass HTTP-Anfrage-Parameter (die Bits manchmal am Ende einer URL angeheftet werden, wenn Sie ein Formular auf einer Webseite absenden) vollkommen akzeptabel sind, wenn Sie diese Funktion ebenfalls verwenden.

Dieses Ereignis wird einen "Rückruf" generieren, der von irgendwelchen HTTP-Ereignissen empfangen wird, in welchem ​​Fall er ein erzeugt ds_map (besser bekannt als "Wörterbuch"), das exklusiv für dieses Ereignis ist und in der speziellen Variablen async_load gespeichert ist. Diese ds_map enthält verschiedene Werte, abhängig davon, ob Daten zurückgegeben werden oder nicht. Wenn Sie beispielsweise eine Datei angefordert haben, wird das Ereignis beim Empfang jedes Datenpakets mehrfach ausgelöst, sodass Sie beispielsweise eine Fortschrittsanzeige anzeigen können. Die allgemeine Struktur für die ds_map lautet wie folgt:

  • ID: Die ID, die vom Befehl zurückgegeben wurde. Wenn Sie eine Reihe von abfeuern http_ Anfragen, dann müssen Sie wissen, auf welche Sie die Antwort bekommen, und so würden Sie diesen Wert verwenden, um mit dem Wert zu vergleichen, den Sie speicherten, als Sie ursprünglich die Anfrage schickten, um die richtige zu finden.

  • status: Gibt einen Wert von weniger als 0 für einen Fehler zurück, 0 für den Abschluss und 1 für den Empfang von Paketen (siehe unten für weitere Details).

  • Ergebnis: Die empfangenen Daten (nur String).

  • URL: Die vollständige URL, die Sie angefordert haben.

  • http_status: Der rohe http-Statuscode (falls verfügbar). Dadurch wird der Standard-Webstatuscode für die meisten Browser zurückgegeben, z. B.: 304 für "Nicht geändert" oder 204 für "Kein Inhalt" usw.

Wenn mehrere Pakete an Ihr Spiel zurückgegeben werden, gibt der Callback-Statusschlüssel 1 zurück. In diesem Fall verfügt die ds_map über die folgenden zusätzlichen Schlüssel:

  1. "contentLength": Dies ist die Größe der Datei, die der Webserver erwartungsgemäß erhalten sollte (kann -1 sein, wenn der Server diese Daten nicht zurückgibt ).

  3. "sizeDownloaded": Die Größe der Daten, die bereits heruntergeladen wurden.
HINWEIS: Sie sollten wissen, dass aufgrund von XSS-Schutz in Browsern Anforderungen an und Versuche, Ressourcen aus verschiedenen Domänen zu laden, blockiert werden und möglicherweise leere Ergebnisse zurückgeben. Weitere Informationen finden Sie im Abschnitt Domänenübergreifende Probleme.


Syntax:

http_post_string(url, string);

Streit Beschreibung
url Die Webadresse des Servers, von dem Sie Informationen erhalten möchten
string Die Zeichenfolge, die Sie an die angegebene URL senden möchten

Erweitertes Beispiel:

Das http_post_string Die Funktion kann von jedem beliebigen Ereignis aus aufgerufen werden, und da sie asynchron ist, kann der Rückruf fast sofort erfolgen oder mehrere Sekunden dauern. Der Aufruf der Funktion ist einfach und würde etwa so aussehen:

var str = "name=" + global.player_name + "&score=" + string(global.player_score);
post = http_post_string("http://www.angusgames.com/game?game_id=" + string(global.game_id), str);

Der obige Code sendet eine Abrufanforderung an die angegebene URL mit den gegebenen Parametern sowie das Senden der zusätzlichen Daten (Spielername und Bewertung in URL-codierter Form), die in der Variablen str gespeichert sind. Dies wird alle definierten asynchronen HTTP-Ereignisse auslösen, wenn ein Rückruf empfangen wird, und Sie können die "ID" überprüfen, die gegen die in der Variablen gespeicherte zurückgegeben wird post um sicherzustellen, dass Sie den richtigen Code ausführen, wenn Sie verschiedene verwendet haben http_post_string() Funktionen. Der folgende Beispielcode zeigt, wie dies geschehen würde:

var r_str = "null";
if ds_map_find_value(async_load, "id") == post
   {
   if ds_map_find_value(async_load, "status") == 0
      {
      r_str = ds_map_find_value(async_load, "result");
      }
   }

Der obige Code überprüft zuerst die ID der erstellten ds_map und überprüft dann den Status des Callbacks. Wenn der Wert gleich 0 ist (Signalisierungserfolg), wird das Ergebnis des Rückrufs dann in einer Variablen für die zukünftige Verwendung gespeichert, andernfalls behält die Variable einfach einen Standardwert (in diesem Fall "Null").
