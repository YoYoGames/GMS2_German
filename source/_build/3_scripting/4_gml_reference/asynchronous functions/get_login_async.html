get_login_async


Beschreibung

Diese Funktion öffnet ein Fenster, in dem der Benutzer aufgefordert wird, einen Benutzernamen und ein Passwort einzugeben. Diese Argumente können als leere Zeichenfolge festgelegt werden, oder Sie können zuvor eingegebene Werte speichern, die Sie verwenden möchten. Dies ist eine asynchrone Funktion, und so blockiert GameMaker Studio 2 nicht das Gerät, auf dem es ausgeführt wird, während auf Antwort gewartet wird, sondern läuft weiter wie gewohnt. Sobald die Benutzereingabe die Details hat und drückte die „Okay“ -Taste, eine asynchrone Benutzerinteraktion Ereignis wird nur ausgelöst, die für die Dauer dieses Ereignisses wird eine ds_map in der Variablen gespeichert haben async_load.

Diese Karte enthält die beiden Schlüssel "Benutzername" und "Passwort", in denen die entsprechende Benutzereingabe gespeichert ist. Wie es ist ds_map das erstellt wurde, kann dies dann beispielsweise von der json_encode Funktion bereit, um an einen Server gesendet oder in eine Datei auf dem ausgewählten Gerät geschrieben zu werden.

Es ist erwähnenswert, dass diese Funktion eine Indexnummer für die erstellte ds_map zurückgibt, die dann in dem entsprechenden Ereignis überprüft werden kann, so dass Sie eine bestimmte ds_map "tarnen" können, wenn Sie mehr als eine asynchrone Karte erwarten eine andere Funktion). Außerhalb der asynchronen Ereignisse gibt async_load ds_map einen "id" -Wert von -1 zurück.


Syntax:

get_login_async(name, password);

Streit Beschreibung
username Der Standardbenutzername
password Das Standardkennwort


Kehrt zurück:

Real


Erweitertes Beispiel:

Das create-Ereignis (zum Beispiel) des Objekts, das den Login unseres Benutzers steuert, würde den folgenden Code haben:

ini_open("Profile.ini");
u = ini_read_string("User","0","");
p = ini_read_string("User","1","");
ini_close();
login = get_login_async(u,p);

Der obige Code öffnet eine Ini-Datei (oder erstellt eine, wenn sie nicht existiert) und erhält den Namen und das Passwort, die in dieser Datei gespeichert sind. Wenn sie nicht vorhanden sind, wird der Standardwert einer leeren Zeichenfolge ("") zurückgegeben. Diese Werte werden dann in verwendet get_login_async() den Benutzer nach seinen Benutzernamen und Passwortdetails fragen, wobei der Anforderungsindex in der Variablen "login" gespeichert wird. Beachten Sie, dass das Spiel und seine Ereignisse weiterhin normal laufen, während der Benutzer auf die notwendigen Informationen wartet.

Jetzt, da der asynchrone Code abgefeuert wurde, müssen wir folgendermaßen auf den Rückgabewert im asynchronen Ereignis für Dialoge prüfen:

if ds_map_find_value(async_load, "id") == login
   {
   u = ds_map_find_value(async_load, "username");
   p = ds_map_find_value(async_load, "password");
   }

Der obige Code überprüft den "ID" -Schlüssel der async_load ds_map und wenn er den gleichen Wert wie den in der "login" -Variablen gespeicherten enthält, werden die Kartendetails in die entsprechenden Variablen eingelesen, die Sie dann speichern oder verwenden können, um zu überprüfen Datenbankwerte usw.
