http_set_request_crossorigin


Beschreibung

Mit dieser Funktion können Sie den Herkunftstyp festlegen, der beim Laden von Bildern aus einer Datei verwendet werden soll. Die Funktion gilt ausschließlich für die HTML5-Plattform und ist nur beim Laden von Sprites aus einer Datei mit der sprite_add Funktion. Wenn Sie auf "use-credentials" setzen, müssen Sie dem geladenen Sprite nicht mehr einen absoluten Pfad angeben, sondern einen relativen Pfad angeben (wie im folgenden Beispiel gezeigt). Standardmäßig ist der Kreuzursprungstyp auf "anonym" gesetzt.


Syntax:

http_set_request_crossorigin(origin_type);

Streit Beschreibung
origin_type Der zu verwendende Herkunftstyp (eine Zeichenfolge)


Kehrt zurück:

N/A


Beispiel:

if string_lower(http_get_request_crossorigin()) != "use-credentials"
    {
    http_set_request_crossorigin("use-credentials");
    }
sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0);

Der obige Code prüft zuerst den aktuell eingestellten Kreuzursprungstyp. Wenn es sich nicht um "use-credentials" handelt, wird er auf "use-credentials" gesetzt. Anschließend wird ein Sprite aus einer Datei hinzugefügt.
