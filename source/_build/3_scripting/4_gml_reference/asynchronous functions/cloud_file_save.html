cloud_file_save


Beschreibung

Diese Funktion wird eine Datei an den ausgewählten Cloud-Dienst zum Speichern übergeben. Die Funktion gibt einen eindeutigen ID- Wert zurück, der dann im entsprechenden asynchronen Ereignis verwendet werden sollte, um die ds_map zu identifizieren, die vom Cloud-Dienst als "Rückruf" zurückgegeben wird. Die Datei sollte alle Informationen enthalten, die Sie für Ihr Spiel speichern müssen, da Sie nur einen einzigen "Datenblob" in der Cloud speichern können. Wenn Sie diese Funktion erneut ausführen, werden alle zuvor gespeicherten Werte überschrieben cloud_string_save Funktion). Die Beschreibung sollte eine kurze Reihe von Informationen sein, die die Speicherung beschreiben, zB: "Level2, Stage2".

Weitere Informationen zu den zurückgegebenen asynchronen Daten finden Sie in der Funktion cloud_synchronise.


Syntax:

cloud_file_save(file, description);

Streit Beschreibung
string Die Datei, die hochgeladen werden soll (als Zeichenfolge).
description Eine kurze Beschreibung der gespeicherten Daten.


Kehrt zurück:

Real


Beispiel:

var t_str = "";
for (var i = 0; i < 10; i++;)
   {
   t_str += string(global.Highscore[i]) + "|"
   }
var file = file_text_open_write("Highscores.txt");
file_text_write_string(file, t_str);
file_text_close(file);
save_check = cloud_file_save("Highscores.txt", "Current Highscores");

Der obige Code erstellt eine Zeichenfolge aus den im globalen Array "Highscores" gespeicherten Werten und schreibt diese Zeichenfolge dann in eine Datei für den lokalen Speicher. Die Datei wird dann zum Speichern an den Cloud-Dienst gesendet.
