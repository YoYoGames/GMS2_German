achievement_load_friends


Syntax:

achievement_load_friends()


Kehrt zurück:

N/A


Beschreibung

Diese Funktion sendet eine Anfrage an den Server für Informationen über alle eingeloggten Benutzerfreunde und löst ein Callback Social Asynchronous Event aus, welches die async_load Karte, die mit den relevanten Schlüssel / Wert-Paaren gefüllt ist. Der ID- Schlüssel davon ds_map wird verwendet, um den richtigen Callback zu identifizieren (es kann mehr als eine Triggerfunktion für ein gegebenes asynchrones Ereignis geben) und wird mit der Konstante gepaart achievement_friends_info sowie eine Anzahl anderer Schlüssel / Wert-Paare für jeden Freund. Der genaue Inhalt der Karte ist unten gezeigt:

  1. " id " - Für diese Funktion sollte es sein achievement_friends_info

  2. " FriendN " - Der Name des Freundes, wobei "N" ein ganzzahliger Wert ist, der ihrer Position innerhalb der Freundesliste entspricht.

  3. " FriendidN " - Die eindeutige Benutzerkennung des Freundes "N".
HINWEIS: Diese Funktion ist nur für iOS verfügbar.


Erweitertes Beispiel:

Der folgende Code wird wahrscheinlich aufgerufen, nachdem der Spieler sich mit seinem Spielkonto eingeloggt hat achievement_login um eine Liste aller Benutzerfreunde zu erhalten:

achievement_friends_info();

Dies wird eine Anfrage nach den Informationen über die Benutzerfreunde senden und einen asynchronen Rückruf mit dem speziellen erzeugen async_load ds_map enthält folgende Daten:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_friends_info
   {
   var numfriends = ds_map_find_value(async_load, "numfriends");
   global.numfriends = numfriends;
   for(var i=0; i < numfriends; i++;)
      {
      global.friendname[i] = ds_map_find_value(async_load, "Friend" + string(i));
      global.friendid[i] = ds_map_find_value(async_load, "Friendid" + string(i));
      achievement_get_pic(global.friendid[i]);
      }
   }

Der obige Code überprüft die zurückgegebenen ds_map im Social Asynchronous Event und wenn seine "id" mit der zu überprüfenden Konstante übereinstimmt, durchläuft es die Map, die alle verschiedenen Werte in einer Reihe von Arrays speichert, und fordert ein Bild für jeden der Einträge in der Map an (siehe achievement_get_pic ).
