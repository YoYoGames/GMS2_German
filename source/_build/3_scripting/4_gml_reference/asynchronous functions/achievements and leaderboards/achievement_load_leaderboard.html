achievement_load_leaderboard


Syntax:

achievement_load_leaderboard(ident, minindex, maxindex, filter)

Streit Beschreibung
ident Die eindeutige ID des Leaderboards, wie im Entwickler-Dashboard angezeigt.
minindex Der Startindexwert, von dem die Leaderboard-Daten abgerufen werden.
maxindex Der maximale Indexwert, aus dem die Ranglisten-Daten abgerufen werden.
filter Ob du die Ergebnisse auf die deiner Freundesliste oder auf alle Spieler filtern willst (siehe Beschreibung unten)


Kehrt zurück:

N/A


Beschreibung

Diese Funktion sendet eine Anfrage an den Server für Informationen über die gegebene Bestenliste. Das Leaderboard muss zuvor auf dem Entwickler-Dashboard für Ihr Spiel erstellt worden sein. Die angegebene ID ist die, die Sie als erstes Argument der Funktion angeben müssen. Es wird ein Callback Social Asynchronous Event ausgelöst, welches die async_load Karte, die mit den relevanten Schlüssel / Wert-Paaren gefüllt ist. Der ID- Schlüssel davon ds_map wird verwendet, um den richtigen Callback zu identifizieren (es kann mehr als eine Triggerfunktion für ein gegebenes asynchrones Ereignis geben) und wird mit der Konstante gepaart achievement_leaderboard_info sowie eine Anzahl anderer Schlüssel / Wert-Paare für jeden Spieler. Der genaue Inhalt der Karte ist unten gezeigt:

  • " id " - Für diese Funktion sollte es sein achievement_leaderboard_info

  • " leaderboardid " - Die eindeutige ID für das Leaderboard, wie im Anbieter-Dashboard definiert.

  • " Numentries " - Die Anzahl der Einträge in der Bestenliste, die Sie erhalten haben.

  • " PlayerN " - Der Name des Players, wobei "N" ein ganzzahliger Wert ist, der ihrer Position in der Ranglisteneintragsliste entspricht.

  • " PlayerIDN " - Die eindeutige Benutzer-ID des Players, "N".
  • " RankN " - Der Rang des Spielers "N" innerhalb der Bestenliste.

  • " ScoreN " - Die Punktzahl des Spielers "N".

Sie können den genauen Teil der Rangliste angeben, indem sie einen minimalen und maximalen Indexwert, sowie Wunsch zu zeigen, dass die Ergebnisse von allen Spielern mit den folgenden Filterkonstanten nur das Scores von deinen Freunden oder diejenigen geschrieben abzurufen gefiltert werden:

  • achievement_filter_friends_only - Erhalten Sie nur die Punktzahlen der Spieler Freunde.

  • achievement_filter_all_players - Erhalte alle Punkte für alle Spieler.

HINWEIS: Diese Funktion funktioniert nicht auf der Android Plattform.


Erweitertes Beispiel:

Der folgende Code wird wahrscheinlich aufgerufen, nachdem der Spieler sich mit seinem Spielkonto eingeloggt hat achievement_login um eine Liste aller Leaderboard-Positionen für Freunde für das Spiel auf Android oder iOS zu erhalten:

if os_type == os_android
   {
   achievement_load_leaderboard("CgkIs9_51u0PEAIQBw", 1, 100, achievement_filter_friends_only);
   }
else
   {
   achievement_load_leaderboard("leaderboard1id", 1, 100, achievement_filter_friends_only);
   }

Dies wird eine Anfrage nach den Informationen auf der gegebenen Bestenliste senden und einen asynchronen Rückruf mit dem speziellen erzeugen async_load ds_map enthält folgende Daten:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_leaderboard_info
   {
   var lbid = ds_map_find_value(async_load,"leaderboardid");
   if lbid == "CgkIs9_51u0PEAIQBw" || lbid == "leaderboard1id"
      {
      global.numentries = ds_map_find_value(async_load,"numentries");
      for(var i = 0; i < numentries; i++;)
         {
         global.playername[i] = ds_map_find_value(async_load, "Player" + string(i));
         global.playerid[i] = ds_map_find_value(async_load, "Playerid" + string(i));
         global.playerrank[i] = ds_map_find_value(async_load, "Rank" + string(i));
         global.playerscore[i] = ds_map_find_value(async_load, "Score" + string(i));
         achievement_get_pic(global.playerid[i]);
         }
      }
   }

Der obige Code überprüft die zurückgegebenen ds_map im Social Asynchronous Event und wenn seine "id" mit der zu überprüfenden Konstante übereinstimmt, überprüft es, ob das gesendete Leaderboard mit dem ursprünglichen Funktionsaufruf übereinstimmt, bevor es die Map durchläuft und alle verschiedenen Werte in einer Reihe von Arrays speichert. Es fordert auch ein Bild für jeden der Einträge in der Karte an (siehe achievement_get_pic ).
