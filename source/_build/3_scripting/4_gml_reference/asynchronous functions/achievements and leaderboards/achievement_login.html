Leistung_login


Syntax:

achievement_login()


Kehrt zurück:

N/A


Beschreibung

Diese Funktion protokolliert den Benutzer in einem entsprechenden Leaderboard und Leistungsservice. Wenn der Dienst nicht verfügbar ist, wird der Benutzer in einem "vorgetäuschten" Spielzentrum angemeldet und alle Errungenschaften und Punkte werden auf dem Gerät gespeichert, so dass diese Details hochgeladen werden können, wenn der tatsächliche Dienst verfügbar ist. Die Funktion löst ein Callback Social Asynchronous Event aus, welches die async_load Karte, die mit den relevanten Schlüssel / Wert-Paaren gefüllt ist. Der ID- Schlüssel davon ds_map wird verwendet, um den richtigen Callback zu identifizieren (es kann mehr als eine Triggerfunktion für ein gegebenes asynchrones Ereignis geben) und wird mit der Konstante gepaart achievement_our_info sowie eine Anzahl anderer Schlüssel / Wert-Paare für jede Herausforderung. Der genaue Inhalt der Karte ist unten gezeigt:

  1. " id " - Für diese Funktion sollte es sein achievement_our_info

  2. " name " - Der Name des Benutzers, der gerade angemeldet ist.

  3. " playerid " - Die eindeutige Player-ID für den aktuell angemeldeten Benutzer.


Erweitertes Beispiel:

Der folgende Code wird wahrscheinlich direkt zu Beginn Ihres Spiels oder von einem speziellen Schaltflächenobjekt, das Sie in einem Raum platziert haben, aufgerufen und fordert den Spieler auf, sich bei seinem Google Game-Dienste- oder Apple Game Center-Konto anzumelden:

achievement_login();

Dies wird eine Anfrage nach der aktuellen Spielerinformation senden und einen asynchronen Rückruf mit dem Spezial erzeugen async_load ds_map enthält folgende Daten:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_our_info
   {
   var name = ds_map_find_value(async_load, "name" );
   var playerid = ds_map_find_value(async_load, "playerid" );
   global.OurName = name;
   global.OurId = playerid;
   if os_type == os_android
      {
      achievement_load_leaderboard("CgkIs9_51u0PEAIQBw", 1, 100, achievement_filter_friends_only);
      }
   else
      {
      achievement_load_leaderboard("leaderboard1id", 1, 100, achievement_filter_friends_only);
      }
   achievement_get_challenges();
   achievement_load_friends();
   achievement_load_progress();
   }

Der obige Code überprüft die zurückgegebenen ds_map im Social Asynchronous Event und wenn seine "id" mit der zu überprüfenden Konstante übereinstimmt, durchläuft es die Map, die alle verschiedenen Werte in Variablen speichert, bevor die restlichen Funktionen aufgerufen werden, um die Bestenlisten, Challenges und Freundeslisten vorzubereiten.
