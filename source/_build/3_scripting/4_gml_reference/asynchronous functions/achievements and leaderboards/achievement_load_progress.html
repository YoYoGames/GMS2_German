achievement_load_progress


Syntax:

achievement_load_progress()


Kehrt zurück:

N/A


Beschreibung

Diese Funktion sendet eine Anfrage an den Server für Informationen über alle verfügbaren Errungenschaften. Es wird ein Callback Social Asynchronous Event ausgelöst, welches die async_load Karte, die mit den relevanten Schlüssel / Wert-Paaren gefüllt ist. Der ID- Schlüssel davon ds_map wird verwendet, um den richtigen Callback zu identifizieren (es kann mehr als eine Triggerfunktion für ein gegebenes asynchrones Ereignis geben) und wird mit der Konstante gepaart achievement_achievement_info sowie eine Anzahl anderer Schlüssel / Wert-Paare für jeden Spieler. Der genaue Inhalt der Karte ist unten gezeigt:

  1. " id " - Für diese Funktion sollte es sein achievement_achievement_info.

  2. " Numerien " - Die Anzahl der Erfolge in der Liste.

  3. " AchN " - Der Name der Errungenschaft, wobei "N" ein ganzzahliger Wert ist, der seiner Stelle in der Eintragsliste entspricht.

  4. " AchCompN " - Wie vollständig ist der Erfolg "N" als Prozentwert von 0 bis 100 (ein String).
HINWEIS: Diese Funktion ist nur für iOS verfügbar.


Erweitertes Beispiel:

Der folgende Code fordert den Fortschritt des Fortschritts an, wenn sich der Benutzer auf einem iOS-Gerät befindet:

if os_type == os_ios
   {
   achievement_load_progress();
   }

Dies sendet eine Anfrage nach den Informationen über die Errungenschaften und generiert einen asynchronen Rückruf mit dem Special async_load ds_map enthält folgende Daten:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_achievement_info
   {
   var numentries = ds_map_find_value(async_load, "numentries");
   for(var i = 0; i < numentries; i++;)
      {
      ach_id[i] = ds_map_find_value(async_load, "Ach" + string(i));
      comp[i] = ds_map_find_value(async_load, "AchComp" + string(i));
      }
   }

Der obige Code überprüft die zurückgegebenen ds_map im Social Asynchronous Event und wenn seine "id" mit der zu überprüfenden Konstante übereinstimmt, durchläuft es die Einträge in der Map für jede der Errungenschaften und speichert deren Namen und Fertigstellungsprozentsatz.
