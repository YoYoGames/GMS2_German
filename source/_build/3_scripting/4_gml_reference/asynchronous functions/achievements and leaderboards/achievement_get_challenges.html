achievement_get_herausforderungen


Syntax:

achievement_get_challenges()


Kehrt zurück:

N/A


Beschreibung

Diese Funktion sendet eine Anfrage an den Server, um Informationen zu allen aktuellen Herausforderungen zu erhalten, und löst ein soziales asynchrones Rückrufereignis aus, das das enthält async_load Karte gefüllt mit den relevanten Schlüssel / Wert-Paaren. Der ID- Schlüssel davon ds_map wird verwendet, um den richtigen Rückruf zu identifizieren (es kann mehr als eine Auslösefunktion für ein bestimmtes asynchrones Ereignis geben) und wird mit der Konstanten gepaart achievement_challenge_list_received sowie eine Reihe weiterer Schlüssel / Wert-Paare für jede Herausforderung. Der genaue Inhalt der Karte wird unten angezeigt:

  1. " id " - Für diese Funktion sollte es sein achievement_challenge_list_received

  2. „ numchallenges “ - Die Anzahl der Herausforderungen (lokal und remote), die derzeit zur Verfügung.

  3. " ChallengeNplayerid " - Die Spieler-ID für die Challenge, wobei "N" eine ganze Zahl ist, EG: "Challenge5playerid" ist die Spieler-ID für die fünfte Herausforderung in der Liste.

  4. " ChallengeNissuerid " - Die ID der Person, die die Challenge ausgegeben hat, wobei "N" eine ganze Zahl ist, EG: "Challenge2issuerid" ist die Emittenten-ID für die zweite Challenge in der Liste.

  5. " ChallengeNstate " - Der Status der Challenge "N", der einen Wert von 0 - 3 (als Zeichenfolge) für ungültig, ausstehend, abgeschlossen oder abgelehnt hat.

  6. " ChallengeNmessage " - Die Textnachricht für die Challenge "N".

  7. " ChallengeNissueddate " - Das Ausgabedatum für die Herausforderung "N".

  8. " ChallengeNcompleteddate " - Das Fertigstellungsdatum für die Challenge "N".

  9. " ChallengeNtype " - Die Art der Herausforderung. Kann eine von zwei Konstanten sein:
    • achievement_type_score_challenge - Eine Herausforderung basierend auf dem Score-Wert.
    • achievement_type_achievement_challenge - Eine Herausforderung, die auf einer Leistung basiert.
  10. " ChallengeNidentifier " - Die identifizierende Zeichenfolge für die Challenge.

  11. " ChallengeNscore " - Die Punktzahl ist mit der Herausforderung verbunden.
HINWEIS: Diese Funktion gilt nur für iOS.


Erweitertes Beispiel:

Der folgende Code wird wahrscheinlich aufgerufen, nachdem der Spieler sich mit seinem Spielkonto angemeldet hat achievement_login:

achievement_get_challenges();

Dadurch wird eine Anforderung für die aktuellen Challenge-Informationen gesendet und mit dem Special ein asynchroner Rückruf generiert async_load ds_map enthält die folgenden Daten:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_challenge_list_received
   {
   var numentries = ds_map_find_value(async_load,"numchallenges");
   for(var i = 0; i < numentries; i++;)
      {
      player_id[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"playerid");
      issuer_id[i] = ds_map_find_value(async_load, "Challenge" + string(i) +"issuerid");
      state[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"state");
      message[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"message");
      date_completed[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"completeddate");
      date_issued[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"issueddate");
      ach_type[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"type");
      ach_ident[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"identifier");
      ach_score[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"score");
   }

Der obige Code überprüft die zurückgegebenen ds_map In dem asynchronen sozialen Ereignis und wenn seine "id" mit der zu überprüfenden Konstante übereinstimmt, durchläuft es die Map und speichert alle verschiedenen Werte in einer Reihe von Arrays. Diese Informationen können dann zum Beispiel zum Erstellen eigener, personalisierter Challenge-Seiten im Spiel verwendet werden.
