achievement_send_challenge


Syntax:

achievement_send_challenge(playerid, challengeid, score, type, message)

Streit Beschreibung
playerid Die eindeutige ID des Spielers, der herausgefordert werden soll.
challengeid Die eindeutige Challenge-ID (wie im Dashboard des Providers zu finden).
score Die zu schlagende Punktzahl.
type die Art der auszustellenden Aufgabe - eine von zwei unten angegebenen Konstanten.
message Die Nachricht, die an die Challenge angehängt werden soll.


Kehrt zurück:

N/A


Beschreibung

Diese Funktion sendet eine Herausforderung über das Netzwerk an den gewählten Spieler. Sie können die playerid mit der achievement_load_friends oder der achievement_load_leaderboard Funktionen und Sie müssen auch die ChallengeID angeben, die den einzigartigen Wert für die Herausforderung darstellt, wenn Sie sie in Ihrem iTunes Connect- oder Google Play-Dashboard erstellt haben. Sie müssen auch eine Partitur, und eine kurze Textnachricht sowie stellen Sie die Herausforderung Typ liefern. Dies kann eine der folgenden Konstanten sein:

  • achievement_type_score_challenge - Eine Herausforderung basierend auf dem Score-Wert.
  • achievement_type_achievement_challenge - Eine Herausforderung basierend auf einer Leistung.

Diese Funktion löst ein soziales asynchrones Rückrufereignis für den Spieler aus, der die Herausforderung erhalten soll. In diesem Fall wird eine ds_map in der Variablen referenziert async_load. Der ID- Schlüssel davon ds_map wird verwendet, um den richtigen Callback zu identifizieren (es kann mehr als eine Triggerfunktion für ein gegebenes asynchrones Ereignis geben) und wird mit der Konstante gepaart achievement_challenge_received sowie eine Anzahl anderer Schlüssel / Wert-Paare für jeden Spieler. Der genaue Inhalt der Karte ist unten gezeigt:

HINWEIS: Der Benutzer kann auch Toastbenachrichtigungen für eingegangene Aufgaben erhalten, dies hängt jedoch von den Werten ab, die Sie mit der Funktion ausgewählt haben achievement_show_challenge_notifications.
  1. " id " - Für diese Funktion sollte es sein achievement_leaderboard_info

  2. " playerid " - Die Spieler-ID für die Herausforderung.

  3. " issuerid " - Die Aussteller-ID für die Anfrage.

  4. " state " - Der Status der Abfrage, der einen Wert von 0 - 3 (als Zeichenfolge) für ungültig, ausstehend, abgeschlossen oder abgelehnt hat.

  5. " Nachricht " - Die Textnachricht für die Herausforderung.

  6. " completeddate " - Das Fertigstellungsdatum für die Challenge.

  7. " Ausgabedatum " - Das Ausgabedatum für die Herausforderung.

  8. " type " - Die Art der gestellten Aufgabe. Kann eine von zwei Konstanten sein:
    • achievement_type_score_challenge - Eine Herausforderung basierend auf dem Score-Wert.
    • achievement_type_achievement_challenge - Eine Herausforderung basierend auf einer Leistung.
  9. " identifier " - Die identifizierende Zeichenfolge für die Challenge, wie sie auf dem Anbieter-Dashboard festgelegt ist

  10. " score " - Die Punktzahl, die mit der Challenge verknüpft ist (falls zutreffend).


Erweitertes Beispiel:

Um eine Herausforderung über das Netzwerk zu senden, hätten Sie diesen Code:

achievement_send_challenge(global.playerid[0], global.challengeid[0], score, achievement_type_score_challenge, "Beat that sucker!");

Diese Anfrage löst dann das Social Event in deinem Spiel für den Spieler aus, an den die Challenge gerichtet war, und dies kann folgendermaßen gehandhabt werden:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_challenge_received
   {
   player_id = ds_map_find_value(async_load, "playerid");
   issuer_id = ds_map_find_value(async_load, "issuerid");
   state = ds_map_find_value(async_load, "state");
   message = ds_map_find_value(async_load, "message");
   date_completed = ds_map_find_value(async_load, "completeddate");
   date_issued = ds_map_find_value(async_load, "issueddate");
   ach_type = ds_map_find_value(async_load, "type");
   ach_ident = ds_map_find_value(async_load, "identifier");
   ach_score = ds_map_find_value(async_load, "score");
   }

Der obige Code überprüft die zurückgegebenen ds_map in dem asynchronen sozialen Ereignis und wenn seine "ID" mit der geprüften Konstante übereinstimmt, extrahiert es dann die relevanten Werte für jeden der Schlüssel in der Karte und speichert sie in Variablen für die zukünftige Verwendung.
