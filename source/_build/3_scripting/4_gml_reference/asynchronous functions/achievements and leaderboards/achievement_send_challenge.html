achievement_send_challenge


Syntax:

achievement_send_challenge(playerid, challengeid, score, type, message)

Streit Beschreibung
playerid Die eindeutige ID des Spielers, der herausgefordert werden soll.
challengeid Die eindeutige Challenge-ID (wie im Provider-Dashboard angegeben).
score Die Punktzahl zu schlagen.
type die Art der Herausforderung, die ausgegeben werden soll - eine der zwei unten angegebenen Konstanten.
message Die Nachricht, die an die Challenge angehängt werden soll.


Kehrt zurück:

N/A


Beschreibung

Diese Funktion sendet eine Herausforderung über das Netzwerk an den ausgewählten Spieler. Sie können die Player-ID mit der achievement_load_friends oder der achievement_load_leaderboard Sie müssen außerdem die Challengeid angeben, die der eindeutige Wert ist, den Sie der Challenge zugewiesen haben, als Sie sie in Ihrem iTunes Connect- oder Google Play-Dashboard erstellt haben. Sie müssen außerdem eine Punktzahl und eine kurze Textnachricht angeben sowie den Challenge- Typ festlegen. Dies kann eine der folgenden Konstanten sein:

  • achievement_type_score_challenge - Eine Herausforderung basierend auf dem Score-Wert.
  • achievement_type_achievement_challenge - Eine Herausforderung, die auf einer Leistung basiert.

Diese Funktion löst ein soziales asynchrones Rückrufereignis für den Spieler aus, der die Challenge erhalten soll. In diesem Fall wird in der Variablen eine ds_map referenziert async_load. Der ID- Schlüssel davon ds_map wird verwendet, um den richtigen Rückruf zu identifizieren (es kann mehr als eine Auslösefunktion für ein bestimmtes asynchrones Ereignis geben) und wird mit der Konstanten gepaart achievement_challenge_received sowie eine Reihe weiterer Schlüssel / Wert-Paare für jeden Spieler. Der genaue Inhalt der Karte wird unten angezeigt:

HINWEIS: Der Benutzer kann auch Toast-Benachrichtigungen für empfangene Herausforderungen erhalten. Dies hängt jedoch von den Werten ab, die Sie mit der Funktion ausgewählt haben achievement_show_challenge_notifications.
  1. " id " - Für diese Funktion sollte es sein achievement_leaderboard_info

  2. " playerid " - Die Spieler-ID für die Challenge.

  3. " issuerid " - Die Emittenten-ID für die Challenge.

  4. " state " - Der Status der Challenge, der einen Wert von 0 - 3 (als Zeichenfolge) für ungültig, ausstehend, abgeschlossen oder abgelehnt hat.

  5. " message " - Die Textnachricht für die Herausforderung.

  6. " completeddate " - Das Fertigstellungsdatum für die Herausforderung.

  7. " issueddate " - Das Ausgabedatum für die Herausforderung.

  8. " type " - Die Art der Herausforderung. Kann eine von zwei Konstanten sein:
    • achievement_type_score_challenge - Eine Herausforderung basierend auf dem Score-Wert.
    • achievement_type_achievement_challenge - Eine Herausforderung, die auf einer Leistung basiert.
  9. „Identifier“ - Die Identifizierungszeichenfolge für die Herausforderung, als Satz am Armaturenbrett Provider

  10. " Punktzahl " - Die Punktzahl, die mit der Herausforderung verbunden ist (falls zutreffend).


Erweitertes Beispiel:

Um eine Challenge über das Netzwerk zu senden, hätten Sie diesen Code:

achievement_send_challenge(global.playerid[0], global.challengeid[0], score, achievement_type_score_challenge, "Beat that sucker!");

Diese Anforderung löst dann das soziale Ereignis in Ihrem Spiel für den Spieler aus, an den die Herausforderung gerichtet wurde. Dies kann auf folgende Weise behandelt werden:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_challenge_received
   {
   player_id = ds_map_find_value(async_load, "playerid");
   issuer_id = ds_map_find_value(async_load, "issuerid");
   state = ds_map_find_value(async_load, "state");
   message = ds_map_find_value(async_load, "message");
   date_completed = ds_map_find_value(async_load, "completeddate");
   date_issued = ds_map_find_value(async_load, "issueddate");
   ach_type = ds_map_find_value(async_load, "type");
   ach_ident = ds_map_find_value(async_load, "identifier");
   ach_score = ds_map_find_value(async_load, "score");
   }

Der obige Code überprüft die zurückgegebenen ds_map Wenn in " Social Asynchronous Event " die "id" mit der zu überprüfenden Konstante übereinstimmt, extrahiert es die relevanten Werte für jeden Schlüssel in der Karte und speichert sie für die zukünftige Verwendung in Variablen.
