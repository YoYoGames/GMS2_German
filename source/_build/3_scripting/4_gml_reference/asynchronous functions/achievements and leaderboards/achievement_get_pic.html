Leistung_get_pic


Syntax:

achievement_get_pic(char)

Streit Beschreibung
char Die ID der Person, von der das Bild erstellt werden soll (entweder aus einer Freundesliste oder einer Bestenliste)


Kehrt zurück:

N/A


Beschreibung

Diese Funktion wird eine Anfrage an das Server für das Bild eines Spielers oder eines Freundes senden und wird ein Callback Social Asynchronous Event auslösen, welches die async_load Karte, die mit den relevanten Schlüssel / Wert-Paaren gefüllt ist. Der ID- Schlüssel davon ds_map wird verwendet, um den richtigen Callback zu identifizieren (es kann mehr als eine Triggerfunktion für ein gegebenes asynchrones Ereignis geben) und wird mit der Konstante gepaart achievement_pic_loaded sowie eine Anzahl anderer Schlüssel / Wert-Paare. Der genaue Inhalt der Karte ist unten gezeigt:

  1. " id " - Für diese Funktion sollte es sein achievement_pic_loaded

  2. " playerid " - Die ID des Spielers oder Freundes, von dem wir das Bild erhalten haben.

  3. " spriteid " - Die Sprite-ID für das Bild, die dann verwendet werden kann, um sie auf dem Bildschirm mit einer der Zeichen-Sprite-Funktionen zu zeichnen

HINWEIS: Diese Funktion ist nur für iOS verfügbar.
WARNUNG: Diese Funktion erstellt Sprites und ihre IDs sollten daher in Variablen gespeichert und dann mit dem Speicher gelöscht werden sprite_delete Funktion sonst wiederholte Verwendung gibt Ihnen ein Speicherleck, das schließlich Ihr Spiel verlangsamen oder zum Absturz bringen wird.


Erweitertes Beispiel:

Der folgende Code würde aufgerufen, nachdem Sie eine Freundesliste oder eine Ranglistenliste mit erhalten haben achievement_load_friends oder achievement_load_leaderboard. Diese Funktionen geben eindeutige ID-Werte für jede Person zurück, die dann in der Funktion verwendet werden, um das Profilbild für sie abzurufen:

achievement_get_pic(global.player_id[0]);

Dies wird eine Anforderung für das Bild senden, das mit der Spieler-ID verknüpft ist, die in dem globalen Feld gespeichert ist, und einen asynchronen Rückruf mit dem Spezial erzeugen async_load ds_map enthält folgende Daten:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_pic_loaded
   {
   var picid = ds_map_find_value(async_load, "playerid");
   var spriteid = ds_map_find_value(async_load, "spriteid");
   if global.playerid[0] == picid)
      {
      global.playerpic[0] = spriteid;
      }
   }

Der obige Code überprüft die zurückgegebenen ds_map in dem asynchronen sozialen Ereignis und wenn seine "ID" mit der zu überprüfenden Konstante übereinstimmt, prüft es dann, ob die mit dem Bild verknüpfte ID die gleiche wie die angeforderte ist, und wenn dies der Fall ist, setzt es ein anderes globales Array für die eindeutige ID das Bild (mit dem das Bild als Sprite gezeichnet werden kann).
