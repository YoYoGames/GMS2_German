show_message_async


Beschreibung

Diese Funktion öffnet ein Fenster und zeigt die Nachricht an, die Sie in der Funktion für den Benutzer definieren. Dies ist eine asynchrone Funktion, und so blockiert GameMaker Studio 2 nicht das Gerät, auf dem es ausgeführt wird, während auf Antwort gewartet wird, sondern läuft weiter wie gewohnt. Sobald der Benutzer die „Okay“ -Taste gedrückt hat, wird ein asynchroner Dialog Ereignis ausgelöst, die für die Dauer des Ereignisses nur eine ds_map in der Variablen gespeichert haben async_load.

Diese Karte enthält die beiden Schlüssel "id" und "status", wobei "id" der Wert ist, der von der Funktion beim Aufruf zurückgegeben wurde, und der "status" entweder true wenn die Taste "Okay" gedrückt wird, oder false wenn die Nachricht abgebrochen wurde (sofern nicht alle Zielplattformen die Löschung von Dialogen erlauben).


Syntax:

show_message_async(string);

Streit Beschreibung
String Die Nachricht, die dem Benutzer angezeigt werden soll.


Kehrt zurück:

Real


Erweitertes Beispiel:

Das linke Mausereignis (zum Beispiel) des Objekts, das die Nachricht anzeigt, würde den folgenden Code haben:

msg = show_message_async("You will now be taken to the store");

Das obige zeigt eine Nachricht mit der angegebenen Zeichenfolge. Die Nachrichten-ID wird in der Variable "msg" gespeichert und wird im asynchronen Dialog " Dialoge " wie folgt verwendet:

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if i_d == msg
   {
   if ds_map_find_value(async_load, "status")
      {
      url_open("https://play.google.com/store");
      }
   }

Der obige Code überprüft die "ID" -Taste des zurückgegebenen ds_map gegen den in der Variable "msg" gespeicherten Wert. Wenn sie gleich sind, prüft sie, ob die Schaltfläche "Okay" gedrückt wurde (anstatt dass das Fenster geschlossen / abgebrochen wird) und ob sie zurückkehrt true Es öffnet eine URL.
