http_get


Beschreibung

Mit dieser Funktion stellen Sie eine Verbindung zur angegebenen URL her, um Informationen abzurufen. Da dies eine asynchrone Funktion ist, blockiert GameMaker Studio 2 nicht, während auf eine Antwort gewartet wird, sondern läuft weiter, es sei denn, es werden Rückrufinformationen erhalten. Diese Informationen haben die Form einer Zeichenfolge und lösen ein asynchrones Ereignis in einer Instanz aus, für die in ihren Objekteigenschaften eine definiert ist. Beachten Sie auch, dass HTTP-Anforderungsparameter (die Bits werden manchmal am Ende einer URL angehängt, wenn Sie ein Formular auf einer Webseite senden) bei der Verwendung dieser Funktion vollkommen akzeptabel sind, z. B.:

http_get("http://www.YoYoGames.com/logon?username="+name);

Übergibt die in der Variablen "name" enthaltene Zeichenfolge an den Server und ruft die Daten von dieser URL ab. Wenn also ein einfaches, kurzes Stück Daten vom Client an den Server übergeben werden soll, wäre dies eine vernünftige Wahl für die zu verwendende Funktion.

Diese Funktion generiert mehrere "Rückrufe", die von beliebigen HTTP-Ereignissen empfangen werden. Diese erzeugen ein ds_map (besser bekannt als "Wörterbuch"), das exklusiv für dieses Ereignis ist und in der speziellen Variablen async_load gespeichert ist. Diese ds_map enthält verschiedene Werte, abhängig davon, ob Daten zurückgegeben werden oder nicht. Wenn Sie beispielsweise eine Datei angefordert haben, wird das Ereignis beim Empfang jedes Datenpakets mehrfach ausgelöst, sodass Sie beispielsweise eine Fortschrittsanzeige anzeigen können. Die allgemeine Struktur für die ds_map lautet wie folgt:

  1. ID: Die ID, die vom Befehl zurückgegeben wurde. Wenn Sie eine Reihe von abfeuern http_ Anfragen, dann müssen Sie wissen, auf welche Sie die Antwort bekommen, und so würden Sie diesen Wert verwenden, um mit dem Wert zu vergleichen, den Sie speicherten, als Sie ursprünglich die Anfrage schickten, um die richtige zu finden.

  3. status: Gibt einen Wert von weniger als 0 für einen Fehler zurück, 0 für den Abschluss und 1 für den Empfang von Paketen (siehe unten für weitere Details).

  5. Ergebnis: Die empfangenen Daten (nur String).

  7. URL: Die vollständige URL, die Sie angefordert haben.

  9. http_status: Der rohe http-Statuscode (falls verfügbar). Dadurch wird der Standard-Webstatuscode für die meisten Browser zurückgegeben, z. B.: 304 für "Nicht geändert" oder 204 für "Kein Inhalt" usw.

Wenn mehrere Pakete an Ihr Spiel zurückgegeben werden, gibt der Callback-Statusschlüssel 1 zurück. In diesem Fall verfügt die ds_map über die folgenden zusätzlichen Schlüssel:

  1. "contentLength": Dies ist die Größe der Datei, die der Webserver erwartungsgemäß erhalten sollte (kann -1 sein, wenn der Server diese Daten nicht zurückgibt ).

  3. "sizeDownloaded": Die Größe der Daten, die bereits heruntergeladen wurden.
HINWEIS: Sie sollten wissen, dass aufgrund von XSS-Schutz in Browsern Anforderungen an und Versuche, Ressourcen aus verschiedenen Domänen zu laden, blockiert werden und möglicherweise leere Ergebnisse zurückgeben. Weitere Informationen finden Sie im Abschnitt Domänenübergreifende Probleme.

if ds_map_find_value(async_load, "id") == get

Der obige Code überprüft zuerst die ID der erstellten ds_map und überprüft dann den Status des Callbacks. Wenn der Wert gleich 0 ist (Signalisierungserfolg), wird das Ergebnis des Rückrufs in einer Variablen zur zukünftigen Verwendung gespeichert, andernfalls wird die Variable auf einen Standardwert gesetzt (in diesem Fall "Null").
