network_send_packet

Syntax:

network_send_packet(socket, buffer, size);


Streit Beschreibung
socket Die ID des zu verwendenden Sockets.
buffer Die ID des Puffers zum Abrufen der Daten.
size Die Größe (in Byte) der Daten.


Rückgabe: Echt


Beschreibung

Mit dieser Funktion können Sie ein Datenpaket über das Netzwerk senden. Die Funktion benötigt die Socket-ID, um eine Verbindung herzustellen. Anschließend müssen Sie die Puffer-ID angeben, die die zu sendenden Daten enthält (weitere Informationen zu Puffern finden Sie unter Referenz - Puffer ) und schließlich die Größe (in Byte) des Datenpakets. Pakete, die mit dieser Funktion gesendet werden, sind so formatiert, dass das GameMaker Studio 2 Spiel, das die Daten empfängt, die Pakete korrekt "aufteilen" kann. Die Funktion gibt die Anzahl der gesendeten Daten zurück oder eine Zahl kleiner als 0, wenn das Senden fehlgeschlagen ist. Es ist erwähnenswert, dass die endgültige Größe der gesendeten Daten, die von dieser Funktion zurückgegeben wird, auch die Header-Informationen von GameMaker enthält, die zusätzliche 12 Bytes sind.


Beispiel:

buff = buffer_load("player_save.dat");
network_send_packet(sock, buff, buffer_get_size(buff));

Die obige Information lädt zuvor gespeicherte Pufferdaten in den Speicher und gibt die Puffer-ID zurück, die in der Variablen "buff" gespeichert werden soll. Dieser vollständige Puffer wird dann als Paket über das Netzwerk gesendet, wobei der Socket verwendet wird, der durch die Variable "sock" identifiziert wird.
