network_set_config

Syntax:

network_set_config(config_value, setting);


Streit Beschreibung
config_value Der einzustellende Konfigurationswert (unten aufgeführt).
setting Die Einstellung des Konfigurationswerts.


Rückgabe: String


Beschreibung

Mit dieser Funktion können Sie verschiedene Netzwerkkonfigurationen einstellen. Diese Konfigurationen werden als Konstanten angegeben und die Einstellung hängt von der gewählten Konstante ab. In der folgenden Tabelle sind die verfügbaren Konstanten und ihre Einstellungen aufgeführt:

Konstante Beschreibung Rahmen
network_config_connect_timeout Legen Sie einen Wert für die Verbindungszeit fest A value in milliseconds
network_config_use_non_blocking_socket Sagen Sie GameMaker Studio 2, dass es beim Verbinden nicht blockieren soll. 0 = block, 1 = don't block
network_config_enable_reliable_udp Aktiviert das Protokoll "zuverlässiges UDP" für einen vorhandenen UDP-Socket A value in milliseconds
network_config_disable_reliable_udp Deaktiviert das "zuverlässige UDP" -Protokoll für eine vorhandene UDP-Socke. 0 = block, 1 = don't block



Die erste Option ( network_config_connect_timeout ) legt einfach das Zeitlimit für die Verbindung zu einem Server fest, ändert jedoch nicht das Verbindungsverhalten, abgesehen von der Wartezeit. Es wird empfohlen, dass Sie dies nicht zu niedrig einstellen und für ein LAN-Spiel bei etwa 1000 oder für das Internet auf etwa 1000 belassen. Andernfalls kann die Verbindung nicht zufällig hergestellt werden. Wenn Sie einen Timeout-Wert für das Senden / Empfangen von Paketen festlegen möchten, verwenden Sie die Funktion network_set_timeout().

Die zweite Option ( network_config_use_non_blocking_socket ) bedeutet, dass alle Netzwerkverbindungsfunktionen a zurückgeben socket_id sofort, aber Sie können an diesem Socket erst senden oder empfangen, wenn Sie ein asynchrones Netzwerkereignis erhalten haben. Für das ausgelöste Ereignis ist die "Typ" -Taste auf gesetzt network_type_non_blocking_connect (Weitere Informationen finden Sie auf der Seite zum asynchronen Netzwerkereignis ). Dies ist eine globale Einstellung, da GameMaker Studio 2 in einer Anwendung keine Mischung aus Blockieren und Nicht-Blockieren unterstützt.

Die dritte und vierte Option ( network_config_enable_reliable_udp. network_config_disable_reliable_udp ) sind für das Aktivieren oder Deaktivieren des GameMaker Studio 2 zuverlässiges UDP-Protokolls und sollten daher nur für UDP-Verbindungen verwendet werden. Was in diesem Fall "zuverlässig" bedeutet, ist, dass GameMaker Studio 2, wenn es an beiden Enden einer Verbindung aktiviert ist, versucht, Ihre Pakete korrekt zu überprüfen und alle Pakete, die nicht ankommen, erneut zu senden (Hinweis: Wir übernehmen keine Garantie) in Ihren Paketen wird in der Reihenfolge der Übertragung ankommen). Wenn Sie einen zuverlässigen Socket-Typ verwenden, wird allen UDP-Paketen ein 12-Byte-Header hinzugefügt, der Informationen enthält, die von GameMaker Studio 2 benötigt werden, um das Paket auf Fehler zu überprüfen und fehlende Pakete erneut zu senden.


Beispiel:

network_set_config(network_config_connect_timeout, 1000);
network_set_config(network_config_use_non_blocking_socket, 1);

Der obige Code setzt das Zeitlimit für die Netzwerkverbindung auf 1000 ms und weist GameMaker Studio 2 an, die Verbindung nicht zu blockieren.
