network_create_server_raw

Syntax:

network_create_server_raw(type, port, max_client);


Streit Beschreibung
type Der Typ des zu erstellenden Servers (siehe die unten aufgeführten Konstanten).
port Der Port, den der Server verwendet.
max_client Die maximale Anzahl von Clients, die gleichzeitig eine Verbindung herstellen können.


Rückgabe: Echt


Beschreibung

Diese Funktion wird verwendet, um einen neuen Netzwerkserver für Ihr Spiel zu erstellen, der eines der zulässigen Verbindungsprotokolle verwendet (siehe die unten aufgeführten Konstanten). Sie geben den Servertyp an, geben ihm einen zu verwendenden Port und schließlich die Anzahl der maximalen Verbindungen, die gleichzeitig mit dem Server erlaubt sein sollten (beachten Sie, dass diese Nummer Ihnen überlassen bleibt, aber zu viele verbundene Clients werden den Server sättigen Netzwerk oder die Geräte-CPU wird nicht in der Lage sein, die Verarbeitung dieser Anzahl von Spielern zu verarbeiten, verwenden Sie also vorsichtig). Die Funktion gibt eine eindeutige ID zurück, die in einer Variablen gespeichert und verwendet werden sollte, um den Server in allen weiteren Netzwerkfunktionen zu identifizieren, oder einen Wert von weniger als 0, wenn die Verbindung fehlschlägt.

Konstante Beschreibung
network_socket_tcp Erstellen Sie einen Web-Socket mithilfe von TCP.
network_socket_udp Erstellen Sie einen Web-Socket mit UDP.
network_socket_bluetooth Erstelle einen Bluetooth-Socket ( momentan nicht verfügbar! ).

Da dies einen "rohen" Server erzeugt, werden keine der eingebauten GameMaker -Datenheader für die GameMaker akzeptiert oder verwendet, und daher sollten Sie die Funktionen verwenden network_send_raw und network_send_udp_raw() um unformatierte Daten an den erstellten Server zu senden.


Beispiel:

var port = 6510;
server = network_create_server_raw(network_socket_tcp, port, 32);
while (server < 0 && port < 65535)
   {
   port++
   server = network_create_server(network_socket_tcp, port, 32);
   }

Der obige Code wird versuchen, einen Server zu erstellen, der TCP über Port 32 verwendet. Wenn dieser Port nicht verfügbar ist, wird er durch die Ports laufen, um einen zu finden, der ist.
