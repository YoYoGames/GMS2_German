network_send_udp

Syntax:

network_send_udp(socket, url, port, buffer, size);


Streit Beschreibung
socket Die ID des zu verwendenden Sockets.
url Die URL oder IP, zu der eine Verbindung hergestellt werden soll (eine Zeichenfolge).
port Der Port, zu dem eine Verbindung hergestellt werden soll.
buffer Die ID des Puffers zum Abrufen der Daten.
size Die Größe (in Byte) der Daten.


Rückgabe: Echt


Beschreibung

Mit dieser Funktion können Sie Daten über das Netzwerk mit UDP an einen Server senden. Die Funktion verwendet die Socket-ID, über die eine Verbindung hergestellt werden soll, die URL, mit der eine Verbindung hergestellt werden soll, und den zu verwendenden Port. Sie müssen dann die Puffer-ID angeben, die die zu sendenden Daten enthält (weitere Informationen zu Puffern finden Sie unter Referenz - Puffer ) und schließlich die Größe (in Byte) der Daten. UDP ist "verbindungslos", da Sie eigentlich keine Verbindung herstellen, sondern ein Paket direkt an eine IP senden, und der Server erhält eingehende Daten von einer IP-Adresse und muss sich "wie besehen" damit befassen. Die Funktion gibt die Anzahl der gesendeten Daten an oder eine Zahl kleiner als 0, wenn der Sendevorgang fehlgeschlagen ist. Es ist erwähnenswert, dass die endgültige Größe der gesendeten Daten, die von dieser Funktion zurückgegeben wird, auch die Header-Informationen von GameMaker enthält, die zusätzliche 12 Bytes sind.


Beispiel:

network_send_udp(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

Der obige Code sendet ein UDP-Paket an den Server, der durch den URL am Port 6510 definiert ist. Die Daten werden aus dem Puffer entnommen, der in der Variablen "buff" indiziert ist.
