Vernetzung

Die Netzwerkfunktionen von GameMaker Studio 2 basieren auf Steckdosen, einem Industriestandard auf allen Plattformen. Mit Sockets können Sie sowohl Clients als auch Server mit einfacher GML erstellen, wodurch Sie auch ein Einzelspieler-Spiel mit dem Client / Server-Modell erstellen können (etwas, das später leicht hinzugefügt werden kann).

Im Grunde ist ein Socket ein Objekt, das Ports im Netzwerk senden, empfangen, verbinden und abhören kann. Diese Funktionen befassen sich nur mit TCP / IP-Verbindungen, auf denen das Internet basiert (genauer gesagt auf IPv4). Im Wesentlichen können Sie damit einen Socket mit einer IP-Adresse (sagen wir 192.168.1.10) mit einem anderen Socket mit einer anderen IP-Adresse (zB 192.168.1.11) verbinden. Einmal verbunden, können diese beiden Steckdosen Daten mit dem Gerät hin und her senden network_send_* Funktionen (wie network_send_packet ) und das dedizierte asynchrone Ereignis, das von ihnen ausgelöst wird. Dieses async-Ereignis generiert eine ds_map, die die empfangenen Daten sowie weitere Details enthält. Die vollständigen Details finden Sie im Abschnitt über das Netzwerk-Async-Ereignis.

Anstatt, dass Programme jedes Bit des Netzwerkverkehrs lesen und verarbeiten müssen, das in eine Maschine gelangt, werden bei IP-Adressen auch Ports verwendet, die von 0 bis 65535 nummeriert sind. Auf diese Weise kann jedes Programm nur die Datenpakete abrufen, die es interessiert in und nicht alles, was von jedem Programm gesendet wurde. Dies spart auch beträchtliche CPU-Zeit, da die Datenverarbeitung erheblich reduziert wird. Mit GameMaker Studio 2 Sie auch Ports "überwachen". Ein Server erstellt also einen Socket und bindet ihn dann an einen bestimmten Port, den er abhören soll. Es wird dann über Verbindungen, Verbindungsabbrüche und Datenübertragungen von diesem Socket informiert.

Die Daten- "Pakete" selbst bestehen aus binären Daten, die aus einem Puffer entnommen werden. Dieser Puffer kann mit den GameMaker Studio 2 Pufferfunktionen erstellt und bearbeitet werden.

HINWEIS: Diese Funktionen sind derzeit für das HTML5- Zielmodul nicht verfügbar.
