Vernetzung

Die GameMaker Studio 2 Netzwerkfunktionen basieren auf Sockets, einem Industriestandard auf allen Plattformen. Mit Sockets können Sie sowohl Clients als auch Server mit einfachem GML erstellen, mit dem Sie sogar ein Einzelspieler-Spiel mit dem Client / Server-Modell erstellen können (wodurch der Multiplayer später einfach hinzugefügt werden kann).

Grundsätzlich ist ein Socket ein Objekt, das Ports im Netzwerk senden, empfangen, verbinden und hören kann. Diese Funktionen behandeln nur TCP / IP-Verbindungen, auf denen das Internet basiert (genauer IPv4). Im Wesentlichen können Sie damit einen Socket über eine IP-Adresse (sagen wir 192.168.1.10) mit einem anderen Socket einer anderen IP-Adresse verbinden (sagen wir 192.168.1.11). Sobald sie verbunden sind, können diese zwei Buchsen Daten mit Hilfe der network_send_* Funktionen (wie network_send_packet ) und das dedizierte asynchrone Ereignis, das von ihnen ausgelöst wird. Dieses asynchrone Ereignis generiert eine ds_map, die die empfangenen Daten sowie weitere Details enthält. Die vollständigen Details finden Sie im Abschnitt zum asynchronen Netzwerkereignis.

Anstatt dass Programme jeden Netzwerkverkehr, der in eine Maschine kommt, lesen und verarbeiten müssen, behandeln IP-Adressen auch Ports, die von 0 bis 65535 nummeriert sind. Dadurch kann jedes Programm nur die Datenpakete erhalten, an denen es interessiert ist. und nicht alles, was von jedem Programm gesendet wurde. Dies spart auch erhebliche CPU-Zeit, da die Datenverarbeitung erheblich beeinträchtigt wird. Mit GameMaker Studio 2 Sie also auch Ports "hören". Ein Server erstellt also einen Socket und bindet ihn dann an einen bestimmten Port, um ihn anzuhören. Es wird dann über Verbindungen, Trennungen und Datenübertragungen von diesem Socket benachrichtigt.

Die Daten "Pakete" selbst bestehen aus binären Daten, die aus einem Puffer stammen. Dieser Puffer kann mit den GameMaker Studio 2 Buffer-Funktionen erstellt und bearbeitet werden.

Hinweis: Diese Funktionen sind derzeit für das HTML5- Zielmodul nicht verfügbar.

Folgende Funktionen können Sie für die Vernetzung verwenden:

  • network_create_server
  • network_create_server_raw
  • network_create_socket
  • network_create_socket_ext
  • Netzwerkverbindung
  • network_connect_raw
  • Netzwerk_Resolve
  • Netzwerk_Set_config
  • network_set_timeout
  • network_send_broadcast
  • network_send_packet
  • network_send_raw
  • network_send_udp
  • network_send_udp_raw
  • Netzwerkzerstörung
