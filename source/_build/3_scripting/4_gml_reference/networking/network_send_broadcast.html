network_send_broadcast

Syntax:

network_send_broadcast(socket, port, buffer, size);


Streit Beschreibung
socket Die ID des zu verwendenden Sockets.
port Der Port, den der Server verwendet.
buffer Die ID des Puffers zum Abrufen der Daten.
size Die Größe (in Byte) der Daten.


Rückgabe: Echt


Beschreibung

Mit dieser Funktion können Sie die Daten aus einem Puffer lokal an eine Reihe von IP-Adressen senden (weitere Informationen zu Puffern finden Sie unter Referenz - Puffer ). Der Bereich ist auf das Gerät beschränkt, auf dem der Server ausgeführt wird. Wenn das Gerät die IP-Adresse 92.168.11.130 hat, werden die Daten über den Bereich 92.168.11. * Übertragen. Die Funktion gibt die Anzahl der gesendeten Daten an oder eine Zahl kleiner als 0, wenn der Sendevorgang fehlgeschlagen ist.

HINWEIS: Diese Funktion funktioniert nur bei Verwendung mit UDP - Ihr Server muss TCP sein und Ihr Client muss einen UDP-Client-Socket haben, mit dem er erstellt wurde network_create_socket_ext() um Broadcasts zu empfangen, die vom Server gesendet wurden.


Beispiel:

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.ServerName);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

Der obige Code schreibt die Namenszeichenfolge des aktuellen Servers (gespeichert in "global.ServerName") und schreibt sie dann in einen binären Puffer mit der ID "broadcast_buffer". Diese Daten werden dann lokal an einen Bereich von IPs (die Geräte-IP wird derzeit als der Übertragungsbasisbereich impliziert) zu Port 6511 gesendet.
