gml_release_mode

Syntax:

gml_release_mode(flag);


Streit Beschreibung
flag Entweder true Freigabemodus oder falsch für allgemeine Entwicklung festlegen.


Rückkehr: N / A


Beschreibung

Wenn Ihr Spiel kompiliert wird, enthält es auch bestimmte interne Funktionen zur Fehlerprüfung während des Laufens. Diese internen Überprüfungen decken viele Aspekte eines Spielecodes ab, einschließlich Bereiche, Parameter und einige allgemeine Aspekte des internen GML-Compilers und erzeugen in vielen Fällen die letzten Fehlermeldungen. Diese Funktion kann jedoch dazu verwendet werden, diese internen Checks zu deaktivieren (dh sie vollständig aus dem kompilierten Spiel zu entfernen), was Ihrem fertigen Projekt einen Geschwindigkeitsschub verleiht.

Bitte beachten Sie, dass das kompilierte Spiel abstürzt, wenn Sie etwas in Ihrem Spielcode tun, das falsch ist, oder es kann etwas sehr unerwartetes Verhalten auftreten. Zum Beispiel, wenn a ds_grid Wenn Sie auf eine Koordinate außerhalb der definierten Größe zugreifen, erhalten Sie im Entwicklermodus eine Compilerfenster-Fehlermeldung, aber die vorhandenen Prüfungen würden es dem Spiel ermöglichen, weiter zu laufen. Im Freigabemodus sind diese Überprüfungen jedoch nicht vorhanden und daher kann der Code entweder einen NULL-Zeiger oder einen schlechten Zeiger zurückgeben, der wiederum auf einen zufälligen Speicher zeigt. Im besten Fall könnten Sie die falschen Daten zurückbekommen, oder im schlimmsten Fall könnten Sie etwas im Speicher überschreiben - Daten, Grafiken oder sogar Code selbst.


Beispiel:

gml_release_mode(true);

Der obige Beispielcode setzt das Spiel so, dass es den Freigabemodus verwendet, wenn es kompiliert wird.
