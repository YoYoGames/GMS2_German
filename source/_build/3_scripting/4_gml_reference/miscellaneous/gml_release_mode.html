gml_release_mode

Syntax:

gml_release_mode(flag);


Streit Beschreibung
flag Entweder true Release-Modus oder false für allgemeine Entwicklung.


Rückgabe: N / A


Beschreibung

Wenn Ihr Spiel kompiliert ist, enthält es auch einige interne Funktionen zur Fehlerprüfung während des Laufs. Diese internen Überprüfungen decken viele Aspekte eines Spielcodes ab, einschließlich Bereiche, Parameter und einige allgemeine Aspekte des internen GML-Compilers, und führen in vielen Fällen zu den endgültigen Fehlermeldungen. Diese Funktion kann jedoch verwendet werden, um diese internen Überprüfungen auszuschalten (dh sie vollständig aus dem kompilierten Spiel zu entfernen), wodurch das endgültige Projekt beschleunigt wird.

Bitte beachten Sie, dass dies bedeutet, dass das kompilierte Spiel einfach abstürzt oder unerwartetes Verhalten auftreten kann, wenn Sie etwas falsches in Ihrem Spielcode tun. Zum Beispiel, wenn a ds_grid Wenn von einer Koordinate außerhalb der definierten Größe zugegriffen wird, wird im Entwicklermodus eine Fehlernachricht für das Compiler-Fenster angezeigt, die Überprüfungen erlauben jedoch, dass das Spiel weiter läuft. Im Freigabemodus sind diese Überprüfungen jedoch nicht vorhanden, sodass der Code entweder einen NULL-Zeiger oder einen fehlerhaften Zeiger zurückgeben kann, der wiederum auf einen zufälligen Speicher zeigt. Im besten Fall könnten Sie also die falschen Daten zurückgeben oder im schlimmsten Fall etwas im Speicher überschreiben - Daten, Grafiken oder sogar Code selbst. Ein anderes Beispiel wäre das Erstellen von Grundelementen: Wenn Sie die Daten für Farbe, Position, Textur usw. in einer anderen Reihenfolge angeben, als Sie sie beim Erstellen des Vertex-Formats definiert haben, kann das Spiel abstürzen oder unerwartete Ergebnisse bei der Veröffentlichung erzeugen Modus ist eingeschaltet.


Beispiel:

gml_release_mode(true);

Der obige Beispielcode legt fest, dass das Spiel beim Kompilieren den Release-Modus verwendet.
