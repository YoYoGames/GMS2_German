os_is_paused

Überprüft, ob das Betriebssystem des Geräts angehalten ist oder nicht.

Syntax:

os_is_paused()


Rückgabe: Boolean


Beschreibung

Wenn Sie für mobile Geräte entwickeln, müssen Sie überprüfen können, ob das Gerät immer noch auf Ihr Spiel ausgerichtet ist oder nicht. Dies gilt insbesondere, wenn sich Ihr Spiel auf einem iOS-, Android oder Windows Phone Gerät befindet. Dann kann es durch Telefonanrufe unterbrochen werden Andere Apps. Mit dieser Funktion können Sie prüfen, ob das Gerät pausiert ist oder nicht.

Es ist erwähnenswert, dass mit dieser Funktion Code ausgeführt wird, der im Codeblock auf Geräten enthalten ist. GameMaker Studio 2 wird jedoch unmittelbar danach angehalten, da das Gerät das Spiel als Hintergrundprozess zwischenspeichert. Wenn Sie beispielsweise über den unten angegebenen Beispielcode verfügen, wird die zu erstellende Instanz tatsächlich nicht angezeigt oder führt keine ihrer Ereignisse aus, bis das Spiel aus dem zwischengespeicherten Zustand neu gestartet wurde. Dies bedeutet, dass Sie es vorziehen, die os_is_paused Funktion rufen Sie einen Alarm auf und erstellen Sie dann im Alarmereignis (das eingestellt wird, wenn das Gerät angehalten wird, aber erst ausgeführt wird, wenn es nicht angehalten wurde) ein Pause-Menü, starten Sie die Musikwiedergabe usw.

Wenn Sie diese Funktion auf Windows-, Ubuntu (Linux) -, MacOS- oder HTML5-Plattformen verwenden, wird die Funktion nur für einen Schritt umgeschaltet true Wenn das Spiel den Fokus verliert, kehrt es zurück false wieder, bis das Spiel wieder im Fokus ist (und es wieder verliert). Dies liegt daran, dass diese Plattformen keine Apps und Spiele zwischenspeichern und daher im Hintergrund weiterlaufen.

HINWEIS: Auf mobilen Geräten werden Sounds und Musik gestoppt, wenn das Betriebssystem angehalten wird. Sie werden nicht erneut gestartet, wenn Sie sie nicht ausdrücklich mit der entsprechenden Funktion starten.


Beispiel:

if os_is_paused()
   {
   if !instance_exists(obj_PauseMenu) instance_create_layer(0, 0, "Controllers", obj_PauseMenu);
   }

Der obige Code prüft, ob das Spiel pausiert wurde. Wenn dies der Fall ist, wird eine Instanz des Objekts "obj_PauseMenu" erstellt, sofern noch keine vorhanden ist.
