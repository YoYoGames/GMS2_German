Verschiedene Funktionen

Die folgenden Funktionen und Variablen sind spezielle, die GameMaker Studio 2 unterstützt, die jedoch nicht in eine bestimmte Hauptkategorie passen:

Wenn Sie plattformübergreifende Spiele erstellen, ist es oft von entscheidender Bedeutung, dass Sie die Details des Betriebssystems erhalten, auf dem das Gerät läuft, auf dem Ihr Spiel läuft. Dinge wie die Sprache, die Version oder der Netzwerkstatus können alle dazu verwendet werden, Ihr Spiel an das System anzupassen, auf dem es ausgeführt wird, und das Endbenutzererlebnis so gut wie möglich zu gestalten. GameMaker Studio 2 verfügt daher über eine Reihe von speziellen Funktionen, die verwendet werden können Holen Sie sich die notwendigen Informationen.

Sie können die folgenden Funktionen verwenden, um bestimmte Details über das Betriebssystem oder den Browser Ihres Spiels zu erhalten:

  • os_browser
  • os_device
  • os_type
  • OS Version
  • os_is_paused
  • os_is_network_connected
  • os_get_config
  • os_get_language
  • os_get_region
  • os_get_info
  • os_powersave_enable
  • os_lock_orientation

Um Sie beim Erstellen und Verwalten von Highscores für Ihre Spiele zu unterstützen, erstellt GameMaker Studio 2 ein globales Highscore-Array mit 10 Stellen, auf die Sie zugreifen, hinzufügen und ändern können, um ohne großen Aufwand Ihre eigenen Highscore-Listen zu erstellen. Diese Funktionalität ist lokal für das Spiel und funktioniert auf allen Plattformen. Sie können also einfach eine einfache Highscore-Tabelle speichern und verwalten, indem Sie diese Funktionen zusammen mit den Dateibearbeitungsfunktionen verwenden.

Es gibt folgende Funktionen, die sich auf die interne Highscore-Liste beziehen.

  • highscore_add
  • highscore_name
  • highscore_value
  • highscore_clear

GameMaker Studio 2 enthält auch eine Funktion zum Zeichnen der Highscores auf dem Bildschirm als Liste. Es ist jedoch sehr einfach. Es wird empfohlen, diese Funktion zum Debuggen vor allem zu verwenden, da das Erstellen einer eigenen Highscore-Tabelle einfach und einfach ist Sie haben viel mehr Kontrolle darüber, wie es angezeigt wird:

  • draw_highscore

Die folgenden Funktionen stehen nur beim Aufruf von Skripts zur Verfügung:

  • script_exists
  • script_get_name
  • script_execute

Bestimmte Funktionen werden vorab zusammengestellt, wenn Sie Ihr Spiel ausführen (dh: Sie legen fest, wie das endgültige Spiel kompiliert wird), und einige werden auch verwendet, wenn das Spiel kompiliert wurde, um bestimmte Details der Laufzeitumgebung zu erhalten. Hier finden Sie die vollständige Liste dieser Funktionen:

  • GM_build_date
  • GM_version
  • GM_runtime_version
  • gml_release_mode
  • gml_pragma
  • parameter_count
  • parameter_string
  • umgebungszielvariable

Wenn Sie mit Erweiterungen arbeiten, müssen Sie externe Funktionen definieren und aufrufen können. So verwenden Sie Folgendes:

  • external_define
  • external_call
  • external_free

Die folgenden Funktionen sind für den allgemeinen Gebrauch bestimmt und passen einfach nicht zu den anderen Kategorien in diesem Dokument:

  • Cursor_sprite
  • Wecker gestellt
  • alarm_get
