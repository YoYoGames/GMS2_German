Verschiedene Funktionen

Die folgenden Funktionen und Variablen sind spezielle Funktionen, die von GameMaker Studio 2 unterstützt werden, aber nicht in eine bestimmte Hauptkategorie passen:

Bei der Erstellung von plattformübergreifenden Spielen ist es oft von entscheidender Bedeutung, dass Sie die Details des Betriebssystems erhalten, das das Gerät verwendet, auf dem das Spiel ausgeführt wird. Dinge wie die Sprache, die Version oder der Netzwerkstatus können verwendet werden, um Ihr Spiel an das System anzupassen und das Endbenutzer-Erlebnis bestmöglich zu gestalten. GameMaker verfügt GameMaker: Studio über eine Reihe von speziellen Funktionen, die Sie nutzen können Holen Sie sich die notwendigen Informationen.

Die folgenden Funktionen können verwendet werden, um bestimmte Details zum Betriebssystem oder Browser zu erhalten, auf dem das Spiel ausgeführt wird:

  • os_browser
  • os_device
  • os_type
  • OS Version
  • os_is_pausiert
  • os_is_network_connected
  • os_get_config
  • os_get_language
  • os_get_region
  • os_get_info
  • os_powersave_enable
  • os_lock_orientation

Um Ihnen beim Erstellen und Verwalten von Highscores für Ihre Spiele zu helfen, erstellt GameMaker Studio 2 ein globales Highscore-Array mit 10 Orten, auf das Sie zugreifen können, das Sie hinzufügen und ändern können, um Ihre eigenen Highscore-Listen zu erstellen. Diese Funktionalität ist lokal für das Spiel und funktioniert auf allen Plattformen, so dass Sie einfach eine grundlegende Highscore-Tabelle mit diesen Funktionen zusammen mit den Dateihandhabungsfunktionen speichern und pflegen können.

Es gibt folgende Funktionen, die sich mit der internen Highscore-Liste befassen.

  • highscore_add
  • highscore_name
  • Highscore-Wert
  • highscore_clear

GameMaker Studio 2 enthält auch eine Funktion zum Zeichnen der Highscores auf dem Bildschirm als eine Liste, aber es ist sehr einfach und es wird empfohlen, dass Sie dies zum Debuggen mehr als alles andere verwenden, da Ihre eigenen Highscore-Tabelle ist einfach und wird geben Sie haben viel mehr Kontrolle darüber, wie es angezeigt wird:

  • draw_highscore

Die folgenden Funktionen dienen nur zum Aufruf von Skripten:

  • script_exists
  • script_get_name
  • script_execute

Bestimmte Funktionen sind vorkompiliert, wenn Sie Ihr Spiel ausführen (dh sie werden verwendet, um festzulegen, wie das endgültige Spiel kompiliert wird), und einige werden auch verwendet, wenn das Spiel kompiliert wurde, um spezifische Details über die Laufzeitumgebung zu erhalten. Hier finden Sie die vollständige Liste dieser Funktionen:

  • GM_Build_Datum
  • GM_version
  • GM_runtime_version
  • gml_release_mode
  • gml_pragma
  • Parameter_Zahl
  • Parameterzeichenfolge
  • environment_get_variable

Wenn Sie mit Extensions arbeiten, müssen Sie externe Funktionen definieren und aufrufen können.

  • external_define
  • External_call
  • external_frei

Die folgenden Funktionen sind für den allgemeinen Gebrauch gedacht und passen einfach nicht in eine der anderen Kategorien in diesem Dokument:

  • cursor_sprite
  • Wecker gestellt
  • alarm_get
