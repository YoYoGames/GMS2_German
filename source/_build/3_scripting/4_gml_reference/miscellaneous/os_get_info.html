os_get_info


Beschreibung

Diese Funktion gibt a zurück ds_map mit detaillierten Informationen über das Betriebssystem, auf dem das Spiel ausgeführt wird. Die genaue Information hängt vom Betriebssystem und vom Gerät ab. Notiere dass der ds_map wird nicht automatisch aus dem Speicher gelöscht und Sie sollten die ds_map_destroy() Funktion, wenn Sie die enthaltenen Daten nicht mehr benötigen.

Beachten Sie, dass Sie unter Windows Desktop und Windows UWP einige zusätzliche Informationen von DX11 mit folgenden Informationen erhalten (nützlich für Erweiterungen):

  • video_d3d11_device - Zeiger auf das DX11-Gerät
  • video_d3d11_context - Zeiger auf den DX11-Kontext
  • video_adapter_vendorid - Zeichenfolge mit der Hersteller-ID des Adapters
  • video_adapter_deviceid - Zeichenfolge mit der Adaptergerät-ID
  • video_adapter_subsysid - Zeichenfolge mit der ID des Adapter-Subsystems
  • video_adapter_revision - Zeichenfolge mit der Adapterrevisionsnummer
  • video_adapter_dedicatedvideomemory - Zeichenfolge mit der Größe des Videospeichers
  • video_adapter_dedicatedsystemmemory - String mit der Größe des Systemspeichers (wird vom DX11-Adapter verwendet)
HINWEIS: Diese Funktion funktioniert nur auf Android, iOS, Mac, Ubuntu und Windows (Desktop + UWP), auf allen anderen Zielplattformen wird sie -1 anstatt a zurückgeben ds_map.


Syntax:

os_get_info()


Kehrt zurück:

Real (ds_map index)


Beispiel:

os_map = os_get_info();
if os_map != -1
   {
   var size, key, i;
   size = ds_map_size(os_map);
   key = ds_map_find_first(os_map);
   for (i = 0; i < size - 1; i++;)
      {
      map_data[i] = ds_map_find_value(os_map, key);
      key = ds_map_find_next(os_map, key);
      }
   ds_map_destroy(os_map);
   }

Der obige Code überprüft die Betriebssysteminformationen, um zu sehen, ob a ds_map ist zurück gekommen. Ist dies der Fall, durchläuft der Code die Karte und weist alle Werte einem Array zu.
