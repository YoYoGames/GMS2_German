gml_pragma

Syntax:

gml_pragma(command, [optional...]);


Streit Beschreibung
command Eine Zeichenfolge mit einem der unten aufgeführten Befehle.
[optional] Einige der verfügbaren Befehle erfordern ein optionales Argument oder Argumente. Diese werden im Folgenden für jeden Befehl erläutert.


Rückkehr: N / A


Beschreibung

Das gml_pragma Funktion beeinflusst, wie das YYC-Ziel Ihren Code kompiliert und sollte mit den verschiedenen Befehlen aufgerufen werden, um die endgültige Kompilierung Ihres Projekts weiter zu optimieren. Diese Befehle werden vor der Kompilierung des Spiels effektiv vorverarbeitet. Die Funktion kann also an einer beliebigen Stelle in Ihrem Projekt platziert werden und wird noch bearbeitet, bevor das Spiel vollständig kompiliert wurde. Die verfügbaren Befehle sind wie folgt:

  • " forceinline " - Wenn die Funktion mit dem Pragine "forceinline" in einer Skript-Ressource aufgerufen wird, kompiliert das YYC das Projekt mit dem Skript inline statt referenziert. Dies wird einen weiteren Verarbeitungsschub geben, aber es muss vorsichtig sein, wenn es verwendet wird, da es auch die endgültige ausführbare Dateigröße aufbläht, insbesondere wenn der eingefügte Code groß ist und / oder an mehreren verschiedenen Orten verwendet wird, sowie stark zunimmt die Kompilierzeit


  • " global ", " [gml code] " - Das "globale" Pragma erlaubt Ihnen, etwas GML-Code vor allem anderen aufzurufen und wird die zweite GML-Zeichenfolge in einem globalen Bereich ausführen, bevor der erste Raum des Spiels ausgeführt wird. Beispielsweise:

    gml_pragma("global", "scr_Init()");


    Dies wird das Skript "scr_Init" aufrufen, bevor der erste Raum des Spiels ausgeführt wird.


  • " PNGCrush " - Das "PNGCrush" -Pragma verwendet das PNGCrush- Programm für jede erstellte Textur. Beachten Sie, dass dies die Zeit, die für das Kompilieren des Spiels benötigt wird, erheblich verlängern kann, damit Sie es nicht ständig verwenden möchten, obwohl dies erhebliche Einsparungen bei der endgültigen Dateigröße bewirken kann.


  • " Texgroup.Scale ", " [TextureGroupName] ", " [Scale Divisor] " - Die "Texgroup.Scale" skaliert die angegebene Texturgruppe beim Kompilieren. Sie müssen hier zwei zusätzliche Argumente angeben: "[TexGroupName]", das ist der Name (eine Zeichenfolge) der zu skalierenden Texturgruppe und der "[Scale Divisor]" (auch ein String), der der Divisor ist Sie möchten für die Skalierung verwenden, dh:

    gml_pragma("Texgroup.Scale", "level1", "2");


    Dadurch werden alle Texturen in der Texturgruppe "level1" halbiert.


  • " UnityBuild ", " [enable / disable] " - Wenn Sie die Funktion mit diesem Pragma aufrufen und das Argument enable / disable auf setzen true Bei der Kompilierung werden dann alle CPP-Dateien im Projekt in einer einzigen Datei zusammengefasst, die dann zum Erstellen aller Dateien verwendet wird, zum Beispiel:

    gml_pragma("UnityBuild", "true");


    Der Vorteil eines Unity-Builds ist, dass Builds schneller sind, aber der Nachteil ist, dass jedes Mal ein vollständiger Build erstellt wird. Selbst wenn Sie einen einzelnen Teil des Codes ändern, wird alles erneut erstellt, ohne zwischengespeicherte Dateien zu verwenden. Dies wurde speziell für den Xbox One-Export mit dem YYC hinzugefügt, obwohl es für andere Builds aufgerufen werden kann. Weitere Informationen zu Unity-Builds finden Sie hier.


Beispiel:

gml_pragma("forceinline");

Der obige Beispielcode erzwingt, dass das Skript, in dem es verwendet wird, beim Kompilieren eingefügt wird.
