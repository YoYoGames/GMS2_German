clickable_add


Syntax:

clickable_add(x, y, tpe, url, target, params);


Streit Beschreibung
x Die x-Position innerhalb des Fensters.
y Die y-Position innerhalb des Fensters.
tpe Der Textur-Seiteneintrag für das zu verwendende Sprite (verwenden Sie sprite_get_tpe ).
url Die URL (Website-Adresse), zu der verlinkt werden soll, oder das Skript, das ausgeführt werden soll.
target Dies ist der Zielbereich zum Öffnen der URL (siehe Beschreibung).
params Die verschiedenen Parameter für das Symbol (siehe Beschreibung).


Rückgabe: Real


Beschreibung

Mit dieser Funktion können Sie ein benutzerdefiniertes, anklickbares Symbol erstellen, das als "Schaltfläche" außerhalb des Spielbereichs, jedoch innerhalb des Browserfensters selbst fungiert. Die Funktion gibt einen ID-Wert zurück, der in einer Variablen gespeichert werden muss, um in allen weiteren Tastenfunktionen verwendet werden zu können, die auf das neue Symbol verweisen. Diese Funktion ist sehr nützlich, um zu verhindern, dass der Browser beim Klicken auf die Schaltfläche ein Popup-Fenster erstellt, da dies alles über GameMaker Studio 2 selbst erfolgt.

Sie müssen zuerst die Position festlegen, die relativ zu (0,0) des Browserfensters (obere linke Ecke) ist, und dann den Eintrag für die Texturseite für das Sprite angeben, das das Schaltflächensymbol haben soll (siehe: sprite_get_tpe ). Sie können dann entweder eine URL zum Öffnen oder ein Skript (aus der Ressourcenstruktur von GameMaker Studio 2 ) zum Öffnen oder Ausführen angeben. Der von Ihnen angegebene "Ziel" -Parameter entspricht dem Standardwert für den JavaScript-Namen, wenn Sie den open() method (Beachten Sie, dass alle außer '_self' dazu führen können, dass der Browser blockiert oder den Benutzer fragt, ob er dies zulassen möchte). Das Argument "params" ist mit dem JavaScript-Parameter "specs" identisch, mit dem festgelegt wird, welche Eigenschaften die neue Eigenschaft ist Fenster / Registerkarte sollte angezeigt werden (möglicherweise unterstützen nicht alle Browser alle Funktionen). Wenn Sie das Symbol zum Ausführen eines internen Skripts verwenden, können diese letzten beiden Argumente als leere Zeichenfolgen "" belassen werden.

Gültige Ziele sind:

Ziel Beschreibung
_leer Öffnet das verknüpfte Dokument in einem neuen Fenster oder einer neuen Registerkarte.
_selbst Öffnet das verknüpfte Dokument in dem Rahmen, auf den geklickt wurde (dies ist die Standardeinstellung).
_Elternteil Öffnet das verknüpfte Dokument im übergeordneten Frame.
_oben Öffnet das verknüpfte Dokument im gesamten Hauptteil des Fensters.

Gültige Parameter sind:

Parameter Beschreibung
'height = [ px ]' Die Höhe des Fensters, wobei der Mindestwert 100 beträgt.
'width = [ px ]' Die Breite des Fensters, wobei der Mindestwert 100 beträgt.
'left = [ px ]' Die linke Position des Fensters.
'top = [ px ]' Die oberste Position des Fensters (nur IE).
'location = [ boolean ]' Ob das Adressfeld angezeigt werden soll oder nicht (Standardeinstellung ist 1).
'menubar = [ boolean ]' Gibt an, ob die Menüleiste angezeigt werden soll oder nicht (Standardeinstellung ist 1).
'resizable = [ boolean ]' Ob das Fenster in der Größe verändert werden kann oder nicht (Standardeinstellung ist 1).
'scrollbars = [ boolean ]' Gibt an, ob Bildlaufleisten angezeigt werden sollen (Standardeinstellung ist 1).
'status = [ boolean ]' Gibt an, ob eine Statusleiste hinzugefügt werden soll oder nicht (Standardeinstellung ist 1).
'titlebar = [ boolean ]' Ob die Titelleiste angezeigt werden soll oder nicht. Dies wird ignoriert, es sei denn, die aufrufende Anwendung ist eine HTML-Anwendung oder ein vertrauenswürdiges Dialogfeld (der Standardwert ist 1).
'toolbar = [ boolean ]' Gibt an, ob die Browser-Symbolleiste angezeigt werden soll oder nicht (Standardeinstellung ist Ja).

Es ist auch erwähnenswert, dass, wenn das URL-Argument, das Sie übergeben, der Name eines Skripts ist (dh: es beginnt nicht mit "http: //") UND das Skript aufgerufen wird gmcallback_* Dann können Sie direkte Feeds in ein HTML5-Ereignis einbauen. Funktionsnamen von GameMaker Studio 2 werden nicht verschleiert, wenn sie mit beginnen gmcallback_.


Beispiel:

home_but = clickable_add(32, 32, sprite_get_tpe(spr_MS_Home, 0), "http://macsweeney_games.com", "_blank", "width=700, height=500, menubar=0, toolbar=0, scrollbars=0");

Der obige Code erstellt ein anklickbares DOM-Symbol an der Position (32, 32) der Seite, auf der der Game Canvas ausgeführt wird. Das Symbol verwendet das von der Textur-Seite referenzierte Sprite als "spr_MS_Home". Wenn Sie darauf klicken, öffnet das Symbol ein neues Fenster für die angegebene URL und die für dieses Fenster definierten Eigenschaften.
