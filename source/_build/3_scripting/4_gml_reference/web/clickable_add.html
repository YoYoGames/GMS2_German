klickbare_add


Syntax:

clickable_add(x, y, tpe, url, target, params);


Streit Beschreibung
x Die x-Position innerhalb des Fensters.
y Die y-Position innerhalb des Fensters.
tpe Der Texturseiteneintrag für das zu verwendende Sprite (verwenden Sie sprite_get_tpe ).
url Die URL (Website-Adresse), auf die verlinkt werden soll, oder das Skript, das ausgeführt werden soll.
target Dies ist der Zielbereich zum Öffnen der URL (siehe Beschreibung).
params Die verschiedenen Parameter für das Symbol (siehe Beschreibung).


Rückgabe: Echt


Beschreibung

Mit dieser Funktion können Sie ein benutzerdefiniertes, anklickbares Symbol erstellen, das als "Schaltfläche" außerhalb des Spielbereichs, jedoch innerhalb des Browserfensters selbst fungiert. Die Funktion gibt einen ID-Wert zurück, der in einer Variablen gespeichert werden muss, um in allen weiteren Schaltflächenfunktionen verwendet zu werden, die auf das neue Symbol verweisen. Diese Funktion ist sehr nützlich, um zu verhindern, dass der Browser ein Popup-Fenster erstellt, wenn er auf die Schaltfläche klickt, da dies alles in GameMaker: Studio selbst geschieht.

Sie müssen zuerst die Position festlegen, die relativ zum (0,0) des Browserfensters (obere linke Ecke) ist und dann den Texturseiteneintrag für das Sprite angeben, das das Schaltflächensymbol haben soll (siehe: sprite_get_tpe ). Sie können dann entweder eine URL zum Öffnen oder ein Skript (aus dem Ressourcenbaum von GameMaker: Studio ) zum Öffnen oder Ausführen angeben. Der von Ihnen angegebene Parameter "target" entspricht bei Verwendung des Befehls "name" dem Standard-JavaScript-Wert open() (Beachten Sie, dass alle außer '_self' dazu führen können, dass der Browser blockiert oder den Benutzer fragt, ob er es zulassen möchte) und das Argument "params" dasselbe wie der Parameter "specs" in JavaScript ist, um die Eigenschaften des neuen zu steuern Fenster / Tab sollte angezeigt werden (nicht alle Browser unterstützen möglicherweise alle Funktionen). Wenn Sie das Symbol verwenden, um ein internes Skript auszuführen, können diese letzten beiden Argumente als leere Zeichenfolgen "" verbleiben.

Gültige Ziele sind:

Ziel Beschreibung
_leer Öffnet das verknüpfte Dokument in einem neuen Fenster oder Tab.
_selbst Öffnet das verknüpfte Dokument in demselben Frame, in dem es angeklickt wurde (dies ist die Standardeinstellung).
_Elternteil Öffnet das verknüpfte Dokument im übergeordneten Frame.
_oben Öffnet das verknüpfte Dokument im gesamten Fensterbereich.

Gültige Parameter sind:

Parameter Beschreibung
'Höhe = [ px ]' Die Höhe des Fensters, wobei der Mindestwert 100 ist.
'Breite = [ px ]' Die Breite des Fensters, wobei der Mindestwert 100 ist.
'links = [ px ]' Die linke Position des Fensters.
'oben = [ px ]' Die oberste Position des Fensters (nur IE).
'Standort = [ Boolesch ]' Gibt an, ob das Adressfeld angezeigt werden soll (Standard ist 1).
'menubar = [ boolean ]' Gibt an, ob die Menüleiste angezeigt werden soll (Standardeinstellung ist 1).
'resizable = [ boolean ]' Gibt an, ob das Fenster skalierbar ist (Standardwert ist 1).
'Bildlaufleisten = [ boolean ]' Gibt an, ob Bildlaufleisten angezeigt werden sollen (Standardeinstellung ist 1).
'Status = [ Boolesch ]' Gibt an, ob eine Statusleiste hinzugefügt werden soll (Standardeinstellung ist 1).
'titlebar = [ boolean ]' Gibt an, ob die Titelleiste angezeigt werden soll oder nicht. Dies wird ignoriert, es sei denn, die aufrufende Anwendung ist eine HTML-Anwendung oder ein vertrauenswürdiges Dialogfeld (Standard ist 1).
'Symbolleiste = [ Boolesch ]' Ob die Browser-Symbolleiste angezeigt werden soll oder nicht (Standard ist ja).

Es ist auch erwähnenswert, dass das übergebene URL-Argument der Name eines Skripts ist (dh es beginnt nicht mit "http: //") UND das Skript wird aufgerufen gmcallback_* Dann können Sie direkte Feeds in ein html5-Event bekommen. GameMaker: Studio Funktionsnamen werden nicht verschleiert, wenn sie mit beginnen gmcallback_.


Beispiel:

home_but = clickable_add(32, 32, sprite_get_tpe(spr_MS_Home, 0), "http://macsweeney_games.com", "_blank", "width=700, height=500, menubar=0, toolbar=0, scrollbars=0");

Der obige Code erstellt ein anklickbares DOM-Symbol an der Position (32, 32) der Seite, auf der das Spielfenster läuft. Das Symbol verwendet das Sprite, auf das von der Texturseite verwiesen wird, als "spr_MS_Home". Wenn Sie darauf klicken, öffnet das Symbol ein neues Fenster für die angegebene URL und mit den definierten Eigenschaften für dieses Fenster.
