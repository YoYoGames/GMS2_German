display_set_gui_size


Beschreibung

Diese Funktion wirkt sich nur auf Elemente aus, die im Draw GUI-Ereignis gezeichnet werden, und kann verwendet werden, um eine bestimmte Breite und Höhe für alle GUI-Komponenten in diesem Ereignis festzulegen. Unabhängig davon, in welcher Größe die Anzeige oder das Fenster zum Anzeigen der GUI verwendet wird, ist dies der Fall skaliert, um richtig zu passen. Wo sich die GUI befindet (dh die Position (0,0)) hängt davon ab, ob Sie die Funktion verwendet haben display_set_gui_maximise() oder nicht. Wenn Sie diese Funktion nicht verwenden (die Standardeinstellung), wird der GUI-Layer mit seinem (0,0) -Punkt an derselben Position wie die Anwendungsoberfläche (der Bereich, in dem Ihr Spiel gezeichnet wird) und der Breite positioniert und die Höhe wird auf diese Fläche gedehnt, so dass das Festlegen einer GUI-Breite von 256 Pixeln mit dieser Funktion, wenn die Anwendungsfläche 512 Pixeln beträgt, diese 256 Pixel dehnen würde, wodurch die Pixel doppelt so groß werden. Dies ist ideal für Spiele, bei denen die Seitenverhältniskorrektur in den globalen Spieleinstellungen festgelegt ist.

Wenn Sie das verwendet haben display_set_gui_maximise() Wenn die Funktion aktiviert ist, wird die Position (0,0) für die GUI-Ebene standardmäßig in der oberen linken Ecke des Spielfensters angezeigt oder (im Vollbildmodus oder auf Geräten) mit dem durch diese Funktion festgelegten Skalierungsfaktor sowie der Breite und Höhe angezeigt von diesem gesetzt. Sie können die GUI-Ebene jederzeit auf die Standardkonfiguration zurücksetzen, indem Sie diese Funktion mit -1 als Breite und Höhe aufrufen. Dadurch wird die Position (0, 0) der Anwendungsoberfläche festgelegt, und Breite und Höhe werden auf zurückgesetzt 1: 1 auch mit der Oberfläche.


Syntax:

display_set_gui_size(width, height);

Streit Beschreibung
width Die Breite der GUI
height Die Höhe der GUI


Kehrt zurück:

N/A


Beispiel:

display_set_gui_size(768, 1024);

Mit dem obigen Code wird das Zeichnungs-GUI-Ereignis auf die angegebene Breite und Höhe festgelegt, und alle Komponenten werden so skaliert, dass sie entweder zur Anwendungsoberfläche oder zur Anzeige passen, wobei dieser Anteil verwendet wird.
