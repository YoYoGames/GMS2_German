screen_save_part


Beschreibung

Mit dieser Funktion können Sie einen Screenshot eines Teils des Spiels speichern, wie er aktuell in der Spielanzeige angezeigt wird. Die Auswahlkoordinaten basieren auf der absoluten Größe der Anzeige (oder des Fensters, wenn nicht im Vollbildmodus). Daher ist (0,0) immer die obere linke Ecke des Bildschirms (oder Fensters). Unten sehen Sie ein Bild, das zeigt, wie diese Funktion funktioniert:

Die Datei wird im Arbeitsverzeichnis des Spiels gespeichert. Es ist wichtig zu beachten, dass die Funktion für das Ende des Draw-GUI-Endereignisses bestimmt ist, wodurch konsistente Ergebnisse für alle Plattformen erzielt werden sollten. Die Verwendung in einem anderen Fall funktioniert möglicherweise nicht wie erwartet und kann zu unterschiedlichen Zielen und sogar Geräten führen.

HINWEIS: Diese Funktion kann nicht für das HTML5-Ziel verwendet werden.


Syntax:

screen_save_part(fname,x,y,w,h);

Streit Beschreibung
fname Der Dateiname für den Screenshot, der gespeichert werden soll
x Die x-Koordinate der oberen linken Ecke des zu erfassenden Bildschirmbereichs
y Die y-Koordinate der oberen linken Ecke des zu erfassenden Bildschirmbereichs
w Die Breite des zu erfassenden Bildschirmbereichs
h Die Höhe des zu erfassenden Bildschirmbereichs


Kehrt zurück:

N/A


Beispiel:

if keyboard_check_pressed(ord("S"))
    {
    screen_save_part("Screens\Screen_" + string(num++) + ".png", 0, 0, 100, 100)
    }

Der obige Code zeigt einen Screenshot eines Teils des Spielbildschirms mit 100 Pixel im Quadrat, wenn der Benutzer die Taste "S" drückt. Dieser Screenshot wird in einem bestimmten Ordner gespeichert und mit einer Variablen nummeriert, die dann inkrementiert wird.
