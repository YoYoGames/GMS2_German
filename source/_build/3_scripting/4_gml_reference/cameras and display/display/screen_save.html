screen_save


Beschreibung

Mit dieser Funktion können Sie einen Screenshot des Spiels speichern, wie es aktuell im Spielfenster angezeigt wird. Die Datei wird im Arbeitsverzeichnis des Spiels gespeichert, in dem sich Windows befindet <drive>:/Users/YOURUSERNAME/AppData/Local/[Game Name]/ und auf dem Mac wäre das so ~/Library/Application Support/[Game Name]/. Das aufgenommene Bild ist das endgültige Rendern der Anwendungsoberfläche des Spiels oder (falls die App-Oberfläche deaktiviert wurde) ein Bild der gesamten Anzeige oder des Fensters. Es ist wichtig zu beachten, dass die Funktion für das Ende des Draw-GUI-Endereignisses bestimmt ist, wodurch konsistente Ergebnisse für alle Plattformen erzielt werden sollten. Die Verwendung in einem anderen Fall funktioniert möglicherweise nicht wie erwartet und kann zu unterschiedlichen Zielen und sogar Geräten führen. Wenn Sie mehr Kontrolle darüber benötigen, was gespeichert wird, sollten Sie die Funktion verwenden screen_save_part.

HINWEIS: Diese Funktion kann nicht für das HTML5-Ziel verwendet werden.


Syntax:

screen_save(fname);


Streit Beschreibung
fname Der Dateiname für den Screenshot, der gespeichert werden soll


Kehrt zurück:

N/A


Beispiel:

if keyboard_check_pressed(ord("S"))
   {
   screen_save(working_directory + "Screens\Screen_" + string(num++) + ".png")
   }

Der obige Code zeigt einen Screenshot des Spiels, wenn der Benutzer die "S" -Taste drückt. Dieser Screenshot wird in einem bestimmten Verzeichnis gespeichert und mit einer Variablen nummeriert, die dann inkrementiert wird.
