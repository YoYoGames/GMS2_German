screen_save

Speichert einen Screenshot des Spiels als *.png.

Syntax:

screen_save(fname);


Streit Beschreibung
fname Der Dateiname für den Screenshot zum Speichern


Rückkehr: N / A


Beschreibung

Mit dieser Funktion können Sie einen Screenshot des Spiels speichern, wie er gerade im Spielfenster oder Browser erscheint. Die Datei wird im Arbeitsverzeichnis des Spiels gespeichert, das sich unter "C: \ Users \ YOURUSERNAME \ AppData \ Local \ YOURGAMENAME \" in Windows befindet. Dies ist ein Bild der gesamten Display-Region und nicht nur eines bestimmten Abschnitts oder Ansicht und Sie sparen, was gerade in den Back Buffer gezogen wird, dh je nachdem, wo Sie die Funktion aufrufen, sind die Ergebnisse unterschiedlich. Wenn Sie die GUI-Ebene nicht möchten, können Sie den Bildschirm vor dem Rendern der GUI speichern (dh im Draw- Ereignis). Wenn Sie jedoch möchten, dass GUI-Elemente enthalten sind, speichern Sie nach dem Draw-GUI- Ereignis. Sollten Sie mehr Kontrolle benötigen, sollten Sie die Funktion verwenden screen_save_part.

Hinweis: diese Funktion funktioniert nicht für das HTML5-Ziel.


Beispiel:

if keyboard_check_pressed(ord("S"))
   {
   screen_save(working_directory + "\Screens\Screen_"+string(num)+".png")
   num += 1;
   }

Der obige Code erstellt einen Screenshot des Spiels, wenn der Benutzer die Taste "S" drückt. Dieser Screenshot wird in einem bestimmten Verzeichnis gespeichert und mit einer Variablen nummeriert, die dann inkrementiert wird.
