window_get_visible_recects

Suchen Sie den überlappenden Bereich des durch (x1, y1) bis (x2, y2) definierten Rechtecks ​​auf jedem der angeschlossenen Displays.

Syntax:

window_get_visible_rects(x1, y1, x2, y2);


Streit Beschreibung
x1 Die linke Kante des zu überprüfenden Rechtecks
y1 Die obere Kante des zu überprüfenden Rechtecks.
x2 Der rechte Rand des zu überprüfenden Rechtecks
y2 Die untere Kante des zu überprüfenden Rechtecks.


Rückgabe: Array


Beschreibung

Mit dieser Funktion können Sie den überlappenden Bereich des durch (x1, y1) bis (x2, y2) definierten Rechtecks ​​auf jedem der angeschlossenen Displays finden. Die Funktion gibt ein Array mit 8 Werten pro Display zurück (dh wenn Sie zwei Displays haben, hat das Array eine Länge von 16 Indizes), wobei die Werte [0... 3] dem Overlapx1, Overlay1, Overlapx2 entsprechen. overlappy2 - definiert den Überlappungsbereich auf dieser Anzeige und wird auf 0,0,0,0 gesetzt, wenn keine Überlappung - und die Werte [4... 7] entsprechen dem Monitorx1, monitory1, monitorx2, monitory2 - den Koordinaten von die Anzeige im virtuellen Anzeigebereich. Dies kann verwendet werden, um zu testen, ob eine gespeicherte Fensterposition sichtbar sein wird oder nicht (der Benutzer hat möglicherweise einen externen Monitor abgeschaltet oder das Fenster vom Bildschirm verschoben, das die Fensterposition verlassen hat, die als nicht gültig gespeichert wurde).


Beispiel:

var wx = window_get_x();
var wy = window_get_y();
var ww = window_get_width();
var wh = window_get_height();
display_data = window_get_visible_rects(wx, wy, wx + ww, wy + wh);
display_num = array_length_1d(display_data) / 8;

Der obige Code erzeugt ein 1D-Array, das in der Variablen enthalten ist display_data enthält Informationen über die Anzeigen und erstellt die Variable display_num um die Anzahl der gefundenen aktiven Anzeigen zu halten.
