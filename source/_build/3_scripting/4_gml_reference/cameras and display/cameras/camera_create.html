camera_create


Beschreibung

Diese Funktion erzeugt eine "leere" Kamera, dh: eine Kamera ohne definierte Sicht- oder Projektionseinstellungen. Nachdem Sie die Kamera erstellt haben, können Sie die verschiedenen Matrixfunktionen verwenden, um die Ansichts- und Projektionsmatrizen festzulegen. Sie können diese Funktion auch verwenden, um die Kamera zu erstellen und dann die andere zu verwenden camera_set_* Funktionen der Ansicht einzurichten, aber beachten Sie, dass Sie zumindest eine Position und Größen liefern müssen, da sonst die resultierende Kamera nicht gültig sein wird und ungerade Ergebnisse geben, wenn auf einen Blick Port angewendet (wenn Sie das Objekt nicht festlegen folgende, Grenzen oder Bewegungsgeschwindigkeit werden diese im Raum-Editor auf die Standardwerte für eine Kamera gesetzt.

Beachten Sie, dass Kameras wie alle dynamischen Ressourcen sind und bei der Erstellung Speicher belegen. Dies bedeutet, dass Sie darauf achten müssen, keine unnötigen Kameras zu erstellen und diejenigen zu zerstören, die Sie nicht benötigen, indem Sie die Funktion aufrufen camera_destroy(). Dies bedeutet auch, dass Kameras von Raum zu Raum fortbestehen, wenn sie einem zugewiesen werden global Variable (sie bleiben erhalten, wenn sie auch Instanz- oder lokalen Variablen zugewiesen sind, aber Sie können nicht auf sie zugreifen, nachdem Sie den Raum gewechselt haben, daher das Speicherleck), so dass Sie z. B. eine neue Kamera erstellen und einer globalen Variable zuweisen können Legen Sie dann zu Beginn jedes Raums die erforderlichen Ansichts- und Projektionsmatrizen und den Ansichtsanschluss fest, den Sie verwenden müssen.


Syntax:

camera_create();


Kehrt zurück:

Real (unique camera ID value)


Beispiel:

view_camera[0] = camera_create();
var viewmat = matrix_build_lookat(640, 240, -10, 640, 240, 0, 0, 1, 0);
var projmat = matrix_build_projection_ortho(640, 480, 1.0, 32000.0);
camera_set_view_mat(view_camera[0], viewmat);
camera_set_proj_mat(view_camera[0], projmat);

Der obige Code erstellt eine neue Kamera und weist ihre ID der Ansicht [0] zu. Die Ansichts- und Projektionsmatrizen werden dann erstellt und in lokalen Variablen gespeichert, bevor sie zum Einstellen der Kamera verwendet werden.
