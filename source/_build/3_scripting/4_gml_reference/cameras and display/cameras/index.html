Kameras

Wenn Sie Räume in GameMaker Studio 2 erstellen, müssen Sie verschiedene Ansichtsports und / oder Kameras einrichten, um zu steuern, was auf dem Player angezeigt wird. Die Viewports sind im Grunde genommen kleine Fenster in deine Spielwelt, die es dir ermöglichen, die Spielerteile eines Raums anzuzeigen, entweder skaliert oder 1: 1, und als solche sind sie essentiell, wenn dein Spielzimmer größer als die Displaygröße ist. Die Kameras definieren genau, was in jedem Ansichtsport angezeigt wird.

GameMaker Studio 2 erlaubt Ihnen 8 unabhängige View-Ports (nummeriert von 0 - 7) und eine unbegrenzte Anzahl von Kameras, von denen nur 8 gleichzeitig aktiv sein können - eine, die jedem der verfügbaren Ports zugeordnet ist - obwohl Sie normalerweise nur benötigen einen oder zwei View-Ports. Kameras können verschiedene Teile des gleichen Raums anzeigen und können jederzeit aktiviert und deaktiviert sowie Ports zugewiesen werden. Dies bedeutet, dass Sie mit Kameras HUD-Elemente zeichnen oder Split-Screen-Effekte erstellen oder Zwischensequenzen erstellen können Beispiel. Im Wesentlichen positionieren Sie eine Kamera in einem Raum und definieren die „Ansicht“ (Fläche) des Raumes, das es sichtbar sein wird, und dann wird diese im Hinblick auf eine Sichtöffnung gezogen - beachten Sie, dass die Sichtöffnung eine andere Größe sein kann, die Kameraansicht und als solche können Sie die Kameraansicht verzerren und skalieren, wenn sie eine andere Größe als 1: 1 mit dem Ansichtsport hat.

WICHTIG! Es ist leicht zu verwirren, wenn man über Kameras, Ansichten und View-Ports spricht, um zu verdeutlichen:

  • Die Kamera: Ein Punkt innerhalb des Raums, der verwendet wird, um festzulegen, wie der Raum angezeigt wird - normalerweise mit Position, Ausrichtung, Sichtfeld und Seitenverhältnis
  • Die Ansicht: Was die Kamera sieht, basierend auf der Position, Projektion und Drehung der Kamera
  • Der View Port: Der Bereich des Bildschirms, in dem die Kameraansicht angezeigt wird

Wenn Sie Kameras über den Raum-Editor hinzufügen, können Sie den Wert für die Kamera-ID mit der Taste aufrufen view_camera Variable. Sie können dann die Ansicht mit den folgenden Funktionen manipulieren und Sie können die Standardkameras bei Bedarf sogar zerstören, obwohl Sie der Ansicht eine neue Kamera zuweisen müssen, da Sie andernfalls ein sehr unberechenbares Verhalten erhalten. Kameras, die einem Ansichtsport im Raum-Editor hinzugefügt wurden, haben einen globalen Gültigkeitsbereich. Das bedeutet, dass sie einmal beim Start des Spiels erstellt werden. Wenn Sie die Räume betreten, werden sie auf die im Raum-Editor eingestellten Werte gesetzt eine Standardkamera in jedem Raum, wird für alle Räume nicht mehr existieren.

Sie sollten vorsichtig sein, wenn Sie mehrere Kameras verwenden, da das Zeichenereignis für alle Instanzen einmal für jede sichtbare Ansicht aufgerufen wird. Wenn also drei Kameraansichten in einem Raum aktiv sind, wird das Zeichenereignis dreimal pro Schritt ausgeführt die Arbeit), die eine Verlangsamung verursachen kann, wenn das Spiel groß oder komplex ist. Das view_current Die Variable kann dazu verwendet werden, diese Zeichenaufrufe zu begrenzen, indem sie prüft, welche Ansicht gezeichnet wird, und nur Elemente zeichnen, die für einen bestimmten Ansichtsport spezifisch sind. Seien Sie auch vorsichtig, wenn Sie Ihre eigenen Kameras erstellen, als ob Sie eine in einem Raum erstellen und nicht mit der Kamera entfernen camera_destroy() Sie können ein Speicherleck bekommen.

Sie können eine Übersicht über alle verfügbaren Funktionen in den verschiedenen Abschnitten unten finden, aber es ist erwähnenswert, dass einige dieser Funktionen die Einrichtung und Verwendung von verschiedenen Matrizen erfordern, so dass Sie vielleicht den Abschnitt des Handbuchs für die Matrix betrachten möchten Funktionen. Beachten Sie auch, dass es einige Raumfunktionen gibt, die verwendet werden können, um Kameras abzurufen und einzustellen und um Port-Werte in anderen als den aktuellen Räumen anzuzeigen (siehe Erstellen und Ändern von Räumen ).

Anzeigen von Ports und Kameraansichtsvariablen


Wenn Sie mit Kameras arbeiten, müssen Sie sie einem Ansichtsport zuweisen, damit sie "aktiv" sind und alles anzeigen können. Diese Ansichtsports sind von 0 bis 7 nummeriert und können mit verschiedenen variablen Arrays und Funktionen geändert und positioniert werden. Sie können auch prüfen, welche gerade gezeichnet wird oder welche Kamera gerade an sie angeschlossen ist.
