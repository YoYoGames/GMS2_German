Kamera_Anwendung


Beschreibung

Diese Funktion wendet die gegebenen Kameraeinstellungen auf die aktuelle Kameraansicht an, die sofort gerendert wird. Während Sie eine beliebige Kamera unter Verwendung der spezifischen einstellen können camera_set_* Funktionen werden die neuen Einstellungen nicht für das Rendering bis zum nächsten Spielrahmen verwendet, aber mit dieser Funktion können Sie diese Änderungen sofort vornehmen. Wenn Sie mehrere Ansichten verwenden, sollte die Funktion im Draw-Ereignis verwendet werden und Sie können die Variable verwenden view_current um die angegebene Kamera nur selektiv auf die gerade wiedergegebene Kameraansicht anzuwenden. Beachten Sie, dass dies NICHT den Kamera-ID-Wert ändert, sondern nur die Einstellungen für diese Kamera. Wenn Sie diese Funktion für die Standardkamera verwenden (dh die interne Kamera, die GameMaker Studio 2 verwendet, wenn keine Ansichtskameras aktiv sind), werden die Änderungen vorgenommen überschrieb den nächsten Spielrahmen durch die internen Einstellungen zum Zeichnen des Raumes.


Syntax:

camera_apply(camera_id)


Argumente:

Streit Beschreibung
camera_id Der eindeutige Kamera-ID-Wert, der beim Erstellen der Kamera zurückgegeben wurde.


Kehrt zurück:

N/A


Beispiel:

if view_current == 0
   {
   camera_apply(cutscene_cam);
   }

Der obige Code (in einem Draw-Event) überprüft, welche Kameraansicht gerade gerendert wird und wenn es sich um die Kameraansicht [0] handelt, werden die Einstellungen für die Kamera, auf die die Variable "cutscene_cam" verweist, darauf angewendet.
