view_set_surface_id


Beschreibung

Mit dieser Variablen können Sie den Inhalt eines Ansichtsanschlusses so einstellen, dass er auf eine Oberfläche gezeichnet wird. Wenn Sie mit Oberflächen arbeiten, ist es oft erforderlich, dass Sie den gesamten sichtbaren Bereich des Bildschirms an der Oberfläche erfassen, und Sie würden ihn mit dieser Funktion einem Ansichtsport zuweisen. Dies bedeutet, dass alles, was in der gewählten Portansicht angezeigt wird, nun auf die zugewiesene Oberfläche gezeichnet wird und der Inhalt dieses Ansichtsports nicht mehr angezeigt wird. Dies bedeutet, dass Sie entweder Folgendes tun müssen:

  • Aktivieren Sie eine neue Ansicht und zeichnen Sie die Oberfläche nur in dieser Ansicht (mit view_current um zu prüfen, welche Ansicht gerade gezeichnet wird)

  • Zeichnen Sie die Oberfläche im Draw-GUI-Ereignis einer Instanz, da die GUI-Ebene unabhängig von Ansichten ist.

Wenn Sie diese Funktion verwenden, geben Sie den Index des Anzeigeports (von 0 bis 7) und einen Oberflächenindex (entweder den application_surface oder der eindeutige Indexwert, der von der Funktion zurückgegeben wird surface_create ) oder, wenn eine Oberfläche zuvor zugewiesen wurde und Sie sie entfernen möchten, der Wert -1. Weitere Informationen zum Einstellen des Anzeige-Ports auf eine Oberfläche finden Sie auch in der Variablen view_surface_id.


Syntax:

view_set_surface_id(view_port, surf)


Argumente:

Streit Beschreibung
view_port Der View-Port zum Targeting (0 - 7)


Kehrt zurück:

Real


Beispiel:

if view_get_surface_id(0) == -1
   {
   view_set_surface_id(0, global.vSurf);
   }

Der obige Code überprüft, ob dem Ansichtsport [0] eine Oberfläche zugewiesen wurde und wenn dies nicht der Fall ist, wird ihm eine zugewiesen.
