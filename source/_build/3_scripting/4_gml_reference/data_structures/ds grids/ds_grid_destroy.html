ds_grid_destroy


Beschreibung

Diese Funktion entfernt die angegebene Gitterdatenstruktur aus dem Speicher, macht die verwendeten Ressourcen frei und entfernt alle darin enthaltenen Werte. Diese Funktion sollte immer verwendet werden, wenn Sie die Verwendung des ds_grid abgeschlossen haben, um Speicherlecks zu verhindern, die das Spiel verlangsamen und zum Absturz bringen können.

WICHTIG! Wenn Sie eine Datenstruktur erstellen, handelt es sich bei dem zu identifizierenden Indexwert um einen ganzzahligen Wert, der bei 0 beginnt. Dies bedeutet, dass Datenstrukturen verschiedener Typen denselben Indexwert haben können. Wenn Sie also Zweifel haben, sollten Sie die ds_exists Funktion, bevor Sie auf sie zugreifen. Beachten Sie auch, dass Indizes wiederverwendet werden. Der Indexwert einer zerstörten Datenstruktur kann anschließend von einem neu erstellten Index verwendet werden. Es wird daher empfohlen, die Variable, die den DS-Index nach der Zerstörung hatte, immer auf -1 zu setzen.


Syntax:

ds_grid_destroy(index);

Streit Beschreibung
index Diesen Index des Gitters zerstören.


Kehrt zurück:

N/A


Beispiel:

if lives = 0
    {
    ds_grid_destroy(Wall_Grid);
    Wall_Grid = -1;
    room_goto(rm_Menu);
    }

Der obige Code prüft den Wert der eingebauten globalen Variablen "life" und wenn er 0 ist, zerstört er das in der Variablen "Wall_Grid" indizierte ds_grid und ändert dann die Räume.
