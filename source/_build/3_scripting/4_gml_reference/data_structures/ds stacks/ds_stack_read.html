ds_stack_read

Liest eine Stapeldatenstruktur aus einer Zeichenfolge.

Syntax:

ds_stack_read(id, str [, legacy] );


Streit Beschreibung
id Die ID der Datenstruktur, in die eingelesen werden soll.
str Die Zeichenfolge, von der gelesen werden soll.
legacy ( optional ) Kann beides sein true oder false oder ganz weggelassen.


Rückkehr: N / A


Beschreibung

Mit dieser Funktion können Sie einen gespeicherten ds_stack neu erstellen (einen, der zuvor mit Hilfe von... als String geschrieben wurde) ds_stack_write ). Sie müssen zuerst einen neuen ds_stack erstellen, in den die Zeichenfolge eingelesen werden kann. Wenn ds_stack bereits existiert und Informationen darin gespeichert sind, wird dies vor dem Lesen gelöscht. Diese Funktion ist von entscheidender Bedeutung, wenn Sie Sicherungs- / Lade-Mechanismen für Ihr Spiel erstellen. Beachten Sie, dass wenn die Datenstruktur mit früheren Versionen von GameMaker, das optionale Argument "Legacy" hinzugefügt werden sollte true da sich das String-Format mit dieser Version geändert hat.


Beispiel:

stack = ds_stack_create();
ini_open("save.ini");
var str = ini_read_string("Stacks", "0", "");
if str != ""
   {
   ds_stack_read(stack, str);
   }
ini_close();

Der obige Code erstellt einen Stapel und speichert den Index in der Variablen "stack". Es öffnet sich dann ein ini file und liest eine Zeichenkette daraus, um sicherzustellen, dass die Zeichenkette nicht zuerst als leer zurückgegeben wird. Diese Zeichenfolge wird dann in den neu erstellten ds_stack eingelesen.
