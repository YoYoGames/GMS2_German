Datenstrukturen


In Spielen müssen Sie Informationen häufig genau und geordnet speichern. Beispielsweise müssen Sie möglicherweise Listen mit Elementen speichern, die eine Person bei sich trägt, oder Sie möchten ein Raster von Orten speichern, die noch besucht werden müssen. Jetzt können Sie die Arrays für diese, aber wenn Sie kompliziertere Operationen tun wollen, wie die Daten sortieren oder für einen bestimmten Artikel suchen, müssen Sie große Stücke von GML Code schreiben, die langsam sein kann auszuführen, schwierig zu debuggen und ein echter Schmerz zu schreiben!

Um dies zu beheben, verfügt GameMaker Studio 2 über eine Reihe von integrierten Datenstrukturen, auf die über spezielle Funktionen zugegriffen werden kann. Es gibt sechs verschiedene Arten von Datenstrukturen, von denen jede ihre eigenen Vorteile hat, abhängig von der Art der Informationen, die Sie speichern möchten, und davon, wie Sie sie später bearbeiten möchten: Stacks, Warteschlangen, Listen, Karten, Prioritätswarteschlangen und Raster.

Im Wesentlichen funktionieren alle Datenstrukturen auf dieselbe Weise - Sie erstellen eine Datenstruktur und speichern ihren Indexwert in einer Variablen. Mit diesem Index referenzieren Sie dann die Datenstruktur in allen weiteren Funktionsaufrufen, die Operationen daran ausführen können. Wenn Sie fertig sind, zerstören Sie die Datenstruktur erneut, um sie aus dem Speicher zu entfernen. Sie können beliebig viele Strukturen gleichzeitig verwenden, und alle Strukturen können sowohl Strings als auch reelle Werte speichern.

HINWEIS: Wie bei allen dynamischen Ressourcen beanspruchen Datenstrukturen Speicherplatz und sollten daher immer dann zerstört werden, wenn sie nicht mehr benötigt werden, um Speicherlecks zu verhindern, die das Spiel verlangsamen und schließlich zum Absturz bringen.
WICHTIG! Wenn Sie eine Datenstruktur erstellen, handelt es sich bei dem zu identifizierenden Indexwert um einen ganzzahligen Wert, der bei 0 beginnt. Dies bedeutet, dass verschiedene Datenstrukturen den gleichen Indexwert haben können. Wenn Sie also Zweifel haben, sollten Sie die ds_exists Funktion, bevor Sie auf sie zugreifen. Beachten Sie auch, dass Indizes wiederverwendet werden. Der Indexwert einer zerstörten Datenstruktur kann anschließend von einem neu erstellten Index verwendet werden. Wir empfehlen, dass Sie jede Variable, die einen DS-Index enthält, auf -1 setzen, nachdem Sie die Datenstruktur zerstört haben.

Bevor Sie Datenstrukturen in Ihrem Spiel verwenden, sollten Sie sich auch bewusst sein, dass es bestimmte Momente gibt, in denen aufgrund von Rundungsfehlern ein Ergebnis erzielt werden kann, das nicht Ihren Erwartungen entspricht. Dies kann durch Ändern der DS-Genauigkeit mithilfe der folgenden Funktion behoben werden:

  • ds_set_precision

Es gibt auch eine spezielle Funktion, um zu prüfen, ob eine Datenstruktur eines bestimmten Typs vorhanden ist:

  • ds_exists
