DS-Listen

Eine ds_list speichert eine Sammlung von Werten in einer bestimmten Reihenfolge.

Eine ds_list ist eine Datenstruktur, die Informationen sequentiell speichert, wenn sie hinzugefügt werden (ähnlich wie ein 1D-Array ). DS-Listen sind sehr flexible Datenstrukturen, mit denen Sie am Ende Werte hinzufügen oder sie irgendwo in der Mitte der Liste einfügen können. Außerdem haben Sie die Möglichkeit, die Reihenfolge zu mischen (Randomisierung der Werte) oder in aufsteigender Reihenfolge zu sortieren oder absteigende Reihenfolge.

Wenn Sie auf DS-Listen-Datenstrukturen zugreifen, sollten Sie immer versuchen, ganzzahlige Werte für die Listenposition zu verwenden, und alle nicht-ganzzahligen Indizes werden von GameMaker Studio 2 mit einem GameMaker Studio 2 wenn Sie dies nicht tun. Wenn Sie das nicht benötigen, müssen Sie die Rundung selbst durchführen, bevor Sie den Index übergeben, den Sie überprüfen möchten.

HINWEIS: Wie bei allen dynamischen Ressourcen belegen auch Datenstrukturen Speicherplatz und sollten daher immer dann zerstört werden, wenn sie nicht mehr benötigt werden, um Speicherlecks zu vermeiden, die Ihr Spiel verlangsamen und schließlich zum Absturz bringen.

Die folgenden Funktionen existieren, die sich mit DS-Listen befassen.

  1. ds_list_create
  2. ds_list_destroy
  3. ds_list_clear
  4. ds_list_empty
  5. ds_list_größe
  6. ds_list_add
  7. ds_list_set
  8. ds_list_delete
  9. ds_list_find_index
  10. ds_list_find_value
  11. ds_list_insert
  12. ds_list_replace
  13. ds_list_shuffle
  14. ds_list_sort
  15. ds_list_copy
  16. ds_list_read
  17. ds_list_write

Ds_lists können auch verwendet werden, ds_maps und andere ds_lists, aber nur für die Verwendung in Verbindung mit dem json_encode Funktion. Dazu müssen Sie den Eintrag in der Liste "markieren" oder "markieren", damit er mit den folgenden Sonderfunktionen korrekt codiert werden kann:

  1. ds_list_mark_as_list
  2. ds_list_mark_as_map

Abgesehen von diesen spezifischen Funktionen können Sie alle einen Ausdruck (einen Accessor genannt ) verwenden, um den Inhalt Ihrer hinzufügen oder ändern ds_list. Dieser Accessor ähnelt einem 1D-Array mit der folgenden Syntax:

list_index[| index]

Weitere Informationen und Beispiele finden Sie auf der GML-Übersichtsseite von Accessors.

HINWEIS: Wenn Sie prüfen müssen, ob eine Datenstruktur vorhanden ist, können Sie die ds_exists() Funktion.
