DS Karten

Eine ds_map ist eine Datenstruktur, die Daten in verknüpften Paaren speichert, die aus einem Schlüssel und einem entsprechenden Wert bestehen.

Die ds_map Datenstruktur ist ein außerordentlich nützliches wie es Sie speichern Schlüssel-Wert - Paare können. Zum Beispiel kann ein Charakter in Ihrem Spiel eine Menge verschiedener Gegenstände ( Schlüssel ) haben und für jeden einzelnen Gegenstand kann er eine Anzahl von ihnen ( Werte ) haben, wie in einem RPG, wo Sie 10 Gesundheitstränke, 5 Manatränke und 100 Gold. Maps verwalten solche Paare alle zusammen an einem Ort, und Sie können Paare zu der Karte hinzufügen und nach dem Wert suchen, der bestimmten Schlüsseln ebenso mit einigen einfachen Funktionen entspricht. Es gibt jedoch ein paar Dinge, die Sie über Karten wissen sollten, bevor Sie sie verwenden! Maps werden nicht in irgendeiner (erkennbaren) Weise sortiert, was bedeutet, dass Sie, um einen bestimmten Schlüssel zu finden, das ganze Ding durchlaufen müssen (was sehr langsam ist). Es gibt auch keine Möglichkeit, zwei Tasten zu halten, die gleich sind, und auch keine Taste zwei Werten zuzuweisen.

HINWEIS: Die ds_map-Funktionen haben sich seit früheren Versionen von GameMaker geändert, sodass importierte Spiele möglicherweise nicht wie erwartet funktionieren oder sich verhalten.

HINWEIS: Wie bei allen dynamischen Ressourcen belegen auch Datenstrukturen Speicherplatz und sollten daher immer dann zerstört werden, wenn sie nicht mehr benötigt werden, um Speicherlecks zu vermeiden, die Ihr Spiel verlangsamen und schließlich zum Absturz bringen.

Die folgenden Funktionen existieren, die sich mit DS-Karten befassen.

  1. ds_map_exists
  2. ds_map_create
  3. ds_map_add
  4. ds_map_clear
  5. ds_map_copy
  6. ds_map_replace
  7. ds_map_delete
  8. ds_map_empty
  9. ds_map_größe
  10. ds_map_find_first
  11. ds_map_find_last
  12. ds_map_find_next
  13. ds_map_find_previous
  14. ds_map_find_value
  15. ds_map_read
  16. ds_map_write
  17. ds_map_destroy

Zum Speichern und Laden einer ds_map stehen vier weitere Funktionen zur Verfügung. Diese Funktionen verschlüsseln die Karte und speichern sie an einem sicheren Ort auf der Zielplattform. Sie können damit In App Purchase Details, Passwörter, sensible Spielinformationen usw. speichern.

  1. ds_map_secure_save
  2. ds_map_secure_save_buffer
  3. ds_map_secure_load
  4. ds_map_secure_load_buffer

Abgesehen von diesen spezifischen Funktionen können Sie alle einen Ausdruck (einen Accessor genannt ) verwenden, um den Inhalt Ihrer hinzufügen oder ändern ds_map. Dieser Accessor ähnelt einem 1D-Array mit der folgenden Syntax:

map_index[? key]

Weitere Informationen und Beispiele finden Sie auf der GML-Übersichtsseite von Accessors.

JSON Und Ds_maps

Eine der Verwendungen von ds_maps ist die Arbeit mit JSON. Daher gibt es einige spezielle Funktionen, die spezifisch sind:

  1. ds_map_add_list
  2. ds_map_add_map
  3. ds_map_replace_list
  4. ds_map_replace_map

Es gibt auch ein paar zusätzliche Funktionen für ds_lists:

  1. ds_list_mark_as_list
  2. ds_list_mark_as_map

HINWEIS: Mit diesen Funktionen können Sie zwar Listen und Maps innerhalb einer Map hinzufügen, sie sind jedoch für nichts anderes als JSON nutzlos, und verschachtelte Maps und Listen werden nicht korrekt gelesen, wenn sie auf die Festplatte geschrieben oder auf andere Weise aufgerufen werden.

HINWEIS: Wenn Sie prüfen müssen, ob eine Datenstruktur vorhanden ist, können Sie die ds_exists() Funktion.
