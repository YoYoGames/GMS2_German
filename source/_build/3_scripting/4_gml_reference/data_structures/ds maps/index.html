DS-Karten

Eine ds_map ist eine Datenstruktur, in der Daten in verknüpften Paaren gespeichert werden, die aus einem Schlüssel und einem entsprechenden Wert bestehen.

Die ds_map Datenstruktur ist ein außerordentlich nützliches wie es Sie speichern Schlüssel-Wert - Paare können. Zum Beispiel kann ein Charakter in Ihrem Spiel eine Menge verschiedener Gegenstände ( Schlüssel ) haben, und für jeden einzelnen Gegenstand kann er eine Anzahl von ihnen ( Werte ) haben, wie in einem RPG, in dem Sie 10 Heiltränke, 5 Manatränke und 100 Gold. Karten behalten diese Paare alle zusammen an einem Ort, und Sie können der Karte Paare hinzufügen und mit einigen einfachen Funktionen nach dem Wert suchen, der bestimmten Tasten entspricht. Es gibt jedoch ein paar Dinge, die Sie über Karten wissen sollten, bevor Sie sie verwenden. Karten werden nicht auf eine (erkennbare) Weise sortiert, was bedeutet, dass Sie zum Suchen eines bestimmten Schlüssels möglicherweise das gesamte Objekt durchlaufen müssen (was sehr langsam ist). Es gibt auch keine Möglichkeit, zwei Tasten zu halten, die gleich sind, und Sie können auch nicht zwei Schlüssel zuweisen.

HINWEIS: Die Funktionen von ds_map haben sich seit den vorherigen Versionen von GameMaker geändert, sodass importierte Spiele möglicherweise nicht wie erwartet funktionieren oder sich verhalten.

HINWEIS: Wie bei allen dynamischen Ressourcen beanspruchen Datenstrukturen Speicherplatz und sollten daher immer dann zerstört werden, wenn sie nicht mehr benötigt werden, um Speicherlecks zu verhindern, die das Spiel verlangsamen und schließlich zum Absturz bringen.

Es gibt folgende Funktionen, die sich mit DS-Karten befassen.

  1. ds_map_exists
  2. ds_map_create
  3. ds_map_add
  4. ds_map_clear
  5. ds_map_copy
  6. ds_map_replace
  7. ds_map_delete
  8. ds_map_empty
  9. ds_map_size
  10. ds_map_find_first
  11. ds_map_find_last
  12. ds_map_find_next
  13. ds_map_find_previous
  14. ds_map_find_value
  15. ds_map_set
  16. ds_map_read
  17. ds_map_write
  18. ds_map_destroy

Es gibt vier weitere Funktionen zum Speichern und Laden einer ds_map. Diese Funktionen verschlüsseln die Karte und speichern sie an einem sicheren Ort auf der Zielplattform. So können Sie diese Informationen zum Speichern von In-App-Kaufdetails, Kennwörtern, vertraulichen Spielinformationen usw. verwenden.

  1. ds_map_secure_save
  2. ds_map_secure_save_buffer
  3. ds_map_secure_load
  4. ds_map_secure_load_buffer

Abgesehen von diesen spezifischen Funktionen können Sie alle einen Ausdruck ( Accessor genannt ) verwenden, um den Inhalt Ihres Moduls hinzuzufügen oder zu ändern ds_map. Dieser Accessor ähnelt einem 1D-Array mit der folgenden Syntax:

map_index[? key]

Weitere Informationen sowie Beispiele finden Sie auf der Seite "GML-Übersicht" unter " Accessoren".

JSON und Ds_maps

Ds_maps wird zum Beispiel bei der Arbeit mit JSON verwendet. Daher gibt es einige spezielle Funktionen, die spezifisch sind:

  1. ds_map_add_list
  2. ds_map_add_map
  3. ds_map_replace_list
  4. ds_map_replace_map

Es gibt auch ein paar kostenlose Funktionen für ds_lists:

  1. ds_list_mark_as_list
  2. ds_list_mark_as_map

ANMERKUNG: Diese Funktionen erlauben zwar das Hinzufügen von Listen und Karten zu einer Karte, sie sind jedoch für andere Zwecke als JSON unbrauchbar. Verschachtelte Karten und Listen werden nicht richtig gelesen, wenn sie auf die Festplatte geschrieben oder auf andere Weise aufgerufen werden.

HINWEIS: Wenn Sie prüfen müssen, ob eine Datenstruktur vorhanden ist, können Sie das verwenden ds_exists() Funktion.
