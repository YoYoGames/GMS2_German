ds_map_read

Liest die Kartendatenstruktur aus einer Zeichenfolge.

Syntax:

ds_map_read(id, str [, legacy]);


Streit Beschreibung
id Die ID der Datenstruktur, in die die Zeichenfolge gelesen werden soll
str Die zu lesende Zeichenfolge
legacy ( optional ) Kann beides sein true oder false oder ganz weggelassen.


Rückkehr: N / A.


Beschreibung

Diese Funktion nimmt eine Zeichenfolge an, die zuvor von der Funktion erstellt wurde ds_map_write und dann lese es in eine zuvor erstellte ds_map. Wenn die Map, in die die Zeichenfolge eingelesen wird, Schlüssel / Wert-Paare enthält, werden diese zuerst gelöscht, bevor die gespeicherte Map neu erstellt wird. Beachten Sie, dass wenn die Datenstruktur mit früheren Versionen von GameMaker Studio 2 erstellt wurde, das optionale Argument "Legacy" hinzugefügt werden muss true da sich das String-Format mit dieser Version geändert hat.


Beispiel:

inventory = ds_map_create();
ini_open("map.ini");
var t_string = ini_read_string("Saved", "0", "");
if t_string != ""
   {
   ds_map_read(inventory, t_string);
   }
ini_close();

Der obige Code erstellt eine neue ds_map und speichert seinen ID-Index in der Variablen "inventory". Es öffnet dann eine Ini-Datei und liest eine Zeichenfolge aus dieser Datei in die temporäre Variable "t_string". Schließlich überprüft es, ob die Zeichenfolge gültig ist (nicht der Standardini-Wert von ""). Wenn dies der Fall ist, liest es die Zeichenfolge in die neu erstellte ds_map, bevor das Ini wieder geschlossen wird.
