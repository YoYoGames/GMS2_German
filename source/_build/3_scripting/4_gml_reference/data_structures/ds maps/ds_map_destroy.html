ds_map_destroy

Zerstört eine bestimmte Karte dauerhaft und entfernt sie aus dem Speicher.

Syntax:

ds_map_destroy(id);


Streit Beschreibung
id Die ID der zu zerstörenden Karte.


Rückgabe: N / A.


Beschreibung

Ds_maps belegen Speicherplatz, der ihnen bei ihrer Erstellung zugewiesen wird. Das bedeutet, dass Sie diesen Speicher auch freigeben müssen, wenn ds_map nicht benötigt wird, um Fehler, Speicherlecks und Leistungsverlust beim Ausführen des Spiels zu vermeiden. Diese Funktion macht genau das.

WICHTIG! Wenn Sie eine Datenstruktur erstellen, handelt es sich bei dem zu identifizierenden Indexwert um einen ganzzahligen Wert, der bei 0 beginnt. Dies bedeutet, dass Datenstrukturen verschiedener Typen denselben Indexwert haben können. Wenn Sie also Zweifel haben, sollten Sie die ds_exists Funktion, bevor Sie auf sie zugreifen. Beachten Sie auch, dass Indizes wiederverwendet werden. Der Indexwert einer zerstörten Datenstruktur kann anschließend von einem neu erstellten Index verwendet werden. Es wird daher empfohlen, die Variable, die den DS-Index nach der Zerstörung hatte, immer auf -1 zu setzen.


Beispiel:

ds_map_destroy(inventory);
inventory = -1;

Der obige Code zerstört die ds_map mit der in der Variablen "inventory" indizierten ID.
