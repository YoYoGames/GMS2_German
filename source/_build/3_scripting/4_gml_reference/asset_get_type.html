Asset_get_type


Beschreibung

Mit dieser Funktion können Sie den Typ des Objekts, auf das verwiesen wird, aus seinem Namen (einer Zeichenfolge) abrufen. Eine der unten aufgeführten Konstanten wird zurückgegeben.


Syntax:

asset_get_type(name);

Streit Beschreibung
name Der Name des Spiel-Assets, um den Typ (eine Zeichenfolge) zu erhalten.


Kehrt zurück:

Constant

Konstante Beschreibung
asset_object Der angegebene Name bezieht sich auf ein Objekt.
asset_sprite Der angegebene Name bezieht sich auf ein Sprite.
asset_sound Der angegebene Name bezieht sich auf einen Sound.
asset_room Der Vorname bezieht sich auf einen Raum.
asset_tile Der angegebene Name bezieht sich auf einen Fliesensatz.
asset_path Der angegebene Name bezieht sich auf einen Pfad.
asset_script Der angegebene Name bezieht sich auf ein Skript.
asset_font Der angegebene Name bezieht sich auf eine Schriftart.
asset_timeline Der angegebene Name bezieht sich auf eine Zeitlinie.
asset_shader Der angegebene Name bezieht sich auf einen Shader.
asset_unknown Der angegebene Name bezieht sich auf ein Asset, das entweder nicht existiert oder nicht zu den oben aufgeführten gehört.


Beispiel:

if asset_get_type("pth_Path_" + string(global.Game)) == asset_unknown
   {
   show_debug_message("Path doesn't exist!!!");
   }
else
   {
   path_index = asset_get_index("pth_Path_" + string(global.Game));
   }

Der obige Code überprüft einen dynamisch erstellten Asset-Namen, um festzustellen, ob das Asset vom richtigen Typ ist. Wenn dies nicht der Fall ist, wird eine Debug-Nachricht angezeigt, andernfalls wird der Asset-Name verwendet, um das Asset der Instanz zuzuweisen.
