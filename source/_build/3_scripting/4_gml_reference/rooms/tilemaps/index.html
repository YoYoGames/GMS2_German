Tilemap Layer Funktionen

Der GameMaker Studio 2 Room Editor ermöglicht das Hinzufügen von Kacheln zu einem beliebigen Raum mithilfe von Ebenen. Wie der Name schon sagt, ist eine Kachelkartenebene einfach eine Sammlung von Kacheln, die alle innerhalb des Raums dieselbe Tiefe haben. Durch das Stapeln von Ebenen können Sie einige Objekte über oder unter andere ziehen. Sie können bestimmte Aspekte von Layern auch über Code steuern, Dinge hinzufügen oder entfernen oder bestimmte Eigenschaften des Layers oder dessen Eigenschaften zur Laufzeit ändern. Beachten Sie, dass im Raum Editor Sie eine tilemap pro Schicht begrenzt sind (die Tile Layer), aber wenn sie mit ihnen im Code arbeiten, können Sie mehrere tilemaps zu einer einzigen Schicht zugeordnet haben, wobei jede Karte eine eigene eindeutige ID und Eigenschaften, damit:

  • Das Kachel - Set ist die Ressource, die Sie Kachel verwenden Schichten und tilemaps erstellen
  • Die Fliesenschicht ist, was Sie im Zimmer Editor erstellen tilesets hinzuzufügen
  • Die Kachelkarte bezeichnet die Sammlung von Kacheln, die einer Ebene entweder im Raumeditor oder über Code als einzelnes Element auf dieser Ebene hinzugefügt werden

Kachelfunktionen

Neben den oben aufgeführten Funktionen zum Festlegen von Tilemaps stehen Ihnen auch eine Reihe von Funktionen zur Verfügung, mit denen Sie die Daten einzelner Kacheln bearbeiten können. Ein "Blob" von Kachel-Daten besteht aus dem Kachel-Index plus ein paar zusätzlichen Bits, um zu zeigen, ob die Kachel gedreht, umgedreht oder gespiegelt wurde. Sie haben auch die Möglichkeit, Ihre eigenen Bitmasken zu den Kacheldaten hinzuzufügen, obwohl dies nur für fortgeschrittene Benutzer ist, da Sie die für den Kachelindex reservierten Bits maskieren - im Wesentlichen sind 19 Bits für Ihre Kachelindizes reserviert, aber wenn Sie verwenden nur 8 von ihnen, dann haben Sie noch 11 Bits übrig, mit denen Sie eine benutzerdefinierte Maske erstellen können. Diese Maske kann dann in Code verwendet und überprüft werden, um benutzerdefinierte Fliesenkollisionen oder was auch immer zu erstellen. Normalerweise müssten Sie jedoch keine eigenen tiledata-Blobs erstellen und würden stattdessen die Funktion verwenden tilemap_get um die tiledata zu erhalten, dann würden Sie sie mit den unten stehenden Funktionen manipulieren, und dann würden Sie sie erneut mit verwenden tilemap_set.
