Tilemap-Layer-Funktionen

Mit dem Room Editor von GameMaker Studio 2 können Sie Kacheln mithilfe von Ebenen in einen beliebigen Raum einfügen. Wie der Name schon sagt, handelt es sich bei einer Tilemap-Ebene einfach um eine Sammlung von Kacheln, die alle innerhalb des Raums dieselbe Tiefe haben. Durch das Stapeln von Ebenen können Sie einige Elemente über oder unter anderen zeichnen. Sie können bestimmte Aspekte von Layern auch über den Code steuern, Elemente hinzufügen oder entfernen oder bestimmte Eigenschaften des Layers oder dessen Inhalt zur Laufzeit ändern. Beachten Sie, dass Sie im Raumeditor auf eine Tilemap pro Layer ( Tile-Layer ) beschränkt sind. Wenn Sie jedoch im Code mit ihnen arbeiten, können Sie mehrere Tilemaps einem einzigen Layer zuweisen, wobei jede Map ihre eigene eindeutige ID und Eigenschaften besitzt. so:

  • Das Kachel - Set ist die Ressource, die Sie Kachel verwenden Schichten und tilemaps erstellen
  • Die Fliesenschicht ist, was Sie im Zimmer Editor erstellen tilesets hinzuzufügen
  • Die Tilemap bezeichnet die Sammlung von Kacheln, die einer Ebene entweder im Raumeditor oder durch Code als einzelnes Element auf dieser Ebene hinzugefügt werden

Kachelfunktionen

Neben den oben aufgeführten Funktionen zum Festlegen von Tilemaps verfügen Sie über eine Reihe von Funktionen, mit denen Sie die Daten einzelner Kacheln bearbeiten können. Ein "Fleck" von Kacheldaten besteht aus dem Kachelindex und ein paar zusätzlichen Bits, um anzuzeigen, ob die Kachel gedreht, gespiegelt oder gespiegelt wurde. Sie haben auch die Möglichkeit, den Kacheldaten eine eigene Bitmaske hinzuzufügen. Dies gilt jedoch nur für fortgeschrittene Benutzer, da Sie die für den Kachelindex reservierten Bits maskieren. Im Wesentlichen sind 19 Bits für Ihre Kachelindizes reserviert Sie verwenden nur 8 davon, dann haben Sie noch elf Bits, mit denen Sie eine benutzerdefinierte Maske erstellen können. Diese Maske kann dann verwendet und im Code eingecheckt werden, um benutzerdefinierte Kacheln oder andere Kollisionen zu erstellen. Normalerweise müssen Sie jedoch keine eigenen Tiledata-Blobs erstellen, sondern verwenden stattdessen die Funktion tilemap_get Um die Tiledata zu erhalten, müssen Sie sie mit den unten stehenden Funktionen bearbeiten und dann erneut mit festlegen tilemap_set.
