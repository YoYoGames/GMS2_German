tilemap_set_global_mask

Beschreibung

Diese Funktion kann verwendet werden, um die Kachel-Bitmaske für alle Fliesenkarten im Spiel festzulegen. Die Idee ist, dass zum Beispiel ziemlich kleine Tilesets von 16 x 16 Tiles insgesamt 256 verschiedene Tiles in jedem Tileset haben, die nur 8 Bit Ihres Indexes in den Tiledaten "Blob" (von verfügbare 19 Datenbits). Diese Bits können "maskiert" werden und dann können die übrigen Kacheldaten verwendet werden, um zusätzliche Werte zu speichern.

Eine einfache Möglichkeit, den anfänglichen Indexmaskenwert zu erhalten, besteht darin, die Anzahl der Kacheln im Kachelsatz zu übernehmen und 1 zu subtrahieren. Ein 64x64-Kachelsatz hat also 256 Kacheln und einen Maskenwert von 255 (oder $ff in hexadezimal) Der Fliesensatz enthält eine Anzahl von Kacheln, die keine Zweierpotenz sind, und rundet dann auf die nächste Potenz von Zwei, um die Maske zu erhalten. Ein Kachelsatz mit 20x20 hat beispielsweise 400 Kacheln, die Sie auf 512 runden und So erhalten Sie einen Maskenindexwert von 511 ( $1ff in Hexadezimal). Abgesehen von der Indexmaske möchten Sie normalerweise auch die Werte für Spiegeln, Drehen und Spiegeln beibehalten, die einer Kachel zugewiesen sind. Um das zu erreichen, können Sie einige wenige Konstanten verwenden. Diese können zusammen mit dem Indexmaskenwert "oder" eingegeben werden, um diese Datenbits zu erhalten. Die folgenden Konstanten können verwendet werden (siehe folgenden Beispielcode zur Verwendung):
Konstante Beschreibung
tile_rotate Wird verwendet, um das Rotationsbit eines Kachel-Datenblobs zu setzen / zu erhalten.
tile_mirror Wird verwendet, um das Spiegelungsbit eines Kachel-Datenblobs zu setzen / zu erhalten.
tile_flip Wird verwendet, um das Flip-Bit eines Kacheldatenblobs zu setzen / zu erhalten.
tile_index_mask Eine spezielle Konstante für "und" mit dem Kachel-Daten-Blob, um den Kachel-Index zu extrahieren.


Die Tilemap-Maskierung ist eine erweiterte Funktion, mit der Sie Bits der Kacheldaten "Blob" für Ihre eigenen Zwecke verwenden können. Zum Beispiel könnte der Benutzer ein Bit reservieren, um "fest" in der Plattenstruktur zu bezeichnen, und dann kann dies auf schnelle Kollisionen überprüft werden. Im Grunde wird die Bitmaske gegen die Kacheldaten "gewendet", wenn die Kachelkarte gezeichnet wird (so dass sie keine der anderen Funktionen beeinflusst), was bedeutet, dass Bits, die nicht in der Maske sind, ignoriert werden, aber trotzdem können vom Benutzer für seine eigenen Zwecke gelesen und beschrieben werden. Beachten Sie, dass es zwei verschiedene Masken gibt - eine globale und eine pro Karte - und diese werden von GameMaker Studio 2 vor dem Zeichnen intern "und" zusammengefügt.


Syntax:

tilemap_set_global_mask(mask)


Argumente:

Streit Beschreibung
mask Der zu verwendende Maskenwert


Kehrt zurück:

N/A


Beispiel:

var mask = tile_mirror | tile_flip | tile_rotate | 255;
tilemap_set_global_mask(mask);

Der obige Code setzt die globale Tilemap-Bitmaske auf 8 Bits, wobei die Werte für Spiegeln, Spiegeln und Drehen für jede Kachel in der Karte beibehalten werden.
