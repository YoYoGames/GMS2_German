Allgemeine Layerfunktionen

Mit dem Room Editor von GameMaker Studio 2 können Sie Ihrem Raum Schichten verschiedener Assets hinzufügen. Dies können Tilemaps (mit Tilesets), Hintergründe, Sprites oder Instanzen sein. Beachten Sie jedoch, dass Sie im Gegensatz zum Raumeditor nicht auf einen einzelnen Asset-Typ pro Layer beschränkt sind und mehrere Asset-Typen demselben Layer hinzufügen können. Dies ist beispielsweise möglich haben einen Hintergrund und eine Tilemap, die sich alle auf derselben Ebene befinden. In diesem Abschnitt werden die Funktionen beschrieben, die beim Arbeiten mit Instanz-Layern verwendet werden, sowie einige allgemeine Funktionen für die Verwendung über alle Layer hinweg:

  • layer_get_id
  • layer_get_depth
  • layer_get_id_at_depth
  • layer_get_name
  • layer_get_all
  • layer_get_all_elements
  • layer_get_element_layer
  • layer_get_element_type
  • layer_get_x
  • layer_get_y
  • layer_get_hspeed
  • layer_get_vspeed
  • layer_create
  • layer_destroy
  • layer_x
  • layer_y
  • layer_hspeed
  • layer_vspeed
  • layer_add_instance
  • layer_has_instance
  • layer_instance_get_instance
  • layer_element_move
  • layer_set_visible
  • layer_get_visible
  • layer_depth
  • layer_force_draw_depth
  • layer_is_draw_depth_forced
  • layer_get_forced_depth
  • layer_script_begin

  • layer_get_script_begin

  • layer_get_shader

Wir haben auch einige zusätzliche Funktionen für das Targeting von Layern in einem anderen Raum. Es kann sein, dass Sie etwas in einem Raum ändern möchten, der nicht der aktuelle Raum ist, oder Sie möchten Räume prozedural generieren oder eine beliebige Anzahl von Dingen. Um dies zu ermöglichen, verwenden wir folgende Funktionen:

  • layer_set_target_room
  • layer_get_target_room
  • layer_reset_target_room

Hier stellen Sie den Zielraum mit der entsprechenden Funktion ein und rufen dann die Layer-Funktionen auf. Diese Layer-Funktionen gelten jetzt nur für den Zielraum und nicht für den aktuellen. Wenn Sie fertig sind, rufen Sie die Reset-Funktion auf, um den Code-Bereich wieder in den aktuellen Raum zu bringen.

WARNUNG! Wenn Sie auf einen anderen Raum abzielen, können Sie alle regulären Ebenenfunktionen verwenden, außer Sie können keine Instanzen mit erstellen instance_create_layer oder instance_create_depth noch wird die Schicht funktionieren layer_add_instance verfügbar sein.


Und dies ist ausschließlich für Instanzen auf Ebenen vorgesehen (obwohl die Ebene selbst andere Elemente enthalten kann, die von den folgenden Funktionen nicht beeinflusst werden), verfügen Sie über eine Reihe von Funktionen, mit denen alle Instanzen deaktiviert oder erneut aktiviert werden können:

  • instance_deactivate_layer
  • instance_activate_layer
