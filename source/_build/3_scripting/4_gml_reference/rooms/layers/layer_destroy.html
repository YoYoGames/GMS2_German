layer_destroy


Beschreibung

Diese Funktion zerstört die angegebene Ebene. Sie geben die Layer-ID an (die Sie erhalten, wenn Sie die Ebene mit erstellen layer_create() ) oder der Layername (als String - dies wirkt sich auf die Leistung aus) und entfernt es aus dem aktuellen Raum. Wenn es sich bei der Ebene um eine Ebene handelt, die im Raumeditor erstellt wurde, wird die Ebene beim nächsten Verlassen des Raums mit dem ursprünglichen Inhalt erneut erstellt. Wenn der Raum jedoch dauerhaft ist, wird die Ebene entfernt, es sei denn die Raumpersistenz ist wieder ausgeschaltet. Wenn du eine Ebene auf diese Weise zerstörst, werden alle ihre Inhalte ebenfalls entfernt, sodass alle Referenz-IDs für Hintergründe oder Tilemaps usw. nicht mehr gültig sind und alle Instanzen, die der Ebene zugewiesen sind, zerstört werden (indem sie ihr Destroy-Ereignis ausführen) zur gleichen Zeit, wenn sie einen haben, sowie das Clean Up Event).


Syntax:

layer_destroy(layer_id)


Argumente:

Streit Beschreibung
layer_id Der eindeutige ID-Wert des zu löschenden Layers (oder der Layername als String)


Kehrt zurück:

N/A


Beispiel:

if !instance_exists(obj_Bullet_Parent)
   {
   layer_destroy(global.Bullet_Layer);
   }

Der obige Code überprüft, ob Instanzen des Objekts "obj_Bullet_Parent" vorhanden sind, und wenn dies nicht der Fall ist, wird der Layer mit der in der globalen Variablen gespeicherten ID zerstört.
