Sprites


Sprites sind im Allgemeinen die visuellen Darstellungen von Objekten in dem Spiel, das Sie erstellen, obwohl sie für viele andere Zwecke verwendet werden können. Ein Sprite ist entweder ein einzelnes Bild oder ein Satz von Bildern, die, wenn sie nacheinander abgespielt werden, wie eine sich wiederholende Animation aussehen. Sprites können in vielen Formaten erstellt werden, wobei das "traditionelle" Sprite aus Frames von Pixelbildern besteht, die als in GameMaker Studio 2 importiert werden *.png Dateien. Sie können jedoch auch Vektor- Sprites importieren, die im gespeichert wurden *.swf Format oder Skelettanimationssprites, die als gespeichert wurden *.json Dateien mit beiliegenden *.atlas und *.png Dateien (weitere Informationen zu diesen Arten von Bilddateien und zum Importieren in GameMaker Studio 2 Sie im Abschnitt Importieren von Nicht-Bitmap-Sprites ).

Da Sprites ein wesentlicher Bestandteil eines Spiels sind, verfügt GameMaker Studio 2 über eine Vielzahl von Funktionen, mit denen Sie Informationen zu bestimmten Sprites abrufen oder die im Spiel befindlichen ändern oder sogar eigene Sprites in eine Anwendung laden können aus einer Quellbilddatei.

Die folgenden Funktionen können verwendet werden, um Informationen zu einem Sprite-Asset abzurufen. Bitte beachten Sie, dass diese Funktionen die Informationen vom Basis- Sprite (dh dem Sprite-Asset) und nicht vom Sprite erhalten, wie es in einer Instanz verwendet wird. Beispielsweise können Sie Instanzvariable verwenden sprite_width um die Breite des Sprites zu ermitteln, das eine Instanz verwendet, und diese Breite berücksichtigt die eventuell angewendete Skalierung. Wenn Sie jedoch die Breite eines Sprites wie gewohnt wissen müssen, ohne dass Transformationen angewendet werden, müssen Sie die sprite_get_width() Funktion unten. Weitere Informationen über das Sprite sehen Instanzvariablen im Abschnitt Instanzvariablen.

Dies bedeutet auch, dass Sie sorgfältig darauf achten müssen, dass ein Sprite vorhanden ist, bevor Sie es referenzieren, oder sicherstellen, dass es nicht vor dem Ändern referenziert wird.
