sprite_add


Syntax:

sprite_add(fname, imgnumb, removeback, smooth, xorig, yorig);

Streit Beschreibung
fname Der Name (ein String-Dateipfad) der hinzuzufügenden Datei.
imgnum Verwenden Sie diese Option, um die Anzahl der Unterbilder anzugeben (1 für ein einzelnes Bild oder für ein *.gif ).
removeback Gibt an, ob alle Pixel mit der Hintergrundfarbe (linkes unteres Pixel) transparent gemacht werden sollen.
smooth Gibt an, ob die Kanten geglättet werden sollen, wenn sie transparent sind.
xorig Geben Sie die x-Position des Ursprungs im Sprite an.
yorig Geben Sie die y-Position des Ursprungs im Sprite an.


Kehrt zurück:

Real (sprite ID value)


Beschreibung

Mit dieser Funktion können Sie ein Bild als Sprite hinzufügen, indem Sie es von einer externen Quelle laden, in der sich die zu ladende Bilddatei immer befinden sollte *.png. *.gif oder *.jpg/jpeg Format. Die Funktion gibt den neuen Sprite-Index zurück, der dann in allen weiteren Codes verwendet werden muss, die sich auf das Sprite beziehen. Wenn Sie diese Funktion mit HTML5 verwenden oder ein Bild von einer URL abrufen, generiert diese Funktion auch ein asynchrones Ereignis " Image Loaded". Weitere Informationen finden Sie auf der Seite Asynchronous Event.

Alle Bilder, die in animierte Sprites umgewandelt werden sollen - mit Ausnahme von *.gif (siehe unten) - sollte ein "Streifen" -Format haben (siehe Bild unten) und es wird in die Anzahl der Unterbilder aufgeteilt, die nach der Regel Sprite width = Streifenbreite / Unterbilder angegeben werden.

Wie Sie im obigen Bild sehen können, wurde das Sprite auf einem dunkelvioletten Hintergrund platziert. Dies kann entfernt werden, indem Sie das Argument "removeback" auf "removeback" setzen true. Dies funktioniert, indem Sie das untere linke Pixel des Sprites auf die dort befindliche Farbe überprüfen und dieses dann als zu entfernende Farbe verwenden. Wenn wir beispielsweise in der obigen Abbildung die untere linke Pixelfarbe als grün hätten, wären alle grünen Teile des Sprites entfernt und der Rest des violetten Hintergrunds ignoriert worden.

Wenn Sie die Option "Entfernen" wählen, möchten Sie möglicherweise, dass GameMaker Studio 2 die Kanten des Sprites glättet, indem Sie das Argument "glatt" auf "Glätten" setzen true. Sie müssen lediglich einen halbtransparenten Rand um die Kanten des Sprites erstellen, nachdem der Hintergrund entfernt wurde.

Schließlich können Sie auch den x- und y- Ursprung für das Sprite angeben. Dies ist der Punkt, an dem das Sprite auf die Instanz, die es verwendet, "fixiert" wird, und wird immer als relativ zur 0,0 oberen linken Ecke eines Sprite-Unterbilds berechnet. So hat beispielsweise ein Sprite mit 32 x 32 Pixeln, deren Werte auf (16,16) gesetzt sind, seinen Ursprung in der Mitte.

Wenn Sie eine hinzufügen *.gif Wenn Sie das Bild formatieren, müssen Sie nur den Dateinamen und die Bildnummer angeben (die für das erste Bild 1 sein sollte) und alle anderen Argumente als 0 belassen. Dabei wird nur das erste Bild der GIF-Datei geladen.

Standardmäßig werden die Begrenzungsrahmen für alle neuen Sprites automatisch berechnet (der genaue Begrenzungsrahmen hängt von der Größe und Transparenz des Sprites ab). Sie können dies jedoch anpassen. In diesem Fall sollten Sie auch die Funktion verwenden sprite_collision_mask.

HINWEIS: Wenn Sie ein Sprite in GameMaker Studio 2 laden, müssen Sie daran denken, es wieder zu entfernen (mit sprite_delete ) wenn es nicht mehr benötigt wird, andernfalls besteht die Gefahr eines Speicherverlusts, der das Spiel verlangsamt und letztendlich zum Absturz bringt.
HINWEIS: Wenn Sie unter iOS eine eingeschlossene Datei aus einem Ordner laden, dh mit einem Pfad wie " Backgrounds/background1.png msgstr "Sie müssen den Ordner NICHT als Teil des Pfads angeben und es reicht aus, den Dateinamen anzugeben. Auf allen anderen Plattformen müssen Sie den Ordner als Teil des Pfads zur Datei angeben.
ANMERKUNG: Wenn Sie unter HTML5 ein Sprite von einem sicheren Server laden, müssen Sie möglicherweise den Herkunftstyp festlegen und einen relativen Pfad anstelle eines absoluten Pfads verwenden. Sehen http_set_request_crossorigin für mehr Details.
HINWEIS: Wenn Sie das Sprite von einer URL aus laden, wird das Laden asynchron ausgeführt und löst ein asynchrones Image Loaded- Ereignis aus.
WICHTIG! Diese Funktion ist nicht mit der Testlizenz des Produkts verfügbar.


Beispiel:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);

Der obige Code lädt ein Sprite in das Spiel und speichert seinen Index in der Variablen "spr".
