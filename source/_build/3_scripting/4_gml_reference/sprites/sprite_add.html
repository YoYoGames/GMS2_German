sprite_add


Syntax:

sprite_add(fname, imgnumb, removeback, smooth, xorig, yorig);

Streit Beschreibung
fname Der Name (ein Zeichenfolgendateipfad) der hinzuzufügenden Datei.
imgnum Verwenden Sie diese Option, um die Anzahl der Teilbilder anzugeben (1 für ein einzelnes Bild oder für ein *.gif ).
removeback Gibt an, ob alle Pixel mit der Hintergrundfarbe (linkes unteres Pixel) transparent gemacht werden sollen.
smooth Gibt an, ob die Kanten geglättet werden sollen, wenn sie transparent sind.
xorig Geben Sie die x-Position des Ursprungs im Sprite an.
yorig Geben Sie die y-Position des Ursprungs im Sprite an.


Kehrt zurück:

Real (sprite ID value)


Beschreibung

Mit dieser Funktion können Sie ein Bild als Sprite hinzufügen und es von einer externen Quelle laden, in der sich die zu ladende Bilddatei immer befinden sollte *.png, *.gif oder *.jpg/jpeg Format. Die Funktion gibt den neuen Sprite-Index zurück, der dann in allen weiteren Codes, die sich auf das Sprite beziehen, verwendet werden muss. Wenn Sie diese Funktion mit HTML5 verwenden oder ein Bild von einer URL erhalten, generiert diese Funktion ebenfalls ein asynchrones Ereignis. Weitere Informationen finden Sie unter Das asynchrone Ereignis.

Alle Bilder, die in animierte Sprites umgewandelt werden sollen - außer für *.gif (siehe unten) - sollte ein "strip" -Format haben (siehe Bild unten) und es wird in die Anzahl der Teilbilder aufgeteilt, die nach der Regel sprite width = strip width / sub images angegeben werden.

Wie Sie im obigen Bild sehen können, wurde das Sprite auf einen dunkelvioletten Hintergrund gesetzt, und dies kann durch Setzen des "Remove Back" -Arguments entfernt werden true. Dies funktioniert, indem Sie das untere linke Pixel des Sprites für die Farbe dort überprüfen und dann diese als die Farbe verwenden, die entfernt werden soll. Wenn beispielsweise im obigen Bild die Pixelfarbe links unten grün wäre, wären alle grünen Teile des Sprites entfernt und der Rest des violetten Hintergrunds ignoriert worden.

Wenn Sie die Option "removeback" wählen, möchten Sie vielleicht auch, dass GameMaker Studio 2 die Kanten des Sprites glättet, indem Sie das Argument "smooth" setzen true. All dies macht einen halbtransparenten Rand um die Kanten des Sprites herum, nachdem der Hintergrund entfernt wurde.

Schließlich können Sie auch den x- und y- Ursprung für das Sprite angeben. Dies ist der Punkt, an dem das Sprite auf die Instanz "fixiert" wird, die es verwendet, und wird immer als relativ zur oberen linken Ecke eines Sprite-Unterbilds berechnet. So wird zum Beispiel ein Sprite, das 32 x 32 Pixel mit diesen auf (16,16) gesetzten Werten hat, seinen Ursprung in der Mitte haben.

Wenn Sie ein hinzufügen *.gif formatieren Sie das Bild, dann müssen Sie nur den Dateinamen und die Bildnummer angeben (was 1 für das erste Bild sein sollte) und können alle anderen Argumente als 0 belassen, wobei darauf hingewiesen wird, dass nur das erste Bild des GIF geladen wird.

Standardmäßig werden bei allen neuen Sprites die Begrenzungsrechteck automatisch berechnet (die genaue Begrenzungsbox hängt von der Größe und Transparenz des Sprites ab). Sie können dies jedoch anpassen. In diesem Fall sollten Sie auch die Funktion verwenden sprite_collision_mask.

HINWEIS: Wenn Sie ein Sprite in GameMaker Studio 2 laden, müssen Sie daran denken, es wieder zu entfernen (mit sprite_delete ) wenn es nicht mehr benötigt wird, besteht sonst die Gefahr eines Speicherlecks, das das Spiel verlangsamt und schließlich zum Absturz bringt.
WICHTIG! Diese Funktion ist mit der Testlizenz des Produkts nicht verfügbar.


Beispiel:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);

Der obige Code lädt ein Sprite in das Spiel und speichert seinen Index in der Variablen "spr".
