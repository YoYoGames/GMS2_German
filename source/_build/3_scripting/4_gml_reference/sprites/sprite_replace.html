sprite_replace


Syntax:

sprite_replace(ind, fname, imgnumb, removeback, smooth, xorig, yorig);

Streit Beschreibung
ind Der Index des Sprites wird permanent ersetzt.
fname Der Dateiname des Bildes, um das neue Sprite zu erstellen.
imgnumb Die Anzahl der Frames, in die das Sprite horizontal geschnitten wird. 1 = ein einzelnes Bild / *.gif.
removeback Gibt an, ob alle Pixel mit der Hintergrundfarbe (linkes unteres Pixel) transparent gemacht werden sollen.
smooth Gibt an, ob die Kanten geglättet werden sollen.
xorig Die x-Koordinate des Ursprungs relativ zur oberen linken Ecke des Sprites.
yorig Die y-Koordinate des Ursprungs relativ zur oberen linken Ecke des Sprites.


Kehrt zurück

N/A


Beschreibung

Diese Funktion funktioniert fast genauso wie sprite_add, anstatt den Index des zu importierenden Sprites zurückzugeben, überschreibt es einen zuvor erstellten Sprite-Index. Wenn Sie diese Funktion verwenden, sollten Sie einen Sprite-Index verwenden, der in einer Variablen mit anderen Funktionen wie z sprite_add oder sprite_create_from_surface, oder auch sprite_duplicate, und nicht ein Sprite-Asset eines Ressourcenbaums. Sie können ein Sprite mithilfe dieser Funktion aus den Spielbestandteilen ersetzen. Dies bedeutet jedoch, dass Sie die Referenz-ID für das Sprite verlieren, das Sie ersetzen, unabhängig davon, ob Sie einen Aufruf ausführen game_restart() oder nicht, und so wird nicht empfohlen. Unabhängig davon, welches Sprite ersetzt wird, erstellt diese Funktion eine neue Texturseite für das Sprite. Daher sollte beim Verwenden des Sprites Vorsicht walten, da dies die Leistung beeinträchtigen kann, indem die Anzahl der erforderlichen Textur-Swaps für das Rendern erhöht wird.

Die zu ladende Bilddatei sollte immer in sein *.png Format und alle Bilder, die in animierte Sprites umgewandelt werden sollen, sollten ein "Streifen" -Format haben (siehe Bild unten). Sie werden in die Anzahl der Teilbilder aufgeteilt, die nach der Regel sprite width = strip width / sub images angegeben sind.

Wie Sie im obigen Bild sehen können, wurde das Sprite auf einen dunkelvioletten Hintergrund gesetzt, und dies kann durch Setzen des "Remove Back" -Arguments entfernt werden true. Dies funktioniert, indem Sie das untere linke Pixel des Sprites für die Farbe dort überprüfen und dann diese als die Farbe verwenden, die entfernt werden soll. Wenn beispielsweise im obigen Bild die Pixelfarbe links unten grün wäre, wären alle grünen Teile des Sprites entfernt und der Rest des violetten Hintergrunds ignoriert worden.

Wenn Sie die Option "removeback" wählen, möchten Sie vielleicht auch, dass GameMaker Studio 2 die Kanten des Sprites glättet, indem Sie das Argument "smooth" setzen true. All dies macht einen halbtransparenten Rand um die Kanten des Sprites herum, nachdem der Hintergrund entfernt wurde.

Schließlich können Sie auch den x- und y- Ursprung für das Sprite angeben. Dies ist der Punkt, an dem das Sprite auf die Instanz "fixiert" wird, die es verwendet, und wird immer als relativ zur oberen linken Ecke eines Sprite-Unterbilds berechnet. So wird zum Beispiel ein Sprite, das 32 x 32 Pixel mit diesen auf (16,16) gesetzten Werten hat, seinen Ursprung in der Mitte haben.

Standardmäßig werden bei allen neuen Sprites die Bounding-Boxen automatisch berechnet (die genaue Größe der Box hängt von der Größe und Transparenz des Sprites ab). Sie können dies jedoch anpassen. In diesem Fall sollten Sie auch die Funktion verwenden sprite_collision_mask.

HINWEIS: Abhängig von der ausgewählten Zielplattform sind Sie eingeschränkt, wo Sie Dateien speichern und laden können. Weitere Informationen finden Sie unter Dateiverarbeitung.


HINWEIS: Wenn Sie diese Funktion in Ihrem HTML5-Zielspiel verwenden, um Ressourcen von einem externen Server zu laden, sollten Sie aufgrund des XSS-Schutzes in Browsern versuchen, Ressourcen aus verschiedenen Domänen zu laden, und möglicherweise zurückkehren leere Ergebnisse.


WICHTIG! Diese Funktion ist mit der Testlizenz des Produkts nicht verfügbar.


Beispiel:

sprite_replace(spr_banner, "gravemaker.png", 1, false, false, 0, 0);

Der obige Code ersetzt das in "spr_banner" indexierte Bild-Asset durch ein anderes, das von einer externen Quelle geladen wurde.
