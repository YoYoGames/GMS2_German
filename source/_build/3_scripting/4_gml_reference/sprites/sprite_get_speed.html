sprite_get_speed


Syntax:

sprite_get_speed(index)

Streit Beschreibung
index Der Index des Sprites, um die Geschwindigkeit von


Kehrt zurück:

Real


Beschreibung

Mit dieser Funktion kann die Sprite-Geschwindigkeit wie im Sprite-Editor für die Sprite-Ressource definiert abgerufen werden. Der zurückgegebene Wert kann dann zum Beispiel verwendet werden, um zu berechnen, wie viele Rahmen für unterschiedliche gezeichnet werden können image_speeds. Beachten Sie, dass der Rückgabewert sehr unterschiedlich ist, abhängig von der Art der Geschwindigkeit, die im Sprite-Editor angewendet wurde, entweder Frames pro Sekunde oder Frames pro Game Frame. Die folgenden zwei Beispiele veranschaulichen dies:

  • Wenn Sie ein Sprite haben, das 1 Bild pro Sekunde zeichnet und die Bildgeschwindigkeit auf 0,5 setzt, wird es mit 0,5 Bildern pro Sekunde gezeichnet. Wenn die Frame-Rate Ihres Spiels 60 Frames pro Sekunde beträgt, zeichnet das Sprite 1 Frame pro 120 Spiel-Frames.

  • Wenn Sie einen Sprit haben, die 1 Bild pro Spiel Rahmen zieht und stellen Sie die Bildgeschwindigkeit auf 0,5 wird es 0,5 Bilder pro Spiel Rahmen ziehen. Wenn die Frame-Rate Ihres Spiels 60 Frames pro Sekunde beträgt, zeichnet das Sprite 30 Frames pro 60 Spiel-Frames.

Sie können herausfinden, welche Art von Animationsgeschwindigkeit verwendet wurde, wenn Sie das Sprite mithilfe der Funktion definieren sprite_get_speed_type und Sie können die Animationsgeschwindigkeit und -art mit der Funktion einstellen sprite_set_speed.


Beispiel:

s_speed = sprite_get_speed(sprite_index);
s_type = sprite_get_speed_type(sprite_index);

Der obige Code erhält die Sprite-Geschwindigkeit und den Sprite-Animationstyp und speichert sie in Instanzvariablen für die zukünftige Verwendung.
