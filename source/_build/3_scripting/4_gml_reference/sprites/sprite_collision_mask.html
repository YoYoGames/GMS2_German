sprite_collision_mask


Syntax:

sprite_collision_mask(ind, sepmasks, bboxmode, bbleft, bbtop, bbright, bbbottom, kind, tolerance);

Streit Beschreibung
ind Der Index des Sprites, für den der Begrenzungsrahmen von festgelegt werden soll.
sepmasks Gibt an, ob für jedes Unterbild des Sprites Kollisionsmasken erstellt werden sollen ( true ) oder eine Maske für alle ( false ).
bboxmode Welche Art von Begrenzungsbox zu verwenden. 0 = automatisch, 1 = Vollbild, 2 = benutzerdefiniert.
bbleft Die maximale linke Position des Begrenzungsrahmens.
bbtop Die maximale obere Position des Begrenzungsrahmens.
bbright Die maximale rechte Position des Begrenzungsrahmens.
bbbottom Die maximale untere Position des Begrenzungsrahmens.
kind Die Art der Maske, eine Konstante (siehe Tabelle in der Beschreibung).
tolerance Gibt die Toleranz im Transparenzwert an (0 = keine Toleranz, 255 = volle Toleranz).


Kehrt zurück

N/A


Beschreibung

Mit dieser Funktion können Sie die Eigenschaften der Kollisionsmaske festlegen, die ein Sprite haben soll. Wenn Sie als Begrenzungsmodus entweder Automatisch (0) oder Vollbild (1) auswählen, können die einzelnen Werte des Begrenzungsrahmens auf 0 gesetzt werden. Bei einer benutzerdefinierten Maske (2) müssen Sie diese Werte jedoch festlegen. Die verschiedenen Begrenzungsrahmenwerte sind immer relativ zur oberen linken Ecke des Sprites (unabhängig vom x- und y-Ursprung), die als Position (0, 0) betrachtet wird.

Durch das Festlegen der Art der Maske wird die allgemeine Form für die Maske selbst festgelegt. Beachten Sie jedoch, dass für andere als rechteckige Masken mehr Verarbeitungsleistung erforderlich ist, um Kollisionen aufzulösen, was zu einem Leistungsabfall führt. Im Allgemeinen sollten Sie nur andere als rechteckige Maskentypen verwenden, wenn dies unbedingt erforderlich ist.

HINWEIS: Diese Funktion lässt die Art der gedrehten Rechteck- Kollisionsmaske nicht zu.


Die Art der Maske, die eingestellt werden kann, ist eine der folgenden Konstanten:

Konstante Beschreibung
bboxkind_rectangular Eine rechteckige (nicht rotierende) Rechteckkollisionsmaskenform
bboxkind_ellipse Eine elliptische Kollisionsmaskenform
bboxkind_diamond Eine Diamantkollisionsmaskenform
bboxkind_precise Eine präzise Kollisionsmaske, bei der die Maske den nicht-transparenten Pixeln des Sprites entspricht, basierend auf dem angegebenen Toleranzwert (siehe unten).

Schließlich wird mit Toleranz festgelegt, wie genau die Maske genau ist (wenn sie mit einer "Vollbild" -Maske verwendet wird, hat dies keine Auswirkung). Die Toleranz 0 bedeutet, dass die Maske jedem einzelnen Pixel folgt, das eine Transparenz aufweist über 0, während andere Werte den Umfang der Kollisionsmaske abhängig von der Transparenz der Pixel verschieben.

HINWEIS: Diese Funktion ist nur für Bitmap- Sprites nützlich und funktioniert nicht mit SWF- oder JSON-Sprites (Spine).
HINWEIS: Diese Funktion funktioniert nur bei duplizierten Sprites und nicht direkt bei vorgefertigten Ressourcen. Sie können Sprites mit der Funktion duplizieren sprite_duplicate.


Beispiel:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);
sprite_collision_mask(spr, true, 1, 0, 0, 0, 0, 0, 0);

Der obige Code lädt ein Sprite aus einer externen Quelle und speichert den neuen Index in der Variablen "spr". Der Code stellt dann das neue Sprite so ein, dass für jedes seiner Teilbilder eine präzise Kollisionsmaske angezeigt wird.
