sprite_collision_maske


Syntax:

sprite_collision_mask(ind, sepmasks, bboxmode, bbleft, bbtop, bbright, bbbottom, kind, tolerance);

Streit Beschreibung
ind Der Index des Sprites, um den Begrenzungsrahmen festzulegen.
sepmasks Ob für jedes Teilbild des Sprites Kollisionsmasken erstellt werden sollen ( true ) oder eine Maske für alle ( false ).
bboxmode Welche Art von Bounding Box zu verwenden. 0 = automatisch, 1 = vollständiges Bild, 2 = benutzerdefiniert.
bbleft Die maximale linke Position der Bounding Box.
bbtop Die maximale obere Position der Begrenzungsbox.
bbright Die maximale rechte Position der Begrenzungsbox.
bbbottom Die maximale untere Position der Begrenzungsbox.
kind Die Art der Maske (0 = präzise, ​​1 = Begrenzungsbox, 2 = Scheibe, 3 = Diamant).
tolerance Gibt die Toleranz im Transparenzwert an (0 = keine Toleranz, 255 = volle Toleranz).

Beschreibung

Mit dieser Funktion können Sie den Typ der Kollisionsmaske festlegen, die ein Sprite haben soll. Wenn Sie als Bounding Box-Modus entweder automatisch (0) oder Vollbild (1) auswählen, können die einzelnen Bounding Box-Werte auf 0 gesetzt werden. Für eine benutzerdefinierte Maske (2) müssen Sie diese Werte jedoch festlegen. Die verschiedenen Bounding Box-Werte beziehen sich immer auf die obere linke Ecke des Sprites (unabhängig vom x- und y-Ursprung), die als Position (0, 0) betrachtet wird.

Wenn Sie die Art der Maske festlegen, wird die allgemeine Form für die Maske selbst festgelegt. Beachten Sie jedoch, dass alles andere als eine rechteckige Maske beim Auflösen von Kollisionen mehr Rechenleistung erfordert und die Leistung dadurch sinkt. Im Allgemeinen sollten Sie nur Maskenarten verwenden, die nicht rechteckig sind, wenn dies unbedingt erforderlich ist.

Schließlich wird Toleranz verwendet, um zu definieren, wie genau die Maske genau ist (wenn sie mit einer "Vollbild" -Maske verwendet wird, hat dies keine Auswirkung), mit einer Toleranz von 0, was bedeutet, dass die Maske jedem einzelnen Pixel folgt, das eine Transparenz aufweist über 0, während andere Werte den Umfang der Kollisionsmaske in Abhängigkeit von der Transparenz der Pixel verschieben.

HINWEIS: Diese Funktion ist nur für Bitmap- Sprites und nicht für SWF- oder JSON (Spine) -Sprites geeignet.
HINWEIS: Diese Funktion funktioniert nur bei duplizierten Sprites und nicht direkt bei vorgefertigten Ressourcen. Sie können Sprites mit der Funktion duplizieren sprite_duplicate.


Beispiel:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);
sprite_collision_mask(spr, true, 1, 0, 0, 0, 0, 0, 0);

Der obige Code lädt ein Sprite aus einer externen Quelle und speichert den neuen Index in der Variablen "spr". Der Code setzt dann das neue Sprite so, dass es für jedes seiner Teilbilder eine genaue Kollisionsmaske hat.
