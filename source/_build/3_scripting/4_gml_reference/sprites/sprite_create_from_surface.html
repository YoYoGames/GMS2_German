sprite_create_from_surface


Syntax:

sprite_create_from_surface(index, x, y, w, h, removeback, smooth, xorig, yorig);

Streit Beschreibung
index Der Index der zu erstellenden Oberfläche.
x Die x-Position, aus der kopiert werden soll.
y Die y-Position, aus der kopiert werden soll.
w Die Breite des zu kopierenden Bereichs (von der x-Position).
h Die Höhe des zu kopierenden Bereichs (von der y-Position).
removeback Gibt an, ob alle Pixel mit der Hintergrundfarbe (linkes unteres Pixel) transparent gemacht werden sollen.
smooth Gibt an, ob die Kanten geglättet werden sollen.
xorig Zeigt die x-Position des Ursprungs im Sprite an.
yorig Zeigt die y-Position des Ursprungs im Sprite an.


Kehrt zurück

Real (unique sprite ID value)


Beschreibung

Mit dieser Funktion können Sie ein Sprite aus einer zuvor initialisierten Oberfläche erstellen (der Oberflächenindex-ID-Wert wird zurückgegeben, wenn Sie die Oberfläche mit erstellen surface_create ). Die X- und Y-Koordinaten, die Sie dann in der Funktion eingeben, sollten relativ zur (0,0) Position der Oberfläche (die obere linke Ecke) sein und nicht das Spielfenster (noch die Ansicht, wenn Sie eine aktiv haben). Die Argumente für Breite und Höhe sind in Pixeln angegeben und definieren die Breite und Höhe des zu verwendenden Teils der Oberfläche.

Setzen Sie das Argument "removeback" auf true entfernt eine Farbe aus dem Sprite, indem Sie das untere linke Pixel des Sprites auf die Farbe dort prüfen und dann diese als zu entfernende Farbe verwenden.

Wenn Sie die Option "removeback" wählen, möchten Sie vielleicht auch, dass GameMaker Studio 2 die Kanten des Sprites glättet, indem Sie das Argument "smooth" setzen true. All dies macht einen halbtransparenten Rand um die Kanten des Sprites herum, nachdem der Hintergrund entfernt wurde.

Schließlich können Sie auch den x- und y- Ursprung für das Sprite angeben. Dies ist der Punkt, an dem das Sprite auf die Instanz "fixiert" wird, die es verwendet, und wird immer als relativ zur oberen linken Ecke eines Sprite-Unterbilds berechnet. So wird zum Beispiel ein Sprite, das 32 x 32 Pixel mit diesen auf (16,16) gesetzten Werten hat, seinen Ursprung in der Mitte haben.

Standardmäßig werden bei allen neuen Sprites die Bounding-Boxen automatisch berechnet (die genaue Größe der Box hängt von der Größe und Transparenz des Sprites ab). Sie können dies jedoch anpassen. In diesem Fall sollten Sie auch die Funktion verwenden sprite_collision_mask.

HINWEIS: Wenn Sie mit dieser Methode ein Sprite in GameMaker Studio 2 erstellen, müssen Sie daran denken, es erneut zu entfernen (mit sprite_delete ) wenn es nicht mehr benötigt wird, besteht sonst die Gefahr eines Speicherlecks, das das Spiel verlangsamt und schließlich zum Absturz bringt.
WICHTIG! Diese Funktion ist mit der Testlizenz des Produkts nicht verfügbar.


Beispiel:

var surf;
surf = surface_create(32, 32);
surface_set_target(surf);
draw_clear_alpha(c_black, 0);
draw_sprite(spr_Body, 0, 0, 0);
draw_sprite(spr_Clothes, 0, 0, 0);
draw_sprite(spr_Hair, 0, 0, 0);
spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
surface_reset_target();
surface_free(surf);

Der obige Code erstellt eine Oberfläche und speichert ihren Index in der lokalen Variablen "surf". Er zielt dann auf diese Oberfläche, löscht sie und zieht mehrere Sprites übereinander. Schließlich erstellt er ein neues Sprite aus dem auf der Oberfläche gezeichneten zusammengesetzten Bild und weist seinen Index "spr_Custom" zu, bevor er den von der Oberfläche verwendeten Speicher freigibt.
