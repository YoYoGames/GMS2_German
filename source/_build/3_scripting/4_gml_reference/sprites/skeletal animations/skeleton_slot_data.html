skeleton_slot_data


Syntax:

skeleton_slot_data(sprite, list);

Streit Beschreibung
sprite Der Sprite-Index der Spine-Skeleton-Animation, von der die Daten stammen.
list Die ID der DS-Liste, die mit den DS-Karten gefüllt werden soll.


Kehrt zurück:

N/A


Beschreibung

Mit dieser Funktion können Sie ein (vordefiniertes) ds_list mit Daten für jeden der verfügbaren Anhang-Slots, die eine Animation enthält. Diese Daten werden im Forum von a zurückgegeben ds_map welche die folgenden Schlüssel / Wert-Paare enthält:

  • "Name": Der Name des Slots.

  • "bone": Der Name des Knochens.

  • "Anhang": Der Name des Anhangs, wenn einer verwendet wird, oder "(keiner)", wenn keiner zugeordnet ist.

Die Werte für jeden Schlüssel sind Zeichenfolgen und können dann in den anderen Skelettanhangsfunktionen für diese Spritearten verwendet werden. Notiere dass der ds_map erstellt werden nicht zerstört, so müssen Sie durch die Schleife ds_list und zerstöre jede der erstellten Karten selbst.

WICHTIG! Diese Funktion ist mit der Testlizenz des Produkts nicht verfügbar.


Beispiel:

var list = ds_list_create();
var open = true;
slot_name = "";
skeleton_slot_data(sprite_index, list);
for (var i = 0; i < ds_list_size(list); i++;)
   {
   var map = list[| i];
   if open
      {
      if map[? "attachment"] == "(none)"
         {
         open = false;
         slot_name = map[? "name"];
         }
      }
   ds_map_destroy(map);
   }
ds_list_destroy(list);

Der obige Code erstellt eine DS-Liste und füllt sie dann mit den Slot-Daten für das Instanz-Sprite auf. Diese Daten werden dann analysiert, um die einzelnen DS-Karten mit den Schlitzdaten zu extrahieren. Dies wird dann überprüft, um festzustellen, ob ein leerer Slot vorhanden ist, und wenn dies der Fall ist, wird der Variablen "slot_name" der Name des leeren Slots zugewiesen, bevor die DS-Map zerstört wird. Schließlich zerstören wir die DS-Liste, da sie nicht mehr benötigt wird.
