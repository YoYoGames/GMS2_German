skeleton_attachment_create


Syntax:

skeleton_attachment_create(name, sprite, ind, xorigin, yorigin, xscale, yscale, rot);

Streit Beschreibung
name Der Name (als Zeichenfolge) des zu erstellenden Anhangs.
sprite Der sprite_index, um das Anhangsbild zu erhalten.
ind Der Image_Index, von dem das Anhangsbild abgerufen werden soll.
xorigin Der x-Ursprung für das verwendete Bild.
yorigin Der y-Ursprung für das verwendete Bild.
xscale Die horizontale Skalierung des Bildes als Multiplikator: 1 = normale Skalierung, 0,5 ist die Hälfte usw.
yscale Die vertikale Skalierung des Bildes als Multiplikator: 1 = normale Skalierung, 0,5 ist die Hälfte usw.
rot Die Drehung des Bildes. 0 = normal, 90 = um 90 Grad gegen den Uhrzeigersinn gedreht usw.


Kehrt zurück:

1 if successful, -1 if not


Beschreibung

Wenn Sie ein Skeleton-Animations-Sprite erstellen, können Sie ihm Anhangs-Slots und -Anhänge zuweisen. Dies sind einfach Bilder, die sich von der Animation unterscheiden, aber wenn sie angebracht sind, werden sie sich mit dem Knochen bewegen, an dem sie befestigt sind. Normalerweise würden Sie diese Anhänge in Ihrem Animationsprogramm (Spine) zuweisen, aber mit dieser Funktion können Sie Ihre eigenen zur Laufzeit mit einem Sprite-Asset aus Ihrem Spiel erstellen. Die Funktion erfordert, dass Sie dem Anhang einen Namen (als String) geben und dann den zu verwendenden sprite_index und image_index sowie den x- und y-Ursprung festlegen (der sich von dem in den Sprite-Eigenschaften des Sprites definierten unterscheiden kann). Außerdem können Sie alle Transformationen festlegen, die auf das Bild angewendet werden sollen, wenn es angehängt wird. Wenn der Anhang erfolgreich erstellt wurde, wird die Funktion zurückkehren 1 Ist dies nicht der Fall (Sie haben einen ungültigen Sprite-Index angegeben oder das Basis-Sprite ist kein Spine-Sprite), wird es zurückgegeben -1.

WICHTIG! Diese Funktion ist mit der Testlizenz des Produkts nicht verfügbar.


Beispiel:

skeleton_attachment_create("sword", spr_Weapons, 0, 0, 80, 1, 1, 90);
skeleton_attachment_create("knife", spr_Weapons, 1, 0, 45, 1, 1, 90);
skeleton_attachment_create("crossbow", spr_Weapons, 0, 10, 30, 1, 1, 0);
skeleton_attachment_set("slot_leftHand", choose("sword", "knife", "crossbow"));

Der obige Code würde den aktuell zugewiesenen Anhangsnamen für den Slot "slot_leftHand" überprüfen und wenn ein leerer String zurückgegeben wird, wird ein neues Sprite angehängt.
